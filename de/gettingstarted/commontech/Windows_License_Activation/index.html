      Windows-Lizenzen in IBM Bluemix Private Cloud aktivieren

      Der von IBM bereitgestellte Imagekatalog enthält Microsoft® Windows-Images, die in einem BYOL-Modell (Bring Your Own License) angeboten werden. Je nach Nutzungsrechten für Microsoft-Service-Provider stellt IBM möglicherweise keine KMS-Aktivierungsservices (Key Management Services, Schlüsselmanagementservices) für Ihr Windows-BYOL-Image bereit.

      Es gibt verschiedene Möglichkeiten, Ihre Windows-Lizenzen zu aktivieren:

      • KMS-Server in der IBM Bluemix Private Cloud-Umgebung erstellen
      • Vorhandenen KMS-Server in Ihrer Site oder an einem anderen Standort verwenden
      • Direkte Online-Aktivierung bei Microsoft
      • Telefonische Offline-Aktivierung bei Microsoft

      Je nach Situation können Sie eine oder mehrere der genannten Methoden verwenden. Im Folgenden sind verschiedene Aspekte beschrieben, die Sie bei der Wahl der Methode berücksichtigen müssen:

      • KMS-Server haben Mindestschwellenwerte bezüglich der Anzahl der aktivierten Clients. Wenn Ihr KMS-Service auf Windows Server 2008 oder Windows Server 2012 gehostet wird, müssen Sie mindestens fünf Clients aktivieren. Wenn Sie weniger als fünf Lizenzen aktivieren, können Sie KMS in IBM Bluemix Private Cloud nicht verwenden.
      • Wenn Sie einen vorhandenen KMS-Server verwenden möchten, der sich außerhalb Ihrer IBM Bluemix Private Cloud-Umgebung befindet, müssen Sie die entsprechende Konnektivität einrichten. Dazu gehören sowohl die Bluemix Private Cloud-Firewalls als auch die Firewalls, die den Bluemix Private Cloud-Windows-Instanzen ermöglichen, Datenverkehr zum KMS-Server über TCP-Port 1688 einzuleiten.
      • Wenn Sie die direkte Online-Aktivierung verwenden möchten, müssen Ihre Windows-Instanzen abgehende Konnektivität zum Internet haben. Instanzen haben standardmäßig keine Internetkonnektivität, aber Sie können den Zugriff in Ihrer Erstbestellung anfordern oder jederzeit ein Support-Ticket einreichen, um den Zugriff zu aktualisieren.

      KMS-Server in IBM Blue Box Cloud erstellen

      Sie können jede Windows Server 2008- oder neuere Instanz für die Bereitstellung des KMS-Service konfigurieren. Damit eine Windows-Instanz als KMS-Server dienen kann, muss die Instanz abgehende Internetkonnektivität haben, um mit Microsoft-Lizenzservern kommunizieren zu können. Außerdem müssen Sie einen KMS-Hostschlüssel bei Microsoft erwerben, um die speziellen Berechtigungen zu erhalten.

      Gehen Sie zum Konvertieren einer Windows 2008 R2-Instanz in einen KMS-Server in IBM Bluemix Private Cloud wie folgt vor:

      1. Laden Sie diese Microsoft-Aktualisierungsdatei in Ihre Instanz herunter.

      2. Installieren Sie die Aktualisierung.

      3. Registrieren Sie den KMS-Hostschlüssel mit dem folgenden Befehl:

         %SystemRoot%\System32\CScript.exe %windir%\system32\slmgr.vbs /ipk <Ihr_KMS-Hostschlüssel>

      4. Aktivieren Sie den neuen KMS-Hostschlüssel mit dem folgenden Befehl:

         %SystemRoot%\System32\CScript.exe %windir%\system32\slmgr.vbs /ato
      5. Starten Sie den Softwarelizenzservice mit dem folgenden Befehl: 
         net stop sppsvc && net start sppsvc
      6. Verwenden Sie den folgenden Befehl, um Port 1688 zu öffnen, sodass die KMS-Clients sich beim KMS-Host registrieren können: 
         netsh firewall set portopening tcp 1688 KMS enable

      Vorhandenen KMS-Server verwenden

      Wenn Sie bereits einen KMS-Server in Ihrer Site oder an einem anderen Standort haben, können Sie diesen verwenden, sofern in diesem die entsprechenden Netzverbindungen eingerichtet sind. Damit der Windows-Server mit dem KMS-Server aktiviert werden kann, muss dieser in der Lage sein, Übertragungen mit dem KMS-Server (gewöhnlich über TCP-Port 1688) einzuleiten. Erstellen Sie ein Support-Ticket, damit die entsprechenden statischen Regeln in der IBM Bluemix Private Cloud-Firewall angewendet werden.

      Windows-Server über einen KMS-Server aktivieren

      Wenn Sie Ihren Windows-Server über einen KMS-Server aktivieren möchten, benötigen Sie die IP-Adresse und Netzkonnektivität zum KMS-Server sowie die entsprechenden KMS-Clientkonfigurationsschlüssel von Microsoft Technet.

      Nachdem Sie aus dem Microsoft Windows Server-Image eine neue Instanz erstellt haben, stellen Sie eine Verbindung zu dieser her und führen dann die folgenden Schritte aus:

      1. Installieren Sie einen KMS-Clientschlüssel in Ihrer Windows-Instanz. Öffnen Sie ein Fenster mit Eingabeaufforderung und führen Sie den folgenden Befehl aus: 
         slmgr /ipk <KMS-Clientschlüssel>

        Sehen Sie sich die KMS-Clientkonfigurationsschlüssel online an. Zum Aktivieren einer Windows 2008 R2 Standard-Instanz verwenden Sie beispielsweise YC6KT-GKW9T-YTKYR-T4X34-R7VHC.

      2. Legen Sie die IP-Adresse und den Port des KMS-Servers in Ihrer Windows-Instanz fest. Der von den Clients standardmäßig verwendete TCP-Port für die Verbindung zum KMS-Host ist 1688: 
         slmgr /skms <IP-Adresse_des_KMS-Servers>:<Port_des_KMS-Servers>
      3. Synchronisieren Sie die Zeit mit Ihrem KMS-Server: 
         net start w32time
 w32tm /config /manualpeerlist:<IP-Adresse_des_KMS-Servers> /update
 w32tm /resync
      4. Aktivieren Sie Ihre Windows-Instanz: 
         slmgr.vbs /ato
      5. Setzen Sie den Zeitserver wieder auf die Standardeinstellung zurück: 
         w32tm /config /manualpeerlist:time.windows.com /update

      Weitere Einzelheiten zur Windows-Aktivierung finden Sie in den folgenden Quellen:

      • Understanding KMS
      • Deploying KMS Activation
      • Determining Product Key needs
      • KMS Client Setup Keys

      Direkte Aktivierung

      Wenn Sie nur sehr wenige Windows-Server aktivieren müssen, müssen Sie nicht unbedingt einen KMS-Server verwenden. Sie können den Server stattdessen direkt bei Microsoft aktivieren, indem Sie entweder telefonisch mit Microsoft Kontakt aufnehmen oder indem Sie den Server online aktivieren. Wenn Sie Ihren Server online aktivieren möchten, muss Ihre Instanz zur Kontaktaufnahme mit dem Microsoft-Lizenzservice abgehende Internetkonnektivität haben.
