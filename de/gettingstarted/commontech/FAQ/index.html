      Wie kann ich die Größe einer vorhandenen Instanz ändern?

      Obwohl die Option Resize Instance in Horizon angezeigt wird, wird diese Funktionalität momentan nicht unterstützt. Wenn Sie die Größe einer Instanz erhöhen müssen, wird empfohlen, eine Momentaufnahme der Instanz zu erstellen und diese Momentaufnahme dann in einer neuen größeren Variante bereitzustellen.

      Wie kann ich eine Instanz auf einem bestimmten Host starten?

      Diese Funktionalität wird über die OpenStack-API unterstützt. Weitere Einzelheiten hierzu finden Sie unter How to deploy an instance to a specific hypervisor node.

      Ich habe eine neue Cloud und es gibt bereits zwei Projekte. Worum handelt es sich bei diesen Projekten?

      Die Projekte admin und service werden zur Unterstützung des grundlegenden Betriebs Ihrer IBM Bluemix Private Cloud-Instanz erstellt. Diese Projekte dürfen nicht gelöscht werden. Ändern Sie diese Projekte nicht und stellen Sie sie nicht bereit.

      Welche Ports/URLs usw. sind in meiner Bluemix Private Cloud-Instanz öffentlich zugänglich und zu welchem Zweck?

      Die folgenden Ports durch die Firewall zu Ihrer Bluemix Private Cloud-Instanz sind offen:

      80 - horizon (OpenStack-Webportal)
443 - horizon (OpenStack-Webportal)
5000 - keystone (OpenStack Authentication-API)
6080 - nova-novnc (OpenStack-VNC-Konsole)
8000 - heat (OpenStack CloudFormation-API)
8004 - heat (OpenStack Orchestration-API)
8774 - nova (OpenStack Compute-API)
8776 - cinder (OpenStack Block Storage-API)
9292 - glance (OpenStack Image-API)
9696 - neutron (OpenStack Network-API)
35357 - keystone-admin (OpenStack Authentication Admin-API)

      Es sind drei weitere Ports aktiviert, wenn Sie die Konfiguration von Swift (OpenStack-Objektspeicher) in Ihrem Cluster angefordert haben: 6000, 6001, 6002

      Alle aufgeführten Ports sind über Ihre angepasste Horizon-URL verfügbar.

      Wenn Sie auch die Ausgangsports auf das Minimum beschränken müssen, müssen Sie mindestens die Ports 67 und 68 für DHCP und den Port 80 für Metadaten öffnen.

      Wie kann ich die E/A-Operationen pro Sekunde auf dem Laufwerk einer Instanz bzw. eines Datenträgers messen?

      Sie können zum Messen der Plattenleistung die unterschiedlichsten Tools verwenden.

      Es ist ein besonders einfach zu verwendendes Python-Script für die IOPS-Messung (Input/Output Operations Per Second, Eingabe-/Ausgabeoperationen pro Sekunden) von Benjamin Schweizer mit Anweisungen und Beispielergebnissen verfügbar.

      https://benjamin-schweizer.de/measuring-disk-io-performance.html

      Die aktuelle Version dieses Scripts ist momentan hier verfügbar.

      Das Script wird wie folgt verwendet:

      iops [-n|--num_threads threads][-t|--time time] <Gerät>

      Parameter:

      threads := Anzahl gleichzeitiger E/A-Threads, Standardwert 1
time    := Zeit in Sekunden, Standardwert 10
Gerät   := ein Blockgerät, z. B. /dev/sda oder \\\\.\\PhysicalDrive0

      Beispiele:

      iops /dev/sda
iops -n 8 -t 2 /dev/disk0
iops --num_threads 1 --time 2 /dev/md1
iops --num_threads 16 --time 2 /dev/md1

      Wie kann ich bei der Verwendung von Sicherheitsgruppen abgewiesene TCP-Verbindungen zu unzulässigen Ports protokollieren?

      Wenn Sie abgewiesene TCP-Verbindungen zu unzulässigen Ports protokollieren möchten, müssen Sie Sicherheitsgruppen öffnen und iptables mit aktivierter Protokollierung für die Instanzen ausführen.

      Wie viele virtuelle Geräte (z. B. Ports) können maximal an eine virtuelle Maschine angehängt werden?

      Es sind maximal 20 virtuelle PCI-Steckplätze verfügbar.

      Wie kann ich eine Liste aller meiner VMs und deren Hostnamen abrufen?

      Sie benötigen eine Liste aller Instanzen in der Cloud und der Hosts, auf denen die Instanzen ausgeführt werden? Sie benötigen auch die Namen der VMs? Wenn Sie cloud_admin sind, können Sie den folgenden Einzeiler verwenden:

      nova list --all-tenants --fields name,OS-EXT-SRV-ATTR:host

      Zeigt der Befehl openstack host show disk name die tatsächliche Festplattennutzung des physischen Hosts an?

      Nein, der Befehl zeigt Informationen zu den Varianten der auf den Hosts ausgeführten Instanzen an. Derzeit sind keine Nutzungsinformationen für den Auslagerungsspeicher enthalten. Im folgenden Abschnitt wird beschrieben, was in der Befehlsausgabe im Mitaka-Release enthalten ist.

      Sehen Sie sich auch die Informationen unter https://derpops.bike/2016/10/07/openstack-nova-and-hypervisor-disk-consumption/ an.

      Zur Beantwortung dieser Frage muss die OpenStack-Dokumentation (http://docs.openstack.org/admin-guide/common/nova-show-usage-statistics-for-hosts-instances.html) herangezogen werden.

      • In der Spalte disk_gb [der Ausgabe von openstack host show] wird die Summe der Größen von Root- und ephemeren Platten (in GB) der auf dem Host ausgeführten Instanzen angezeigt. [Dieser Wert] wird anhand der Informationen zu den Varianten der auf den Hosts ausgeführten Instanzen berechnet. Dieser Befehl fragt nicht die Festplattennutzung des physischen Hosts ab.
      • Die tatsächliche Variantenberechnung finden Sie unter https://github.com/openstack/nova/blob/c521ba7fa679b6e9c790d84ea22f04fb999987db/nova/notifications/base.py#L448-L454 disk_gb = root_gb + ephemeral_gb.
      • ephemeral_gb = Gibt die Größe einer sekundären ephemeren Datenplatte an. Hierbei handelt es sich um eine leere, nicht formatierte Platte, die nur so lange existiert wie die Instanz. Der Standardwert ist 0.
      • root_gb = Gibt den für die Rootpartition (/) zu verwendenden Plattenspeicherplatz (in GB) an.
      • Die Spalte disk_gb für Nova enthält momentan keine Informationen zum Auslagerungsspeicher. (Das bedeutet, dass die Zuordnung von Auslagerungsspeicher für die Instanz optional ist. Der Standardwert ist 0.)

      Es gibt Arbeitsvorschläge für das OpenStack-Release Newton, die den Einschluss dieses optionalen Auslagerungsspeichers gemäß der Spezifikation https://specs.openstack.org/openstack/nova-specs/specs/newton/implemented/resource-providers-allocations.html beinhalten:

      “Wenn der Rechenknoten lokalen Speicher für Instanzplatten verwendet oder wenn der Rechenknoten über den Datenträger gebootet wurde, muss der verwendete Wert die Summe der Werte aus den Feldern root_gb, ephemeral_gb und swap für die Variante sein. Das Feld resource_provider_uuid muss die UUID des Rechenknotens enthalten. Für Instanzen, die über den Datenträger gebootet wurden, hat root_gb den Wert 0.”

      Wie kann ich einen Datenträger aus einer Momentaufnahme erstellen und Abhängigkeiten entfernen?

      Wenn Sie einen Datenträger aus einer Momentaufnahme erstellen, kann die ursprüngliche Momentaufnahme abhängige (Meta-)Daten enthalten, die der Datenträger benötigt. Mit dem Befehl openstack volume show $volumeid können Sie die ID der übergeordneten Momentaufnahme ermitteln. Wenn Sie versuchen, den Datenträger zu löschen, kann ein Fehler auftreten, weil die abhängige Momentaufnahme zuerst gelöscht werden muss:

      Failed to delete snapshot with name or ID ...

      Sie können dieses Problem umgehen, indem Sie ein Image aus dem Datenträger und dann einen Datenträger aus diesem Image erstellen:

      Schritt 1: Temporäres Image aus einem Datenträger erstellen:

      openstack image create --volume $volumeid $image_name

      Schritt 2: Datenträger aus dem temporären Image erstellen:

      openstack volume create --image $imageID --size $x

      Anschließend können Sie das temporäre Image löschen und Sie haben einen unabhängigen Datenträger.
