      Benutzerhandbuch für Cloud-Images (IBM Bluemix Private Cloud with Red Hat)

      • by Ying Tang, Xiong Chen ● 31. März 2017

      • Tags:

        • userguides

        • Benutzerhandbücher

        • Images

        • Glance

        • image-create

        • Instanz

        • rhosp

        • Newton

      Im Lieferumfang von IBM Bluemix Private Cloud with Red Hat sind vorab mit Cirros 0.3.3 x86_64 konfigurierte Images enthalten.

      Wenn Sie weitere Informationen zu den von IBM bereitgestellten Images benötigen, lesen Sie die Releaseinformationen für Cloud-Images.

      Wenn Sie mit Red Hat-Gastimages arbeiten möchten, können Sie die Imagedateien über die folgenden URLs herunterladen:

      • Red Hat Enterprise Linux 7 KVM Guest Image ( https://access.redhat.com/downloads/content/69/ver=/rhel—7/x86_64/product-downloads)
      • Red Hat Enterprise Linux 6 KVM Guest Image

      Weitere Informationen zur Verwendung von Images mit OpenStack sind auf der OpenStack-Website verfügbar: https://docs.openstack.org/image-guide/

      Voraussetzungen

      Wenn Sie beabsichtigen, die Befehlszeilentools zu verwenden, vergewissern Sie sich, dass der OpenStack Glance-Client installiert ist. Installationsanweisungen finden Sie in der Knowledge Base unter Einführung in die OpenStack-API.

      Beim aktuellen Release von IBM Bluemix Private Cloud with Red Hat können Sie eine Imagedatei über die Befehlszeile oder über das Horizon-Dashboard hochladen. Das Hochladen über eine Imageposition oder URL wird nicht unterstützt.

      Alternative, von IBM Bluemix Private Cloud bereitgestellte, Cloud-Images auswählen

      Im Lieferumfang des aktuellen Release von IBM Bluemix Private Cloud with Red Hat sind vorab mit Cirros 0.3.3 x86_64 konfigurierte Images enthalten. Es sind jedoch weitere Cloud-Images verfügbar, die IBM Bluemix Private Cloud zu Ihrem Komfort bereitstellt.

      • CentOS 6.8 x86_64
      • CentOS 7.3 x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Ubuntu Server 16.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      Image in das Glance-Repository hochladen

      Sie können separat gekaufte Betriebssystemsoftware hochladen und installieren, z. B. Red Hat Enterprise Linux 7 KVM Guest Image, oder Sie können eines der folgenden Betriebssystemimages bereitstellen, das IBM Ihnen als VM-Instanz zur Verfügung stellt. Sie sind dafür verantwortlich, alle entsprechenden Lizenzbedingungen, die für das Betriebssystem gelten, einzuhalten, und die ordnungsgemäßen Berechtigungen für jede VM-Instanz anzufordern.

      Von IBM bereitgestellte Cloud-Image-Downloads von Box Panel abrufen

      1. Melden Sie sich bei Box Panel mit den Box Panel-Berechtigungsnachweisen an. Die Seite “Cloud-Images” in Box Panel steht Ihnen zur Verfügung, wenn Sie mindestens eine Cloud haben und entweder ein primärer oder ein technischer Kundenkontakt sind.

      2. Klicken Sie im Box Panel-Dashboard auf “Services” und wählen Sie den Eintrag “Cloud-Images” aus.

      3. Daraufhin wird die Seite “Cloud-Images” angezeigt. Jedes Image wird als Karte angezeigt.

      4. Klicken Sie auf die Karte, die das Image anzeigt, für das Sie sich interessieren. Es wird ein Modalfenster geöffnet, das die temporäre URL (tempURL) enthält.

      5. Die angezeigte temporäre URL (tempURL) ist 24 Stunden lang gültig, um das Cloud-Image herunterzuladen.

      6. Stellen Sie sicher, dass das tempUrl-Token mit Escapezeichen versehen oder in Anführungszeichen gesetzt ist. Beispiel:

          curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
  curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      7. Laden Sie die Dateien .qcow2 und .md5sum für das Image herunter.

      8. Laden Sie die .qcow2-Imagedatei und die .md5sum-Kontrollsummendatei über die tempURL herunter und speichern Sie diese beiden Dateien in demselben Ordner. Beispiel:

          ubuntu-guest-image-14.04-20160301-x86_64.qcow2
  ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      9. Überprüfen Sie die heruntergeladene .qcow2-Imagedatei mit der .md5sum-Kontrollsummendatei. Beispiel:

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

      10. Wenn als Ergebnis “OK” zurückgegeben wird, ist das heruntergeladene qcow2-Image gültig.

      Image über die Befehlszeile hochladen

      Sie können Images über den Glance-Client hochladen. Weitere Informationen finden Sie unter OpenStack-Clients installieren.

      Erstellen Sie ein Image.

      • Für Linux-Images: -- min-disk 5 --min-ram 512

      • Für Windows-Images: --min-disk 30 --min-ram 2048

        $ glance image-create --os-version <os_version> --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --file <image_file>

      Überprüfen Sie, ob das Image erfolgreich erstellt wurde. Das Image wird für den Upload in eine Warteschlange gestellt. Es kann eine Weile dauern, bis sich der Status von “Queued” in “Active” ändert.

      $ glance image-show <image-id>

      Weitere Informationen zu diesem Befehl finden Sie unter Glance-Befehlszeilenclient.

      Image über das OpenStack-Dashboard hochladen

      1. Melden Sie sich am OpenStack-Dashboard von IBM Bluemix Private Cloud an.

      2. Erweitern Sie in der Anzeige Project den Eintrag Compute und klicken Sie auf Images.

      3. Klicken Sie auf der Seite auf Create Image. Geben Sie auf der neuen Seite die folgenden Parameter an und wählen Sie Create Image aus.

        Element Beschreibung
        Name Geben Sie für das Image einen Namen ein.
        Description Geben Sie optional eine kurze Beschreibung des Image ein.
        File Wählen Sie den Eintrag Imagedatei aus.
        Format Wählen Sie das richtige Format für das Image aus (z. B. QCOW2).
        Kernel Geben Sie optional die Kernel-ID des in Glance gespeicherten Image an, das als Kernel verwendet werden soll, wenn ein Image des Typs AMI gebootet wird.
        Ramdisk Geben Sie optional die ID des in Glance gespeicherten Image an, das als Kernel verwendet werden soll, wenn ein Image des Typs AMI gebootet wird.
        Architecture Geben Sie optional die Architektur an (z. B. i386 für eine 32-Bit-Architektur oder x86_64 für eine 64-Bit-Architektur).
        Minimum Disk(GB) Geben Sie für ein Linux-Image 5 und für ein Windows-Image 30 an.
        Minimum RAM(MB) Geben Sie für ein Linux-Image 512 und für ein Windows-Image 2048 an.
        Public Wählen Sie dieses Kontrollkästchen aus, um das Image allen Benutzern in allen Projekten öffentlich zugänglich zu machen.
        Protected Wählen Sie dieses Kontrollkästchen aus, um sicherzustellen, dass nur berechtigte Benutzer das Image löschen können.

        Beispiel:

         Name: Ubuntu Server 14.04 LTS x86_64 - 20160301	   Description: Ubuntu Server 14.04 LTS x86_64 cloud image released on 2016/03/01		
 File: C:\Documents\ubuntu-guest-image-14.04-20160301-x86_64.qcow2	
 Format: QCOW2 - QEMU Emulator	
 Architecture: x86_64	
 Minimum Disk(GB): 5	
 Minimum RAM(MB): 512	
 Public: True

      4. Überprüfen Sie, ob das erstellte Image in der Imageliste angezeigt wird. Das Image wird für den Upload in eine Warteschlange gestellt. Es kann eine Weile dauern, bis sich der Status von “Queued” in “Active” ändert.
