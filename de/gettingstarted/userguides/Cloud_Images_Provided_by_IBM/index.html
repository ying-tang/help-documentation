      Benutzerhandbuch für Cloud-Images (Community-Edition)

      • by Yanqi Dou ● 9. März 2016

      • Tags:

        • userguides

        • Benutzerhandbücher

        • Images

        • Glance

        • image-create

        • Instanz

      Im Lieferumfang von IBM Bluemix Private Cloud 3.0 sind vorab mit Cirros 0.3.3 x86_64 konfigurierte Images enthalten. IBM Bluemix Private Cloud stellt weitere für den Download verfügbare Images bereit, damit Sie Instanzen mit einem Betriebssystem Ihrer Wahl ausführen können. In diesem Dokument erfahren Sie, wie Sie diese Images herunterladen und mit ihnen arbeiten. Wenn Sie sicherstellen möchten, dass Sie mit den aktuellsten Images arbeiten, können Sie jederzeit die Releaseinformationen für Cloud-Images abrufen.

      Weitere Informationen zur Verwendung von Images mit OpenStack sind auf der OpenStack-Website verfügbar: https://docs.openstack.org/image-guide/

      Voraussetzungen

      1. Wenn Sie beabsichtigen, die Befehlszeilentools zu verwenden, vergewissern Sie sich, dass der OpenStack Glance-Client installiert ist. Installationsanweisungen finden Sie in der Knowledge Base unter Einführung in die OpenStack-API.

      2. Stellen Sie sicher, dass Sie auf Box Panel zugreifen können, um die temporären Download-URLs für die von IBM bereitgestellten Cloud-Images abzurufen.

      3. Das Hochladen eines Image in Glance mit tempUrl als Imageposition wird nur unter Bluemix Private Cloud 2.1.0 und höher unterstüzt.

      Grundlegende Konzepte

      Element Beschreibung
      Image Ein Image einer virtuellen Maschine, in diesem Dokument einfach als Image bezeichnet, ist eine einzelne Datei, die eine virtuelle Platte mit einem bootfähigen Betriebssystem enthält. Verwenden Sie Images, um Instanzen einer virtuellen Maschine in der Cloud zu erstellen.
      Instanz Instanzen sind virtuelle Maschinen (VMs), die in der OpenStack-Cloud ausgeführt werden.
      Flavor Der Begriff Flavor beschreibt die Größe einer Instanz und gibt die vordefinierten Ressourcenspezifikationen für die einer Instanz zur Verfügung stehende Größe der vCPU und des RAM sowie den verfügbaren Plattenspeicher an.

      Alternative von IBM Bluemix Private Cloud bereitgestellte Cloud-Images auswählen

      Im Lieferumfang des aktuellen Release von IBM Bluemix Private Cloud sind vorab mit Cirros 0.3.3 x86_64 konfigurierte Images enthalten. Es sind jedoch weitere Cloud-Images verfügbar, die IBM Bluemix Private Cloud zu Ihrem Komfort bereitstellt.

      Sie können separat gekaufte Betriebssystemsoftware hochladen und installieren, oder Sie können eines der folgenden Betriebssystemimages bereitstellen, das IBM Ihnen als VM-Instanz zur Verfügung stellt. Sie sind dafür verantwortlich, alle entsprechenden Lizenzbedingungen, die für das Betriebssystem gelten, einzuhalten, und die ordnungsgemäßen Berechtigungen für jede VM-Instanz anzufordern.

      • CentOS 6.8 x86_64
      • CentOS 7.2 x86_64
      • Ubuntu Server 12.04 LTS x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      Für alle VM-Instanzen mit einem beliebigen Microsoft Windows-Betriebssystem gelten die folgenden bestimmten Kundenbedingungen: Der Kunde muss einen eigenen Microsoft Windows-Lizenzserver besitzen und verwalten, um diese VM-Instanzen mit Windows-Betriebssystemen aktivieren zu können. Weitere Informationen finden Sie unter Vorgehensweise beim Aktivieren von Windows-Lizenzen IBM Bluemix Private Cloud.

      Cloud-Image-Downloads von Box Panel abrufen

      1. Melden Sie sich bei Box Panel mit den Box Panel-Berechtigungsnachweisen an. Die Seite “Cloud-Images” in Box Panel steht Ihnen zur Verfügung, wenn Sie mindestens eine Cloud haben und entweder ein primärer oder ein technischer Kundenkontakt sind.

      2. Klicken Sie im Box Panel-Dashboard auf Services und wählen Sie den Eintrag Cloud-Images aus.

      3. Daraufhin wird die Seite “Cloud-Images” angezeigt. Jedes Image wird als Karte angezeigt.

      4. Klicken Sie auf die Karte, die das Image anzeigt, für das Sie sich interessieren. Es wird ein Modalfenster geöffnet, das die temporäre URL (tempURL) enthält.

      5. Die angezeigte temporäre URL (tempURL) ist 24 Stunden lang gültig, um das Cloud-Image herunterzuladen.

      Image in das IBM Bluemix Private Cloud Glance-Repository mit Ihrer Cloud-Image-tempURL hochladen

      Anmerkungen:

      • Nur Benutzer mit der Rolle cloud_admin können für andere Benutzer und für das Projekt ein Image veröffentlichen.
      • Das direkte Hochladen eines Image in Glance mit tempUrl als Imageposition kann nicht verwendet werden, wenn bei IBM Bluemix Private Cloud der abgehende Internetzugriff über die OpenStack-Steuerebene blockiert ist.
      • Das direkte Hochladen eines Image in Glance mit tempUrl als Imageposition wird nur in IBM Bluemix Private Cloud 2.1.0 und höher unterstützt, und nicht in Releases vor Version 2.1.0.
      • Befolgen Sie bei IBM Bluemix Private Cloud für Releases vor 2.1 oder ohne abgehenden Internetzugriff die Anweisungen unter Image in ein IBM Bluemix Private Cloud-Glance-Repository mithilfe heruntergeladener Imagedateien hochladen.

      Sie können entweder ein Image über das OpenStack-Dashboard oder über den Befehlszeilenclient hochladen.

      Image über das OpenStack-Dashboard hochladen

      1. Melden Sie sich am OpenStack-Dashboard von IBM Bluemix Private Cloud an.
      2. Erweitern Sie in der Anzeige Project den Eintrag Compute und klicken Sie auf Images.

      3. Klicken Sie auf der Seite auf Create Image. Geben Sie auf der neuen Seite die folgenden Parameter an und klicken Sie auf Create Image.

        Element Beschreibung
        Name Geben Sie für das Image einen Namen ein.
        Description Geben Sie optional eine kurze Beschreibung des Image ein.
        Image Source Wählen Sie eine Position für das Image aus.
        Image Location Geben Sie die tempURL für den Cloud-Image-Download an.
        Format Wählen Sie das richtige Format für das Image aus (z. B. QCOW2).
        Architecture Geben Sie die Architektur an (z. B. i386 für eine 32-Bit-Architektur oder x86_64 für eine 64-Bit-Architektur).
        Minimum Disk(GB) Geben Sie für ein Linux-Image 5 und für ein Windows-Image 30 an.
        Minimum RAM(MB) Geben Sie für ein Linux-Image 512 und für ein Windows-Image 2048 an.
        Copy Data Diese Option muss ausgewählt sein, um Imagedaten in den Image-Service zu kopieren.
        Public Wählen Sie dieses Kontrollkästchen aus, um das Image allen Benutzern in allen Projekten bereitzustellen.
        Protected Wählen Sie dieses Kontrollkästchen aus, um sicherzustellen, dass nur berechtigte Benutzer das Image löschen können.

        Beispiel:

         Name: Ubuntu Server 14.04 LTS x86_64 - 20160301
 Description: Ubuntu Server 14.04 LTS x86_64 cloud image released on 2016/03/01
 Image Source: Image Location
 Image File: <tempURL_for_QCOW2>
 Format: QCOW2 - QEMU Emulator
 Architecture: x86_64
 Minimum Disk(GB): 5
 Minimum RAM(MB): 512
 Copy Data: True
 Public: True
      4. Überprüfen Sie, dass das erstellte Image in der Imageliste angezeigt wird. Das Image wird für den Upload in eine Warteschlange gestellt. Es kann eine Weile dauern, bis sich der Status von “Queued” in “Active” ändert.

      **Anmerkung: ** Nur Benutzer mit der Rolle cloud_admin können anderen Benutzern und dem Projekt ein Image öffentlich zugänglich zu machen.

      Image mit dem OpenStack-Befehlszeilenclient hochladen

      Sie können Images über den Glance-Client hochladen. Weitere Informationen finden Sie unter OpenStack-Clients installieren.

      Erstellen Sie ein Image.

      • Für Linux-Images: -- min-disk 5 --min-ram 512

      • Für Windows-Images: --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --copy-from  <tempURL_to_qcow2_image_file>

      Überprüfen Sie, ob das Image erfolgreich erstellt wurde. Das Image wird für den Upload in eine Warteschlange gestellt. Es kann eine Weile dauern, bis sich der Status von “Queued” in “Active” ändert.

      $ glance image-show <image-id>

      Image in ein IBM Bluemix Private Cloud-Glance-Repository mithilfe heruntergeladener Imagedateien hochladen

      Anmerkungen:

      • Nur Benutzer mit der Rolle cloud_admin können anderen Benutzern und dem Projekt ein Image zur Verfügung stellen.

      • Stellen Sie beim Herunterladen des Cloud-Image über die tempUrl sicher, dass das tempUrl-Token mit Escapezeichen versehen oder in Anführungszeichen gesetzt ist. Beispiel:

        curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      Führen Sie die folgenden Schritte aus:

      1. Laden Sie die Dateien .qcow2 und .md5sum für das Image herunter.

      2. Laden Sie die Imagedatei .qcow2 und die Kontrollsummendatei .md5sum über die tempURL herunter und speichern Sie diese beiden Dateien in demselben Ordner. Beispiel:

         ubuntu-guest-image-14.04-20160301-x86_64.qcow2
 ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      3. Überprüfen Sie die heruntergeladene Imagedatei .qcow2 mit der Kontrollsummendatei .md5sum. Beispiel:

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

        Wenn als Ergebnis “OK” zurückgegeben wird, ist das heruntergeladene Image qcow2 gültig.

      4. Laden Sie das Image in Glance hoch. Sie können das Image entweder über das OpenStack-Dashboard oder über den Befehlszeilenclient hochladen.

      Image über den OpenStack-Befehlszeilenclient hochladen

      Weitere Informationen finden Sie unter OpenStack-Clients installieren.

      Erstellen Sie ein Image.

      • Für Linux-Images: -- min-disk 5 --min-ram 512

      • Für Windows-Images: --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --file <path_to_qcow2_image_file>

      Überprüfen Sie, ob das Image erfolgreich erstellt wurde. Das Image wird für den Upload in eine Warteschlange gestellt. Es kann eine Weile dauern, bis sich der Status von “Queued” in “Active” ändert.

      Anmerkung: Bei großen Images kann beim Web-Upload das zulässige Zeitlimit überschritten werden. In solchen Fällen wird empfohlen, dass Sie eine Instanz in Ihrer Cloud starten, das Image in diese Instanz herunterladen und es anschließend von dieser Instanz in Glance hochladen. Der gesamte Datenverkehr bleibt in SoftLayer und der Upload findet in der eigenen Cloud statt.

      2. Erweitern Sie die Anzeige Project, klicken Sie anschließend im Navigationsbereich links auf das Register Compute und klicken Sie dann auf Instances.

      3. Suchen Sie Ihre Instanz in der Liste der aktiven Instanzen und notieren Sie die zugehörige IP-Adresse, wie z. B. 192.168.100.237.

      4. Stellen Sie eine Verbindung zu Ihrer Instanz her, indem Sie die zugehörige IP-Adresse angeben.

      Verbindung zu Linux-Instanzen herstellen

      Stellen Sie über SSH oder VNC eine Verbindung zu Ihrer Instanz her. Die Standardbenutzer-ID für die von IBM bereitgestellten Linux-Images ist ibmcloud. Sobald Sie eine Verbindung zu Ihrer Linux-Instanz hergestellt haben, können Sie den Befehl sudo verwenden, um Befehle auszuführen, für die in der Regel Rootzugriff erforderlich ist. Sudo ist so konfiguriert, dass kein Kennwort für die Standardbenutzer-ID erforderlich ist.

      • Verwenden Sie den von Ihnen bevorzugten SSH-Client und stellen Sie eine Verbindung zur richtigen IP-Adresse der VM-Instanz her, indem Sie den privaten SSH-Schlüssel des Schlüsselpaars verwenden, den Sie bei der Bereitstellung angegeben haben:

          ssh -i <Pfad_Ihrer_privaten_SSH-Schlüsseldatei> <Benutzer-ID>@<IP-Adresse_der_Instanz>

      • Zeigen Sie die VNC-Konsole der Instanz an, indem Sie auf den Instanznamen im OpenStack-Dashboard von IBM Bluemix Private Cloud klicken und anschließend auf das Register Console klicken. Die VNC-Konsole stellt HTTPS-Verbindungen her.

      Anmerkung: Damit Sie sich über die Konsole an einer neuen Linux-Instanz als Benutzer ibmcloud anmelden können, müssen Sie während der Bereitstellung im Anpassungsscript ein Kennwort angegeben haben.

      Verbindung zu Windows-Instanzen herstellen

      Stellen Sie über Remote Desktop oder VNC eine Verbindung zu Ihrer Instanz her. Sie müssen die während der VM-Bereitstellung im Anpassungsscript angegebene Benutzer-ID und das Kennwort verwenden. Bei Ihrer ersten Anmeldung erscheint eine Aufforderung zum Ändern Ihres Kennworts. Ändern Sie Ihr daraufhin Ihr Kennwort.

      • Verwenden Sie den von Ihnen bevorzugten Remote-Desktop-Client, um eine Verbindung herzustellen.
      • Zeigen Sie die VNC-Konsole der Instanz an, indem Sie auf den Instanznamen im OpenStack-Dashboard von IBM Bluemix Private Cloud OpenStack klicken und anschließend auf das Register Console klicken. Die VNC-Konsole stellt HTTPS-Verbindungen her.

      Image aktualisieren

      Wenn Sie eine Benachrichtigung vom IBM Bluemix Private Cloud-Operations-Team erhalten, in der Sie darüber informiert werden, dass die Cloud-Images aktualisiert werden, können Sie das vorhandene Image inaktivieren und ein neues Image neu erstellen, indem Sie die Download-URL verwenden, die Sie über Box Panel für das aktualisierte Image abgerufen haben.

      Verwenden Sie den OpenStack-Befehlszeilenclient, um das vorhandene Image zu inaktivieren.

      1. Rufen Sie die vorhandene Image-ID wie folgt ab:

         $ glance image-list

      2. Inaktivieren Sie das Image wie folgt:

        $ glance image-deactivate <Image-ID>

      3. Zeigen Sie den Imagestatus wie folgt an:

        $ glance image-show <Image-ID>

      4. Nachdem Sie das vorhandene Image inaktiviert haben, erstellen Sie das neue Image neu, indem Sie die Schritte in diesem Handbuch unter Image in das IBM Bluemix Private Cloud-Glance-Repository mit Ihrer Cloud-Image-tempURL hochladen ausführen.

      Support für Cloud-Images vom IBM Bluemix Private Cloud-Operations-Team anfordern

      Wenn Sie für ein bestimmtes Cloud-Image Support vom IBM Bluemix Private Cloud-Operations-Team anfordern möchten, ist es hilfreich, wenn Sie die Build-ID des Image angeben können. Rufen Sie die Build-ID des Image ab und erstellen Sie anschließend ein Support-Ticket beim IBM Bluemix Private Cloud-Operations-Team und geben Sie hierbei die Build-ID an. Sie finden die Build-ID wie folgt:

      • Aus dem Namen der Imagedatei:
        Beispiel: ubuntu-guest-image-14.04-20160301-x86_64.qcow2

      • Aus der Build-ID-Datei, in der aus dem Cloud-Image bereitgestellten VM:

        • Für Linux-Instanzen: Die Build-ID-Datei befindet sich im Verzeichnis /etc/buildID.

        • Für Windows-Instanzen: Die Build-ID-Datei befindet sich im Verzeichnis C:\buildID.
