      IBM Bluemix Private Cloud Customer

      Releaseinformationen

      Release 2.1.0 vom 31. März 2016

      Zusammenfassung der neuen Features im aktuellen Release

      Dieses Dokument enthält eine grundlegende technische Beschreibung der neuesten IBM Bluemix Private Cloud-Angebote, einschließlich der technischen Daten für die Angebote Dedicated und Local. Zu den neuen Funktionen von IBM Bluemix Private Dedicated und Local Cloud gehören die folgenden:

      1. Es sind weitere Typen von Erweiterungsrechenknoten verfügbar.

      2. Es sind Hybridblockspeichercluster und Erweiterungsknoten verfügbar.

      3. Direct Link ist in IBM Bluemix Private Dedicated Cloud in drei Varianten und für Hybridcloudanwendungen verfügbar.

      4. Der Zugriff auf das private Netz des IBM Cloud-Rechenzentrums ist verfügbar.

      5. Verwendung der rollenbasierten Zugriffssteuerung (RBAC, Role-Based Access Control) für die Definition von Berechtigungen.

      6. Es werden weitere Cloud-Images von IBM Bluemix bereitgestellt.

      7. Es sind jetzt HIPAA-kompatible Optionen für IBM Bluemix Private Dedicated Cloud verfügbar.

      Informationen zu weiteren Benutzerdokumentationen finden Sie auf der Haupthilfetextseite.

      In diesem Dokument ist eine allgemeine Produktübersicht für IBM Bluemix Private Cloud Version 2.0 und höher verfügbar..

      Eine Zusammenfassung der Features aus früheren Releases, die sich auf Benutzer dieses Release auswirken, finden Sie am Ende dieses Dokuments.

      Bausteine von IBM Bluemix Private Cloud

      Jede IBM Bluemix Private Cloud 2.1.0-Instanz, dediziert oder lokal, wird mit einem Cluster von Controller- und Rechenknoten gestartet:

      IBM Bluemix Private Cloud:

      • Eine Cloud kann als Cluster mit drei Knoten und zwei hyperkonvergierten Knoten (Controller- und Rechenknoten) und einem reinen Rechenknoten gestartet werden.  • Eine Cloud kann als Cluster mit fünf Knoten und einem Paar dedizierter reiner Contollerknoten und drei reinen Rechenknoten gestartet werden.

      Lokale IBM Bluemix Private Cloud-Instanz:

      • Jede Cloud muss mindestens einen dedizierten reinen Controllerknoten und drei reine Rechenknoten haben.

      Ausgehend von jeder dieser Ausgangssituationen (z. B. 3 Knoten mit Hyperkonvergenz oder 5 Knoten mit dedizierten Controllern) können Sie Rechenknoten beliebigen Typs zusammen mit Blockspeicher (Ceph) und/oder Objektspeicher (Swift) hinzufügen.

      Weitere Bausteine:

      • In eine dedizierte Cloud mit Standardkapazität werden standardmäßig dedizierte Gateway-Firewalls eingeschlossen.  • Eine Cloud mit unternehmensorientierter Kapazität bietet das Doppelte an Ressourcen und 10 Gb/s Netzkonnektivität.  • Dedizierte Controllerknoten (oder zusätzliche Controllerknoten)  • Rechenknoten (oder zusätzliche Knoten): Standard- oder Unternehmensknoten, E/A-Knoten oder Knoten mit hohem Speicher (neu)

      Optionale Erweiterungsrechenknoten sind jetzt verfügbar

      Optional können Sie die folgenden Rechenknotenerweiterungen zu Ihrer Standard- oder Unternehmenscloud hinzufügen:

      • Sie können spezielle Erweiterungsrechenknoten auswählen: IO oder HIGH STORAGE (neu in diesem Release)  • Sie können weitere Standard- oder Unternehmensrechenknoten auswählen: STANDARD COMPUTE oder ENTERPRISE COMPUTE

      Erweiterungsrechenknoten erfüllen Ihre Anforderungen in Bezug auf schnelleren ephemeren E/A-Speicher, höhere Arbeitsspeicherkapazität und um Größenordnungen mehr ephemeren Speicher. Beide Typen von Erweiterungsknoten, d. h. für E/A optimierte Knoten und Knoten mit hoher Speicherkapazität, ermöglichen eine bessere Abdeckung von Instanztypen und Workloads. Ab dem Release 2.1.0 sind Rechentypen zur einfacheren Zuordnung von Workloads zum entsprechenden Rechentyp in Verfügbarkeitszonen gruppiert.

      Optionale Hybridblockspeicherknoten sind jetzt verfügbar

      Hybridblockspeicherknoten bieten Blockspeicher zu geringeren Kosten und mit höherer Dichte.

      • Sie können Blockspeicher auswählen: SSD oder HYBRID.  • Sie können Objektspeicher auswählen.

      Direct Link ist jetzt in IBM Cloud-Rechenzentren verfügbar

      Für Dedicated Cloud steht Ihnen die zusätzliche Netzoption der Direct Link-Konnektivität zur Verfügung:

      • Sie können eine Zugriffsmethode auswählen (Public, VPN oder Direct Link)

      Direct Link ist in IBM Cloud-Rechenzentren verfügbar. Mit diesem Feature können Sie eine Verbindung zum Backbone des privaten IBM Cloud-Rechenzentrums oder zum Netz herstellen können, indem Sie das öffentliche Internet bei der Verbindungsherstellung zu Ihren dedizierten Clouds umgehen.

      Direct Link kann verwendet werden, um eine Hybrid-Cloud zu erstellen, wenn Sie Ihre lokale Cloud mit einer dedizierten Cloud verbinden möchten, die in einem IBM Cloud-Rechenzentrum ausgeführt wird. Es ist in 1-Gb/s- und 10-Gb/s-Inkrementen verfügbar.

      Verfügbar:

      • IBM Bluemix Private Direct Link Cloud Exchange 1 Gb/s
• IBM Bluemix Private Direct Link Cloud Exchange 10 Gb/s
• IBM Bluemix Private Direct Link Colocation 1 Gb/s
• IBM Bluemix Private Direct Link Colocation 10 Gb/s
• IBM Bluemix Private Direct Link Network Service Provider (NSP) 1 Gb/s
• IBM Bluemix Private Direct Link Network Service Provider (NSP) 10 Gb/s

      Zugriff auf das private Netz des IBM Cloud-Rechenzentrums

      Über den Zugriff auf das private Netz des IBM Cloud-Rechenzentrums können Sie die Konnektivität zwischen zwei IBM Bluemix Private Cloud-Instanzen über die private Netzfunktion des IBM Cloud-Rechenzentrums einrichten, wenn die beiden zu verbindenden IBM Bluemix Private Cloud-Instanzen unter demselben IBM Cloud-Konto bereitgestellt werden. Sie wählen diesen Service aus, um unbegrenzten privaten Netzverkehr mit geringen Latenzzeiten zwischen IBM Bluemix Private Cloud-Instanzen zu ermöglichen.

      Weitere Informationen finden Sie im Hauptdokument zu diesem Thema.

      Eine bekannte Einschränkung ist hier beschrieben.

      Rollenbasierte Zugriffssteuerung (RBAC, Role-Based Access Control)

      Es werden standardmäßig mehrere Rollen erstellt, die Ihnen bestimmte Zugriffsberechtigungen in Ihrer Cloud erteilen. In dieser Version von IBM Bluemix Private Cloud wird das Konzept von Gruppen eingeführt. Weitere Informationen zu Benutzern, Rollen und Gruppen finden Sie in diesem Dokument.

      Weitere Cloud-Images sind verfügbar

      Im Lieferumfang von IBM Bluemix Private Cloud 2.1.0 sind vorab mit Cirros 0.3.3 x86_64 konfigurierte Images enthalten. Es sind jedoch weitere Cloud-Images verfügbar, die IBM Bluemix zu Ihrem Komfort bereitstellt, wie z. B.:

      • CentOS 6.7 x86_64
• CentOS 7.2 x86_64
• Ubuntu Server 12.04 LTS x86_64
• Ubuntu Server 14.04 LTS x86_64
• Windows Server 2008 R2 SP1 Datacenter
• Windows Server 2008 R2 SP1 Enterprise
• Windows Server 2008 R2 SP1 Standard
• Windows Server 2012 R2 Datacenter
• Windows Server 2012 R2 Standard

      Wenn Sie eines dieser Images verwenden möchten, finden Sie weitere Informationen in diesem Dokument.

      HIPAA und IBM Bluemix Private Cloud

      • Sie können die HIPAA-Aktivierung für Ihre IBM Bluemix Private Cloud-Instanz auswählen

      Eine Cloud, in der HIPAA aktiviert ist, hat dieselben Konfigurations- und Erweiterungsoptionen wie jedes andere IBM Bluemix Private Cloud-Angebot, aber wir aktivieren über 50 weitere Steuerelemente für die Informationssicherheit, die unsere Kunden unterstützen, die persönliche Gesundheitsdaten speichern müssen. Aufgrund dieser erhöhten Informationssicherheit muss eine Cloud bereits während der Erstellung für die HIPAA-Unterstützung aktiviert werden. Es ist nicht möglich, ein Upgrade einer Cloud auf das HIPAA-Protokoll durchzuführen. Das Downgrade einer IBM Bluemix Private Cloud mit HIPAA ist ebenfalls nicht möglich. Die vorhandene HIPAA-Cloud muss gemäß HIPAA-Protokoll bereinigt werden, und Sie müssen eine neue Cloud mit Controllern ohne HIPAA-Unterstützung bestellen.

      Im Wesentlichen gleicht eine IBM Bluemix Private Cloud-Instanz mit HIPAA jeder anderen IBM Bluemix Private Cloud-Instanz. Die Ausnahmen sind im Folgenden beschrieben:

      1.  Jede IBM Bluemix Private Cloud-Instanz mit HIPAA muss dedizierte Controller (keine konvergierten Controller) haben. 
2.  Pro Cluster und pro Asset, das verwaltet wird, fallen für HIPAA zusätzliche wiederkehrende monatliche Gebühren an. 
3.  Es werden zusätzliche (50+) Steuerelemente aktiviert und die Zuständigkeit für diese Steuerelemente teilen Sie sich mit IBM Blue Box.

      IBM Bluemix Private Cloud-Instanzen mit HIPAA können in jedem IBM Cloud-Rechenzentrum bereitgestellt werden, in dem IBM Bluemix Private Cloud-Instanzen bereitgestellt werden können.

      Voraussetzung: Bevor eine Bestellung einer IBM Bluemix Private Cloud-Instanz mit HIPAA abgesetzt werden kann, muss ein BAA unterzeichnet werden. Sie müssen Ihre Cloudbestellung mit der Mindestkonfiguration für HIPAA-Kunden absetzen. Nach der Serviceinitialisierung kann HIPAA nicht mehr aktiviert werden.

      Zusammenfassung der neuen Features aus führen Releases, die sich auf Kunden dieses Release auswirken

      In diesem Abschnitt sind die Features aus früheren IBM Bluemix Private Cloud-Releases, hauptsächlich aus dem Release 2.0.1, aufgelistet, die sich auf Ihr Erlebnis mit diesem 2.1.0-Release auswirken, wenn Sie ein Upgrade von einem 1.x-Release durchführen. Weitere Informationen zu diesen Features erhalten Sie von unserem IBM Bluemix Private Cloud-Support-Team.

      Features, die sich auf Benutzer, Gruppen und Rollen in diesem Release auswirken:

      Ausblenden der Anzeige Host Aggregates für ‘cloud_admin’: Diese Horizon-Anzeige wurde aus der Sichtbarkeit von cloud_admin-Benutzern entfernt. Es ist nicht mehr zulässig, dass cloud_admin-Benutzer Hostaggregate bearbeiten, und deshalb war diese Anzeige in Horizon nicht mehr angebracht.

      Unterbinden von Verwaltungsoperationen für Hostaggregate durch ‘cloud_admin’: Siehe die Beschreibung zur Anzeige Host Aggregates für Horizon.

      Bereitstellung weiterer ‘admin’-Anzeigen für ‘cloud_admin’: Es wurden weitere Verwaltungsanzeigen für die cloud_admin-Benutzer in Horizon aktiviert. Dazu gehören “Overview,” “Hypervisors,” “Instances,” “Volumes,” “Flavors,” “Images” und “Defaults”.

      Korrektur des Benutzermanagements nach Namen in der Befehlszeile: Ein Fehler, der das Management von Benutzern nach Namen über die Befehlszeile verhindert hat, wurde behoben. In den früheren Releases haben viele Versuche, einen Benutzer anhand seines Namens zu referenzieren, zu einem Fehler geführt.

      Veröffentlichung von Glance-Images durch ‘cloud_admin’: Die Rolle cloud_admin hat jetzt die erforderlichen Berechtigungen zur Veröffentlichung von Glance-Images.

      Features, die sich auf Horizon, Glance, Nova und Heat in diesem Release auswirken:

      Aktualisierung der Horizon-Codebasis, um Programmfehler in Bezug auf Datenträgersicherungen zu beheben: Ein Fehler in Horizon, der die Erstellung von Cinder-Datenträgersicherungen verhindert hat, wurde behoben.

      Aktualisierung der Horizon-Codebasis, um das Hochladen von Images über die Swift-URL (Bereitstellung gepflegter IBM Bluemix-Images) zuzulassen: Ein Fehler in Glance, der das Hochladen von Images über eine URL verhindert hat, wurde behoben, sodass URLs in Box Panel für das Hochladen von Images in Glance verwendet werden können.

      Neuer DiskFilter für die Nova-Planung: Ein Fehler, der dazu führt, dass Nova Builds in Hypervisors plant, in denen nicht genügend Speicherplatz für den Build verfügbar ist, wurde behoben.

      Optimierung von Heat für die Unterstützung großer Vorlagen: bugs.launchpad.net/heat/+bug/1491185: Ein Fehler, der die Bereitstellung von Heat-Stacks mit großen Vorlagen verhindert, wurde behoben.

      Unterstützung der Änderung von Instanzgrößen: Wenn Sie die Größe einer Instanz ändern, können Sie deren Variante in eine größere Variante ändern. Diese Operation wird nicht live durchgeführt. Zum Durchführen der Operation wird die Instanz gestoppt und anschließend erneut gestartet. Weitere Informationen sind in diesem Dokument verfügbar.

      Künftige Releaseinformationen

      In regelmäßigen Abständen werden Releaseinformationen veröffentlicht, in denen Sie Details zu den vorgenommenen Änderungen finden und nachlesen können, wie sich diese Änderungen auf Ihr Erlebnis mit Ihrem IBM Bluemix Private Cloud-Angebot auswirken können. Wenn beispielsweise eine neue Version von OpenStack installiert wird, in der neue Features bereitgestellt werden, erhalten Sie Releaseinformationen mit den Details.
