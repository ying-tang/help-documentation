      Hoch verfügbare IP-Adresse konfigurieren

      • by Eric French ● 21. September 2016

      • Tags:

        • neutron

        • Neutron

        • IP-Adresse konfigurieren

        • HA

        • hoch verfügbar

        • zulässige Adresspaare

        • l2 POP

      Dieser Artikel enthält Informationen zum Einrichten hoch verfügbarer variabler IP-Adressen in einer IBM Bluemix Private Cloud Local-Umgebung. In diesen Anweisungen wird davon ausgegangen, dass Sie die OpenStack-Python-API-Clients bereits installiert haben.

      ANMERKUNG: Dieses Feature funktioniert NUR mit IBM Bluemix Private Local Cloud. Es gilt nicht für das Angebot IBM Bluemix Private Dedicated Cloud. Das Netz des IBM Bluemix-Rechenzentrums setzt derzeit die Verwendung von L2 POP voraus, eines Treibers, der für die Unterstützung der Skalierung großer Netze konzipiert ist. Um einen der IBM Entwickler zu zitieren:

      “Der Treiber für den L2Population-Mechanismus, den wir verwenden müssen, damit VXLAN in den Netzen von IBM Bluemix-Rechenzentren funktioniert, ist mit dem MAC Learning und den ARP-Antworten, die von HA-IP-Methoden verwendet werden, nicht kompatibel. IP-Adressen werden Hosts mithilfe zweier Übersetzungsmechanismen zugeordnet: ARP übernimmt die Zuordnung von IP-Adressen zu MAC-Adressen und Switches lernen, welche MAC-Adresse jedem einzelnen Port zugeordnet ist. Beim Failover muss für jede HA-IP-Adresse mindestens eine dieser beiden Zuordnungen geändert werden.”

      Als mögliche Ausweichlösung im OpenStack-Release Mitaka gibt es die Option, die Hälfte des L2-POP-Responders zu inaktivieren, was Ihnen ermöglicht, den Gratuitous-ARP-Typ der Hochverfügbarkeit einzurichten. (Lesen Sie dazu den folgenden Artikel: https://review.openstack.org/278597.) Wenn Sie die Hochverfügbarkeit wirklich benötigen, sollten Sie am besten einen Load Balancer wie Neutron LBaaS v2 verwenden. Als andere mögliche Ausweichlösung, die Sie verwenden können, wenn Ihre Cloud in einem einzigen SoftLayer-Pod bereitgestellt wird und nie über diesen Pod hinauswächst, könnten Sie einen angepassten Typ von HA-IP-Adressen implementieren.

      OpenStack-Instanzen konfigurieren

      Die Arbeit für OpenStack setzt sich aus zwei Teilen zusammen. Für den ersten Teil werden Verwaltungsberechtigungsnachweise zum Einrichten von Verfügbarkeitszonen benötigt und der zweite Teil kann von einem regulären Benutzer ausgeführt werden. Der erste Teil muss nur ein einziges Mal ausgeführt werden.
