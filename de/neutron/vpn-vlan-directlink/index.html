      VPN, VLAN und Direct Link

      • by Ying Tang ● 17. April 2017

      • Tags:

        • neutron

        • Neutron VLAN

        • Direct Link

      Übersicht

      VPN

      VPN ist ein sicherer Tunnel zwischen zwei Netzen. Im Folgenden ist ein ganz typischer Anwendungsfall für VPN mit IBM Bluemix Private Cloud beschrieben:

      Wenn Sie Ihre internen Bluemix Private Cloud-Instanzen verbinden müssen und es bevorzugen, diese virtuellen Maschinen nicht im Internet bereitzustellen, können Sie einen Bastionshost mit OpenVPN einrichten und dann mit OpenVPN eine Verbindung zu Ihrer Cloudinstanz herstellen.

      VLAN

      Ein VLAN ist eine logische Gruppierung von Switch- (oder Bridge-)Ports, die sich über einen oder mehrere Switches erstrecken, die eine allgemeine isolierte L2-Broadcastdomäne darstellen, d. h. eine Broadcastdomäne, in der ein Broadcast-Frame alle Ports im VLAN erreichen muss.

      Aus technischer Sicht kann ein Neutron-Netz ein VLAN sein. Das Neutron-Netz wurde mit verschiedenen Overlay- und Tunneltechnologien und erst vor Kurzem zu einer Gruppe von Neutron-Ports erweitert, die über L3 (z. B. IP-Protokollpakete) erreichbar sind. Diese Unterscheidung wurde für große Provider vorgenommen, die L3-Routing verwenden möchten, um für eine bessere Skalierung die Größe der L2-Broadcastdomäne verringern möchten.

      Gewöhnlich können Sie das Horizon-Dashboard oder die OpenStack-Befehlszeile verwenden, um weitere Netze mit OpenStack Neutron zu erstellen.

      Direct Link

      Direct Link ist ein SoftLayer-Feature, mit dem Sie eine Verbindung zum Backbone des privaten SoftLayer-Netzes oder zum Netz herstellen können, indem Sie das öffentliche Internet bei der Verbindungsherstellung zu Ihren dedizierten Clouds umgehen. Es ist in 1-Gb/s- und 10-Gb/s-Inkrementen verfügbar.

      Direct Link kann verwendet werden, um eine Hybrid-Cloud zu erstellen, wenn Sie Ihre lokale Cloud mit einer dedizierten Cloud verbinden möchten, die in einem SoftLayer-Rechenzentrum ausgeführt wird. Weitere Informationen zu Direct Link finden Sie auf der SoftLayer-Website.

      Die gängigste Verwendung von Direct Link ist die Übernahme einer eigenen IP-Adresse in die IBM Bluemix Private Cloud-Instanz.

      VPN anfordern

      In den folgende Fällen müssen Sie sich wegen des Setups eines VPN an das IBM Bluemix Private Cloud-Personal wenden:

      Wenn Sie eine eigene SoftLayer-IP-Adresse (BYOIP) in die IBM Bluemix Private Cloud-Instanz übernehmen und diese Ihrer Cloudinstanz zuweisen möchten, benötigen Sie entweder eine OpenVPN-Verbindung oder einen IPSec-Tunnel zwischen Ihrer Kundensite und der Cloudinstanz. Öffnen Sie ein Support-Ticket und geben Sie die folgenden Informationen an:

      • SoftLayer-IP-Adresse, die Sie Ihrer Cloudinstanz zuweisen möchten
      • VPN-Endpunkt der fernen Site

      Wenn Sie eine eigene variable IP-Adresse (BYOFIP) in die IBM Bluemix Private Cloud-Instanz übernehmen und von Ihrer Kundensite aus eine Verbindung zu den Cloudinstanzen über das BYOFIP-Netz herstellen möchten, muss ein Tunnel zwischen dem BYOFIP-Netz und der Cloudinstanz vom IBM Bluemix Private Cloud-Personal eingerichtet werden. Öffnen Sie ein Support-Ticket und geben Sie die folgenden Informationen an:

      • Variable IP-Adresse, die Sie einbringen möchten
      • VPN-Endpunkt der fernen Site

      VLAN anfordern

      Die Wahrscheinlichkeit, dass Sie gezielt eine VLAN-Anforderung absetzen müssen, ist sehr gering, weil das VLAN normalerweise standardmäßig vom IBM Bluemix Private Cloud-Personal eingerichtet ist, wenn die Cloud eingerichtet wird. Sollte dieser Fall jedoch eintreten, können Sie ein Support-Ticket öffnen.

      Direct Link anfordern

      Wenn Sie Direct Link für BYOIP anfordern, kann Direct Link als Alternative zum eingerichteten Tunnel betrachtet werden, wie unter VPN anfordern beschrieben.

      Zum Anfordern von Direct Link müssen Sie die folgenden Informationen vorbereiten und ein Support-Ticket mit diesen Informationen öffnen:

      • SoftLayer-Rechenzentrum, zu dem eine Verbindung hergestellt werden soll
      • Direct Link-Endpunkt der fernen Site
