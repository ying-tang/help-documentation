      Variable IP-Adresse mit der OpenStack-API erstellen

      • by Jason Kennedy ● 13. August 2015

      • Tags:

        • openstack

        • OpenStack

        • variable IP-Adresse

        • API

      In diesem Artikel wird beschrieben, wie Sie vorgehen, wenn Sie einer Ihrer Instanzen eine variable IP-Adresse hinzufügen möchten. Am Ende des Artikels In der Horizon-Anzeige zusätzliche Netze in OpenStack erstellen wurde diese Vorgehensweise bereits angedeutet. Das Beispiel aus diesem Artikel wird jetzt verwendet, um die Vorgehensweise über das Terminal zu veranschaulichen. Zunächst ist zu beachten, dass das Erstellen einer variablen IP-Adresse die Adresse nicht tatsächlich erstellt, sondern eine Adresse aus dem zugeordneten Pool zuweis. Hier sehen Sie eine solche Zuordnung:

      Die IP-Adresse wurde erstellt und kann nun der Instanz zugeordnet werden. Hierzu benötigen Sie die Port-ID der internen IP-Adresse der Instanz und die ID der variablen IP-Adresse aus dem obigen Beispiel.

      Der Befehl zeigt die Instanz mit der zugehörigen internen IP-Adresse an. Mithilfe dieser Informationen können Sie jetzt den Port suchen.

      Nachdem dieser Schritt abgeschlossen ist, wird nur noch die ID der variablen IP-Adresse benötigt. Die variable IP-Adresse wurde bereits erstellt. Sie finden Sie unter floating_network_id im obigen Beispiel: 953cda96-1a33-4645-a4e2-1b00d6e06736. Der letzte Schritt besteht darin, die Adressen zuzuordnen.

      Der Befehl nova list zeigt die beiden Ihrer Instanz zugeordneten IP-Adressen an:

      Sie können jederzeit die Zuordnung einer variablen IP-Adresse aufheben, indem Sie den entsprechenden Befehl unter Angabe der ID der betreffenden IP-Adresse absetzen. Im folgenden Beispiel wird die Zuordnung der gerade zugeordneten Adresse aufgehoben.

      Nun ist diese Adresse Ihrer Instanz nicht mehr zugeordnet.

      • In OpenStack sind variable IP-Adressen öffentlich routbare IP-Adressen, die Sie Ihren virtuellen Maschinen zuordnen. An der virtuellen Maschine ändert sich durch das Hinzufügen der variablen IP-Adresse nichts. Wenn Sie sich über SSH anmelden und die Netzkonfiguration anzeigen, wird weiterhin eine einzelne Netzschnittstelle mit einer konfigurierten IP-Adresse angezeigt. Die gesamte Konfiguration beschränkt sich auf den Compute-Hypervisor selbst. OpenStack Neutron konfiguriert die Netzadressumsetzung (Network Address Translation, NAT) zwischen den festen und variablen IP-Adressen der virtuellen Maschine.

      Ein Vorteil der variablen IP-Adressen ist, dass sie bei einem Ausfall der virtuellen Maschine einer anderen virtuellen Maschine zugeordnet werden können.

      Weitere Informationen finden Sie unter https://www.mirantis.com/blog/configuring-floating-ip-addresses-networking-openstack-public-private-clouds/.
