      Erste Schritte mit der OpenStack-API

      • by Eric French ● 16. November 2015

      • Tags:

        • api

        • API

        • erste Schritte

        • OpenStack

      Ihre Instanz von IBM Bluemix Private Cloud wurde mit API-Endpunkten für alle in der Instanz ausgeführten OpenStack-Hauptservices bereitgestellt. Um auf diese Services zugreifen zu können, richten Sie Ihr System folgendermaßen ein:

      Unter macOS:

      Python installieren

      Anmerkung: macOS El Capitan hat einen Mechanismus zum Verhindern von Beschädigungen der Betriebssystemdateien eingeführt. /System/Library/Frameworks/Python.framework/Versions/2.7/share ist eine der geschützten Speicherpositionen. Ein Standardbenutzer verschiebt bzw. schreibt in der Regel keine Dateien in dieses Verzeichnis. Daher sollte sudo pip nicht verwendet werden, um Python zu installieren.

      Installieren Sie stattdessen ein Python-Paket, beispielsweise IPython, lokal im Ausgangsordner Ihres Benutzers. Die einfachste Methode besteht darin, eine virtuelle Umgebung zu erstellen, diese zu aktivieren und anschließend pip in der virtuellen Umgebung auszuführen.

      Beispiel:

      Lokale stackrc-Datei einrichten

      Eine stackrc-Datei definiert die Umgebungsvariablen, mit denen Ihr Nova-Client eine Verbindung zur Nova-API herstellt:

      Erstellen Sie Ihre eigene stackrc-Datei, indem Sie die Felder OS_USERNAME, OS_PROJECT_NAME, OS_TENANT_NAME, OS_USER_DOMAIN_NAME, OS_PROJECT_DOMAIN_NAME, OS_CACERT und OS_PASSWORD durch Ihre eigenen Werte ersetzen.

      Anmerkung: Wenn Sie Ihr Kennwort nicht in dieser Datei speichern möchten, lassen Sie das Kennwortfeld leer. Sie werden zur Eingabe eines Kennworts aufgefordert, wenn Sie die Datei stackrc im entsprechenden Kontext ausführen.

      Sie können außerdem eine OpenStack-RC-Datei von Horizon herunterladen. Melden Sie sich dazu in Horizon an, navigieren Sie zur Seite Access & Security und wählen Sie die Registerkarte API Access aus.

      Klicken Sie auf die Schaltfläche zum Herunterladen der OpenStack-RC-Datei und nehmen Sie die gewünschten Änderungen vor.

      **Befehl “source” mit der Datei “stackrc” im entsprechenden Kontext ausführen **

      Nova-Client testen

      Dieser Befehl gibt eine Liste der Images in Ihrem Projekt aus, die das Booten einer Instanz (virtuelle Maschine) zulassen.

      OpenStack-API mit cURL abfragen

      F: Wie erhalte ich über die v3-Api ein OpenStack-Token?

      A: Sie können den Befehl cURL verwenden:

      Weitere Informationen:

      Ein API-Handbuch mit Beispielen für das Abrufen einer Serverliste mit cURL finden Sie hier.

      Jeder Service hat seinen eigenen Port für den API-Zugriff. Wenn Sie sich an der Web-GUI anmelden und zur Registerkarte Zugriff & Sicherheit => API-Zugriff wechseln, wird angezeigt, welcher Port zu welchem Service gehört.

      Bevor Sie einen anderen API-Service (Nova, Neutron, Cinder usw.) abfragen können, müssen Sie zuerst ein Token vom Authentifizierungsservice (Keystone) abrufen. Sie können dieses Token für beliebig viele API-Aufrufe verwenden, bis es abgelaufen ist.

      Angenommen, Port 9696 gehört zu Neutron. Um ein Token abzurufen, benötigen Sie jedoch (für den auszuführenden Befehl) eine Keystone-Authentifizierungsanforderung, die über Port 5000 erfolgen muss.

      Führen Sie die folgenden Schritte aus:

      1. Rufen Sie über eine HTTP-POST-Anforderung ein Token von $OS_AUTH_URL/tokens ab. (Sie können hierfür keine GET-Anforderung verwenden, da Sie $OS_USERNAME und $OS_PASSWORD, $OS_TENANT_NAME angeben müssen, um ein Token abrufen zu können.) Die Antwort enthält ein Token und einen Katalog der verschiedenen Services.

      2. Rufen Sie dieses Token ab und prüfen Sie, ob das zugehörige Ablaufdatum in der Zukunft liegt.

      3. Rufen Sie die URL des Service aus dem zurückgegebenen Katalog ab, für den Sie einen API-Aufruf absetzen möchten. In diesem Beispielfall suchen Sie nach dem Katalogeintrag mit dem Typ network. Dieser sollte den Namen neutron haben.

      4. Geben Sie dieses Token im X-Auth-Token-Header an, wenn Sie eine andere HTTP-GET-Anforderung für diese URL absetzen. Sie können den Befehl neutron --debug security-group-list ausführen, um anzuzeigen, wie diese POST-Anforderung aussehen sollte. Sie sollte ähnlich der folgenden Anforderung sein:

      curl -g -i --cacert "/etc/ssl/certs/ca-certificates.crt" \
-X GET https://example.openstack.blueboxgrid.com:9696/v2.0/security-groups.json \
-H "User-Agent: python-neutronclient" \
-H "Accept: application/json" \
-H "X-Auth-Token: {SHA1}11111a1111aaa1a111aaaaaa1111aa1a111a1111"
