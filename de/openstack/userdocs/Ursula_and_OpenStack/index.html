      Ursula und OpenStack

      Die meisten Benutzer sind vermutlich bereits mit OpenStack vertraut, wohingegen dies möglicherweise bei Ursula noch nicht der Fall ist. Ursula wird für die Installation einer spezialisierten Version von OpenStack für Ihre IBM Bluemix Private Cloud-Instanz verwendet. Mithilfe automatisierter Installationen sind schnellere, ordnungsgemäße und anpassbare OpenStack-Installationen möglich. Außerdem können so Benutzerfehler vermieden werden. Die Informationen in diesem Blog sollen Sie bei der Arbeit mit Ursula unterstützen und Ihnen einen Überblick über Ursula geben. Alle Ursula-Playbooks sind quelloffen und unter GitHub verfügbar. Sie können daher Playbooks auch für Ihre eigenen Zwecke anpassen.

      Was ist Ursula?

      • Ursula ist eine Sammlung von Ansible-Playbooks, die von Blue Box erstellt wurden, um eine OpenStack-Cloud bereitzustellen und zu verwalten.

      Es gibt zwei Möglichkeiten für die Installation von Ursula: Sie verwenden Vagrant oder Sie führen eine manuelle Bereitstellung durch. Die Informationen in diesem Beitrag konzentrieren sich auf die Methode der manuellen Bereitstellung.

      Ursula enthält Anweisungen zur Durchführung einer manuellen Bereitstellung. Wenn Sie neu bei OpenStack sind, können diese Anweisungen jedoch kompliziert erscheinen. In diesem Beitrag erfahren Sie, welche Erfahrungen der Autor bei der Installation von Ursula gemacht hat, und erhalten einige ausführliche Anweisungen zu seiner Vorgehensweise sowie Hinweise zur Fehlerbehebung, falls Probleme auftreten sollten.

      Voraussetzungen: Zwei virtuelle private Server (VPS)

      • 1 VPS mit konfiguriertem Ansible. Verwendet wurde ein VPS mit CentOS 6.x, 2 GB und 2 Kernen.
      • 1 VPS als Bereitstellungsziel. Für dieses Bereitstellungsziel sind weitere, bestimmte Voraussetzungen erforderlich:
        • Ubuntu 14 LTS (Verwenden Sie nicht Ubuntu 12 LTS.)
        • 8 GB RAM (Eine Bereitstellung mit weniger RAM ist evtl. auch möglich, könnte dann jedoch sehr langsam sein.)
        • 4 Kerne
        • Eine IP-Adresse. In diesem Beispiel wurde als öffentliche IP-Adresse für den VPS 69.87.123.456 verwendet. (Dies ist ganz eindeutig keine gültige IP-Adresse.)
