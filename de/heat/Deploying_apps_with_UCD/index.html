      Anwendungen mit IBM UrbanCode Deploy und IBM Bluemix Private Cloud bereitstellen

      Dieser Artikel entstand aus dem folgenden Bluemix-Blogbeitrag: Get Started with IBM UrbanCode Deploy and IBM Bluemix Private Cloud

      IBM UrbanCode Deploy (UCD) ist ein Tool für die Automatisierung der Anwendungsbereitstellung, das ein schnelles Feedback und die Continuous Delivery in einer agilen Entwicklung vereinfachen und gleichzeitig die erforderlichen Auditprotokolle, Versionskontrollen und Genehmigungen für Produktionsumgebungen bereistellen soll. Durch die Verwendung von UCD mit IBM Bluemix Private Cloud können Entwickler die Anwendungsbereitstellung und das Lebenszyklusmanagement in mehreren Cloudumgebungen beschleunigen.

      Dieser Artikel enthält Kurzanleitungen für die folgenden Aufgaben:

      • Bereitstellung von Anwendungen mit IBM UrbanCode Deploy in IBM Bluemix Private Cloud
      • Verwendung von OpenStack Heat-Blueprints zum Einrichten einer Bereitstellung mit vollständigem Stack
      • Verwaltung von Anwendungen mit OpenStack-APIs aus IBM UrbanCode Deploy

      UCD-Software installieren

      Wenn Sie kein bestehender UCD-Kunde sind, müssen Sie die UCD-Software zuerst herunterladen und die Installationsanweisungen ausführen. Wenn Sie Ihr Paket auf VMs in einer Cloudumgebung installieren, müssen Sie sicherstellen, dass diese Instanzen öffentliche variable IP-Adressen haben und die UCD-Systemvoraussetzungen erfüllen.

      Anmerkung: Wir empfehlen Ihnen, sich das folgende Dokument anzusehen, bevor Sie beginnen. In diesem Dokument finden Sie ausführliche Installationsdetails: IBM UrbanCode Deploy mit IBM Bluemix Private Cloud verwenden.

      Blueprint in UCD erstellen

      Zum Einrichten einer Bereitstellung mit vollständigem Stack können Sie einen Blueprint Zum Installieren Ihrer Anwendungen erstellen. In der UCD-Netzschnittstelle können Sie Ihrem Blueprint Ihre Netze, VMs und die installierte Software hinzufügen und den Blueprint dann anzeigen, sodass Sie das Setup und die Bereitstellung schnell durchführen können.

      Klicken Sie zum Erstellen Ihres Blueprints oben links auf die Schaltfläche New. Daraufhin wird ein neues Modalfenster geöffnet, in dem Sie Ihrem Blueprint einen Namen geben können. Klicken Sie anschließend, wie in der folgenden Abbildung gezeigt, auf Save.

      Klicken Sie oben in der Anzeige in der Symbolleiste auf das Pluszeichen (+), um eine Suche durchzuführen. Wählen Sie dann das Netz, die VMs und die Anwendungen aus, die Sie installieren möchten. Sie können Ihren Blueprint sogar noch schneller erstellen, indem Sie die Komponenten, die Sie installieren möchten, über die Palette auf der rechten Seite der Anzeige ziehen und übergeben. Ihre VMs werden automatisch an das Netz gebunden und Ihre Anwendungen werden automatisch an Ihre VMs gebunden, sobald Sie sie an die gewünschte Position ziehen.

      Auf der Registerkarte Policies auf der rechten Seite können Sie das Symbol Deployment Sequence verwenden, um die Installationsreihenfolge für die neu hinzugefügten Anwendungen festzulegen.

      Stellen Sie sicher, dass Sie Ihren Anwendungen Ihre IP-Adresse hinzugefügt haben. Für diesen Schritt können Sie Ihre Anwendung auswählen, deren Eigenschaften anzeigen und dann Ihre IP-Adresse eingeben.

      Jetzt können Sie Ihren Blueprint bereitstellen. Klicken Sie zum Starten der Bereitstellung oben in der Anzeige auf die Schaltfläche Save und klicken Sie dann auf Provision. Daraufhin wird ein Modalfenster geöffnet, in dem Sie vor der Bereitstellung eine Variante definieren, einen SSH-Schlüssel festlegen und Ihre Berechtigungsnachweise aktualisieren können.

      Verbindung zu OpenStack Heat herstellen

      Während Sie Ihre Umgebung bereitstellen, stellen Sie eine Verbindung zu einer OpenStack Heat-Engine her, die die Übernahme der VMs in IBM Bluemix Private Cloud automatisiert. Von diesem zentralen Blickpunkt aus können Sie, wie in der folgenden Abbildung gezeigt, Ihre UCD-Anwendungen und deren Bereitstellungsstatus sehen:

      Sie können in der OpenStack Heat-Engine auch auf das Symbol Details klicken, um den schrittweisen Fortschritt jeder Installation anzuzeigen, wie in der folgenden Abbildung gezeigt:

      Ihre neuen Instanzen werden über UCD bereitgestellt und der UCD-Agent wird installiert und für den Host gebootet. Sobald Ihre Instanzen online sind, werden die Anwendungen automatisch installiert.

      Während des gesamten Installationsprozesses können Sie den OpenStack-API-Client verwenden, um Installationsprotokolle für den Server und die Anwendung abzurufen. Anhand dieser Protokolle können Sie feststellen, ob es Probleme mit dem Installationsprozess gibt.

      Zugriff auf Anwendungen in OpenStack Horizon

      Der UCD-Agent stellt den vollständigen Stack innerhalb von Sekunden bereit. Sobald Ihre Anwendungen über OpenStack Heat bereitgestellt wurden, können Sie auf Ihre Instanzen und deren Anwendungen in Ihrer IBM Bluemix Private Cloud-Umgebung zugreifen.
