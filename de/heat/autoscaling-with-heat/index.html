      Heat für automatische Skalierung verwenden

      • by Blue Box Support ● 11. Mai 2016

      • Tags:

        • heat

        • Heat

        • automatische Skalierung

        • bereitstellen

      OpenStack stellt über Heat Features für die automatische Skalierung bereit. Wenn Sie diese Features verwenden, müssen Sie im Voraus manuell weniger Instanzkapazität bereitstellen. Mithilfe von Heat-Ressourcen können Sie erkennen, wenn ein Ceilometer-Alarm ausgelöst wird, und je nach Auslöser eine neue VM bereitstellen oder die Bereitstellung einer VM zurücknehmen. Diese VM-Gruppen müssen von einem Load Balancer verwaltet werden, der die Last auf die VMs in der Skalierungsgruppe verteilt.

      Egal, ob Sie eine einzige Instanz oder Tausende ausführen, können Sie die automatische Skalierung verwenden, um Instanzen ohne manuelle Intervention zu erkennen, zu erweitern, zu verringern und zu ersetzen.

      In den folgenden Beispielen werden die folgenden beiden Richtlinien definiert:

      • Wenn die CPU-Auslastungsrate über 50 % liegt, wird automatisch eine neue Instanz erstellt, bis die Anzahl von sechs Instanzen erreicht ist.
      • Wenn die CPU-Auslastungsrate unter 15 % liegt, wird jeweils eine Instanz beendet, bis die Anzahl von einer Instanz erreicht ist.

      Stacks erstellen

      1. Laden Sie die Datei simplescale herunter.
      2. Melden Sie sich mit der Rolle heat_stack_owner am Horizon-Dashboard an.
      3. Klicken Sie auf Project -> Orchestration -> Stacks.
      4. Klicken Sie auf Launch Stack.
      5. Klicken Sie auf der Seite Select Template auf Choose File und wählen Sie die Datei ‘simplescale’ auf Ihrer lokalen Maschine aus.
      6. Aktualisieren Sie die Image-ID, die Netz-ID, den Schlüsselpaarnamen und die Sicherheitsgruppen-ID basierend auf Ihrer Cloudinstanz.
      7. Klicken Sie auf Launch.

      Stacks überwachen

      1. Melden Sie sich mit der Rolle heat_stack_owner am Horizon-Dashboard an.
      2. Klicken Sie auf Project -> Instance und suchen Sie die Instanz, die als Teil des Stacks erstellt wurde.
      3. Stellen Sie mit SSH-Verbindungstools wie putty eine Verbindung zu der Instanz her.
      4. Führen Sie den Befehl top aus, um die CPU-Auslastungsrate zu überprüfen. Die Auslastungsrate für den dd-Prozess muss mindestens bei 99 % liegen. Bei diesem Wert wird der Alarm für hohe CPU-Auslastung ausgelöst.
      5. Warten Sie ungefähr zehn Minuten und vergewissern Sie sich, dass eine neue Instanz im Stack erstellt wurde.
      6. Melden Sie sich bei beiden Instanzen an, suchen Sie den dd-Prozess und beenden Sie ihn. Daraufhin wird der Alarm für geringe CPU-Auslastung ausgelöst.
      7. Warten Sie ungefähr zehn Minuten und vergewissern Sie sich, dass eine Instanz beendet wurde und nur eine einzige Instanz verblieben ist.

      Stacks entfernen

      1. Melden Sie sich mit der Rolle heat_stack_owner am Horizon-Dashboard an.
      2. Klicken Sie auf Project -> Orchestration -> Stacks.
      3. Wählen Sie den Stack aus, den Sie entfernen möchten, und klicken Sie auf Löschen.
