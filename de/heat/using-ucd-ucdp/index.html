      IBM UrbanCode Deploy mit IBM Bluemix Private Cloud verwenden

      • by Ying Tang, Pan Xia Zou ● 7. Juni 2016

      • Tags:

        • heat

        • Heat

        • bereitstellen

        • UCD

        • UCDP

      IBM UrbanCode Deploy with Patterns (UCDP) ist eine Lösung für das Management und die Bereitstellung von Umgebungen, mit der Sie Anwendungen in mehreren Clouds bereitstellen und deren Lebenszyklen verwalten können.

      Während UCDP einen grafischen Editor und einen Texteditor bereitstellt, in dem Sie Ihre Umgebungen mit vollständigem Stack entwerfen, testen und hochstufen können, ist IBM UrbanCode Deploy (UCD) eine umfassendere Lösung für die vollständige Konfiguration Ihrer Komponenten.

      Die Heat-Engine in IBM Bluemix Private Cloud kann mit UCDP und UCD interagieren. Wenn Sie UCDP und UCD in Ihrer Instanz installiert haben, können Sie eine sichere Verbindung von Ihrer Instanz zur Heat-Engine herstellen und anschließend eine Anwendung über UCDP und UCD schnell in Ihrer Instanz bereitstellen.

      Sie können die UCDP- und UCD-Server entweder in Instanzen in derselben IBM Bluemix Private Cloud-Umgebung, die Sie integrieren möchten, oder auf externen Servern außerhalb von IBM Bluemix installieren.

      In den folgenden Abschnitten werden beide Fälle abgedeckt. Es wird beschrieben, wie UCDP und UCD installiert und dann mit IBM Bluemix Private Cloud integriert werden.

      Als Beispiele werden in den folgenden Abschnitten UCD 6.1.1 und UCDP 6.1.1 verwendet. Informationen zu den verschiedenen Versionen finden Sie im IBM Knowledge Center.

      Voraussetzungen

      1. Laden Sie die UCDP- und UCD-Installationspakete von IBM Fix Central herunter.

      2. Wenn Sie UCD oder UCDP in Instanzen in einer Cloudumgebung installieren möchten, müssen diese Instanzen öffentliche variable IP-Adressen haben.

      3. Stellen Sie zum Installieren von UCDP und UCD auf externen Servern sicher, dass Ihre Server die Systemvoraussetzungen für UCDP und UCD erfüllen. Weitere Einzelheiten hierzu finden Sie unter UCDP system requirements und UCD system requirements.

      UCDP installieren

      Führen Sie zum Installieren von UCDP die folgenden Schritte aus:

      1. Installieren Sie eine Datenbank. Weitere Informationen hierzu finden Sie im Kapitel Installing the database im IBM Knowledge Center zu UCDP.
      2. Installieren Sie den Design-Server im interaktiven Modus. Weitere Informationen hierzu finden Sie im Kapitel Installing the design server im IBM Knowledge Center zu UCDP.

      Anmerkungen:

      • Es werden nur die Standardports (8080 und 8443) unterstützt. Wenn Sie andere Ports verwenden möchten, öffnen Sie ein Support-Ticket.
      • Akzeptieren Sie während der Installation den Standardpfad für Probelizenzen, 27000@localhost, sofern Sie keine eigene Lizenz haben. Sie können die Lizenz nach der Installation aktualisieren.
      • Wenn die Frage “What host name will users access the web UI at?” erscheint und Ihr Hostname nicht registriert ist, geben Sie anstelle des Hostnamens die IP-Adresse an.
      • Ein bewährtes Verfahren ist, konsistente Versionen von Datenbankserver und Datenbanktreiber zu verwenden.

      Die vollständige UCDP-Dokumentation finden Sie im IBM Knowledge Center zu UCDP.

      UCD installieren

      Führen Sie zum Installieren von UCD die folgenden Schritte aus:

      1. Installieren Sie eine Datenbank. Weitere Informationen hierzu finden Sie im Kapitel Installing the database im IBM Knowledge Center zu UCD.
      2. Installieren Sie den Server im interaktiven Modus. Weitere Informationen hierzu finden Sie im Kapitel Install the server im IBM Knowledge Center zu UCD.

      Die vollständige UCD-Dokumentation finden Sie im IBM Knowledge Center zu UCD.

      Anmerkung:

      • Es werden nur die Standardports (7918, 8080 und 8443) unterstützt. Wenn Sie andere Ports verwenden möchten, öffnen Sie ein Support-Ticket.
      • Ein bewährtes Verfahren ist, konsistente Versionen von Datenbankserver und Datenbanktreiber zu verwenden.

      UCD mit UCDP integrieren

      Integrieren Sie UCD mit UCDP anhand der Anweisungen unter Connecting to IBM UrbanCode Deploy.

      Verbindung zu IBM Bluemix Private Cloud mit UCDP und UCD herstellen

      1. Nur für UCD und UCDP auf externen Servern: Verwenden Sie einen OpenVPN-Client, um eine Verbindung zu der zu integrierenden Cloudumgebung herzustellen. Weitere Informationen zum Zugriff auf die Umgebung finden Sie unter Setting up OpenVPN server on your instance.

      2. Verbinden Sie den UCDP-Server anhand der Anweisungen unter Connecting to OpenStack-based clouds with Keystone authentication mit IBM Bluemix Private Cloud.

      Anmerkung: Wenn Sie den UCDP-Server mit IBM Bluemix Private Cloud verbinden, erstellen Sie einen Authentifizierungsrealm, der auf den Keystone-Server verweist und lassen Sie beim Konfigurieren des Realms das Kontrollkästchen Use default orchestration engine ausgewählt.

      Nachdem die Verbindung hergestellt wurde, können Sie Anwendungen für IBM Bluemix Private Cloud in IBM UrbanCode Deploy und IBM Urban Code Deploy with Patterns bereitstellen.

      Anmerkung: Wenn Sie ein Image verwenden, das einen Punkt (.) im Imagenamen enthält, aktualisieren Sie den im Blueprint erstellten VM-Namen im Feld Properties und entfernen Sie den Punkt (.). Andernfalls enthält der erstellte VM-Name standardmäßig einen Punkt (.), was zum Fehlschlagen der Softwarebereitstellung führt.

      Weitere Informationen zu IBM Bluemix Private Cloud und UCD finden Sie in diesem Video zur Bereitstellung von Anwendungen mit IBM Bluemix & IBM UrbanCode Deploy.
