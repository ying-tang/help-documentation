      In der Horizon-Anzeige zusätzliche Netze in OpenStack erstellen

      • by Jason Kennedy ● 5. August 2015

      • Tags:

        • getting started

        • erste Schritte

        • zusätzliche Netze

      Sie haben die Schlüssel für Ihre neue OpenStack-Cloud erhalten und Sie haben explizit angefragt, die Cloud mit VXLAN einzurichten, weil Sie eine Entwicklungs-/Testumgebung und Ihr Produktionsnetz separieren möchten. Können Sie hierfür Sicherheitsgruppen verwenden? Nein? Dann lesen Sie weiter.

      Eine typische IBM Bluemix Private Cloud-Instanz hat standardmäßig ein Setup mit einem internen und einem externen Netz mit einem Standardrouter (namens default), der als Brücke zwischen den Netzen dient. Zum Erstellen Ihres eigenen Netzes müssen Sie dieses Setup spiegeln.

      Stellen Sie sicher, dass Sie VXLAN verwenden, bevor Sie beginnen. Andernfalls funktioniert der Prozess nicht.

      Zuerst erstellen Sie ein Netz. Der Vollständigkeit halber wird sowohl die Methode über die Befehlszeilenschnittstelle (CLI, Command-Line Interface) als auch die Methode mit Horizon beschrieben. Zunächst wird die Horizon-Methode gezeigt. Die Methode über die Befehlszeilenschnittstelle wird in einem anderen Beitrag beschrieben.

      Wechseln Sie in Ihrem Projekt auf die Registerkarte Network und wählen Sie Networks aus. Daraufhin werden, wie zuvor schon erwähnt, die internen und externen Netze angezeigt.

      Klicken Sie oben rechts auf Create Network, woraufhin die Popup-Anzeige Create Network erscheint.

      Geben Sie Ihren Netznamen im Feld Network Name ein (übernehmen Sie im Feld Admin State den Eintrag UP) und klicken Sie dann auf das Register Subnet (oder wählen Sie Next) aus.

      Geben Sie den Namen Ihres Teilnetzes im Feld Subnet Name (in diesem Beispiel werden test_subnet als Teilnetzname und test als Netzname verwendet), die Netzadressen im Feld Network Addresses (in diesem Beispiel 192.168.10.0/28) und die Gateway-IP-Adresse im Feld Gateway IP (in diesem Beispiel 192.168.10.1) ein.

      Klicken Sie jetzt auf Subnet Detail oder Next. Wenn Sie in dieser Anzeige nur einen Teil Ihres Pools reservieren oder wenn Sie DNS-Server oder Hostrouten festlegen möchten, tun Sie dies.

      Klicken Sie anschließend auf die blaue Schaltfläche Create. Der Abschnitt Networks sollte jetzt wie folgt aussehen:

      Jetzt fügen Sie einen Router hinzu.

      Klicken Sie in der Navigation unter Networks auf Routers. Sie sollten dort keine Router sehen. Jetzt erstellen Sie einen Router, indem Sie auf Create Router klicken. Daraufhin erscheint das Fenster Create Router. Alles, was Sie jetzt tun müssen, ist einen Routernamen im Feld Router Name auszuwählen (in diesem Beispiel ist dies test_router). Klicken Sie auf Create Router, woraufhin Folgendes angezeigt werden sollte:

      Wenn Sie sich Ihren Router ansehen, stellen Sie fest, dass er keinem externen Netz zugeordnet ist, was aber erforderlich ist, damit der Router seine Aufgaben ausführen kann. Klicken Sie auf der rechten Seite unter Actions auf Set Gateway. Daraufhin erscheint das Fenster Set Gateway unterhalb des Dropdown-Felds External Network. Wählen Sie External aus und klicken Sie dann auf die blaue Schaltfläche Set Gateway. Ihr Gateway ist jetzt dem externen Netz zugeordnet.

      Jetzt können Sie überprüfen, was Sie bisher getan haben. Wenn Sie auf der linken Seite auf Network Topology klicken, sehen Sie jetzt, dass Ihr Netz ebenso wie der Router, den Sie erstellt haben, existiert. Dieser Router ist wie geplant dem externen Netz zugeordnet. Das Einzige, das jetzt noch fehlt, ist die Zuordnung dieses Routers zu Ihrem Netz, damit der Router den Datenverkehr für Ihre Instanz weiterleitet. Wenn Sie den Mauszeiger über Ihren Router bewegen, werden verschiedene Routerdetails aus dem Abschnitt Network Topology angezeigt, darunter die Optionen Add Interface und Delete Router. Sie möchten jetzt eine Schnittstelle hinzufügen, und so gehen Sie vor:

      Klicken Sie auf der linken Seite auf Routers und klicken Sie dann auf den Router. Daraufhin sollte die Anzeige Router Details, wie im Folgenden gezeigt, erscheinen:

      Diese Anzeige enthält zahlreiche Informationen zu Ihrem Router. Außerdem sehen Sie, dass Ihre externe Gateway-Schnittstelle hinzugefügt wurde. Klicken Sie auf Add Interface, um das Dialogfenster Add Interface aufzurufen. Wählen Sie in der Dropdown-Liste Subnet das Netz aus, das Sie erstellt haben. Optional können Sie dieser Schnittstelle selbst eine IP-Adresse zuweisen oder Sie können die Zuweisung der Cloud überlassen. In diesem Beispiel lassen Sie die IP-Adresse leer und lassen sie von der Cloud zuweisen. Klicken Sie auf die blaue Schaltfläche Add Interface, um die Aktion abzuschließen.

      An diesem Punkt sind ein paar Informationen zu Schnittstellen und IP-Adressen hilfreich. Jede Schnittstelle, die Sie einem Netz hinzufügen, erfordert eine IP-Adresse. Als Sie Ihren Router dem externen Netz zugeordnet haben, mussten Sie eine IP-Adresse angeben. Dies sehen Sie oben, weil Ihre externe Gateway-Schnittstelle eine feste IP-Adresse in Ihrem Bereich variabler IP-Adressen hat. Die interne Schnittstelle, die Sie soeben erstellt haben, muss eine eigene feste IP-Adresse aus dem erstellten Pool für interne Netze erhalten. Beachten Sie, dass anfänglich der Status DOWN (Inaktiv) angezeigt wird. Dieser wird jedoch in ACTIVE (Aktiv) geändert, sobald dieser Prozess abgeschlossen ist.

      Wenn Sie wieder in den Abschnitt Network Topology auf der linken Seite zurückkehren, sollten Sie eine Verbindung zwischen dem externen Netz und dem von Ihnen erstellten Netz über den erstellten Router, der mit jedem Netz verbunden ist, sehen. Jetzt können Sie eine Instanz in diesem Netz starten, die dann extern erreichbar ist, sobald Sie eine variable IP-Adresse zugewiesen haben.

      Klicken Sie im Abschnitt Compute unter Ihrem Projekt auf Instances. Klicken Sie auf der rechten Seite auf Launch Instance, woraufhin das Dialogfenster Launch Instance erscheint. Geben Sie einen Instanznamen im Feld Instance Name (in diesem Fall test_instance) ein und wählen Sie Boot From Image in der Dropdown-Liste Instance Boot Source aus. Jetzt erstellen Sie eine einfache Cirros-Instanz. Wählen Sie dazu Cirros (12.6MB) in der Dropdown-Liste Image Name aus.

      Klicken Sie jetzt oben auf die blau unterlegten Einträge Access & Security. Sie sollten bereits ein Schlüsselpaar haben. Stellen Sie sicher, dass dieses ausgewählt ist. Vergewissern Sie sich auch, dass Sie die richtige Sicherheitsgruppe verwenden, und klicken Sie dann oben auf den nächsten blauen Abschnitt, Networking.

      Wählen Sie das Testnetz aus. Klicken Sie auf Launch. Daraufhin sollte eine Anzeige ähnlich der folgenden erscheinen:

      Beachten Sie, dass die IP-Adresse 192.168.10.1 für das interne Schnittstellengateway in Ihrem Router verwendet wird und die IP-Adresse damit in Instanzen nicht verwendet werden kann. Klicken Sie auf den Instanznamen, um in die Anzeige Overview zu wechseln. Klicken Sie dort auf das Register Console. Klicken Sie auf der daraufhin angezeigten Registerkarte auf den dunkleren blauen Abschnitt “Connected” und drücken Sie dann die Eingabetaste. Daraufhin sollte ein Anmeldedialog erscheinen. Melden Sie sich mit den Cirros-Standardanmeldedaten an der Instanz an: Benutzername: cirros, Kennwort cubswin:).

      Setzen Sie einen Pingbefehl an eine externe Adresse ab und sehen Sie sich Ihre IP-Adresse an. Jetzt müssen Sie nur noch eine variable IP-Adresse zuweisen.

      Wählen Sie rechts unter Actions die Option Associate Floating IP aus, woraufhin das Dialogfenster Manage Floating IP Associations erscheint. Wenn dies das erste Mal ist, dass Sie eine IP-Adresse zuweisen, müssen Sie das Pluszeichen (‘+’) neben der Dropdown-Liste auswählen, weil die Liste leer ist. Wählen Sie External from the Pool aus und klicken Sie auf die blaue Schaltfläche Allocate IP und dann auf Associatec. Jetzt sollten Sie zwei IP-Adressen sehen, die Ihrer Instanz zugeordnet sind.

      Jetzt können Sie über die zugewiesene variable IP-Adresse und den zugeordneten Schlüssel eine ssh-Verbindung zu Ihrer Instanz und über Ihren eigenen Router eine Verbindung zu einer Instanz in Ihrem eigenen Netz herstellen.
