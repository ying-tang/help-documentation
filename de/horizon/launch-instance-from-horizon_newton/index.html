      Instanz über das Horizon-Dashboard starten (OpenStack Newton-Release)

      • by Ying Tang ● 31. März 2017

      • Tags:

        • horizon

        • Horizion

        • Instanz

        • starten

        • Newton

        • rhosp

      Schritte

      1. Melden Sie sich am Horizon-Dashboard an.
      2. Erweitern Sie unter Project den Eintrag Compute und klicken Sie dann auf Instances. Wenn Sie vorhandene Instanzen haben, können Sie eine Liste Ihrer Instanzen zusammen mit deren Details wie Name, IP-Adresse und Status anzeigen. Informationen zum Verwalten Ihrer Instanzen finden Sie unter Instanzen verwalten.
      3. Klicken Sie im oberen rechten Abschnitt des Horizon-Dashboards auf Launch Instance. Das Dialogfeld Launch Instance wird angezeigt.

      Navigieren Sie durch die folgenden Registerkarten und geben Sie die Werte an. Klicken Sie anschließend unten rechts im Dialogfenster auf Launch Instance.

      Werte im Dialogfenster Launch Instance eingeben

      Registerkarte Details

      Werte Beschreibung
      Instance Name Der Instanzname. Weisen Sie Ihrer virtuellen Maschine einen Namen zu. Der Name, den Sie hier zuweisen, wird als anfänglicher Hostname des Servers verwendet. Servernamen sind bei ihrer Erstellung nicht unbedingt eindeutig. Es kann also vorkommen, dass Sie zwei Instanzen mit demselben Hostnamen haben.
      Availability Zone Die Verfügbarkeitszone. Standardmäßig wird die vom Host-Provider vergebene Verfügbarkeitszone festgelegt, z. B. compute_standard.
      Count Die Anzahl. Wenn Sie mehrere Instanzen starten möchten, geben Sie einen Wert größer als 1 ein. Der Standardwert ist 1.

      Klicken Sie auf Next oder auf das Register Source, um fortzufahren.

      Registerkarte Source

      Im Feld Select Boot Source werden die folgenden Optionen angeboten:

      Option Beschreibung
      Image Wählen Sie einen Eintrag in der Liste privater und öffentlicher Images in der Tabelle Available aus und klicken Sie dann auf das Pluszeichen (+), um ihn in die Tabelle Allocated zu verschieben. Optional können Sie neben Create New Volume auf Yes klicken, um weitere Optionen für die Datenträgererstellung zu aktivieren. Falls Sie einen neuen Datenträger mit dieser Instanz erstellt haben, können Sie neben Delete Volume on Instance Delete auf Yes klicken, wenn Sie diesen Datenträger beim Beenden der Instanz löschen möchten.
      Instance Snapshot Wählen Sie eine Momentaufnahme in der Tabelle Available aus und klicken Sie dann auf das Pluszeichen (+), um ihn in die Tabelle Allocated zu verschieben.
      Volume Wählen Sie einen Datenträger in der Tabelle Available aus und klicken Sie dann auf das Pluszeichen (+), um ihn in die Tabelle Allocated zu verschieben. Optional können Sie neben Delete Volume on Instance Delete auf Yes klicken, wenn dieser Datenträger beim Beenden der Instanz gelöscht werden soll.
      Volume Snapshot Wählen Sie eine Datenträgermomentaufnahme in der Tabelle Available aus und klicken Sie dann auf das Pluszeichen (+), um ihn in die Tabelle Allocated zu verschieben. Optional können Sie neben Delete Volume on Instance Delete auf Yes klicken, wenn dieser Datenträger beim Beenden der Instanz gelöscht werden soll.

      Anmerkung: Die Option Image ist standardmäßig gesetzt, d. h., Sie müssen sie vor dem Booten nicht auswählen. Wenn Sie die Option Image verwenden, können Sie ein Image in der Liste der von IBM bereitgestellten öffentlichen Images auswählen oder Sie können Ihr eigenes Image hochladen und verwenden. Informationen zum Hochladen eines Image in IBM Bluemix Private Cloud finden Sie unter Von IBM Bluemix Private Cloud bereitgestellte Cloud-Images.

      Registerkarte Flavor

      Wählen Sie eine Variante (Flavor) in der Liste aus und klicken Sie dann auf das Pluszeichen (+), um sie in die Tabelle Allocated zu verschieben.

      Registerkarte Networks

      Wählen Sie ein Netz in der Liste aus und klicken Sie dann auf das Pluszeichen (+), um es in die Tabelle Allocated zu verschieben. Sie können der Instanz mehrere Netze zuordnen.

      Registerkarte Network Ports

      Wenn Sie keinen Netzport angeben, wird der Instanz ein Standardport zugewiesen, wenn die Instanz erstellt und gebootet wird.

      Gelegentlich müssen Sie möglicherweise vorab über APIs Ports erstellen. Nachdem Sie einen Port erstellt und diesen einem der auf der Registerkarte Networks ausgewählten Netze zugeordnet haben, sehen Sie diesen Port in der Tabelle Available. Sie können auf das Pluszeichen (+) klicken, um den Port in die Tabelle Allocated zu verschieben.

      Registerkarte Security Groups

      Wählen Sie eine Sicherheitsgruppe in der Liste aus und klicken Sie dann auf das Pluszeichen (+), um sie in die Tabelle Allocated zu verschieben.

      Registerkarte Key Pair

      Geben Sie ein Schlüsselpaar an. Sie können zwischen den folgenden Optionen wählen:

      • Wählen Sie ein gespeichertes Schlüsselpaar in der Liste aus und klicken Sie dann auf das Pluszeichen (+), um es in die Tabelle Allocated zu verschieben.
      • Klicken Sie auf Import Key Pair, um Ihr eigenes Schlüsselpaar zu importieren. Kopieren Sie Ihren öffentlichen Schlüssel und fügen Sie ihn ein und klicken Sie dann auf Import Key Pair.

      • Klicken Sie auf Create Key Pair, um ein neues Schlüsselpaar zu erstellen und zu speichern. Geben Sie den Schlüsselpaarnamen an und klicken Sie dann auf Create Key Pair.

        Sie werden aufgefordert, das neue Schlüsselpaar auf dem lokalen Host zu speichern.

      Registerkarte Configuration

      Angepasste Scripts, die Instanzen zugeordnet werden, können bestimmte Aktionen ausführen, wenn die Instanz gestartet wird. Sie können Ihr Script direkt in das Feld Customization Script eingeben.

      Für Linux-Instanzen: Der Standardbenutzername ist ibmcloud. Zum Aktivieren der Kennwortauthentifizierung über die Konsole und SSH verwenden Sie die folgenden Scriptdaten für das Anpassungsscript (ersetzen Sie <IHR_KENNWORT> durch das relevante Kennwort):

      #cloud-config password: <IHR_KENNWORT> chpasswd: { expire: False } ssh_pwauth: True

      Für Windows-Instanzen: Zum Festlegen Ihres anfänglichen Benutzernamens und Ihres Kennworts verwenden Sie die folgenden Scriptdaten für das Anpassungsscript (ersetzen Sie <IHR_BENUTZERNAME> und <IHR_KENNWORT> durch den relevanten Benutzernamen und das relevante Kennwort).

      rem cmd
net user <IHR_BENUTZERNAME> <IHR_KENNWORT> /logonpasswordchg:yes /add /y
net localgroup administrators <IHR_BENUTZERNAME> /add

      Wenn Ihr Browser die HTML5 File-API unterstützt, können Sie auf Browse klicken, um Ihr Script aus einer Datei zu laden. Ihr Script sollte nicht größer als 16 Kb sein.

      Im Dropdown-Menü Disk Partition können Sie Automatic auswählen, um die Größe der Platte zu ändern und eine einzige Partition festzulegen, oder Sie können Manual auswählen, um mehrere Partitionen auf der Platte zu erstellen.

      Wählen Sie das Kontrollkästchen Configuration Drive aus, wenn Metadaten auf ein spezielles Konfigurationslaufwerk geschrieben werden sollen, und geben Sie dann Ihre Metadaten auf der Registerkarte Metadata an. Wenn Sie die Instanz booten, wird sie dem Konfigurationslaufwerk zugeordnet, damit sie auf Ihre Metadaten zugreifen kann.

      Registerkarte Server Groups

      Mit Servergruppen können Sie die Affinitäts- und Anti-Affinitätsplanungsrichtlinien für eine Gruppe von Servern (Instanzen) steuern.

      Gelegentlich müssen Sie möglicherweise vorab über APIs Servergruppen erstellen. Nachdem Sie eine Servergruppe erstellt haben, sehen Sie diese in der Tabelle Available. Sie können auf das Pluszeichen (+) klicken, um den Port in die Tabelle Allocated zu verschieben.

      Registerkarte Scheduler Hints

      Die Schedulerhinweise können verwendet werden, um zusätzliche verteilungsrelevante Informationen an den Nova-Scheduler zu übergeben. Sie können beispielsweise einen Schedulerhinweis verwenden, um sicherzustellen, dass eine Instanz auf einem Host aus einer bestimmten Servergruppe geplant wird. Sie können auch einen angepassten Schedulerhinweis im JSON-Format erstellen. Weitere Einzelheiten zum Planungsmechanismus und zu den unterstützten Hinweistypen finden Sie unter Compute Schedulers.

      Registerkarte Metadata

      Glance stellt eine allgemeine API für die Definition von Schlüssel/Wert-Paaren bereit, die Metadaten für verschiedene Typen von Ressourcen (Images, Artefakte, Datenträger, Varianten, Aggregate usw.) markieren. Wenn das ausgewählte Image vordefinierte Metadaten enthält, sehen Sie diese in der Spalte Available metadata.

      Sie können im Feld Custom weitere Schlüssel/Wert-Paare erstellen und dann auf das Pluszeichen (+) klicken, um sie in die Liste Existing metadata zu verschieben. Angepasste Metadaten sind beispielsweise ein Parameter, den Sie an die Anwendung übergeben können, damit sie zusammen mit der Instanz installiert werden kann.

      Weitere Informationen zu Glance-Metadaten finden Sie unter Metadata Definition Concepts und unter Using Glance’s Metadata Definitions Catalog Public APIs.

      Verbindung zu einer Instanz herstellen

      Anmerkung: Stellen Sie sicher, dass die richtigen Sicherheitsgruppenregeln für die Herstellung einer Verbindung zu Ihrer Instanz über ssh oder rdp definiert sind.

      1. Melden Sie sich am Horizon-Dashboard von IBM Bluemix Private Cloud an.

      2. Rufen Sie die Anzeige Project und dann die Registerkarte Compute im linken Navigationsbereich auf und klicken Sie dann auf Instances.

      3. Suchen Sie Ihre Instanz in der Liste aktiver Instanzen und notieren Sie deren IP-Adresse, z. B. 192.168.100.237.

      4. Stellen Sie eine Verbindung zu Ihrer Instanz her, indem Sie deren IP-Adresse angeben.

      Verbindungen zu Linux-Instanzen herstellen

      Stellen Sie über SSH oder VNC eine Verbindung zu Ihrer Instanz her. Die Standardbenutzer-ID für die von IBM bereitgestellten Linux-Images ist ibmcloud. Sobald die Verbindung zu Ihrer Linux-Instanz hergestellt ist, können Sie den Befehl sudo verwenden, um Befehle auszuführen, die normalerweise Rootzugriff erfordern. Sudo ist so konfiguriert, dass kein Kennwort für die Standardbenutzer-ID erforderlich ist.

      • Öffnen Sie über den von Ihnen bevorzugten SSH-Client eine Verbindung zur richtigen IP-Adresse der VM-Instanz. Verwenden Sie dazu den privaten SSH-Schlüssel des Schlüsselpaars, das Sie während der Bereitstellung angegeben haben.

          ssh -i <Pfad_Ihrer_privaten_SSH-Schlüsseldatei> <Benutzer-ID>@<IP-Adresse_der_Instanz>

      • Rufen Sie im Horizon-Dashboard von IBM Bluemix Private Cloud die VNC-Konsole der Instanz auf, indem Sie in der Dropdown-Liste in der Spalte Actions der Instanz auf Console klicken. Die VNC-Konsole verbindet Sie mit HTTPS.

      Anmerkung: Wenn Sie sich als Benutzer ibmcloud über die Konsole an einer neuen Linux-Instanz anmelden möchten, müssen Sie während der Bereitstellung ein Kennwort im Anpassungsscript angegeben haben.

      Verbindungen zu Windows-Instanzen herstellen

      Stellen Sie über Remote Desktop oder VNC eine Verbindung zu Ihrer Instanz her. Sie müssen die Benutzer-ID und das Kennwort verwenden, das Sie während der VM-Bereitstellung im Anpassungsscript angegeben haben. Bei Ihrer ersten Anmeldung erscheint eine Aufforderung zum Ändern Ihres Kennworts. Ändern Sie Ihr daraufhin Ihr Kennwort.

      • Verwenden Sie den von Ihnen bevorzugten Remote Desktop-Client für die Herstellung der Verbindung.
      • Rufen Sie im Horizon-Dashboard von IBM Bluemix Private Cloud die VNC-Konsole der Instanz auf, indem Sie in der Dropdown-Liste in der Spalte Actions der Instanz auf Console klicken. Die VNC-Konsole verbindet Sie mit HTTPS.

      Instanzen verwalten

      Erweitern Sie unter Project den Eintrag Compute und klicken Sie dann auf Instances.

      Im Dashboard wird die Liste der Instanzen für das aktuelle Projekt angezeigt. In der Spalte Actions werden, wie im Folgenden gezeigt, Ihre Optionen in einer Dropdown-Liste angezeigt:

      • Creates snapshot: Erstellt eine Momentaufnahme der Instanz.*
      • Associate Floating IP: Ordnet einem Port Ihrer Instanz eine verfügbare variable IP-Adresse zu.
      • Attach an Interface: Ordnet die Schnittstelle einem Netz zu.
      • Detach an Interface: Entfernt einen Port, den Sie für die Instanz erstellt haben.
      • Edit instance: Ändert den Instanznamen oder lädt die Sicherheitsgruppen hoch.
      • Update metedata: Aktualisiert die angepassten Metadaten, die Sie für die Instanz angegeben haben.
      • Edit Security Groups: Entfernt eine vorhandene Sicherheitsgruppe oder weist eine neue Sicherheitsgruppe zu.
      • Console: Zeigt die VNC-Konsole der Instanz an.
      • View Log: Zeigt das Instanzprotokoll an.
      • Pause instance: Hält die Instanz an und speichert deren Status im Hauptspeicher. Bei dieser Option werden die Ressourcen (vCPUs und Hauptspeicher) beibehalten.
      • Suspend instance: Setzt die Instanz aus und speichert deren Status auf der Platte. Bei dieser Option werden die Ressourcen (vCPUs und Hauptspeicher) freigegeben.
      • Shelve instance: Stellt das Image zurück und beendet die Instanz.
      • Resize instance: Aktualisiert die Varianteneinstellung der Instanz.
      • Lock instance: Sperrt die Instanz.
      • Unlock instance: Entsperrt die Instanz.
      • Soft Reboot Instance: Führt eine ordnungsgemäße Beendigung und einen Neustart der Instanz durch.
      • Hard Reboot Instance: Führt die einem Kaltstart des Servers funktional entsprechende Aktion aus.
      • Shut off Instance: Schaltet die Instanz aus.
      • Rebuild Instance: Erstellt die Instanz mit einem anderen Image erneut.
      • Delete Instance: Löscht die Instanz.

      Klicken Sie auf die Option für die Operation, die Sie anwenden müssen.
