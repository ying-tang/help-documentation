      管理块存储器

      • by Ying Tang ● November 21, 2016

      • Tags:

        • cinder

      本文根据以下 Bluemix 博客帖子进行改编：Managing Cinder Volumes with IBM Bluemix Private Cloud。

      IBM Bluemix Private Cloud 提供多个存储器模型以满足客户的独特需求。如果您具有在 Bluemix Private Cloud 上托管的可扩展文件系统和数据管理系统之类的工作负载，那么块存储器可能非常适合特定于您环境的用例。本文阐述如何开始使用块存储器模型。

      OpenStack Cinder

      块存储器是可连接到虚拟服务器以及与虚拟服务器拆离的独立持久存储器。借助块存储器，数据持久存储时间可超过虚拟服务器的生命周期。例如，如果虚拟服务器意外停止，那么块存储器中存储的数据将受到保护且保持不变。

      因为 Bluemix Private Cloud 由 OpenStack 提供支持，我们提供 OpenStack Cinder 作为客户的块存储器选项。Cinder 支持开发人员使用 OpenStack Horizon 仪表板 UI 和标准化 OpenStack API 来管理其块云硬盘。

      对于运行 IBM Bluemix Private Cloud V3.1.0 和更高版本的客户，Cinder 支持多个后端：其调度程序可灵活地查找适合的云硬盘类型。默认情况下，Cinder 块存储器配置为使用 Ceph 集群。

      Ceph 如何与 Bluemix Private Cloud 一起使用

      Ceph 是基于 Reliable Autonomic Distributed Object Store (RADOS) 的分布式存储解决方案，可在单个统一的存储器集群中为应用程序提供对象、块和文件系统存储器。

      Ceph 集群通常由监视节点和存储节点（OSD 节点）组成。监视节点保留集群映射的副本以确保高可用性，并且跟踪集群运行状况。

      借助 Ceph，Bluemix Private Cloud 以增加的容量、优异的性能和更低的成本提供块存储器选项。根据您的特定工作负载需求，您还可以针对 Ceph 集群自定义以下节点类型：

      • 使用仅含 SSD 的 OSD 磁盘和并置 Ceph 日志的仅 SSD 选项具有最佳 I/O 性能。它适合预计高 IOPS 的存储工作负载。

      • 使用 SATA OSD 磁盘和基于 SSD 的 Ceph 日志的混合选项具有良好的 IO 性能和更低的成本，适合中等 IOPS 工作负载。

      • 混用仅 SSD 和混合节点的多层选项非常适合大型环境。

      在使用适合的多后端和多层选项配置块存储器后，您即准备好使用 OpenStack Horizon 仪表板或 OpenStack Cinder 客户端创建 Cinder 云硬盘。

      使用 OpenStack API 创建 Cinder 云硬盘

      要使用 OpenStack 客户端创建云硬盘，请运行以下 Cinder 命令：

      $  cinder create <size> --display-name <name> --volume-type <type> \
--image-id <imageid> --availablity-zone <az>

      以下命令显示如何为 Bluemix Private Cloud 自定义新创建的云硬盘的示例。

      • <size>：云硬盘的大小（以 GiB 为单位）。
      • <name>：云硬盘的名称。
      • <type>：CEPH_HYBRID 或 CEPH_SSD。您可以使用 cinder type-list 命令来获取可用云硬盘类型的列表。
      • <imageid>：（可选）指定想要用于云硬盘的镜像 ID。您可以使用 nova image-list 命令来列出现有镜像。
      • <az>：（可选）指定想要在其中创建云硬盘的可用性区域。您可以使用 cinder availability-zone-list 命令来获取可用性区域的列表。

      有关本主题的更多信息，您可以参阅以下文档：使用命令行创建云硬盘。

      使用 Horizon 创建 Cinder 云硬盘

      要使用 Horizon（OpenStack 仪表板）创建块存储器云硬盘，您可以从以下步骤开始：

      1. 登录到 Horizon 仪表板。

      2. 在项目部分下，选择计算，然后单击云硬盘。如果您目前具有云硬盘，那么可从仪表板 UI 查看云硬盘列表及其详细信息（例如，名称、描述、大小和状态）。

      单击 Horizon 仪表板右上方的创建云硬盘。这样会显示“创建云硬盘”对话框。

      提供有关云硬盘的以下详细信息：

      • 云硬盘名称：输入云硬盘的名称。
      • 描述：输入云硬盘的描述。
      • 云硬盘源：无源、空白云硬盘或镜像。如果选择空白云硬盘，那么空白云硬盘不包含文件系统或分区表。如果选择镜像选项，那么将显示新字段“使用镜像作为源”，并且您可以从列表中选择镜像。
      • 类型：Ceph_SSD 或 Ceph_Hybrid。
      • 大小 (GiB)：选择云硬盘的大小（以 GiB 为单位），例如，1。
      • 可用性区域：选择可用性区域，例如，nova。

      在完成后，单击创建云硬盘。现在，您就能够将 Cinder 云硬盘连接到虚拟服务器。

      有关本主题的更多信息，您可以参阅以下文档：从 Horizon 仪表板创建云硬盘。

      在创建 Cinder 云硬盘并将其连接到虚拟服务器后，您可以随容量需求增长轻松调整和扩展云硬盘大小。您还能够拆离块云硬盘并将其重新连接到需要持久存储器的其他虚拟服务器，同时存储的数据仍受保护。

      故障诊断

      openstack volume list –name vs. cinder list –name

      如果您是具有非 cloud_admin 角色的用户，那么 openstack volume list --name 命令无法为您正确运行。要避免此问题，您可以让具有 cloud_admin 角色的用户按名称过滤云硬盘，或者也可以使用适合任何用户的 cinder list --name 命令。我们已针对此问题提出了内部缺陷 374319，并且我们正在与工程团队一起解决此问题。

      在运行 cinder 或 openstack 云硬盘命令时，发生找不到 volumev2 服务的 publicURL 端点错误

      如果遇到此错误，那么通常是因为您在当前集群中无任何块存储器（Cinder 云硬盘）节点。如果想要订购块存储器，请建立申请问题单，我们的销售团队将会帮助您添加该存储器。

      找不到 volumev3 服务的 publicURL 端点…

      在端点列表中包含先前版本的云硬盘服务时，可能会发生此错误。要解决此错误，请向 RC 文件添加 export OS_VOLUME_API_VERSION=2。

      为何 Cinder 中分配的所有云硬盘的总大小与 Ceph 的原始总大小不匹配？

      如果您具有在 Ceph 中已删除但仍存在于池中的云硬盘，并且在删除这些云硬盘之前从中创建了其他云硬盘，那么可能会发生此情况。Ceph 的工作方式为，克隆的云硬盘是从中进行克隆的云硬盘的写入时复制子云硬盘，这意味着必须保留父云硬盘直至删除所有子代（以及孙代和重孙代等）。既然如此，使用量将不等于当前活动云硬盘大小之和。

      可在以下地址找到有关 Ceph 写入时复制分层的更多信息：http://docs.ceph.com/docs/hammer/dev/rbd-layering/
