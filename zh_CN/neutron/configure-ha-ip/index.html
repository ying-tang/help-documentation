      如何配置高度可用的 IP

      • by Eric French ● September 21, 2016

      • Tags:

        • neutron

        • neutron

        • 配置 ip

        • ha

        • 高度可用

        • 允许的地址对

        • l2 pop

      本文提供了有关如何在 IBM Bluemix Private Cloud Local 环境中设置 HA 浮动 IP 的资源。这些指示信息假定您已安装 OpenStack Python API 客户端。

      注：此功能仅适用于 IBM Bluemix Private Local Cloud。它不适用于 IBM Bluemix Private Cloud (Dedicated Cloud) 产品服务。IBM Bluemix 数据中心网络当前要求使用 L2 POP，这是旨在支持向大型网络扩展的驱动程序。引用我们的一位工程师所说的话：

      “为了使 VXLAN 能够在 IBM Bluemix 数据中心网络上正常运行而需要使用的 L2Population 机制驱动程序，与 HA IP 方法所使用的 MAC 学习或 ARP 响应不兼容。IP 通过两种转换机制映射到主机：ARP 将 IP 映射至 MAC 地址，交换机了解与每个端口关联的 MAC。当发生故障转移时，任何 HA IP 都必须更改这两种映射中的至少一种。”

      作为一种可行的变通方法，在 OpenStack Mitaka 发行版中，有一个禁用一半 L2 POP 响应程序的选项，该选项允许您设置免费 ARP 类型的 HA。（请参阅此文：https://review.openstack.org/278597）如果您确实需要 HA，最好对其使用负载均衡器，例如，Neutron LBaaS v2。另一种可能的变通方法是，如果您的云部署在单个 SoftLayer Pod 中，并且将永不扩展至此单个 Pod 外，那么可以自定义方式实施某些类型的 HA IP。
