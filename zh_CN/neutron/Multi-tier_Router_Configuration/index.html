      IBM Bluemix Private Cloud 中的多层路由器配置

      • by Ulysses Kanigel and Leslie Lundquist ● August 31, 2016

      • Tags:

        • neutron

        • neutron

        • 网络

        • 路由器

        • 配置

        • 多层

        • add_interface

      本文解释了如何测试多层路由器配置，以及为何需要此配置。

      在以下某个用例中，您可能想要为云设置多层路由器配置：

      1. 您想阻止这些（子代）子网上的主机使用浮动 IP。
      2. 您想要为这些（子）网络创建网络隔离，用于将其与云中的其他项目或租户网络隔离，这通过在主路由器下为其中每个（子）网络提供各自的（子）路由器来实现。
      3. 您需要每个被隔离的项目网络拥有因特网入口和出口，同时仍然与云中其他网络隔离，并且不使用浮动 IP。

      初始建立多层网络的方法是建立从主路由器到每个“子”路由器的静态路由。这种测试的主要构想是确保这些静态路由为持久路由。在此示例中，我们设置两个此类型的子网络，分别称为 child-net1 和 child-net2。

      假设您要测试多层路由器配置，并确认您设置的路由器在故障转移期间持久存在。

      以下是您执行此测试所需的步骤：

      
# 创建网络和子网

neutron net-create parent-net

neutron net-create child-net1

neutron net-create child-net2

neutron subnet-create parent-subnet 162.16.0.0/24

neutron subnet-create child-subnet1 162.16.1.0/24

neutron subnet-create child-subnet2 162.16.2.0/24

# 创建路由器

neutron router-create parent-router

neutron router-create child-router1

neutron router-create child-router2

# 设置外部网络的父路由器的网关，并在 parent-router 子网上添加接口

neutron router-gateway-set parent-router external

neutron router-interface-add parent-subnet b490f691-364f-4f7c-866c-1c45e7b8f8cf  # 子网的唯一标识

# 为 parent-router、child-router1 和 child-router2 子网添加接口

neutron port-create parent-router

neutron port-create child-router1

neutron port-create child-router2

neutron router-interface-add child-subnet1 port=c1b305a1-f634-4828-a6ce-d5a6a058824e  # 端口的唯一标识

neutron router-interface-add child-subnet1 2476ad51-e2cb-4d4e-bbd8-db18cc06ea88   # 子网的唯一标识

neutron router-interface-add child-subnet2 port=c1b467b1-f444-4325-a5dc-d4b3a04624d   # 端口的唯一标识

neutron router-interface-add child-subnet2 2378ad51-e4cc-2d3e-bcc5-db34cd05fa48    # 子网的唯一标识

# 将子路由器上的默认路由设置为 parent-router 子网网关地址：它们都与父路由器对话

neutron router-update child-router1 --routes type=dict list=true destination=0.0.0.0/0,nexthop=162.16.0.1

neutron router-update child-router2 --routes type=dict list=true destination=0.0.0.0/0,nexthop=162.16.0.1

      应可轻松扩展此设计以包含多个连接到每个子路由器的子网，因为路由是在子路由器上实现的。

      注：请记住，连接到子路由器的任何云主机都将无法使浮动 IP 与之关联，因为此子路由器并未直接连接到外部网络。

      故障诊断：“错误：无法执行 add_interface：”

      如果您尝试将某个接口连接到 tenant-B-network 的 tenant-A-router，那么可能遇到此错误：错误：无法执行 add_interface：请求失败：处理您的请求时发生内部服务器错误。

      我们的 Neutron 策略实际上允许将路由器接口添加到具有 admin、cloud_admin 或 project_admin 角色的用户，但失败原因是 Neutron API 已显式编码为不允许对除您已通过其认证的租户以外的其他租户所拥有的资源执行操作，除非您具有 admin 角色，如以下代码中所示：

      https://github.com/openstack/neutron/blob/stable/mitaka/neutron/api/v2/attributes.py#L971

      如果您需要的网络来自连接到其他租户中路由器的某个租户，支持团队将需要代表您执行此操作。请提供您尝试运行的抛出错误的 CLI 命令，我们将为您运行此命令。
