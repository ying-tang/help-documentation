      IBM Bluemix Private Cloud 客户

      发行说明 3.0.0

      2016 年 6 月 30 日

      本文档提供了最新 IBM Bluemix Private Cloud 产品服务的基本技术描述，包括 Dedicated 和 Local 产品服务的细节。可在每个部分中找到指向详细文档的链接。

      最新发行版摘要

      IBM Bluemix Private Cloud R3.0.0 包含对基于其构建每个 Bluemix Private Cloud 的底层 OpenStack 平台的许多更新，以及对 Bluemix Private Cloud 和 Bluemix Private Cloud Local 的基础架构和功能的升级。最值得注意的是迁移至 OpenStack Mitaka 发行版，此发行版支持许多新功能。我们包含了一些常规产品改进，例如性能基准和经过更新的 Rally 测试套件，旨在实现更顺畅的操作和部署。此发行版还包含对文档进行的许多改进。

      每个 IBM Bluemix Private Cloud V3.0.0 都具有以下新功能：

      1. Openstack Mitaka 支持。
      2. 在多个 Bluemix Private Cloud 之间实现 Keystone-to-Keystone 联合身份。
      3. 经过更新的文档 Web 站点现在可通过 Box Panel 访问。
      4. 针对 IBM Bluemix Private Cloud 镜像的定期更新。

      IBM 云数据中心内的 IBM Bluemix Private Cloud 3.0.0 具有以下特性：

      1. 支持 IBM 云数据中心 VLAN 生成和连接至客户裸机硬件。
      2. 具有 10G 网络的 Vyatta。

      IBM Bluemix Private Cloud Local 3.0.0 具有以下特性：

      1. 混合块存储器节点。
      2. 新的计算节点类型。
      3. 支持 HIPAA 的 IBM Bluemix Private Cloud Local。
      4. 支持 Bluemix 本地系统平台（Beta 发行版）上的 IBM Bluemix Private Cloud Local。

      此发行版的已知限制

      要获取其他用户文档，请参阅客户帮助主页，可通过 help.blueboxcloud.com 随时在线访问此页面。

      提供了有关 IBM Bluemix Private Cloud V2.0 和更高版本的常规产品概述。

      每个 IBM Bluemix Private Cloud 3.0.0 都具有以下新功能：

      Openstack Mitaka 支持：基于迁移至 OpenStack Mitaka，一些功能现已可用。

      • 现在可以使用 Heat 模板和 Horizon 仪表板设置自动扩展组和 LBaaS (v2)。
      • Mitaka 发行版引入了全新的 aodh 警报服务，可安装在启用 Ceilometer 的任何环境中。
      • 有关更多信息，请参阅 OpenStack Mitaka 发行说明。

      在 Mitaka 发行版中，为每个 IBM Bluemix Private Cloud V3.0.0 启用特定 OpenStack 服务：

      • 块存储器 API 和扩展 (Cinder v2.0)
      • 计算 API 和扩展 (Nova v2.1)
      • 仪表板 (Horizon) API（不适用）
      • 身份 API 和扩展 (Keystone v3.0)
      • 镜像服务 API (Glance v2.2)
      • 网络 API 和扩展（Neutron v2.0 和 Neutron LBaaS v2）
      • 对象存储器 API 和扩展 (Swift v1.0)
      • 编排 API (Heat v1.0)
      • 遥测 API (Ceilometer v2.0)（和 aodh）

      在多个 IBM Bluemix Private Cloud 之间实现 Keystone-to-Keystone 联合 Keystone-to-Keystone (K2K) 联合支持您使用某个 Keystone 身份提供者上存储的用户凭证登录多个云。我们已为您撰写了 K2K 设置文章。

      经过更新的文档 Web 站点现在可通过 Box Panel 访问：我们已更新并重新设计了客户文档 Web 站点，此站点现在位于 help.blueboxcloud.com 上，可通过 Box Panel 界面轻松访问。请参阅 Box Panel 用户指南以获取更多信息。

      针对 IBM Bluemix Private Cloud 镜像的定期更新：针对受支持的 IBM Bluemix Private Cloud 镜像的定期更新现在可通过 Box Panel 界面获取。所有个别镜像更新的客户发行说明都在此处发布。我们还撰写了关于云镜像使用的用户指南。

      IBM 云数据中心内的 IBM Bluemix Private Cloud 3.0.0：

      支持 IBM 云数据中心 VLAN 生成和客户管理的裸机硬件连接：此功能是为以下类型的任意 IBM Bluemix Private Cloud 客户设计的：裸机服务器性能成为一项严格要求，但也需要通过 IBM 云数据中心的专用网络访问其裸机基础架构与 OpenStack 云主机网络之间的安全通信。注：要使用此功能，裸机硬件必须位于独立的、客户拥有且管理的 SoftLayer 帐户中。

      具有 10G 网络的 Vyatta：此功能是为对云出口和入口具有高带宽需求的客户设计的。

      IBM Bluemix Private Cloud Local 3.0.0：

      新的计算节点类型：扩展计算节点可满足您对更快临时 IO、更多 RAM 以及更大数量级临时存储的需求。扩展节点支持更好地覆盖云主机类型和工作负载。通过此发行版，计算类型可分组为“可用性区域”，从而便于将工作负载定位到相应的计算类型。 您可以从以下计算节点扩展中进行选择，以添加到您的 IBM Bluemix Private Cloud Local 中：

      • 您可以选择专业的扩展计算节点：IO 或 HIGH STORAGE（此发行版中的新增功能）
      • 您可以选择其他标准或企业计算节点：STANDARD COMPUTE 或 ENTERPRISE COMPUTE（先前可用）
      • 您可以选择 POWER 计算节点 (Habañero)：这些节点尤其支持 IBM Watson 应用程序和其他应用程序。

      混合块存储器节点：混合块存储器节点提供更低成本和更高密度的块存储器。我们添加了混合 Ceph 存储器节点类型，以提供更低成本且更高密度的块存储器。它的容量是同价位 SSD 的 4 倍，具有相同的软件堆栈和引用架构。请注意，每个集群只能使用一种类型的块存储器：SSD 块存储器和混合块存储器不能在同一个集群中混用。

      • 您可以选择块存储器：SSD 或混合

      支持 HIPAA 的云：支持 HIPAA 的云的配置和扩展选项与任何其他 IBM Bluemix Private Cloud 产品服务相同，但我们启用超过 50 个额外的信息安全控件，为必须存储个人健康信息 (PHI) 的客户提供支持。由于增加了信息安全功能，支持 HIPAA 的云必须从一开始就支持 HIPAA：不能将现有云升级为支持 HIPAA 协议。另外，您无法从支持 HIPAA 的 Bluemix Private Cloud Local 降级：现有 HIPAA 云必须按 HIPAA 协议擦除，您必须订购不含任何 HIPAA 控制器的新的 Local Cloud。

      • 现在，您可以为 IBM Bluemix Private Cloud Local 选择 HIPAA 支持。

      支持 Bluemix 本地系统（Beta 发行版）上的 Bluemix Private Cloud：Bluemix Local 是现有的 IBM 硬件管理系统（以前称为 PureApp）。通过其 Beta 发行版，我们将其作为 IBM Bluemix Private Cloud Local 的平台提供，作为已经可用的 Cisco 硬件的补充。

      此发行版的已知限制

      Horizon 仪表板和 LBaaS v2：

      • LBaaS v2 Horizon 仪表板将不允许您创建 https 侦听器。
      • 无法通过 Horizon 界面使用池共享。
      • 您无法通过 Horizon 将成员权重设置为 0。
      • L7 内容切换对默认驱动程序不可用。(HaproxyOnHostPluginDriver)
      • 不支持 TLS 终止。

      针对 LBaaS v2 的 OpenStack 社区未解决的问题：

      • 默认情况下，LBaaS VIP 与默认安全组关联。变通方法是在默认组中打开所需端口。https://bugs.launchpad.net/neutron/+bug/1295424
      • 删除 LBaaSv2 无法删除 LBaaS 名称空间。https://bugs.launchpad.net/neutron/+bug/1495430
      • 运行状况监视器状态不会反应运行状况监视器运行成功还是失败。https://bugs.launchpad.net/neutron/+bug/1464229
      • PING 监视器发送 HTTP 请求。当针对 PING 配置 LBaaS v2 运行状况监视器时，它会发送 HTTP 请求。https://bugs.launchpad.net/neutron/+bug/1463816
      • 如果已删除 LBaaSv2 侦听器，将无法删除其运行状况监视器。变通方法是从命令行删除孤立的运行状况监视器。https://bugs.launchpad.net/neutron/+bug/1571097
      • 可删除 lbaas-pool，但 lbaas-hm 仍处于连接状态。发生此情况时，即使 admin 也无法删除剩余的运行状况监视器，除非直接进入数据库。https://bugs.launchpad.net/neutron/+bug/1526964

      Horizon 仪表板和项目：

      • 在 Horizon 仪表板上，cloud_admin 无法在“项目编辑方式”窗口中查看配额编辑页面。仍允许 cloud_admin 使用 OpenStack CLI 或 API 修改配额。
      • 在 Horizon 仪表板上的“身份面板”中，以 cloud_admin 角色创建新项目时，会显示一个错误弹出菜单 (Unable to set project quotas)。但仍会创建此项目并且此项目会收到默认配额。具有 cloud_admin 角色的用户可随后使用 OpenStack CLI 或 API 更新配额。
      • 在 Horizon 仪表板上的“身份面板”中，以具有 cloud_admin 角色的用户身份修改现有项目时，会显示一个错误弹出菜单 (Unable to modify project)。事实上将更新此项目。显示弹出菜单的原因是无法设置项目配额。

      Horizon 仪表板和管理面板：

      • 在 Horizon 仪表板上的“管理面板”中，以具有 cloud_admin 角色的用户身份单击虚拟机管理器时，会显示一个错误弹出菜单：“Unable to get Nova services list”。此行为是预期行为，因为不允许具有 cloud_admin 角色的用户对这些项执行操作。

      其他已知限制：

      • Neutron RBAC-create：CLI 命令 neutron rbac-create 返回“500”错误状态码，除非在初始创建点指定了 --target-tenant。
      • 当在具有混合 Ceph 配置 (HDD + SSD bcache) 的 30 节点性能测试环境中进行测试时，存在块存储器性能限制
      • Keystone-to-Keystone 联合：由联合用户创建 trust-for-trust 授权目前存在问题。为了避免此问题，须向联合用户显式分配 heat_stack_owner 角色。
