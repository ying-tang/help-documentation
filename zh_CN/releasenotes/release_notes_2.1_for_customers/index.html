      IBM Bluemix Private Cloud 客户

      发行说明

      R2.1.0 创建日期 2016 年 3 月 31 日

      最新发行版中的新增功能摘要

      本文档提供了最新 IBM Bluemix Private Cloud 产品服务（即 Dedicated 和 Local Cloud 产品服务）的基本技术描述。IBM Bluemix Private Dedicated 和 Local Cloud 的新功能包括：

      1. 提供了额外类型的扩展计算节点。

      2. 提供了混合块存储器集群和扩展节点。

      3. 在 IBM Bluemix Private Dedicated Cloud 上提供了直接链接（有三种），适用于混合云应用程序。

      4. 提供了 IBM 云数据中心专用网络访问权。

      5. 使用基于角色的访问控制 (RBAC) 来定义权限。

      6. IBM Bluemix 提供了额外的云镜像。

      7. 现在针对 IBM Bluemix Private Dedicated Cloud 提供了符合 HIPAA 要求的选项。

      要获取其他用户文档，请参阅帮助主页。

      在此文档中提供了有关 IBM Bluemix Private Cloud V2.0 和更高版本的常规产品概述。

      在本文档末尾提供了将影响此发行版用户的先前发行版的功能摘要。

      IBM Bluemix Private Cloud 构建块

      每个 IBM Bluemix Private Cloud 2.1.0（无论是 Dedicated 还是 Local）都从控制器和计算节点集群开始：

      IBM Bluemix Private Cloud：

      • 云能够以一个三节点集群开始，其中包含两个超融合节点（控制器 + 计算节点）和一个仅限计算的节点。
• 云能够以一个五节点集群开始，其中包含一对仅限专用控制器的节点和三个仅限计算的节点。

      IBM Bluemix Private Cloud Local：

      • 每个云都必须具有至少一个仅限专用控制器的节点和三个仅限计算的节点。

      在上述任意起点（例如，3 节点超融合或含专用控制器的 5 节点）中，都可以添加任意类型的计算节点以及块存储器 (Ceph) 和/或对象存储器 (Swift)。

      其他构建块：

      • 对于标准容量 Dedicated Cloud，默认情况下包含专用网关防火墙
• 企业容量包含两倍资源和 10 Gbps 的网络连接
• 专用控制器节点（或额外节点）
• 计算节点（或额外节点）：标准或企业，IO 或高存储容量（新增）

      可选扩展计算节点现已可用

      （可选）您可以从以下计算节点扩展中进行选择，以添加到您的标准云或企业云中：

      • 您可以选择专业的扩展计算节点：IO 或 HIGH STORAGE（此发行版中的新增功能）
• 您可以选择其他标准或企业计算节点：STANDARD COMPUTE 或 ENTERPRISE COMPUTE

      扩展计算节点可满足您对更快临时 IO、更多 RAM 以及更大数量级临时存储的需求。这两种类型的扩展节点 - IO 优化和高存储容量 - 都支持更好地覆盖云主机类型和工作负载。通过此 2.1.0 发行版，计算类型可分组为“可用性区域”，从而便于将工作负载定位到相应的计算类型。

      可选混合块存储器节点现已可用

      混合块存储器节点提供更低成本和更高密度的块存储器。

      • 您可以选择块存储器：SSD 或混合
• 您可以选择对象存储器

      直接链接在 IBM 云数据中心内现已可用

      对于 Dedicated Cloud，您具有选择直接链接连接功能的额外网络选项：

      • 您可以选择访问方式（公共、VPN 或直接链接）

      在 IBM 云数据中心内提供直接链接。它支持您连接到 IBM 云数据中心的专用网络主干或网络，绕过公共因特网以连接到 Dedicated Cloud。

      如果您要将自己的 Local Cloud 链接到 IBM 云数据中心内运行的 Dedicated Cloud，直接链接可用于创建混合云。它可按 1Gbps 和 10Gbps 增量提供。

      可用内容：

      • IBM Bluemix Private Direct Link Cloud Exchange 1 Gbps
• IBM Bluemix Private Direct Link Cloud Exchange 10 Gbps
• IBM Bluemix Private Direct Link Colocation 1 Gbps
• IBM Bluemix Private Direct Link Colocation 10 Gbps
• IBM Bluemix Private Direct Link Network Service Provider (NSP) 1 Gbps
• IBM Bluemix Private Direct Link Network Service Provider (NSP) 10 Gbps

      IBM 云数据中心专用网络访问

      如果将要连接的两个 IBM Bluemix Private Cloud 部署在同一个 IBM 云帐户下，IBM 云数据中心专用网络访问支持您使用 IBM 云数据中心专用网络功能，在这两个 IBM Bluemix Private Cloud 之间设置连接。您可以选择此服务以在 IBM Bluemix Private Cloud 之间启用低延迟、未计量的专用网络流量。

      有关更多信息，请参阅有关本主题的主要文档。

      要了解已知限制，请查看此处。

      基于角色的访问控制 (RBAC)

      默认情况下已创建多个角色，在云中为您提供特定访问功能。此版本的 IBM Bluemix Private Cloud 引入了组的概念。有关用户、角色和组的更多信息，请参阅本文档。

      其他云镜像可用

      IBM Bluemix Private Cloud 2.1.0 预填充了 Cirros 0.3.3 x86_64 镜像。提供了其他云镜像，由 IBM Bluemix 出于善意为客户提供。包括：

      • CentOS 6.7 x86_64
• CentOS 7.2 x86_64
• Ubuntu Server 12.04 LTS x86_64
• Ubuntu Server 14.04 LTS x86_64
• Windows Server 2008 R2 SP1 Datacenter
• Windows Server 2008 R2 SP1 Enterprise
• Windows Server 2008 R2 SP1 Standard
• Windows Server 2012 R2 Datacenter
• Windows Server 2012 R2 Standard

      本文档中提供了有关使用其中任意镜像的更多信息。

      HIPAA 和 IBM Bluemix Private Cloud

      • 您可以为自己的 IBM Bluemix Private Cloud 选择 HIPAA 支持

      支持 HIPAA 的云的配置和扩展选项与任何其他 IBM Bluemix Private Cloud 产品服务相同，但我们启用超过 50 个额外的信息安全控件，为必须存储个人健康信息 (PHI) 的客户提供支持。由于增加了信息安全功能，支持 HIPAA 的云必须从一开始就支持 HIPAA：不能将现有云升级为支持 HIPAA 协议。另外，您无法从 IBM Bluemix Private HIPAA Cloud 降级：现有 HIPAA 云必须按 HIPAA 协议擦除，您必须订购不含任何 HIPAA 控制器的新的云。

      本质上，IBM Bluemix Private HIPAA Cloud 与任何其他 IBM Bluemix Private Cloud 都是相似的，但具有以下例外：

      1.  每个 IBM Bluemix Private HIPAA Cloud 都必须具有专用控制器（非融合控制器）。
2.  额外 HIPAA 每月定期费用根据集群和管理的资产而累积增加。
3.  支持额外（超过 50 种）控件，您与 IBM Blue Box 共同负责这些控件。

      IBM Bluemix Private HIPAA Cloud 可部署在已准备好部署 IBM Bluemix Private Cloud 的任何 IBM 云数据中心内。

      先决条件：必须先签署 BAA，然后才能开始订购 IBM Bluemix Private HIPAA Cloud。您必须使用 HIPAA 客户的最低配置开始订购云。启动服务后，无法再完成 HIPAA 支持。

      将影响客户使用此发行版的先前发行版中的新功能摘要

      本部分简要列出了 IBM Bluemix Private 先前发行版（主要为 2.0.1 发行版）的功能，它们将影响您使用此 2.1.0 发行版（尤其是当您从 1.x 发行版升级时）的体验。有关这些功能的更多信息，请联系您的 IBM Bluemix Private Cloud 支持团队。

      将影响此发行版中的用户、组和角色的功能：

      对 cloud_admin 隐藏“主机聚集”面板：此 Horizon 面板已隐藏，对 cloud_admin 用户不可见。我们将不再允许 cloud_admin 操作主机聚集，因此不适合在 Horizon 中显示此面板。

      禁止 cloud_admin 管理主机聚集：已涵盖在上述有关 Horizon 的“主机聚集”面板的讨论中。

      向 cloud_admin 显示更多 admin 面板：我们在 Horizon 中为 cloud_admin 用户启用了更多管理面板。包括“概述”、“虚拟机管理器”、“云主机”、“云硬盘”、“云主机类型”、“镜像”和“默认值”。

      在命令行上修复了按名称管理用户的问题：我们修复了阻止从命令行按名称管理用户的缺陷。以前，许多按名称引用用户的尝试都会导致错误。

      允许 cloud_admin 公开 Glance 镜像：现在，cloud_admin 角色具有公开 Glance 镜像的特权。

      将影响此发行版中的 Horizon、Glance、Nova 和 Heat 的功能：

      刷新 Horizon 代码库以修复云硬盘备份错误：Horizon 中阻止创建 Cinder 云硬盘备份的缺陷已得到解决。

      刷新 Horizon 代码库以允许从 Swift URL 上传镜像（经过组织的 IBM Bluemix 镜像交付）：Glance 中阻止通过 URL 上传镜像的缺陷已得到解决，允许使用来自 Box Panel 的 URL 将镜像上传至 Glance 中。

      添加 DiskFilter 用于 Nova 调度：我们解决了可能会导致在没有足够可用磁盘空间来满足构建要求的虚拟机管理器上构建 Nova 调度的缺陷。

      调优 Heat 以提供大型模板支持：bugs.launchpad.net/heat/+bug/1491185：我们解决了阻碍使用大型模板进行 Heat 堆栈部署的缺陷。

      支持云主机大小调整（增大）：云主机大小调整允许云主机将其云主机类型更改为更大的云主机类型。调整大小操作并非实时操作：它将停止并重新启动云主机。在本文档中提供了更多信息。

      未来发行说明

      发行说明将定期发布，提供有关更改内容及其对您的 IBM Bluemix Private Cloud 产品服务使用体验有何影响的详细信息。例如，安装可能启用新功能的新版本 OpenStack 时，您将收到包含详细信息的发行说明。
