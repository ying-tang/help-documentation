      管理用户和项目

      基本概念

      用户：用户是具有 OpenStack 环境登录凭证的个人或服务。一个用户可以具有一个或多个角色，属于一个或多个项目。

      项目：项目是云中的组织单元，也称为租户或帐户。每个用户可以是一个或多个项目的成员。在一个项目内，用户负责创建并管理各种资源，例如云主机和云硬盘。管理员可以对允许一个项目创建的资源（云主机、vCPU、云硬盘等）量设置限制。

      角色：角色是一组特权，这些特权允许用户执行一组特定的操作。

      组：组充当用户与角色之间的纽带。用户可以是一个或多个组的成员，组可以具有一个或多个关联的角色。

      IBM Bluemix Private Cloud 安装带有四个预定义角色：cloud_admin、project_admin、_member_ 和 heat_stack_owner。第五个角色 heat_stack_user 是自动分配的。

      |角色| 描述 |:———|:———– | cloud_admin | 该角色支持云级别的访问控制。此角色能让您执行 API 执行任务，与项目无关。此角色可以创建和管理配额、组、用户和项目，并且可以执行管理云硬盘操作。| project_admin | 此角色支持项目级别的访问控制。此用户可以在指定项目内执行用户和项目管理。| _member_ | 此角色能让用户使用分配给项目的资源（如云主机和云硬盘）。| heat_stack_owner | 此角色能让用户在项目中创建 Heat 堆栈。此角色必须手动分配，并且不得分配给同时分配了 heat_stack_user 角色的用户。| heat_stack_user | 这是由 Heat 自动分配给其所创建用户的角色。所有 API 访问均限制此角色，它不得显式分配给任何用户。 cloud_admin 角色限制为只能执行云和项目管理操作。将专门向 admin 角色授予一组更高的权限，包括针对服务、域和角色、网络以及身份提供者的管理操作。admin 角色仅供 IBM 使用，不会授予客户。

      在安装 IBM Bluemix Private Cloud 时，您将收到一个拥有 cloud_admin 特权的用户，该用户可以创建其他项目和用户。cloud_admin 可以为用户授予 cloud_admin 和更低特权的角色。

      使用 Keystone V3 组功能

      IBM Bluemix Private Cloud 中的 Keystone API V3 引入了组的概念。使用组可以轻松地同时进行多项分配。

      例如，可能有一个支持团队，需要访问环境中的每个项目。 可以创建一个名为支持的组，为该组赋予针对所有项目的 _member_ 角色，然后向该组添加所有支持人员，而不需要分别向每个项目添加每个成员。

      另一个常见的用例是创建一个 cloud_admin 组，为该组提供针对环境中每个项目的 cloud_admin 角色。 此安排通过临时将用户添加到 cloud_admin 组，让 cloud_admin 轻松提升用户的特权。它可以用于假期代理之类的情况。

      禁用项目的影响

      具有 cloud_admin 角色的用户可以启用和禁用项目。 在禁用项目时，将产生以下影响：

      • 在仪表板中，用户不再能够通过“项目”选项卡上的“当前项目”列表来访问项目。
      • 只有已禁用项目的成员用户不再能够进行登录。
      • 您无法启动已禁用项目的云主机。不会自动终止已运行的云主机。必须手动执行停止操作。
      • 已禁用项目的数据会保留，因此可随时重新启用项目。

      使用仪表板

      接下来的部分将告诉您如何使用仪表板来添加、更新、修改和删除项目或用户帐户。

      添加项目

      1. 在导航栏中，打开“身份”面板，然后单击项目。
      2. 单击创建项目。
      3. 在“创建项目”窗口的“项目信息”选项卡上，输入项目的名称和描述。默认情况下，项目已启用。
      4. 在“项目成员”选项卡上，向项目添加成员并分配适当的角色。
      5. 在“项目组”选项卡上，向项目添加用户组并分配适当的角色。
      6. 在“配额”选项卡上，编辑配额值。
      7. 单击创建项目。“项目列表”显示项目，包括分配的 ID。

      更新项目

      您可以更新项目以更改其名称或描述，以及启用或临时禁用项目。

      1. 在导航栏中，打开“身份”面板，然后单击项目。
      2. 选择想要更新的项目。
      3. 从“操作”列下的下拉列表中选择编辑项目。
      4. 在“编辑项目”窗口中，可以更新项目以更改其名称或描述，以及启用或临时禁用项目。默认情况下，项目已启用。要临时禁用，请清除已启用复选框。要启用已禁用的项目，请选择已启用复选框。
      5. 单击保存。

      修改项目的用户分配

      如果您具有 cloud_admin 角色，那么可以分配对任何项目的角色。如果您具有 project_admin 角色，那么可以对应用 project_admin 角色的项目修改分配情况。

      1. 在导航栏中，打开“身份”面板，然后单击项目。
      2. 单击想要修改的项目的管理成员按钮。

      “编辑项目”窗口显示这些用户的列表：

      • 所有用户：可分配给当前项目的用户。
      • 项目成员：已分配给当前项目的用户。您还可以在“项目组”选项卡中分配用户组。

      删除项目

      1. 在导航栏中，打开“身份”面板，然后单击项目。
      2. 选择想要删除的项目。
      3. 单击删除项目以确认删除。

      创建用户帐户

      如果您具有 cloud_admin 或 project_admin 角色，那么可以创建新用户帐户。在创建用户帐户时，必须将帐户分配给主项目。您还可以选择将帐户分配给其他项目。

      1. 在导航栏中，打开“身份”面板，然后单击用户。
      2. 单击创建用户。
      3. 在“创建用户”窗口中，输入用户的用户名、电子邮件和初始登录密码。确认密码。从“主项目”列表中选择一个项目。从“角色”列表中为用户选择一个角色。默认角色为 _member_。
      4. 单击创建用户以确认更改。仪表板会向用户分配 ID，这样用户就会出现在“用户”类别中。

      禁用或启用用户

      如果您具有 cloud_admin 或 project_admin 角色，那么可以启用和禁用用户帐户。在禁用用户帐户后，用户就不再能够登录。但是，用户数据会保留，因此可随时重新启用该用户。

      1. 在导航栏中，打开“身份”面板，然后单击用户。
      2. 在“用户”列表中找到想要禁用或启用的用户。
      3. 从“操作”列下的下拉列表中选择禁用用户或启用用户。在“已启用”列中，“已启用”状态的值会更新为“是”或“否”。
