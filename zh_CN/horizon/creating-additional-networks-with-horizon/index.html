      使用 Horizon 面板在 OpenStack 中创建其他网络

      • by Jason Kennedy ● August 5th, 2015

      • Tags:

        • getting started

        • 入门

        • 其他网络

      您已经掌控了自己全新 OpenStack 云的重要内容，同时还明确要求使用 VXLAN 来设置，因为您希望将开发/测试环境与生产网络隔离开来。首先，您可以使用安全组执行此操作吗？不能？好，请继续阅读。

      典型 IBM Bluemix Private Cloud 默认情况下已设置内部和外部网络，并使用默认路由器（恰当命名为 default）进行桥接。要创建我们自己的网络，我们需要为此设置制作镜像。

      （免责声明：此处有风险！执行此操作前请确保您实际上正在使用 VXLAN - 否则无法生效。此外，有可能地球突然停止自转，把我们全都抛向太空。）

      首先，我们要创建网络。出于理智原因，我会同时展示命令行界面 (CLI) 和 Horizon 方法，以便尽可能提供帮助。我们先从 Horizon 开始，很快我会在另一个帖子中展示 CLI。

      在您的项目下，转到网络选项卡并选择网络。如前所述，您将看到“内部”和“外部”。

      在这里单击右上角的创建网络，以显示创建网络弹出屏幕。

      插入您的网络名称（将管理状态保留为 UP），然后单击子网选项卡（或选择下一步）。

      在此处插入您的子网名称（我使用的是 test_subnet，而 test 作为网络名称本身）、网络地址（我使用的是 192.168.10.0/28）和网关 IP（我的示例中为 192.168.10.1）。

      现在，单击子网详细信息或下一步。在此屏幕上，如果您仅仅分配一部分池，或您想指定 DNS 服务器或主机路由，请继续。

      只需从此处单击蓝色的创建按钮。您的网络部分现在应类似如下：

      现在我们来添加路由器。

      在导航中，单击网络下的路由器；您在此处应该看不到任何路由器，所以让我们单击创建路由器来创建一个路由器。此时将显示创建路由器对话窗口。您需要做的只是选取路由器名称（我们将使用 test_router）。单击创建路由器，您将看到以下内容：

      查看我们的路由器，我们可以看到路由器没有连接到外部网络，该路由器必须发挥一般路由器的作用。在右侧的操作下，让我们单击设置网关。这样会显示设置网关窗口；在外部网络下拉菜单下， 选择外部，然后单击蓝色的设置网关按钮。您的网关现已连接到外部网络。

      从这里我们可以退后一步，检查目前为止做了哪些操作。如果单击左侧的网络拓扑，您将看到您的网络现在已存在，您创建的路由器也同样存在。该路由器已按照计划连接到外部网络。现在剩下的唯一操作是将此路由器连接到您的网络，使其路由您的云主机流量。如果您将鼠标悬停在路由器上，可以从网络拓扑部分中看到有关此路由器的一些详细信息，包括添加接口和删除路由器功能。我们需要添加接口，但这个过程比较长：

      单击左侧的路由器，然后单击您的路由器。您现在应位于如下所示的路由器详细信息屏幕中：

      此屏幕提供了有关路由器的很多信息，我们可以看到外部网关接口已经添加。让我们单击添加接口，此时将显示添加接口对话框。在子网下拉菜单中，选择您创建的网络。您可以根据需要为此接口分配 IP 或让云进行分配。我们将保留 IP 地址空白，让云进行分配。单击蓝色的添加接口按钮完成操作。

      继续之前，可以从此处添加有关接口和 IP 地址的一些信息。您添加到网络的每个接口都需要 IP；将路由器连接到需要 IP 地址的外部网络；如上所示，您的外部网关接口在浮动 IP 范围内有一个固定 IP。您刚刚创建的内部接口应该具有自己的来自所创建内部网络池的固定 IP。请注意，状态最初为 DOWN，但完成后将转变为 ACTIVE。

      如果从此处回到左侧的网络拓扑，我们应该会看到外部网络和我们所创建网络之间的连接，它通过我们创建的路由器来连接并连接到每个网络。现在，我们应该能够在此网络上启动云主机，并在向其分配浮动 IP 后使其能够从外部进行访问。

      从项目下的计算中单击云主机。在右侧单击启动云主机，这将显示启动云主机对话窗口。插入云主机名称（此案例中为 test_instance），然后从云主机引导源下拉菜单中选择从镜像引导。我们要创建的是简单的 Cirros 云主机，因此请从镜像名称下拉菜单中选择 Cirros (12.6MB)。

      现在单击顶部蓝色的访问和安全性。您应该已经有密钥对；请确保选择该密钥对。另外，请确保您使用的是相应的安全组，然后单击顶部接下来的蓝色部分网络。

      这里，我们选择自己的测试网络。单击启动。您应该看到类似以下所示的内容：

      请注意，IP 地址 192.168.10.1 将用于路由器的内部接口网关，因此不能用在云主机中。单击云主机名称以进入概述。从此处单击控制台选项卡。显示控制台时，单击深蓝色的“已连接”部分，然后按 Enter 键，应该会向您显示登录提示。使用默认 Cirros 登录信息登录云主机（用户名为 cirros，密码为 cubswin:)）

      您应该能够 ping 外部地址并查找您的 IP。我们现在要做的只是为其分配一个浮动 IP。

      在操作下的右侧选择关联浮动 IP，这将打开管理浮动 IP 关联对话框。如果这是您第一次关联 IP，您必须选择下拉列表旁边的“+”图标，因为此列表将为空。从池中选择“外部”并单击蓝色的分配 IP，然后单击关联。您现在应该看到两个 IP 地址与您的云主机相关联。

      现在，您应该能够使用云主机的浮动 IP 和关联密钥通过 ssh 登录到云主机，使用您自己的路由器登录到自己网络上的云主机。
