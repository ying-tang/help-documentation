      如何使用静态 IP 创建云主机

      • by Jason Kennedy ● September 2, 2016

      • Tags:

        • openstack

        • openstack

        • 静态 ip

        • 云主机

      本文说明了如何在 OpenStack 中创建新端口，并在引导时将其手动附加到云主机以向云主机分配“静态”IP。

      首先，此方法的大部分操作都在命令行中执行，因为命令行是可与 IBM Bluemix Private Cloud 配合使用的通用型工具。必要时，需要进行少量备份并使用 OpenStack CLI & API 进行仔细检查，从而使此指令集看起来更合理一些。我们来直接介绍如何完成此任务。此任务将在测试集群上完成，但也可以在您自己的集群上的屏幕共享中重复这些步骤（如果能保证完成这些步骤）。

      首先，Bluemix Private Cloud 集群具有两个预先生成的网络：internal 和 external。为实现目标，我们想要在内部网络中创建此新端口，下面通过命令行快速完成此操作：

      bash-3.2$ openstack network list | grep internal
| 81e829d0-5e35-4808-9f7a-d22159faa9d1 | internal | c6560f00-ae29-40b5-93bb-6e7b1ffd3720

      这可获取内部网络的 ID。在讨论 CLI 一会儿后，我打算使用 python-openstackclient（“新版”客户端）来完成此任务。此客户端中较少使用 ID，但可以找到给定网络、租户或重要对象的 ID，因为在必要时可以使用旧版工具（python-novaclient 等）。但是，此时还是不要过于详细地讨论这一点。

      我们需要查看在此情况下要使用的子网，我们来看一看以下内容：

      bash-3.2$ openstack network show internal
