      为何选择 OpenStack？

      • by Leslie Lundquist ● April 14, 2016

      • Tags:

        • OpenStack

        • OpenStack

        • 参考

      OpenStack™ 是一个功能广泛的开源软件平台，用于提供“基础架构即服务”(IaaS)。OpenStack 由 Rackspace 和 NASA 从 2010 年开始合作开发。OpenStack 可管理数据中心内的小型或大型计算、存储和联网资源池，用户可通过一个集中式仪表板来使用和扩展这些资源。

      历史和愿景： OpenStack 是有史以来发展最快的开源项目，现在有超过 2000 万行代码。OpenStack 由 OpenStack 基金会管理，该基金会致力于促进 OpenStack 的开发、分发和应用。OpenStack 项目包括超过 34,000 名个人贡献者和超过 550 家参与公司，他们都是该基金会的会员。该基金会的个人会员资格是免费的。公司会员资格有两种付费级别：白金会员和黄金会员。该基金会由董事会管理，由个人和参与公司代表组成。

      许可和发行版：OpenStack 软件依据 Apache 许可证提供，每年发行两次（4 月/5 月和 10月/11 月）。发行版代码按字母顺序命名。最新发行版命名为“Ocata”。与此同时，正在开发下一个发行版。与一年两次的发行活动对应，OpenStack 社区每年召开两次 OpenStack 峰会，主要是规划下一个发行版并展示用例或成果。

      硬件需求：OpenStack 适用于标准或商用硬件，无需使用专门的供应商硬件。对于想要使用特定企业级硬件的客户，提供了相应的驱动程序。可以在 OpenStack Web 站点的 Marketplace 中找到这些驱动程序的列表。

      软件项目方法 (Big Tent)：从 Liberty 发行版开始，OpenStack 社区使用“Big Tent”暗指 OpenStack 软件开发。项目由社区成员提出，并受“Big Tent”保护，社区成员决定协作后便可获取资源。在初步阶段不会正式批准或正式拒绝任何项目。

      OpenStack 社区：OpenStack 的最大优势在于其社区充满活力。此社区拥有来自超过 177 个国家的 34,000 多名个人贡献者，在全球有 70 多个用户组。OpenStack 用户组会定期会面。他们在会场上分享彼此的观点、开展技术讨论并促进社区成员间的友好关系。如果您是 OpenStack 的新用户，那么加入这些用户组是从其他用户那里学到知识的最快方式。

      市场趋势和堆栈状态：OpenStack 正成为私有云部署的选择平台。其市场收入到 2016 年预计将达到 13 亿美元，到 2018 年将超过 30 亿美元。采用 OpenStack 的前三大驱动因素是：在 OpenStack IaaS 平台上快速创新 (DevOps) 的能力、实现云部署自动化以便于扩展的能力以及 OpenStack 是一项开放技术的事实。软件开发和测试是最常用的工作负载，其次是基础架构服务。企业工作负载不断增加，特别是在 OpenStack API 方面，这已成为“基础架构即服务”(IaaS) 的标准。从更广泛的角度来看，OpenStack 在学术研究、电信、金融、媒体以及其他市场领域的应用都在不断增加。

      OpenStack 文档：对于 OpenStack 的新用户，提供了适用于不同平台（Ubuntu、RHEL、Debian 和 SUSE）的安装指南。这些指南会随每个发行版一起更新。OpenStack 项目提供面向系统管理员、云管理员、最终用户、架构师和云开发人员的综合联机文档。同时还提供了有关高级主题（如安全性和高可用性）的文档。这些高级主题文档会定期更新，但不会与 OpenStack 发行版同步更新。

      Ask OpenStack：如果您对 OpenStack 有任何疑问，请在 Ask OpenStack（一个基于热门网站 StackOverflo 建模的综合问答站点）上咨询专家。

      OpenStack Marketplace：OpenStack 客户有很多选择：服务提供商、供应商、系统集成商、分销商、培训师和顾问等。为帮助您做出明智决策，OpenStack Marketplace 提供一个中央存储库，以用于存储与 OpenStack 供应商、产品、培训、顾问和其他服务相关的信息和评价。

      OpenStack：组件概述

      OpenStack 平台是一系列服务，细分为若干个功能组件。几个核心组件在您的数据中心内联合提供计算、对象存储器和块存储器以及联网功能，用户可通过一个集中式仪表板来管理所有这些资源。这些核心服务通过一组支持服务集成在一起，如本文档中的后续部分中所述。下图说明了 OpenStack 组件之间的关系。

      图 4：OpenStack 高级架构（按组件列出）

      OpenStack 基础架构即服务 (IaaS)：核心项目

      通常，核心项目是 OpenStack 社区最先开发的一批项目。这些项目已在生产环境中使用多年且经过了测试，目前仍然是部署最广泛的 OpenStack 组件。这些组件针对同一开放平台以及支持公共云和私有云全球网络的 API 进行了标准化。此外，这些核心项目提供了可行的战略性方案来替代基础架构方面的传统资本支出。

      OpenStack 核心基础架构 (IaaS) 项目包括：

      • Nova（计算）
      • Cinder（块存储器）
      • Swift（对象存储器）
      • Neutron（网络）
      • Glance（镜像）
      • Keystone（身份）

      事实上，这些服务（不包含 Swift）最开始是属于 Nova，而 Nova 是一项计算服务，也是第一个 OpenStack 项目（以及 Swift）。每个项目要么独立于 Nova（Glance 和 Cinder），要么在稍后重新实施 (Neutron)。Glance、Cinder 和 Swift 共同构成了 OpenStack 的数据处理和存储功能。Neutron 将 OpenStack 联网功能整合成易于部署的形式。Ironic 协助自动完成硬件配置，从而实现完整的 IaaS 功能和可扩展性。

      Nova：Nova 是 OpenStack 的计算服务。它为虚拟机提供按需配置和管理。它支持多个虚拟机管理器，包括 KVM、XenServer®、VMware ESXi 和 Microsoft Hyper-V。它还支持诸如 LXC 的 Linux 容器。

      Glance：Glance 提供了一个可访问 API 的服务，用于处理磁盘和服务器镜像（包括发现、注册和交付）。Glance 可以将镜像存储在各种后端安装（包括 OpenStack Swift）中。Glance 服务支持多种镜像格式（包括 Raw、VHD、VMDK 和 VDI）。

      Cinder：Cinder 是 OpenStack 的块存储器模块，为虚拟云主机提供虚拟存储器。通过使用 Cinder，可创建设备、将设备连接到虚拟服务器以及从虚拟服务器拆离设备。Cinder 与 OpenStack 的 Nova 计算服务完全集成，可以从 Horizon 仪表板中进行管理，支持通过特定驱动程序与企业级存储平台（如 Nimble®、Solidfire® 或 EMC®）进行集成。它还支持快照。

      Swift：Swift 可处理分布式的非结构化数据。Swift 提供了一个功能强大的平台，用于存储图像、视频、虚拟机镜像和归档。Swift 提供可访问 API 的分布式、可扩展的冗余存储器，这种存储器经济实惠，是备份、归档和保留数据的理想选择。

      Neutron：Neutron 提供了由 API 驱动的可插拔平台，以用于管理网络和 IP 地址。Neutron 支持多种网络模型（Flat、VLAN 和 VXLAN）、静态 IP 和 DHCP。可以通过利用诸如 OpenFlow 的 SDN 平台来使用高级联网功能。

      Keystone：Keystone 提供了认证和授权功能。它与现有后端目录服务（如 LDAP）集成。Keystone 提供一个中央目录，其中存储了用户以及用户到其有权使用的服务的映射。

      实际上，Keystone 是任何 OpenStack 部署的必需组件。每个 OpenStack 部署需要使用某种形式的认证服务，Keystone 是最常见的用例。（某些极端测试用例可能存在无操作认证。）例如，Swift 也有自己的认证，但在除了 Keystone 不可用的测试部署之外并不常用。

      OpenStack 将计算、存储、联网和配置组件等模块相互集成，并提供额外的支持服务，从而针对“基础架构即服务”(IaaS) 提供可扩展的统一用户体验。请注意，也可通过统一的 RESTful API 来访问这些相同的 OpenStack 服务；可以选择使用 API 界面创建自己的用户体验 (UX)。

      OpenStack 支持 IaaS 服务

      OpenStack 支持服务扩展了 OpenStack 核心 IaaS 的功能。其中包括安全服务、使用监控和自动部署服务。以下项目符号列出了这些额外的 OpenStack 服务、其大致的部署频率以及先前部分中列出的基本 OpenStack IaaS 服务。

      支持 IaaS 服务包括：

      • Ceilometer（遥测）
      • Heat（编排）
      • Horizon（仪表板）
      • Ironic（裸机配置）

      通过在所有 OpenStack 组件中整合常见的使用和功能信息，Ceilometer、Horizon 和 Heat 共同丰富了 OpenStack 云环境。（注：在 OpenStack 的最新发行版中，始终会自动部署库代码 Oslo，因为这些服务需要将其用作公共库。）

      Ceilometer：Ceilometer 汇聚了 OpenStack 服务中的使用情况和性能数据。

      Heat：Heat 提供基于模板的功能，用于描述和自动完成相对复杂的基础架构的部署。例如，如果您需要部署简单的 Web 应用程序，并且其中包含一个作为应用程序服务器运行的云主机、一个存储在 Trove 中的数据库以及一个负载均衡器，那么将使用表示应用程序云主机数的参数来创建 Heat 模板。与 Ceilometer 遥测服务结合使用时，Heat 提供自动缩放功能，因此在上述示例中，可以在需要时部署更多的云主机。

      Horizon：Horizon 提供集中式图形用户界面，可用于查看，配置和管理云资源。可以通过 Horizon 仪表板界面轻松集成第三方服务（如计费和监控）。OpenStack 仍可在没有 Horizon 的情况下完全正常运行，并且可以使用底层 OpenStack API 访问相同的使用情况信息。

      Ironic：Ironic 允许您自动配置和管理物理机器。从传统意义上讲，公司需要花钱购买或构建这种服务，但可以认为这是一种计算服务。

      OpenStack 应用程序支持

      本部分中涉及的每个 OpenStack 项目都用作特定应用程序的包装器。这些模块非常适用于简化云端以实现特定目的：数据库访问、大数据分析或共享信息。尤其是，这些 OpenStack 模块会加快已在运行所需的特定应用程序的新 Nova 云主机。这些 OpenStack 项目将在容器（例如 Docker 容器）内正常运行，主要是因为在大多数情况下，用户不能直接访问这些云主机：

      • Trove（数据库）
      • Sahara（数据处理 - Hadoop）
      • Manila（文件 - CIFS）

      Trove：Trove 在 OpenStack 云基础架构上提供“数据库即服务”(DBaaS) 功能。

      Sahara：Sahara 提供可扩展的数据处理堆栈及相关管理接口。

      Manila：Manila 提供共享的文件系统服务。它适用于 Samba（Windows 文件共享的 Linux 实现）等。

      持续改进：Big Tent OpenStack 服务中的其他项目不断进行孵化、实施和改进，从而满足上述广泛部署的 OpenStack 模块（即，集成 IaaS、支持 IaaS 和 PaaS）所建立的质量标准。Big Tent 下当前孵化的一些项目包括部署服务 (TripleO)、DNS 即服务 (Designate)、密钥管理 (Barbican)、负载均衡即服务 (Octavia)、防火墙即服务、网络功能虚拟化 (NFV) 等。

      OpenStack 云交付方法

      您可能会问：我现在已了解所有这些内容，实际该如何获取 OpenStack Cloud？目前使用三种主要方式将 OpenStack 软件提供给客户：

      1. 作为软件包集合（称为“分发版”）：OpenStack Web 站点提供了可下载的标准 OpenStack 分发版。除了默认 OpenStack 分发版外，许多供应商还提供了专有分发版，其中包括一些增值服务。分发版可帮助完成复杂的 OpenStack 安装和升级过程。但是，在简化 OpenStack 云操作方面，分发版不是特别有用。成功使用分发版的客户通常都非常熟悉 OpenStack。分发版由多达 25 家供应商创建，包括 Mirantis、Red Hat、HP 和 Platform 9 等。OpenStack Marketplace 中提供了分发版供应商列表，其中还列出了可帮助您部署 OpenStack 的咨询专家。

      2. 作为服务：OpenStack 即服务，这意味着您可以从经验丰富的云运营商（如 IBM 或 Rackspace）处购买（私有）云服务。云运营商将为其客户提供支持，这样，您就可以自由地“使用”OpenStack，而不是在 OpenStack 上执行操作。这是“OpenStack 云即服务”(IaaS) 提供的主要优势：它可以让您专注于自己的业务应用程序，而不必维护云。

      3. 标准 OpenStack 分发版：标准 OpenStack 分发版将 OpenStack 服务捆绑为可供下载的软件包。标准软件包可用于多个平台，包括 Ubuntu®、CentOS、Debian 等。OpenStack 软件会持续更新，可从干线上进行下载。这些持续更新并不稳定；只有一年两次的发行版才是稳定的。最新的稳定版本以及干线上提供的工作版本始终可供下载。

      分发版

      第三方分发版：参与的供应商会提供自定义的 OpenStack 分发版，其中包括该供应商提供的某些优势功能。自定义分发版可以包括增值服务和专有服务，还可以与其他供应商产品进行独特集成。OpenStack Marketplace 上提供了第三方分发版的完整列表。最热门的第三方分发版有 Mirantis OpenStack、Red Hat Enterprise Linux OpenStack Platform、Rackspace® Private Cloud Software 和 Ubuntu OpenStack。

      OpenStack 云即服务

      少数供应商通过上述的托管或“即服务”交付机制来提供 OpenStack 云，使得 OpenStack 云可以在托管环境中按需提供。对于那些希望在无运营或其他基础架构开销的情况下使用 OpenStack 的客户，OpenStack 托管云是一个很好的选择。

      托管云的客户将获得单租户 OpenStack 云及其所有优势，包括在无部署或运营开销的情况下进行 API 访问。OpenStack Marketplace 上提供了 OpenStack 托管云提供商的完整列表。下面是一些受欢迎的“OpenStack 云即服务”供应商：

      IBM Bluemix Private Cloud：Blue Box 在 2015 年 6 月被 IBM 收购，成为 IBM 云产品的一部分，现在称为 IBM Bluemix Private Cloud。Blue Box 是提供托管的 OpenStack 私有云即服务的最早提供商之一，为 IBM Bluemix 团队提供了大量的专门技术。IBM Bluemix Private Cloud 团队负责主持 OpenStack 西雅图用户组会议，并积极为 OpenStack 社区做出贡献，尤其是针对 OpenStack Operators 工作组。IBM Bluemix Private Cloud Local 是其提供的最新产品，可向您的数据中心引入 OpenStack 私有云硬件，整个由 IBM Bluemix Private Cloud 专家管理。

      Cisco Metapod：Cisco 是另一家提供“OpenStack 云即服务”的供应商，此服务以前由 Metacloud 开发。他们从基于 OpenStack 的“开箱建云”或“私有云设备”开始，后来提供托管的私有云。他们的自定义 OpenStack 分发版也以卓越的用户体验 (UX) 而闻名。

      Rackspace：在其数据中心内提供托管的专用私有云环境。此产品以卓越的性能著称。

      两种托管环境：公共云和私有云

      公共云：可通过试用某个基于 OpenStack 的公共云服务提供商提供的产品来了解 OpenStack 云的运作方式。许多提供商都提供了免费的试用预订或低廉的招揽价格。OpenStack Marketplace 上提供了 OpenStack 公共云服务提供商列表。

      私有云：私有云趋向于以下列两种形式提供：托管云（表示供您专用的远程云硬件和软件由云服务提供商代替您运营）；本地云（表示您拥有自己的硬件并雇用其他人为您运行）。

      托管：OpenStack 托管云产品提供了一种开始使用 OpenStack 的更简单的选择。例如，当您向 IBM Bluemix Private Cloud 这样的提供商注册时，可能会在几小时内为您运行托管的私有云。

      本地云：本地云表示您拥有的云硬件位于您自己的数据中心内，但由诸如 IBM Bluemix 的托管服务提供商代替您运行。在 2016 年，有几家提供商进入了这个市场，提供了新的本地云服务产品。OpenStack Marketplace 上提供了提供商列表。

      有趣的部分：OpenStack 为何对您很重要

      想要证实么？开始使用 DevStack。DevStack 是以最低配置安装 OpenStack 测试云的最简单方法。如果想要测试一下本地云环境（无需服务的合同承诺，只是为了证实其功能），可轻松开始使用 DevStack。这是一个易于设置的一体化配置，可从 OpenStack Web 站点下载。

      DevStack 旨在用作原型设计环境。它提供了一体式虚拟机、一体式机器和多节点配置。但是，DevStack 不适合运行生产云工作负载；例如，在重新引导机器后，它不会恢复到完全运行状态。

      使用 DevStack 启动的云不受监控或管理，默认情况下已启用所有调试日志记录。这很实用，例如，比较不同的 OpenStack 版本以了解软件在升级之前是否可正常用于新的 OpenStack 发行版。它目前可用于 Ubuntu、Fedora 和 CentOS/RHEL 平台。

      如果您对云有特殊要求，那么可以部署仅满足您当前需求的 OpenStack 项目。OpenStack 提供了用于继续添加所需功能的路径。例如，假设您在自己的专用 Web 服务器上运行一个 Web 站点，仅运行诸如 Tumblr 的服务，用于存储博客帖子和图像。您只需使用一种方式来存储大量数据。可以仅将 Keystone 和 Swift 作为 OpenStack 配置进行部署，这将提供认证和存储服务，无需计算！

      又例如，假设您只需要在云中包含计算基础架构，因为您的站点只需要临时存储功能（重新引导后会丢失数据）。您可以部署仅使用 Keystone、Nova 和 Glance 项目的 OpenStack 云。在这种情况下，Nova 中的联网功能将为您提供简单的联网功能。要实现更丰富的联网功能，请将 OpenStack 的 Neutron 组件添加到部署中。您已经获得了非常完美的计算基础架构。

      要部署完整功能的 OpenStack 云以及完整的计算和存储功能，可以使用所有核心 OpenStack IaaS 模块，并可以添加 Keystone 来提供认证服务。然后，可以进一步自定义云，使其完全符合所需的操作类型，例如，数据库或数据分析集群。可以根据需要来继续添加和升级 OpenStack 功能。

      常见的 OpenStack 用例

      OpenStack 适用于许多行业垂直领域以及各种用例。下面是 OpenStack 最成功的一些用例：

      云服务提供商：在此用例中，OpenStack 平台提供了更大规模的“虚拟基础架构即服务”(IaaS)。这种类型的典型堆栈包括若干个开源工具，例如，KVM（虚拟机管理器）、Ubuntu (OS)、OpenStack（云管理层）、Nagios（监控）、LogStash（日志记录）、Kibana（分析）、Chef（编排）、HAProxy（负载均衡器）以及本地组件。商用硬件通常用于提供计算功能和某些类型的存储服务；但是，可以部署多种硬件配置。Rackspace Public Cloud 就是这种用例的一个很好的例子。

      电信服务提供商：电信服务提供商 (Telcos) 早期就使用了 OpenStack 来满足其私有云需求。这些用例通常为内部组织或合作伙伴组织提供“IT 即服务”。OpenStack 社区一直致力于实现电信特有的高级功能（如网络功能虚拟化或 NFV）。

      应用程序开发云：OpenStack 的另一个常见用例是，在应用程序开发云中实现持续集成/持续开发 (CI/CD)。在这种情况下，OpenStack 云实现了业务敏捷性，并提升了客户对在 OpenStack 云上运行行业 (LOB) 应用程序的信心。这些 LOB 应用程序和工作负载通常涉及某种源代码存储库（例如 github.com）、测试工具（例如 Jenkins）、错误数据库（例如 Atlassian）和部署工具。

      将第三方工具与 OpenStack 配合使用

      OpenStack 与可用的第三方热门云运行、开发和管理工具进行了良好集成。第三方工具可包括针对 OpenStack API 的软件开发者工具包 (SDK)、编排工具、监控工具、日志记录工具和配置工具。在实际云部署中，肯定会使用一些其他开源工具。本部分列出了与 OpenStack 一起广泛使用的一些类型的工具。请注意，某些专有 OpenStack 分发版可能不需要这些工具，因为它们通常包括自定义的等效工具。

      SDK：OpenStack 提供两种方式来管理数据中心内的计算、存储和联网资源：仪表板和服务端点。OpenStack 服务端点通过可编程接口 (REST API) 来公开。可以通过任何 REST 客户端（例如，作为 OpenStack 项目一部分提供的客户端工具）访问这些 API。同时还提供了适用于各种编程语言的 SDK 来使用这些 API。

      OpenStack 客户端工具：OpenStack 项目提供了各种项目（包括仪表板、计算、对象存储器、身份服务和镜像服务）的正式客户端，而适用于其他项目的其他客户端正在开发中。

      OpenStack SDK：提供了正式 OpenStack SDK for Python，还提供了适用于各种编程语言（包括 C、C ++ 和 Java .NET 等）的非正式 SDK。

      编排工具：虽然 OpenStack 包含自己的编排服务 (Heat)，但许多部署会使用其他热门编排工具（例如，Chef、Puppet 和 Ansible）。

      Chef：Chef 是 Chef（以前称为 Opscode）提供的热门配置管理工具。它广泛用于 OpenStack 部署。Chef 采用客户端/服务器架构。提供了正式手册以帮助您开始使用 Chef 部署 OpenStack。

      Puppet：Puppet 是 PuppetLabs 提供的另一种热门配置管理工具。Puppet 也采用客户端/服务器架构。可以使用正式 Puppet 模块来帮助部署大型 OpenStack 云。

      Ansible：Ansible 软件是无代理的配置管理工具。虽然未提供适用于 OpenStack 的正式 Ansible playbook，但是用户可以开始使用这些 Blue Box playbook。

      监控工具：运行大型 OpenStack 云需要经常监控各种组件，包括计算资源、虚拟机和虚拟网络。通过监控，管理员还可主动地引发警报，例如，在计算节点的 CPU 利用率接近 100% 时。OpenStack 提供了遥测服务 (Ceilometer)，此服务提供基于事件的基础架构来实现监控；但是，它未提供完整的监控功能。因此，OpenStack 云部署通常会使用其他开源监控工具（例如，Nagios）。

      日志记录工具：要提供高级日志管理功能，可以使用 LogStash 等开源工具。LogStash 提供了一种集中式功能来进行日志记录、日志解析和日志存储。与 ElasticSearch 等搜索工具结合使用时，它会为您的 OpenStack 云提供可扩展的强大日志管理解决方案。

      配置：部署 OpenStack 云包括将裸机服务器配置为控制器节点或资源节点。用于配置的热门工具包括 Chef、Puppet、Ubuntu MaaS 和 Crowbar。OpenStack 拥有自己的裸机服务 Ironic，也可以使用此服务。

      云词汇表

      访问密钥/密钥：两个自动生成的配对的加密密钥，它们组合使用时可访问计算云主机并与其通信。密钥用于对每个请求进行数字签名，访问密钥用于验证签名。

      可用性区域：云中一组独立的虚拟机管理器。可用性区域为虚拟机提供容错。OpenStack 支持可对服务进行逻辑分离或集群化的可用性区域。通常，为确保电力可达性或出于地理方面的原因，才会创建这些区域。启动新云主机时，可以选择一个区域。

      块存储器：支持云硬盘、云硬盘快照和云硬盘类型的存储类型。块存储器云硬盘是不直接连接到正在运行的云主机的块设备。它可以一次连接到一个云主机，如果云主机本身发生故障或关闭，它仍然会运行。通常，OpenStack 云硬盘连接到现有 SAN，因此您可以使用 API 来与其进行交互。

      容器：在对象存储器中组织和存储对象。在 OpenStack 中，容器包括诸如 Docker 的第三方软件。

      固定/静态 IP：每次重新引导时与相同 VM 云主机相关联的 IP 地址。在云中，使用这种类型的地址来管理 VM。客户通常无法访问此地址。

      云主机类型：用于描述虚拟机镜像的一组参数。参数包括 CPU、内存大小和存储大小等。

      浮动 IP：与 VM 云主机相关联的 IP 地址，设置此地址后，每次重新引导时云主机都具有相同的公共 IP 地址。浮动 IP 会创建可达的公共 IP 地址，然后将来自该地址的流量定向到已配置的云主机。实际上，此 IP 不作为云主机的一部分创建；而是由云提供间接关系。首先，必须从提供的池中分配地址，然后可将其附加到云主机。

      虚拟机管理器：用于仲裁和控制虚拟机对底层实际硬件的访问的软件。它表示在特定云主机中配置的物理计算节点。

      镜像：针对特定操作系统创建或重建服务器所需的文件集合。OpenStack 支持多种镜像格式（AMI、VMDK、VHD 和 QEMU）。例如，您可以通过服务器镜像来创建预配置镜像，以供以后使用。这些镜像可用于启动新的服务器云主机，并在列出镜像时显示。

      云主机：虚拟机。要启动（或“加速”）云主机，至少需要以下信息：用作模型的磁盘镜像、云主机类型（可视为“规模”）和云主机名称（例如“MyInstance4032”）。多数 OpenStack 环境还会要求您在启动云主机时提供对网络设备的引用。

      元数据：云主机元数据可用于对云主机进行排序或分类。元数据是按云主机列出的自由格式的“键=值”存储器。

      NAT（网络地址转换）：用于将云中一种类型的 IP 地址更改为另一种类型的 IP 地址的协议。

      网络：虚拟网络是提供实体（例如，云中的云主机）之间连接的第二层网络。

      对象：对象是在对象存储器中存储的任何类型的数据，可以为任何格式（文件、音频、视频或二进制文件）。

      对象存储器：可支持最终一致、冗余且非结构化的数据的存储器类型。OpenStack 对象存储器不是传统的文件系统，而是用于静态数据（例如，虚拟机镜像、照片存储、电子邮件存储、备份和归档）的分布式存储系统。它没有“根”目录、中央“大脑”或主控制点，因而实现了更好的可扩展性、冗余性和耐久性。数据由对象表示，对象存储在容器中。

      项目/租户：项目是用户的逻辑分组；通常也称为“租户”。

      配额：按项目对计算和存储资源设置的资源限制。通常，在多租户托管云中，每个租户的配额都基于其合同协议。配额可以按项目或类进行设置，用于限制诸如 RAM、IP 地址、内核数和存储容量等资源的使用量。OpenStack 可以对资源使用量设置配额。

      角色：角色包含一组权限和特权。具有特定角色的用户将继承其相关权限和特权。

      安全组：应用于计算云主机的一组过滤规则。安全组就像是一组系统的防火墙。您可以提供用于引用其他组或 CIDR 块的访问规则。这些组可以附加到云主机，以提供对该云主机的访问。

      服务器：用于指代虚拟机 (VM) 云主机的通用术语。虚拟机云主机的另一个通用术语是主机。

      服务：OpenStack 服务会提供若干个端点，用户可通过这些端点来访问和控制资源。

      SSH 密钥：SSH 密钥用于（通过 ssh 命令）安全地连接到使用 OpenStack 启动的云主机。OpenStack 提供一种内置方法，用于将 SSH 密钥对的公共部分插入到新引导的云主机的授权密钥文件中。Nova 还可为您创建密钥的私有部分，或者您可上传现有密钥对的公共部分。

      快照：存储云硬盘或镜像的时间点副本。快照与镜像的功能相同。

      用户：用户使用云资源。每个用户可以是一个或多个项目的成员。在一个项目内，用户负责创建并管理各种资源，例如云主机和云硬盘。

      虚拟机 (VM)在 OpenStack 云中运行的云主机；VM 云的另一个术语是服务器或主机。

      云硬盘：基于磁盘的数据存储器，可以为持久或临时（重新引起时会丢失数据）。该存储器通常表示为其文件系统支持扩展属性的 iSCSI 目标。将云硬盘连接到服务器时，可以依赖系统在云主机中自动分配设备 ID 以连接到新云硬盘。也可以选择指定设备 ID。如果连接了新云硬盘，那么该云硬盘必须格式化后才能使用。后续使用该云硬盘时不需要进行格式化。
