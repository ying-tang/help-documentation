      OpenStack 网络概念

      • by Ying Tang ● May 25, 2016

      • Tags:

        • OpenStack

        • OpenStack

        • 概念

      本文档提供了一些常见网络概念的基本介绍。使用 IBM Bluemix Private Cloud 时，您会遇到这些概念。

      本文档中使用的术语

      概念 描述
      L2 ISO 7 层因特网架构的第 2 层，在其中路由数据包。
      VLAN 虚拟局域网，尤其是子网。
      IP 因特网协议，通常用作 IP 地址缩写，其指定了网络中数据包的目的地。
      IPv4 因特网协议 V4，格式为 xxx.xxx.xxx 的因特网地址，现在通常在末尾添加 /xx 以使用 CIDR 协议指示子网规模。
      CIDR 无类域间路由或超网，用于延长 IPv4 地址的有效寿命，全局池中的 IPv4 地址可能不够用，因而需要购买 IPv4 地址。下面是指向 CIDR 更多相关信息的链接：http://searchnetworking.techtarget.com/definition/CIDR。下面是指向 CIDR 子网规模计算器的链接，可使用此计算器来了解有多少个 IP 超出特定范围：http://www.subnet-calculator.com/cidr.php（请记住，其中一些 IP 是预先分配的，不可用于云）。
      IPv6 因特网协议 V6，一种较新的因特网寻址协议，其提供大量的网络地址，但会增加网络架构的复杂性。
      NIC 网络接口卡，用于将服务器连接到网络的一块硬件。此外，虚拟 NIC（尤其是工作方式与 NIC 类似的软件）往往用于云中，而术语“网络接口”通常是指虚拟 NIC，除非上下文指示其专门指代硬件。
      MAC 个人标识符，类似于硬件序列号，用于唯一标识网络中的每个主机。

      基本网络概念

      概念 描述
      网络 一个隔离的二级 (L2) 网段，类似于常规网络中的 VLAN。一个网络可以具有一个或多个子网。
      子网 IPv4 地址范围。
      端口 一个连接点，用于将单个设备（例如虚拟服务器的 NIC）连接到虚拟网络。术语“端口”还描述了关联的网络配置，例如要在该端口上使用的 MAC 和 IP 地址。
      固定 IP 地址 在引导时分配给云主机的 IP 地址（例如，需要访问公共因特网的云主机的网络 IP 地址）。
      浮动 IP 地址 从池中按需分配给云主机的 IP 地址，通常为 IPv6 格式。
      配额 配额用于限制可用于云的资源（例如，计算或磁盘分配）的数量。
      安全组 一组特定于项目的网络过滤规则，通过指定哪些 IP 可以向其他 IP 或端口发送数据包以及哪些 IP 不能发送来将这些规则应用于云主机的网络交互。
      安全组规则 协议、端口和 IP 地址规范的列表，这些规范确定允许流量如何流入或流出网络中的任何特定云主机；这些是安全组的具体内容。

      云网络类型

      通常，云包含一个或多个内部和外部网络。这句话可理解如下：云中的云主机可在内部网络中互相“通信”，这些云主机可与外部网络中私有云外部的源进行“通信”。

      可以使用多种术语来描述内部网络和外部网络。例如，内部网络有时称为“租户网络”。用于与公共因特网通信的外部网络有时称为“提供者网络”。

      内部网络

      内部网络在云中“通信”，它在所连接的所有虚拟网络接口 (NIC) 之间提供连接。每个内部网络接口与一个虚拟机 (VM) 相关联，一个 VM 可具有多个关联的网络接口，这表示它可在很多网络中“通信”。

      可以通过将一系列 VM 连接到同一内部网络以在这些 VM 之间实现通信。内部网络与单个项目（租户）相关联，仅对该项目可见且可供该项目访问。

      外部网络

      外部网络可在云内部和外部“通信”。这类网络可包含公共因特网、云的外部网络以及 IBM Cloud Data Center 网络（其中包括 IBM Cloud Data Center 路由器）。

      • 云的“公用网络”或“外部网络”可与公共因特网相互访问。
      • SoftLayer 专用网络用于提供对 SoftLayer 中其他服务的访问，包括可能希望与其通信的其他云。（请参阅发行说明，以获取有关对 SoftLayer 专用网络的访问权的更多信息。）
      • SoftLayer 路由器网络会连接到 SoftLayer 后端内核路由器 (BCR)，可通过此路由器来访问您的私有云环境外部的资源、可能位于您站点的数据中心 (Blue Box Local) 的资源、您拥有的另一个私有云环境中的资源、SoftLayer 数据中心中的资源或其他位置（Blue Box Dedicated 或您自己的旧云）中的资源。
      • 公用网络和 SoftLayer 专用网络都是物理网络。

      不同专用子网中虚拟机之间的连接？

      假设您有两个专用网络。您已在每个子网上创建了虚拟机并添加了必需的安全组，以便可以接受入局流量。如何在不同专用子网中的 VM 之间建立连接？假设您有一个名为 operations 的网络和一个名为 internal 的网络。可通过以下几种方式在它们之间建立通信：

      • 将 operations 网络连接到默认路由器。通过此连接，可在默认 internal 网络和 operations 网络之间传输流量。现在，可将云主机部署到 internal 网络，为其分配公共浮动 IP 地址，使用此云主机连接到目标云主机。
      • 或者，可使用相似的方法，在 internal 网络和 operations 网络之间创建新的内部路由器。将云主机与 internal 网络中公共 IP 配合使用，以便访问 operations 网络中的 VM。通过创建内部路由器，可确保 operations 网络中的 VM 没有到因特网的外部连接。
      • 也可以创建多宿主云主机。创建一个同时连接到 internal 和 operations 网络的云主机。为其分配公共浮动 IP。使用此 VM 通过另一个专用接口提供对 operations 网络中云主机的访问权。

      应该对具有公共浮动 IP 的机器采取哪些安全预防措施？

      开始时可遵循以下最佳实践：

      • 配置 sshd 以仅允许密钥认证，并禁用 root ssh 访问权。
      • 不要针对 SSH 使用标准或默认系统用户名（ubuntu、deploy 及类似名称）。
      • 使用诸如 fail2ban 或 denyhosts 的系统服务以阻止可能出现的蛮力攻击。最受欢迎的 Linux 分发版中已经打包了这两个工具。
      • 使用安全组规则将 SSH 访问限制为数目受限的 IP 地址。
      • 定期更新在公共云主机上运行的系统。
      • 不要在公共云主机上运行任何不需要的服务（例如，http、ftp、smb…）
