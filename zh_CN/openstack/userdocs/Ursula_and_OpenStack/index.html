      Ursula 和 OpenStack

      你们中的多数也许熟悉 OpenStack，但不是每个人都熟悉 Ursula。对于 IBM Bluemix Private Cloud，我们使用 Ursula 来安装 OpenStac 专用版本。自动安装可减少用户错误数，同时可进行更快速、更干净且定制程度更高的 OpenStack 安装。此博客将引导您开始理解和使用 Ursula。所有 Ursula playbook 都是开源的，且在 GitHub 上提供。最后，您可自定义一些内容以供自己使用。

      Ursula 是：

      • Blue Box 为部署和管理 OpenStack 云而创建的 Ansible playbook 的集合。

      可通过两种方式安装 Ursula：一种方式是使用 Vagrant，另一种方式是通过“手动部署”。此帖子重点介绍了手动部署方法。

      Ursula 提供了有关如何执行手动部署的指示信息；但是，新手使用 OpenStack 时可能存在很大挑战。在此帖子中，我将分享关于安装 Ursula 的一些经验，并提供有关如何操作以及在遇到问题时如何进行故障诊断的一些详细指示信息。

      需求：两个虚拟专用服务器 (VPS)

      • 1 个 VPS 用于设置 Ansible。我选择在 CentOS 6.x VPS 上执行此操作。我已使用 2GB 双核 VPS。
      • 1 个 VPS 用作部署目标。此目标具有更具体的需求：
        • Ubuntu 14 LTS（不要使用 Ubuntu 12 LTS。）
        • 8GB RAM（可在更少的 RAM 上部署，但速度可能相当慢。）
        • 4 个内核
        • 1 个 IP 地址。在此示例中，VPS 的公共 IP 是 69.87.123.456（明显不是有效 IP 地址！）

      步骤 3：尽情体验！
