      今天，我们将运行一遍向某个云主机添加浮动 IP 地址所需的过程。我在使用 Horizon 面板在 OpenStack 中创建其他网络内容结束时涉及到了如何完成此过程，因此我们通过终端运行此示例。首先要注意的是，**创建浮动 IP 地址不会实际创建地址，而只是从已分配的池分配一个地址。让我们看一下已分配的地址：

      既然已创建了 IP，我们就可以继续操作，将其连接到云主机。我们将需要云主机内部 IP 地址的端口 ID 和上述可见的浮动 IP 的 ID。

      这里有我们的云主机（请忽视格式）及其内部 IP。我们现在使用此信息来查找端口。

      就是这样！现在，我们只需浮动 IP 的 ID。我们在创建时已在上述 floating_network_id 下有了该 ID：953cda96-1a33-4645-a4e2-1b00d6e06736。最后的步骤是进行关联。

      nova list 命令现在应向您显示与云主机相关联的两个 IP 地址：

      在所需的任何时候，您也可以通过运行该命令，后跟所考虑地址的 ID，解除与浮动 IP 的关联。我们将解除与刚刚分配给以下云主机的地址的关联。

      您将注意到该地址不再与云主机相关联。

      • 在 OpenStack 中，浮动 IP 是分配给 VM 的公共可路由 IP。添加浮动 IP 后，在 VM 内会发生什么？答案是……什么都不会发生。如果通过 SSH 登录并显示网络配置，那么您将看到仍有一个配置了固定 IP 的网络接口。所有设置都在计算虚拟机管理器本身完成。OpenStack Neutron 设置 VM 的固定 IP 和浮动 IP 之间的 NAT。

      浮动 IP 的一大优势是：如果一个 VM 发生故障，那么可通过连接到其他 VM 来复用浮动 IP。
