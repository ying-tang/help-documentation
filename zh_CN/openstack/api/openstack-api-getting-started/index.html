      在 macOS 上：

      安装 Python

      注：macOS El Capitan 引入了一种机制以避免损坏操作系统文件。/System/Library/Frameworks/Python.framework/Versions/2.7/share 是其中一个受保护的位置。普通用户无理由在此放置或写入任何文件。因此，我们建议不要使用 sudo pip 来安装 Python。

      相反，在用户主文件夹本地安装 Python 软件包，例如，IPython。最简单的方式是创建并激活虚拟环境，然后在虚拟环境中运行 pip。

      设置本地 stackrc 文件

      stackrc 文件定义用于将 Nova 客户端连接到 Nova API 的环境变量：

      为您自己创建 stackrc，将 OS_USERNAME、OS_PROJECT_NAME、OS_TENANT_NAME、OS_USER_DOMAIN_NAME、OS_PROJECT_DOMAIN_NAME、OS_CACERT 和 OS_PASSWORD 字段替换为自己的字段。

      注：如果您担心在此文件中存储您的密码，请保留密码字段为空；在获取 stackrc 文件后，将要求您输入密码。

      如何使用 cURL 查询 OpenStack API

      问：我如何通过 v3 API 获取 OpenStack 令牌？

      答：您可以使用 cURL 命令：

      了解更多信息：

      此处链接的 API 指南包含使用 cURL 获取服务器列表的示例。

      每个服务都有各自的端口用于 API 访问。如果登录到 Web GUI 并转至访问和安全性 => API 访问选项卡，那么可能会看到与服务相对应的端口。

      要查询任何其他 API 服务（Nova、Neutron、Cinder 等），必须先从认证服务 (Keystone) 获取令牌。您可以复用该令牌来执行所需数量的 API 调用，直至令牌到期。

      例如，9696 为 Neutron，但是获取令牌（尝试运行的命令）需要 Keystone 认证请求，这需要位于端口 5000 上。

      因此，部分步骤如下：

      1. 通过 HTTP POST 从 $OS_AUTH_URL/tokens 访存令牌（不是 GET，因为您需要对 $OS_USERNAME 和 $OS_PASSWORD, $OS_TENANT_NAME 执行 POST 才能够检索令牌）。响应将包含令牌以及不同服务的目录。

      2. 检索此令牌并检查令牌到期日期是否为未来日期。

      3. 从返回的目录中检索想要针对其执行 API 调用的服务的 URL。在您的案例中，您将在目录中查找类型为 network 的条目。它应该具有名称 neutron。

      4. 在对此 URL 执行不同 HTTP GET 操作时，包含此令牌作为 X-Auth-Token 头。您可以运行 neutron --debug security-group-list，以查看 POST 看起来应该是什么样子，它应该类似如下：

      curl -g -i --cacert "/etc/ssl/certs/ca-certificates.crt" \
-X GET https://example.openstack.blueboxgrid.com:9696/v2.0/security-groups.json \
-H "User-Agent: python-neutronclient" \
-H "Accept: application/json" \
-H "X-Auth-Token: {SHA1}11111a1111aaa1a111aaaaaa1111aa1a111a1111"
