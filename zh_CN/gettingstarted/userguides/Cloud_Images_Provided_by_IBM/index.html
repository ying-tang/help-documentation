      云镜像用户指南 (Community Edition)

      • by Yanqi Dou ● March 9th, 2016

      • Tags:

        • userguides

        • 用户指南

        • 镜像

        • glance

        • image-create

        • 云主机

      IBM Bluemix Private Cloud 3.0 预先填充了 Cirros 0.3.3 x86_64 镜像。IBM Bluemix Private Cloud 还提供了其他可下载的云镜像，以便您可以使用所选操作系统运行云主机。本文档介绍了在下载和使用这些镜像时需要注意的事项。为确保使用的是最新镜像，可以随时查看云镜像发行说明。

      有关将镜像用于 OpenStack 的更多信息，请访问 OpenStack Web 站点：https://docs.openstack.org/image-guide/

      先决条件

      1. 如果您计划使用命令行 (CLI) 工具，请确保已安装了 OpenStack Glance Client。可以在知识库中找到相关的安装指示信息：开始使用 OpenStack API。

      2. 确保您有权访问 Box Panel，以便可以获取 IBM 提供的云镜像的临时下载 URL。

      3. 只有在 Bluemix Private Cloud 2.1.0 和更高版本中才支持使用 tempUrl 作为镜像位置将镜像上传到 Glance。

      基本概念

      项 描述
      **镜像  
      ** 在本文档中，将虚拟机镜像简称为镜像，这是单个文件，其中包含安装了可启动操作系统的虚拟盘。可以使用镜像在云中创建虚拟云主机。
      **云主机  
      ** 云主机是在 OpenStack 云中运行的虚拟机 (VM)。
      云主机类型 术语云主机类型描述了云主机的规模，并为云主机可用的 vCPU、RAM 和磁盘空间量定义预先确定的资源规范。

      从 IBM Bluemix Private Cloud 提供的备用云镜像中进行选择

      此版本的 IBM Bluemix Private Cloud 预先填充了 Cirros 0.3.3 x86_64 镜像。IBM Bluemix Private Cloud 还提供了其他云镜像以便于客户使用。

      可以上传并安装独立获取的操作系统软件，也可以供应作为虚拟机云主机提供的以下任何操作系统镜像。您必须遵守所有适用的操作系统许可条款，并获得每个虚拟机云主机的适当权利。

      • CentOS 6.8 x86_64
      • CentOS 7.2 x86_64
      • Ubuntu Server 12.04 LTS x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      对于任何 Microsoft Windows 操作系统虚拟计算云主机，这些都是特定的客户规定：要求客户拥有并维护自己的 Microsoft Windows 许可证服务器，以激活此类 Windows 操作系统虚拟计算云主机。请参阅如何在 IBM Bluemix Private Cloud 中激活 Windows 许可证，以获取更多信息。

      从 Box Panel 下载云镜像

      1. 使用 Box Panel 凭证登录到 Box Panel。如果您至少具有一个云，并且是“主要”或“技术”客户联系人，那么可以使用 Box Panel 中的“云镜像”页面。

      2. 在 Box Panel 仪表板中，单击服务，然后选择云镜像。

      3. 此时会显示“云镜像”页面。每个镜像都显示为一张卡片。

      4. 单击显示您感兴趣的镜像的卡片。此时会弹出一个包含您的 tempURL 的模态窗口。

      5. 显示的 tempURL 在接下来的 24 小时内有效，以便您可以下载云镜像。

      使用云镜像 tempURL 将镜像上传到 IBM Bluemix Private Cloud Glance 存储库

      注：

      • 只有具有 cloud_admin 角色的用户才能够向其他用户和项目公开镜像。
      • 在 OpenStack 控制面板中已阻止出站因特网访问的 IBM Bluemix Private Cloud 无法使用以下功能：直接使用 tempUrl 作为镜像位置来将镜像上传到 Glance。
      • 只有在 Bluemix Private Cloud 2.1.0 和更高版本中才支持直接使用 tempUrl 作为镜像位置来将镜像上传到 Glance，而 2.1.0 之前的发行版中不支持此功能。
      • 对于 2.1 之前发行版中的或不具有出站因特网访问权的 IBM Bluemix Private Cloud，请遵循使用已下载的镜像文件将镜像上传到 IBM Bluemix Private Cloud Glance 存储库中的指示信息。

      可以使用 OpenStack 仪表板或命令行客户端来上传镜像。

      使用 OpenStack 仪表板上传镜像

      1. 登录到 IBM Bluemix Private Cloud OpenStack 仪表板。
      2. 在项目面板下，展开计算，然后单击镜像。

      3. 在此页面中单击创建镜像。在新页面中指定以下参数，然后单击创建镜像。

        项 描述
        名称 输入镜像的名称。
        描述 （可选）输入镜像的简要描述。
        镜像源 选择镜像位置。
        镜像地址 指定云镜像下载 tempURL。
        格式 为镜像选择正确的格式（例如 QCOW2）。
        架构 指定架构（例如，i386 表示 32 位架构，或 x86_64 表示 64 位架构）。
        最小磁盘 (GB) 对于 Linux 镜像，请使用 5；对于 Windows 镜像，请使用 30。
        最小 RAM (MB) 对于 Linux 镜像，请使用 512；对于 Windows 镜像，请使用 2048。
        复制数据 必须选中此选项才可以将镜像数据复制到镜像服务。
        公共 选中此复选框以使该镜像成为所有项目上所有用户的公共镜像。
        受保护 选中此复选框以确保只有具有相应许可权的用户才可以删除该镜像。

        例如：

         名称：Ubuntu Server 14.04 LTS x86_64 - 20160301
 描述：在 2016/03/01 发布的 Ubuntu Server 14.04 LTS x86_64 云镜像
 镜像源：镜像位置
 镜像文件：<tempURL_for_QCOW2>
 格式：QCOW2 - QEMU Emulator
 架构：x86_64
 最小磁盘 (GB)：5
 最小 RAM (MB)：512
 复制数据：True
 公共：True
      4. 检查所创建的镜像是否显示在镜像列表中。镜像已排队等待上传。可能需要一段时间，状态才会从“已排队”变为“活动”。

      注：只有具有 cloud_admin 角色的用户才能够向其他用户和项目公开镜像。

      使用 OpenStack 命令行客户端上传镜像

      可以通过 Glance 客户端上传镜像。请参阅安装 OpenStack 客户端，以获取更多信息。

      创建镜像。

      • 对于 Linux 镜像，请使用 -- min-disk 5 --min-ram 512

      • 对于 Windows 镜像，请使用 --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --copy-from  <tempURL_to_qcow2_image_file>

      检查是否成功创建了该镜像。镜像已排队等待上传。可能需要一段时间，状态才会从“已排队”变为“活动”。

      $ glance image-show <image-id>

      使用已下载的镜像文件将镜像上传到 IBM Bluemix Private Cloud Glance 存储库

      注：

      • 只有具有 cloud_admin 角色的用户才能够向其他用户和项目公开镜像。

      • 使用 tempUrl 下载云镜像时，请确保对 tempUrl 标记进行了转义或用引号将其括起。例如：

        curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      请执行以下步骤：

      1. 下载镜像的 .qcow2 和 .md5sum 文件。

      2. 使用 tempURL 下载 .qcow2 镜像文件和 .md5sum 校验和文件，并将这两个文件放置在同一个文件夹中。例如：

         ubuntu-guest-image-14.04-20160301-x86_64.qcow2
 ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      3. 使用 .md5sum 校验和文件来验证已下载的 .qcow2 镜像文件。例如：

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

        如果结果返回“OK”，表明已下载的 qcow2 镜像有效。

      4. 将该镜像上传到 Glance。可以使用 OpenStack 仪表板或命令行客户端来上传镜像。

 最小磁盘 (GB)：5	
 最小 RAM (MB)：512	
 公共：True

      4. 检查所创建的镜像是否显示在镜像列表中。镜像已排队等待上传。可能需要一段时间，状态才会从“已排队”变为“活动”。

      使用 OpenStack 命令行客户端上传镜像

      请参阅安装 OpenStack 客户端，以获取更多信息。

      创建镜像。

      • 对于 Linux 镜像，请使用 -- min-disk 5 --min-ram 512

      • 对于 Windows 镜像，请使用 --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --file <path_to_qcow2_image_file>

      检查是否成功创建了该镜像。镜像已排队等待上传。可能需要一段时间，状态才会从“已排队”变为“活动”。

      $ glance image-show <image-id>

      注：对于大型镜像，Web 上传可能会超时。对于这些镜像，我们建议用户在其云中启动一个云主机，将镜像下载到该云主机，然后从该云主机上将镜像上传到 Glance。所有流量都保留在 SoftLayer 内，而上传是在用户自己的云中进行。

      使用 OpenStack 仪表板创建 VM 云主机

      在“创建后”选项卡中设置初始用户名和密码

      自定义脚本字段指定在启动云主机后运行的自定义脚本。

      镜像更新

      当您从 IBM Bluemix Private Cloud 运营团队处收到关于已更新云镜像的通知时，可以停用现有镜像，并使用从 Box Panel 获取的已更新镜像的下载 URL 来重新创建新镜像。

      使用 OpenStack 命令行客户端停用现有镜像

      1. 获取现有的镜像 ID：

         $ glance image-list

      2. 停用镜像：

        $ glance image-deactivate <image_id>

      3. 显示镜像状态：

        $ glance image-show <image_id>

      4. 停用现有镜像后，请按照本指南中使用云镜像 tempURL 将镜像上传到 IBM Bluemix Private Cloud Glance 存储库中的步骤来重新创建新镜像。
