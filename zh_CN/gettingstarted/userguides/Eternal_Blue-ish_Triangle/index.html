      本文档介绍了 Box Panel 和站点控制器的基本概念和术语，还介绍了它们如何协作来共同控制和管理 IBM Bluemix Private Cloud Local 环境。本文档提供了指向外部文档的链接和指针，以供您查找更多信息。“术语常见问题解答”部分中定义的部分术语比您日常需要了解的术语技术性更强一些，但当您与我们的支持团队沟通时，了解这些术语对您很有帮助。

      Box Panel 是什么？

      Box Panel 是用于交付 IBM Bluemix Private Cloud 的应用程序套件。Box Panel 可供内部支持人员和 IBM 客户使用。我们用它来管理云基础架构。

      Box Panel 与其他功能一起构成所有数据中心、机器资产、集群、IP 地址和客户帐户信息的记录系统。

      Box Panel 套件以 SaaS 应用程序形式提供，可通过 Web 浏览器和自定义 API 进行访问。Box Panel 套件包括支持、票务和交谈等应用程序，同时还提供了指向产品文档的链接。

      有关 Box Panel 的更多信息，可以查看 Box Panel 用户指南。

      Box Panel 站点控制器是什么？

      Box Panel 站点控制器是一种软件工具，用于为 Box Panel 套件以及 Bluemix Private Cloud 支持和工程团队提供分布式控制面板操作、配置管理和自动化服务。

      每个本地和专用数据中心内都会运行 Box Panel 站点控制器，此站点控制器通过中央控制 Pod 来与 Box Panel 软件即服务 (SaaS) 通信。

      Box Panel 站点控制器的功能包括：

      • 管理远程访问
      • IPMI 代理
      • 裸机配置
      • 中央软件包镜像
      • 监控/警报
      • Sensu
      • Graphite
      • PagerDuty
      • 日志管理 (ELK)
      • 自动化服务
      • HTTP 代理服务

      术语常见问题解答：什么是…..

      问：Box Panel 队列是什么？

      答：Box Panel 队列是 Box Panel 提供的管理功能所使用的作业队列。 这些功能包括自动软件修补、新节点部署和服务器重新引导。

      问：Box Panel 工作程序是什么？

      答：该工作程序是一个从 Box Panel 队列接收作业请求、执行这些请求并将结果返回到 Box Panel 队列的进程。

      问：中央控制 Pod 上的 Sensu 是什么？

      答：用于监控 IBM Cloud 和客户拥有的数据中心内的远程站点控制器。

      问：站点控制器上的 Sensu 是什么？

      答：监控 Bluemix Private Cloud 物理节点 (pNode) 并将警报回送到中央控制 Pod。pNode 包括控制节点、计算节点和存储节点。

      问：PagerDuty 是什么？

      答：一个事件报告工具，用于在夜间当您的云需要关注时唤醒我们。

      问：Graphite 是什么？

      答：一种图形工具，专门用于制作显示 API 是否启动的 SLA 报告。可以在 Horizon 中查看此信息，因为 Graphite 将正常运行时间数据提供给 Grafana 仪表板。

      问：IPMI 代理是什么？

      答：提供对云节点的安全 IPMI 访问。

      问：HTTP 代理是什么？

      答：提供用于访问站点控制器上 ELK 和 Sensu 用户界面的 HTTP 接口。

      问：镜像是什么？

      答：云操作软件的中央软件存储库。该存储库不包含客户端工作负载。

      问：Bastion 是什么？

      答：Bastion 主机通过基于角色的访问控制 (RBAC) 提供对站点控制器和客户云节点的安全远程访问。有关 RBAC 的更多信息，请参阅以下文档：

      • 管理用户和项目
      • RBAC 定义
      • 联合身份服务

      问：Yama 是什么？

      答：提供用于访问客户云的多因子认证，并提供云支持人员的访问记录。

      问：ELK (Elasticsearch, Logstash & Kibana) 是什么？

      答：用于云资源的日志管理。这些日志会保留在客户站点上。

      问：Boot-Strapper 是什么？

      答：提供服务器的裸机配置。Boot-Strapper 是镜像存储库的副本。

      安全性常见问题解答：

      问：谁能从“IBM 支持”访问 Bluemix Private Cloud Local？

      答：OpenStack 工程和运营团队的成员有权访问控制器、计算和节点、OpenStack API 和 Horizon。他们利用这种访问权来部署和运行私有云服务。这些支持工程师无法访问虚拟机管理器上的客户数据或工作负载。

      问：IBM 支持代表可以访问客户资源吗？

      答：IBM Bluemix Private Cloud 工程和运营团队没有相应的凭证，因此无法访问云中部署的任何云 VM 或应用程序。

      问：可通过 VPN 传递哪些信息？

      答：通过 VPN 只能传递部署和运行私有云服务所需的那些命令和控制消息。在 VPN 与客户数据面板（VM、工作负载、存储器、Neutron 网络）之间不路由任何数据。由 OpenStack 支持人员发起的安全管理远程访问会话。用于安装和修补私有云服务器的软件包。由云节点发出的监控警报和消息。在客户端设备上维护所有日志记录数据。

      问：谁可以获得这些信息？

      答：IBM Bluemix Private Cloud OpenStack 工程和运营团队成员可以访问这些信息以进行云管理和故障诊断。

      问：支持团队的访问控制策略是什么？

      答：支持团队在授予、重新评估和撤销客户云访问权时，遵循严格的 IBM Cloud 安全策略。

      问：Bluemix Private Cloud 能否使用客户注册过程？

      答：Bluemix Private Cloud 目前不支持，我们的流程符合 ISO27002 标准。

      问：Bluemix Private Cloud 可管理哪些项？

      答：Bluemix Private Cloud 可管理从裸机服务器到虚拟机管理器在内的 OpenStack 云。我们的流程、策略和工具均遵循 IBM Cloud 安全策略 (CSP)。CSP 符合行业标准 ISO27002，这是信息安全控制（即安全技术）方面的信息技术实践标准。
