      云网络和软件定义的网络 - 最佳实践白皮书

      • by Dustin Lundquist and Leslie Lundquist ● June 7, 2016

      • Tags:

        • sdn

        • sdn

        • 网络

        • 最佳实践

        • vlan

        • vxlan

        • l2pop

        • arp

        • 配置

      概述：

      在许多“企业”数据中心网络和许多下一代无线提供商网络中，正在从完全分布式的网络控制转变到通过软件定义的网络 (SDN) 实现集中式网络控制的网络环境。这种转变是为了应对网络行为中涌现的复杂性，而繁重的网络流量加上路由器和交换机的分布式本地记忆共同造成了这种复杂性。

      涌现的这些复杂性会影响云网络的设计，所以需要特别关注这些复杂性。工程师们致力于实现虚拟覆盖网络，并依赖于底层硬件网络的实施细节，而这可能导致生产云环境中出现不确定行为。我们为此提供了解决方案。

      本文提供了一些背景信息，解释了针对“软件定义的网络”作出一些选择（看似容易做出但可能导致后来面临困难）的原因和方式。它基于简单网络硬件和软件的已知行为准则，提供了一些用于创建可预测的稳定网络环境的替代方案。通常，它会检查 SDN 的最佳适用范围，并描述用户如何选择要用于 SDN 的正确技术。

      第 2 层“以太网交换机”的背景

      首先，我们来看一看以太网交换机的设计和运行原理，将这作为一个起点来开始了解繁忙网络环境中复杂性的提升速度。将只包含交换机最突出的特征。首先，请记住以太网交换机是在第二层 (L2) 协议级别运行，这意味着它将使用 MAC 地址而不是 IP 地址。另外请记住，在 L2 上，数据报称为“帧”，而在第 3 层 (L3) 上，数据报称为“数据包”。后面还会继续使用这一术语来简化讨论内容。

      首先将每次交换机接收到帧时发生的事情分解为简单的细节，这有助于我们了解随复杂性提升而发生的情况。在接收到帧时：

      1. 记住该帧的源地址，并存储端口信息。

      2. 根据已知地址列表检查目标地址。

      3. 如果在列表中找到目标，那么将该帧发送到相关端口。

      4. 如果未找到目标地址，那么将该帧发送到交换机中除传入端口以外的所有其他已知端口。此操作称为“泛洪”(Flooding)。将该帧复制所需的次数，以便向每个已知端口发送一个副本。

      这是一种很基本的方法，即使将许多交换机（数百个）连接到一个树中，只要树中没有循环，此方法就有效。“生成树”协议会检测并防止循环，因此，所有帧都不会出现循环。（这些帧称为“bpdu 帧”，即网桥协议数据单元。）这些帧可能由于网络拥塞或链路错误（例如，校验和无效）而被丢弃。

      作为此过程中的一个附加因素，在单个交换机上，如果在预先确定的时间段（通常为 5 分钟到 4 小时）内与已知地址之间未传输任何帧，那么这些已知地址可视为“已老化”。不难看出，随着不断接收新帧并且旧地址不断老化，所有交换机的状态不会总是一成不变。它们的已知地址列表会略有不同，这也是设计的一部分。

      通常由于大量新传入帧或泛洪而发生拥塞时，L2 上就会开始涌现复杂性。例如，假设存在下列情况：一个新帧上升到树中的某个点，因此树中其他部分的其他交换机不会将该 MAC 地址作为源。当来自相同源的帧进入另一个交换机时，会由于未知地址而导致泛洪。更多的交换机将忙于处理泛洪，因此，其他帧就有可能进入树中之前尚未处理过该源 MAC 地址的另一个交换机。 更多的泛洪相继发生，直到大多数或所有的交换机都记住（或重新记住）新帧的源 MAC 地址后，泛洪流量才会减少。

      与此同时，还会出现性能下降和其他问题。

      第 3 层“ARP 消息”的背景

      地址解析协议（或 ARP）在第 3 层 (L3) 上工作，其使用 IP 地址来帮助处理子网中“邻居”间的数据包路由。

      ARP 回答的问题是：如何向本地邻居发送数据包？（也就是说，如何识别并发送到同一子网中的主机？）

      脚注：在本文中，子网是指共享 IP 地址 (XX.XX.XX.me) 的网络（掩码）部分的一组主机。要访问子网以外的地址，会将数据包发送到作为网关的某个主机，这意味着该主机已经连接到较大的网络。

      ARP 实际上非常简单。它包括四种类型的消息，例如：

      1. “谁具有 192.0.2.10？”（请求）
      2. “我具有 192.0.2.10”（回复）
      3. “X 具有 192.0.2.10”（作为代理回复）
      4. 探针：为我分配此子网中的所有地址

      虽然可以广播免费的 ARP 消息，但通常将 ARP 回复单点广播到请求者。代理 ARP 可与路由配合使用。代理可回复远程主机上的条目，因为它可以将回复发送到远程主机：它具有该主机的 IP 地址。

      ARP 消息有 4 种类型，因为主机可以代表自身或其他主机回复，还可能有 ARP 探针。所有的 ARP 请求都将发送到以太网广播地址：FF:FF:FF:FF:FF:FF（全都为 F，2 个一组，共 6 组）。以太网子网上的任何主机都可以断言自己具有特定 IP 地址或其同级具有特定 IP 地址，等等。

      因此，正如您所推断，ARP 根本不安全。希望您信任自己的子网邻居。通常，这个问题称为“第一跳安全”，因为它与主机数据包能否通过第一跳安全地传递到路由器有关。

      如上一部分所述，以太网交换协议基于主机的 MAC 地址 (L2)（而不是其 IP 地址 (L3)）在子网内传送帧。因此，每个主机针对子网上与其通信的本地邻居，需要从 MAC 地址 (L2) 到 IP 地址 (L3) 的映射表。

      这里实际存在两级映射，如以下示意图中所示：

      IP      o    o     o
          \ /      |
MAC        o       o
           |       |
端口       o       o

      MAC 地址到端口是一对一映射，但 MAC 地址到 IP 地址可以是一对多映射。（这种映射差异在云部署中很常见，其中 VM 可具有内部和外部 IP 等，针对其连接的每个网络各有一个 IP。）

      在繁忙的子网中，可能有大量的 ARP 请求！每个主机会将其邻居的地址最多记忆 5 分钟。（也就是说，L3 上的子网邻居会像 L2 上的以太网交换机上的 MAC 地址一样“老化”。）事实上，如果将 ARP 和以太网交换机端口老化设置为相同值，那么子网上会发生有趣的情况；必须将其中一个设置为较长时间，否则最终出现的网络流量全都是广播！

      联合 L2 和 L3：SDN 顺势而生

      在出现大量广播流量之前，或者在过多的 MAC 地址开始在网络中通信以让交换机容纳其表中的所有地址之前，以太网交换和 ARP 请求的这些系统都还表现不错。（如果用完 MAC 地址段，许多交换机将开始泛洪。）为了突破这些限制，许多数据中心设计人员将本地网络分解成较小的 L2 段，并将其与路由器连接。换句话说，他们通过将问题分解成较小的 L2 域来攻克了问题。

      通过使用这种“分层路由”技术，可以将问题分解成 log N 个较小问题。

      然而，仍有许多客户希望当子网在其自己的数据中心内时，其所有主机都属于同一 L2 域，就像自然配置的那样。因此，许多云解决方案通过部署覆盖网络（指在提供商的 L3 基础架构上承载客户的第 2 层网络的一些技术）来将主机连接到子网域。

      例如，VxLAN 就是一种覆盖网络，通过此封装在 UDP 数据包中承载以太网帧。参与 VXLAN 覆盖网络的每个主机都具有一个 VTEP（VxLAN 隧道端点），这是一个包含 UDP 端口和 IP 地址的侦听 UDP 套接字。

      此情况适用于 SoftLayer 数据中心内的许多 IBM Bluemix Private Cloud 部署。由于 SoftLayer 将数据中心组织到本地化的硬件“pod”中，因此必须部署软件定义的覆盖网络 (SDN) 以连接尚未在同一硬件 pod 中建立的任何客户云部署。IBM Bluemix Private Cloud 3.0（及更高版本）已实施了许多软件定义的网络解决方案，可让客户将其云连接在一起，从而创建多种可行的混合云解决方案。（对于混合云，我们指的是将两个或更多个私有或公共计算机集群绑定在一起的能力，其中这些集群都旨在创建虚拟机及相关网络的集合 - 即组合多个云环境。）

      配置

      有时，我们会收到来自客户的问题，涉及一次可以部署多少个 VLAN 和 VxLAN 的终极限制。下面是有关配置的一些基本信息：

      在整个交换光纤网中配置 VLAN，因此光纤网中的每个交换机（可能有数百个）都能够访问所有 4096 个 VLAN。但是，如果需要使用第 4097 个 VLAN，那么必须在新的交换光纤网上完全配置该 VLAN。这是以太网的基本限制，与我们的平台无关。

      VxLAN 可突破此限制，它能够在以太网的全部 4096 个 VLAN 上各另外配置 65,000 个虚拟 VLAN。这在单个光纤网上最大可实现 268,435,456 个 VxLAN。简而言之，VLAN 耗尽实际并不会构成限制。

      对于 IBM Bluemix Private Cloud 标准配置，单个光纤网将支持 3000 多名客户，每个客户各有 65,000 个 VxLAN。

      在 Box Panel 中，可以通过名称（能反映与 VLAN 相关的交换光纤网）来跟踪 VLAN。我们从未超出光纤网的 VLAN 限制，所以只是命名为 VL###（例如 VL101）。为区分光纤网，可将其命名为“fabric1-VL101”、“fabric2-VL101”等。

      脚注：软件定义的网络 (SDN) 被开发为能够在中央数据库中定义策略集，然后使用计算来自动实施这些策略。此术语广泛用于指代网络覆盖的用途。
