      如何调整我的云主机的大小？

      功能：云主机大小调整（增加）

      描述：自 Bluemix Private Cloud 1.3 发行版起，我们已支持云主机大小调整（增加）。调整大小操作并非实时操作：它会停止并重新启动云主机。调整大小可在相同主机上进行，这对于小型环境很有用。

      云主机大小调整允许云主机将其云主机类型更改为更大的云主机类型。云主机只能调整为更大的云主机类型。此更改会造成短时间中断，此时停止云主机、迁移至更大的云主机类型并重新启动。此大小调整还可能将云主机重新定位到新的虚拟机管理器。

      请勿在调整大小过程进行时更新目标云主机类型（云主机的调整大小目标），否则调整大小操作可能失败并且云主机可能进入 ERROR 状态。

      调整大型云主机的大小可能相当麻烦，因为如果没有足够的磁盘空间用于新磁盘镜像，那么可能会导致故障。

      注：在调整云主机大小或进行迁移且其处于 VERIFY_RESIZE 状态后，在恢复联网之前可能有短暂延迟。

      有关更多信息，请访问 http://docs.openstack.org/user-guide/enduser/cli_change_the_size_of_your_server.html 以获取有关如何执行大小调整的文档。

      故障诊断

      如果大型云主机的调整大小操作所用时间非常长（例如，某些日志可能显示操作在 09:30 开始并在 14:41 出现认证错误），那么在此期间 Nova 用于完成此操作的认证令牌可能实际已到期。

      如果尝试启动新的调整大小操作，那么很可能再次发生相同错误。在此情况下，我们建议使用不同的方法。如果集群启用了块存储器，那么建议采取以下步骤：

      1. 关闭目标 VM。
      2. 使用 nova image-create 生成目标 VM 的快照。
      3. 使用 cinder create --image-id <SNAPSHOT UUID> 从该快照生成云硬盘。
      4. 使用 OpenStack image delete 或 glance image-delete 删除快照。
      5. 使用 nova boot --boot-volume 从云硬盘引导新的 VM。

      如果想要保留相同的 IP 地址，那么您可以：

      1. 使用以下命令获取引导 VM 的云硬盘的端口 UUID：neutron port-list | grep <IP of new VM>（第一列是端口 UUID）。

      2. 使用 nova interface-detach 从旧的 VM 除去端口

      3. 通过以下命令对新的 VM 上的端口进行更新以使用旧的 VM IP：neutron port-update --fixed-ip ip_address=<old VM IP> <port UUID>

      在对此交换感到满意后，您可以使用 nova delete 清除旧的 VM。

      此过程可能比您的预期更加复杂。但是，切换到作为访客后端的块存储器后具有多种优势：

      • 云主机类型大小调整将大幅加快，因为可跳过磁盘大小调整。
      • 使用块存储器可非常快速地迁移至新的虚拟机管理器（新的虚拟机管理器只连接到块存储器集群）。
      • 您可以使用 cinder extend 命令在运行时扩展 VM 的磁盘。

      如果不想使用块存储器，并且想要调整为较小或较大的云主机类型，那么可使用所需大小云主机类型创建新的云主机，并对所需数据执行 rsync 操作，将其从旧的云主机同步到新的云主机。

      为何 OpenStack 不允许减小云主机大小？

      您可以在 https://blueprints.launchpad.net/nova/+spec/no-downward-resize 处查看原因 - 基本上来说，允许减小云主机大小操作可能会影响数据完整性，因此不允许。作为安全的变通方法，您仍可以使用较小的云主机类型创建新 VM，然后对所需数据执行 rsync 操作，将其从较大的云主机同步到较小的云主机。

      在调整从云硬盘引导的云主机的大小时，云主机类型发生更改但是大小保持相同。为什么？

      这是预期的行为。云主机类型的磁盘大小不是从云硬盘引导时根磁盘的磁盘大小。您可以在以下地址检查此无效的 OpenStack 错误：https://bugs.launchpad.net/nova/+bug/1412993，其中包含进一步阐述。

      要调整磁盘大小，请访问 https://docs.openstack.org/user-guide/dashboard-manage-volumes.html#edit-a-volume 以调整云硬盘的大小。如果由于某些原因导致这无效，您还可以尝试 https://mcwhirter.com.au/craige/blog/2014/Resizing_a_Root_Volume_for_an_Openstack_Instance/ 中列出的步骤。
