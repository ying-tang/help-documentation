      Démarrage d'une instance à partir du tableau de bord Horizon (édition Newton d'OpenStack)

      • by Ying Tang ● 31 mars 2017

      • Tags:

        • horizon

        • horizon(horizon)

        • instance(instance)

        • launch(démarrer)

        • newton(newton)

        • rhosp(rhosp)

      Etapes

      1. Connectez-vous au tableau de bord Horizon.
      2. Sous la section Projet, développez Calcul et cliquez sur Instances. Si vous disposez d’instances, vous pouvez afficher leur liste et leurs détails, tels que leur nom, leur adresse IP et leur statut. Pour plus d’informations sur la gestion de vos instances, voir Comment gérer vos instances.
      3. Cliquez sur Lancer l’instance dans la partie supérieure droite du tableau de bord Horizon. La boîte de dialogue Lancer l’instance s’affiche.

      Parcourez les onglets suivants en renseignant les différentes valeurs, puis cliquez sur Lancer l’instance dans la partie inférieure droite de la boîte de dialogue.

      Saisie des valeurs de la boîte de dialogue Lancer l’instance

      Onglet Détails

      Valeurs Description
      Nom de l’instance Attribuez un nom à votre machine virtuelle. Le nom que vous lui attribuez devient le nom d’hôte initial du serveur. Il n’est pas garanti que les noms de serveur soient uniques lors de la création ; ainsi, deux instances peuvent porter le même nom d’hôte.
      Zone de disponibilité Cette valeur est définie par défaut sur la zone de disponibilité donnée par le fournisseur de cloud, compute_standard par exemple.
      Nombre Pour démarrer plusieurs instances, saisissez une valeur supérieure à 1. La valeur par défaut est 1.

      Pour continuer, cliquez sur Suivant ou sur l’onglet Source.

      Onglet Source

      La zone Sélectionner une source d’amorçage contient les options suivantes :

      Options Description
      Image Choisissez une image privée ou publique dans la liste de la table Disponibles, puis cliquez sur le signe plus (+) pour la déplacer vers la table Allouées. Vous pouvez éventuellement définir l’option Créer un volume sur Oui afin d’activer d’autres options de création de volume. Si vous avez créé un volume avec cette instance, vous pouvez définir l’option Supprimer le volume après avoir supprimé l’instance sur Oui pour supprimer ce volume une fois l’instance supprimée.
      Image instantanée d’instance Choisissez une image instantanée dans la table Disponibles, puis cliquez sur le signe plus (+) pour la déplacer vers la table Allouées.
      Volume Choisissez un volume dans la table Disponibles, puis cliquez sur le signe plus (+) pour le déplacer vers la table Allouées. Vous pouvez éventuellement définir l’option Supprimer le volume après avoir supprimé l’instance sur Oui pour supprimer ce volume une fois l’instance supprimée.
      Image instantanée de volume Choisissez une image instantanée de volume dans la table Disponibles, puis cliquez sur le signe plus (+) pour la déplacer vers la table Allouées. Vous pouvez éventuellement définir l’option Supprimer le volume après avoir supprimé l’instance sur Oui pour supprimer ce volume une fois l’instance supprimée.

      Remarque : l’option Image est définie par défaut, vous n’avez donc pas à en sélectionner une avant l’amorçage. Si vous utilisez l’option Image, vous pouvez choisir une image publique fournie par IBM, ou transférer et utiliser votre propre image. Pour plus d’informations sur le transfert d’une image vers IBM Bluemix Private Cloud, voir Images de cloud fournies par IBM Bluemix Private Cloud.

      Onglet Version

      Choisissez une version dans la liste, puis cliquez sur le signe plus (+) pour la déplacer vers la table Allouées.

      Onglet Réseaux

      Choisissez un réseau dans la liste, puis cliquez sur le signe plus (+) pour le déplacer vers la table Allouées. Vous pouvez allouer plusieurs réseaux à l’instance.

      Onglet Ports réseau

      Si vous ne spécifiez aucun port réseau, une valeur par défaut sera affectée à l’instance au moment de sa création et de son amorçage.

      Il se peut que vous ayez parfois à anticiper la création de ports par le biais d’API. Après avoir créé un port et l’avoir associé à l’un des réseaux sélectionnés dans l’onglet Réseaux, ce port apparaîtra dans la table Disponibles. Vous pouvez cliquer sur le signe plus (+) pour déplacer le port vers la table Allouées.

      Onglet Groupes de sécurité

      Choisissez un groupe de sécurité dans la liste, puis cliquez sur le signe plus (+) pour le déplacer vers la table Allouées.

      Onglet Paire de clés

      Spécifiez une paire de clés. Vous disposez des options suivantes :

      • Choisissez une paire de clés enregistrée dans la liste, puis cliquez sur le signe plus (+) pour la déplacer vers la table Allouées.
      • Cliquez sur Importer une paire de clés pour importer votre propre paire de clés. Copiez et collez votre clé publique, puis cliquez sur Importer une paire de clés.

      • Cliquez sur Créer une paire de clés pour créer et enregistrer une nouvelle paire de clés. Spécifiez le nom de la paire de clés, puis cliquez sur Créer une paire de clés.

        Vous serez invité à sauvegarder la nouvelle paire de clés sur le système hôte local.

      Onglet Configuration

      Les scripts personnalisés connectés aux instances peuvent exécuter des actions spécifiques au démarrage de l’instance. Vous pouvez saisir votre script directement dans la zone Script de personnalisation.

      Pour les instances Linux : le nom d’utilisateur par défaut est ibmcloud. Pour activer l’authentification par mot de passe via la console et SSH, utilisez les données de script suivantes pour le script de personnalisation (remplacez <VOTRE_MOT_DE_PASSE> par le mot de passe approprié) :

      #cloud-config password: <VOTRE_MOT_DE_PASSE> chpasswd: { expire: False } ssh_pwauth: True

      Pour les instances Windows : pour configurer vos nom d’utilisateur/mot de passe initiaux, utilisez les données de script suivantes pour le script de personnalisation (remplacez <VOTRE_NOM_UTILISATEUR> et <VOTRE_MOT_DE_PASSE> par le nom d’utilisateur et le mot de passe appropriés).

      rem cmd
net user <VOTRE_NOM_UTILISATEUR> <VOTRE_MOT_DE_PASSE> /logonpasswordchg:yes /add /y
net localgroup administrators <VOTRE_NOM_UTILISATEUR> /add

      Si votre navigateur prend en charge l’API HTML5 File, vous pouvez cliquer sur Parcourir pour charger votre script à partir d’un fichier. La taille de votre script ne doit pas dépasser 16 ko.

      Dans le menu déroulant Partition de disque, vous pouvez sélectionner Automatique pour redimensionner le disque et le définir sur une seule partition, ou Manuel pour créer plusieurs partitions sur le disque.

      Cochez la case Unité de configuration pour écrire des métadonnées sur une unité de configuration spéciale, puis spécifiez vos métadonnées dans l’onglet Métadonnées. Au moment de l’amorçage de l’instance, elle se connecte à l’unité de configuration pour accéder à vos métadonnées.

      Onglet Groupes de serveurs

      Les groupes de serveurs peuvent être utilisés pour contrôler les règles de planification d’affinité et d’anti-affinité d’un groupe de serveurs (instances).

      Il se peut que vous ayez parfois à anticiper la création de groupes de serveurs par le biais d’API. Après avoir créé un groupe de serveurs, il apparaîtra dans la table Disponibles. Vous pouvez cliquer sur le signe plus (+) pour déplacer le port vers la table Allouées.

      Onglet Indicateurs de planificateur

      Les indicateurs de planificateur peuvent être utilisés pour transmettre des informations de placement supplémentaires au planificateur Nova. Par exemple, vous pouvez utiliser un indicateur de planificateur pour vérifier qu’une instance est planifiée sur un hôte à partir d’un certain groupe de serveurs. Vous pouvez également créer un indicateur de planificateur personnalisé au format JSON. Pour plus de détails sur le mécanisme de planification et les types d’indicateur pris en charge, voir Compute Schedulers (en anglais).

      Onglet Métadonnées

      Glance propose une API commune permettant de définir des paires de clés et de valeurs qui balisent les métadonnées pour différents types de ressource (images, artefacts, volumes, versions, agrégats, etc). Si l’image que vous sélectionnez contient des métadonnées prédéfinies, elles s’afficheront dans la colonne Métadonnées disponibles.

      Vous pouvez créer des paires de clés et de valeurs supplémentaires dans la zone Personnalisé, puis cliquer sur le signe plus (+) pour les déplacer vers la liste Métadonnées existantes. Par exemple, vous souhaiterez peut-être transmettre les métadonnées personnalisées à l’application, de sorte à pouvoir les installer avec l’instance.

      Pour plus d’informations sur les métadonnées Glance, voir Metadata Definition Concepts (en anglais) et Using Glance’s Metadata Definitions Catalog Public APIs (en anglais).

      Comment se connecter à votre instance

      Remarque : assurez-vous d’avoir défini les règles de groupe de sécurité appropriées pour la connexion à votre instance via ssh ou rdp.

      1. Connectez-vous au tableau de bord Horizon d’IBM Bluemix Private Cloud.

      2. Développez le panneau Projet, puis l’onglet Calcul dans la zone de navigation de gauche, et cliquez sur Instances.

      3. Recherchez votre instance dans la liste des instances en cours et notez son adresse IP, 192.168.100.237 par exemple.

      4. Créez une connexion à votre instance en spécifiant son adresse IP.

      Connexion aux instances Linux

      Connectez-vous à votre instance via SSH ou VNC. L’ID utilisateur par défaut des images Linux fournies par IBM est ibmcloud. Une fois connecté à votre instance Linux, vous pouvez utiliser la commande sudo pour exécuter des commandes qui nécessitent en temps normal des droits d’accès d’utilisateur root. Sudo est configuré pour ne pas demander de mot de passe pour l’ID utilisateur par défaut.

      • A l’aide de votre client SSH favori, ouvrez une connexion à l’adresse IP appropriée de l’instance de machine virtuelle en utilisant la clé SSH privée de la paire de clés spécifiée au cours de la mise à disposition :

          ssh -i <chemin_de_votre_fichier_clé_privée_SSH> <ID_utilisateur>@<IP_instance>

      • A partir du tableau de bord Horizon d’IBM Bluemix Private Cloud, affichez la console VNC de l’instance en cliquant sur Console dans la liste déroulante située dans la colonne Actions de l’instance. La console VNC vous connecte via HTTPS.

      Remarque : pour vous connecter à une nouvelle instance Linux via la console en tant qu’utilisateur ibmcloud, vous devez avoir spécifié un mot de passe dans le script de personnalisation lors de la mise à disposition.

      Connexion aux instances Windows

      Connectez-vous à votre instance via Remote Desktop ou VNC. Vous devrez utiliser l’ID utilisateur et le mot de passe spécifiés dans le script de personnalisation lors de la mise à disposition de la machine virtuelle. Lorsque vous vous connectez pour la première fois, une invite vous demandant de changer votre mot de passe apparaît. Faites-le.

      • Utilisez votre client Remote Desktop favori pour vous connecter.
      • A partir du tableau de bord Horizon d’IBM Bluemix Private Cloud, affichez la console VNC de l’instance en cliquant sur Console dans la liste déroulante située dans la colonne Actions de l’instance. La console VNC vous connecte via HTTPS.

      Comment gérer vos instances

      Sous la section Projet, développez Calcul et cliquez sur Instances.

      Le tableau de bord contient la liste des instances pour le projet en cours. Dans la colonne Actions, une liste déroulante répertorie vos options, comme suit :

      • Créer une image instantanée : crée une image instantanée de l’instance.*
      • Associer l’IP flottante : associe une IP flottante disponible à un port de votre instance.
      • Connecter une interface : connecte l’interface à un réseau.
      • Déconnecter une interface : supprime un port créé pour l’instance.
      • Editer l’instance : modifie le nom de l’instance ou transfère les groupes de sécurité.
      • Mettre à jour les métadonnées : met à jour les métadonnées personnalisées spécifiées pour l’instance.
      • Editer les groupes de sécurité : supprime un groupe de sécurité existant ou affecte un nouveau groupe de sécurité.
      • Console : affiche la console VNC de l’instance.
      • Afficher le journal : affiche le journal de l’instance.
      • Mettre l’instance en pause : met l’instance en pause et sauvegarde son état en mémoire. Cette option conserve les ressources (UC virtuelles et mémoire).
      • Interrompre l’instance : interrompt l’instance et sauvegarde son état sur le disque. Cette option libère des ressources (UC virtuelles et mémoire).
      • Abandonner l’instance : abandonne l’image et arrête l’instance.
      • Redimensionner l’instance : met à jour le paramètre de version de l’instance.
      • Verrouiller l’instance : verrouille l’instance.
      • Déverrouiller l’instance : déverrouille l’instance.
      • Redémarrer l’instance : procède à un arrêt approprié et redémarre l’instance.
      • Réinitialiser l’instance : équivaut à une réinitialisation du serveur.
      • Arrêter l’instance : met l’instance hors tension.
      • Régénérer l’instance : régénère l’instance avec une autre image.
      • Supprimer l’instance : supprime l’instance.

      Cliquez sur l’option associée à l’opération à appliquer.
