      Création de réseaux supplémentaires dans OpenStack à l'aide du panneau Horizon

      • by Jason Kennedy ● 5 août 2015

      • Tags:

        • getting started

        • getting started(initiation)

        • additional networks(réseaux supplémentaires)

      Vous avez accès à votre tout nouveau cloud OpenStack et vous avez expressément demandé à ce qu’il soit configuré avec VXLAN car vous voulez séparer l’environnement de développement/test de votre réseau de production. Pouvez-vous le faire en utilisant des groupes de sécurité ? Non ? D’accord, poursuivez la lecture.

      Une instance typique d’IBM Bluemix Private Cloud possède un réseau interne et un réseau externe configurés par défaut, reliés par un routeur par défaut (nommé default). Pour créer votre propre réseau, vous devez reproduire cette configuration.

      (Attention : avant de continuer, assurez-vous d’utiliser VXLAN car sinon, la procédure échouera. Vous risquez également d’arrêter la Terre de tourner et de tous nous projeter dans l’Espace)

      Commençons par créer un réseau. Pour vous aider le mieux possible, je décrirai les méthodes d’interface de ligne de commande (CLI) et Horizon. Commençons par la méthode Horizon. La méthode CLI sera traitée dans un autre article prochainement.

      Sous Projet, accédez à l’onglet Réseau et sélectionnez Réseaux. Les réseaux interne et externe apparaissent, comme décrit précédemment.

      A partir d’ici, cliquez sur Créer un réseau dans le coin supérieur droit pour afficher la fenêtre en incrustation Créer un réseau.

      Saisissez votre nom de réseau (laissez Etat admin défini sur UP), puis cliquez sur l’onglet Sous-réseau (ou sélectionnez Suivant).

      Saisissez ici votre nom de sous-réseau (nous allons utiliser test_subnet, et test pour le nom du réseau), les adresses réseau (192.168.10.0/28) et l’IP de la passerelle (192.168.10.1 dans mon exemple).

      Cliquez maintenant sur Détails du sous-réseau ou Suivant. Ignorez cet écran si vous n’allouez qu’une partie de votre pool ou si vous souhaitez désigner des serveurs DNS ou des routes hôte.

      A partir d’ici, cliquez simplement sur le bouton Créer bleu. La section Réseaux devrait ressembler à ce qui suit :

      Ajoutons à présent un routeur.

      Cliquez sur Routeurs sous Réseaux dans l’arborescence de navigation ; aucun routeur ne devrait s’afficher, nous allons donc en créer un en cliquant sur Créer un routeur. La fenêtre Créer un routeur s’affiche. Choisissez un nom de routeur (nous allons utiliser test_router). Cliquez sur Créer un routeur. L’écran suivant devrait apparaître :

      En examinant notre routeur, vous remarquerez qu’il n’est pas connecté à un réseau externe, ce qui est pourtant nécessaire pour son fonctionnement. Sous Actions dans la partie droite de l’écran, cliquez sur Définir la passerelle. La fenêtre Définir la passerelle s’affiche sous le menu déroulant Réseau externe. Sélectionnez Externe, puis cliquez sur le bouton Définir la passerelle bleu. Votre passerelle est désormais connectée au réseau externe.

      Prenons maintenant du recul et examinons ce que nous avons fait jusqu’ici. Si vous cliquez sur Topologie de réseau à gauche, vous verrez votre réseau ainsi que le routeur que vous avez créé. Ce routeur est connecté au réseau externe, comme prévu. Il vous reste maintenant à connecter ce routeur à votre réseau, de sorte qu’il puisse router le trafic de votre instance. Si vous survolez votre routeur à l’aide du curseur, des détails s’affichent dans la section Topologie de réseau, notamment la possibilité d’ajouter une interface et de supprimer le routeur. Avant d’ajouter une interface, procédez comme suit :

      Cliquez sur Routeurs à gauche, puis cliquez sur votre routeur. L’écran Détails du routeur s’affiche, comme suit :

      Cet écran contient des informations sur votre routeur. Vous remarquez notamment que notre interface de passerelle externe a été ajoutée. Cliquez sur Ajouter une interface pour afficher la boîte de dialogue Ajouter une interface. Dans le menu déroulant Sous-réseau, sélectionnez le réseau que vous avez créé. Vous pouvez éventuellement affecter une IP à cette interface ou laisser le cloud s’en charger. Laissez la zone d’adresse IP vide pour que le cloud en affecte une. Cliquez sur le bouton Ajouter une interface bleu pour terminer.

      Avant de continuer, voici quelques informations sur les interfaces et les adresses IP. Chaque interface que vous ajoutez à un réseau nécessite une IP et la connexion de votre routeur au réseau externe nécessitait une adresse IP. Dans l’écran ci-dessus, votre interface de passerelle externe possède une IP fixe comprise dans votre plage d’IP flottantes. L’interface interne que vous venez de créer doit posséder sa propre IP fixe issue du pool de réseaux internes que vous avez créé. Notez que le statut sera initialement défini sur DOWN. Il basculera sur ACTIVE une fois la procédure terminée.

      A partir d’ici, si vous cliquez sur Topologie de réseau à gauche, vous devriez apercevoir qu’une connexion a été établie entre le réseau externe et celui que vous avez créé, via le routeur que vous avez créé et ancré à chaque réseau. Vous devriez à présent pouvoir démarrer une instance sur ce réseau et activer son accès en externe après lui avoir affecté une IP flottante.

      A partir de Calcul sous Projet, cliquez sur Instances. Dans la partie droite, cliquez sur Lancer l’instance pour afficher la fenêtre Lancer l’instance. Saisissez un nom d’instance (test_instance ici), puis sélectionnez Amorcer à partir d’une image dans le menu déroulant Source d’amorçage de l’instance. Nous allons créer une instance Cirros simple. Sélectionnez Cirros (12.6MB) dans le menu déroulant Nom de l’image.

      Cliquez à présent sur Accès et sécurité en bleu en haut de l’écran. Vous devriez déjà posséder une paire de clés ; assurez-vous qu’elle est sélectionnée. Vérifiez également que vous utilisez le groupe de sécurité approprié et cliquez sur la section suivante, Mise en réseau, en bleu en haut de l’écran.

      Sélectionnez ici votre réseau test. Cliquez sur Lancer. L’écran suivant devrait apparaître :

      Vous remarquez que l’adresse IP 192.168.10.1 est actuellement utilisée pour la passerelle d’interface interne sur votre routeur, elle ne peut donc pas être utilisée sur les instances. Cliquez sur le nom de l’instance pour accéder à Présentation. A partir d’ici, cliquez sur l’onglet Console. Une fois l’onglet ouvert, cliquez sur la section bleu foncé qui indique “Connected” et appuyez sur Entrée. Une invite de connexion devrait s’afficher. Connectez-vous à l’instance à l’aide des identifiants de connexion Cirros par défaut : nom d’utilisateur cirros, mot de passe cubswin:)

      Vous devriez pouvoir exécuter un ping sur une adresse externe et consulter votre IP. Il vous reste à l’affecter à une IP flottante.

      Sous Actions à droite, sélectionnez Associer l’IP flottante pour ouvrir la boîte de dialogue Gérer les associations d’IP flottantes. Si c’est la première fois que vous associez une IP, la liste sera vide. Vous devrez donc cliquer sur l’icône ‘+’ en regard de la liste déroulante. Sélectionnez Externe au pool, cliquez sur le bouton Allouer une IP bleu, puis sur Associer. Deux adresses IP devraient à présent être associées à votre instance.

      Vous devriez à présent pouvoir utiliser ssh dans votre instance à l’aide de son IP flottante et de votre clé associée, dans une instance sur votre propre réseau à l’aide de votre propre routeur.
