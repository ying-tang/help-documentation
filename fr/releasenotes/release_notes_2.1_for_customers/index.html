      Notes sur l’édition

      Edition 2.1.0 créée le 31 mars 2016

      Récapitulatif des nouvelles fonctionnalités de la dernière édition

      Le présent document fournit une description technique de base des dernières offres IBM Bluemix Private Cloud, notamment les offres Dedicated et Local Cloud. Les nouvelles fonctionnalités d’IBM Bluemix Private Dedicated et Local Cloud incluent :

      1. Types de noeud de calcul d’extension supplémentaires.

      2. Cluster de stockage par blocs hybride et noeuds d’extension.

      3. Direct Link désormais disponible dans IBM Bluemix Private Dedicated Cloud (trois options) et pour les applications de cloud hybride.

      4. Accès au réseau privé du centre de données IBM Cloud.

      5. Utilisation du contrôle d’accès basé sur les rôles (RBAC) pour définir les droits.

      6. Images de cloud supplémentaires fournies par IBM Bluemix.

      7. Options conforme à la norme HIPAA désormais disponibles pour IBM Bluemix Private Dedicated Cloud.

      Pour obtenir une documentation utilisateur complémentaire, accédez à la page d’aide principale.

      Une présentation générale du produit IBM Bluemix Private Cloud version 2.0 et ultérieure est disponible dans le présent document.

      Un récapitulatif des fonctionnalités issues des éditions précédentes qui auront un impact sur les utilisateurs de la présente édition est disponible à la fin du présent document.

      Blocs fonctionnels IBM Bluemix Private Cloud

      Chaque instance d’IBM Bluemix Private Cloud 2.1.0, Dedicated ou Local, démarre avec un cluster de noeuds de contrôleur et de traitement :

      IBM Bluemix Private Cloud :

      • Un cloud peut démarrer en tant que cluster à trois noeuds avec deux noeuds d'hyper-convergence (contrôleur + traitement) et un noeud de calcul uniquement. • Un cloud peut démarrer en tant que cluster à cinq noeuds avec une paire de noeuds de contrôleur uniquement dédiés et trois noeuds de calcul uniquement.

      IBM Bluemix Private Cloud Local :

      • Chaque cloud doit comporter au moins un noeud de contrôleur uniquement dédié et trois noeuds de calcul uniquement.

      Vous pouvez, à partir de l’un de ces emplacements de démarrage (par ex., cluster d’hyper-convergence à trois noeuds ou cluster avec contrôleurs dédiés à cinq noeuds), ajouter des noeuds de calcul de n’importe quel type, ainsi que le stockage par blocs (Ceph) et/ou le stockage d’objets (Swift).

      Blocs fonctionnels supplémentaires :

      • Pour un cloud dédié de capacité Standard, les pare-feux de passerelle dédiée sont inclus par défaut • La capacité Enterprise permet d'inclure le double des ressources ainsi que 10 Gbits/s de connectivité réseau • Noeuds de contrôleur dédiés (ou noeuds supplémentaires) • Noeuds de calcul (ou noeuds supplémentaires) : Standard ou Enterprise, IO ou High Storage (nouveau)

      Noeuds de calcul d’extension facultatifs désormais disponibles

      Vous pouvez éventuellement sélectionner l’une des extensions de noeud de calcul suivantes à ajouter à votre cloud Standard ou Enterprise :

      • Noeuds de calcul d'extension spécialisés : IO ou HIGH STORAGE (nouveauté de cette édition) • Noeuds de calcul Standard ou Enterprise supplémentaires : STANDARD COMPUTE ou ENTERPRISE COMPUTE

      Les noeuds de calcul d’extension permettent de remplir vos exigences dans le sens où ils fournissent un débit de stockage éphémère en E-S plus rapide, un volume de mémoire RAM plus élevé et un espace de stockage éphémère plus étendu. Les deux types de noeud d’extension (IO et HIGH STORAGE) permettent de mieux couvrir les types d’instance et les charges de travail. Dans cette édition 2.1.0, les noeuds de calcul sont regroupés en zones de disponibilité pour pouvoir mettre en correspondance les charges de travail avec le type de noeud de calcul approprié plus facilement.

      Noeuds de stockage par blocs hybrides facultatifs désormais disponibles

      Les noeuds de stockage par blocs hybrides permettent d’augmenter la densité du stockage par blocs à coût réduit.

      • Vous pouvez sélectionner le type de stockage par blocs SSD ou HYBRID • Vous pouvez sélectionner Stockage d'objets

      Direct Link désormais disponible dans les centres de données IBM Cloud

      Pour Dedicated Cloud, vous pouvez également sélectionner la connectivité Direct Link dans les options de réseau :

      • Vous pouvez sélectionner une méthode d'accès parmi Public, VPN ou Direct Link

      Direct Link est disponible dans les centres de données IBM Cloud. Il permet de se connecter au réseau principal privé ou au réseau du centre de données IBM Cloud en ignorant l’Internet public pour la connexion à vos clouds Dedicated.

      Direct Link peut être utilisé pour créer un cloud hybride, si vous souhaitez relier votre cloud Local à un cloud Dedicated qui s’exécute dans un centre de données IBM Cloud. Il est disponible par incréments de 1 Gbits/s et 10 Gbits/s.

      Disponibles :

      • IBM Bluemix Private Direct Link Cloud Exchange 1 Gbit/s
• IBM Bluemix Private Direct Link Cloud Exchange 10 Gbits/s
• IBM Bluemix Private Direct Link Colocation 1 Gbit/s
• IBM Bluemix Private Direct Link Colocation 10 Gbits/s
• IBM Bluemix Private Direct Link Network Service Provider (NSP) 1 Gbit/s
• IBM Bluemix Private Direct Link Network Service Provider (NSP) 10 Gbits/s

      Accès au réseau privé du centre de données IBM Cloud

      L’accès au réseau privé du centre de données IBM Cloud vous permet de configurer la connectivité entre deux clouds IBM Bluemix Private, à l’aide de la fonctionnalité du réseau privé du centre de données IBM Cloud, lorsque les deux instances d’IBM Bluemix Private Cloud qui seront connectées sont déployées sous le même compte IBM Cloud. Sélectionnez ce service pour activer un trafic réseau privé illimité à faible temps d’attente entre vos instances d’IBM Bluemix Private Cloud.

      Pour plus d’informations, voir le document principal dans le présent article.

      Pour obtenir une limitation connue, cliquez ici.

      Contrôle d’accès basé sur les rôles (RBAC)

      Plusieurs rôles, qui offrent des fonctionnalités d’accès spécifiques au sein de votre cloud, sont créés par défaut. Cette version d’IBM Bluemix Private Cloud introduit le concept de groupes. Pour plus d’informations sur les utilisateurs, les rôles et les groupes, voir ce document.

      Images de cloud supplémentaires disponibles

      IBM Bluemix Private Cloud 2.1.0 est pré-rempli avec des images x86_64 Cirros 0.3.3. Des images de cloud supplémentaires sont disponibles, fournies à titre gracieux au client par IBM Bluemix. Ces images sont les suivantes :

      • CentOS 6.7 x86_64
• CentOS 7.2 x86_64
• Ubuntu Server 12.04 LTS x86_64
• Ubuntu Server 14.04 LTS x86_64
• Windows Server 2008 R2 SP1 Datacenter
• Windows Server 2008 R2 SP1 Enterprise
• Windows Server 2008 R2 SP1 Standard
• Windows Server 2012 R2 Datacenter
• Windows Server 2012 R2 Standard

      Pour utiliser l’une de ces images, consultez les informations supplémentaires dans ce document.

      HIPAA et IBM Bluemix Private Cloud

      • Vous pouvez sélectionner l'activation HIPAA pour votre instance d'IBM Bluemix Private Cloud

      Un cloud conforme à la norme HIPAA possède les mêmes options de configuration et d’extension que toute autre offre IBM Bluemix Private Cloud, mais nous activons plus de 50 contrôle de sécurité des informations supplémentaires afin d’aider nos clients qui doivent stocker des renseignements médicaux personnels (PHI). A cause de cette amélioration de la fonctionnalité de sécurité des informations, un cloud conforme à la norme HIPAA doit être l’être à compter de sa création : vous ne pouvez pas mettre à niveau un cloud existant pour activer le protocole HIPAA. De même, vous ne pouvez pas rétromigrer un cloud HIPAA IBM Bluemix Private : le cloud HIPAA existant doit être nettoyé (par protocole HIPAA) et vous devez commander un nouveau cloud doté de contrôleurs non HIPAA.

      Un cloud HIPAA IBM Bluemix Private ressemble principalement à tout autre cloud IBM Bluemix Private, à quelques exceptions :

      1.  Chaque cloud HIPAA IBM Bluemix Private doit être doté de contrôleurs dédiés (et non de contrôleurs de convergence).
2.  Des frais périodiques mensuels HIPAA supplémentaires sont imputés par cluster et par actif géré.
3.  Plus de 50 contrôles supplémentaires sont activés et la responsabilité de ces contrôles est partagée entre vous-même et IBM Blue Box.

      Les clouds HIPAA IBM Bluemix Private peuvent être déployés dans n’importe quel centre de données IBM Cloud prêt à déployer IBM Bluemix Private Cloud.

      Prérequis : vous devez signer un accord de partenariat avant de commander un cloud HIPAA IBM Bluemix Private. Vous devez passer votre commande de cloud en utilisant la configuration minimale pour les clients HIPAA. Vous ne pourrez pas activer HIPAA après avoir initié le service.

      Récapitulatif des nouvelles fonctionnalités issues des éditions précédentes qui auront un impact sur les utilisateurs de la présente édition

      La présente section répertorie brièvement les fonctionnalités issues des éditions précédentes d’IBM Bluemix Private, principalement l’édition 2.0.1, qui auront un impact sur votre utilisation de l’édition 2.1.0, en particulier si vous effectuez une mise à niveau à partir d’une édition 1.x. Pour plus d’informations sur ces fonctionnalités, contactez votre équipe de support IBM Bluemix Private Cloud.

      Fonctionnalités qui auront un impact sur les utilisateurs, les groupes et les rôles de la présente édition :

      Panneau Masquer les agrégats d’hôtes de cloud_admin : ce panneau Horizon a été supprimé afin que les utilisateurs cloud_admin aient plus de visibilité. Etant donné que les utilisateurs cloud_admin ne sont plus autorisés à manipuler les agrégats d’hôtes, l’affichage de ce panneau dans Horizon est inapproprié.

      Les utilisateurs cloud_admin ne sont plus autorisés à gérer les agrégats d’hôtes : voir ci-dessus.

      Affichage d’un plus grand nombre de panneaux admin pour les utilisateurs cloud_admin : nous avons activé plus de panneaux d’administration pour les utilisateurs cloud_admin dans Horizon. Ces panneaux incluent “Présentation,” “Hyperviseurs,” “Instances,” “Volumes,” “Versions,” “Images” et “Valeurs par défaut.”

      Gestion des utilisateurs par nom sur la ligne de commande : nous avons corrigé un incident qui empêchait les utilisateurs d’être gérés par nom à partir de la ligne de commande. Auparavant, les tentatives de référencement d’un utilisateur par nom généraient une erreur.

      Les utilisateurs cloud_admin sont autorisés à rendre les images Glance publiques : le rôle cloud_admin possède désormais les privilèges lui permettant de rendre les images Glance publiques.

      Fonctionnalités qui auront un impact sur Horizon, Glance, Nova et Heat dans la présente édition :

      Actualisation du codebase Horizon pour corriger les bogues liés aux sauvegardes du volume : dans Horizon, un incident qui empêchait la création de sauvegardes du volume Cinder a été corrigé.

      Actualisation du codebase Horizon pour autoriser le transfert d’images à partir de l’URL Swift (distribution d’images IBM Bluemix organisée) : dans Glance, un incident qui empêchait le transfert d’images via une URL a été corrigé. Les URL issues de Box Panel peuvent désormais être utilisées pour transférer des images dans Glance.

      Ajout de DiskFilter pour la planification Nova : nous avons résolu un incident selon lequel les générations de planification Nova sur les hyperviseurs ne disposaient pas d’un espace disque libre suffisant pour satisfaire la génération.

      Optimisation de Heat pour la prise en charge des modèles de grande taille : bugs.launchpad.net/heat/+bug/1491185 : nous avons résolu un incident qui entravait les déploiements de pile Heat avec des modèles de grande taille.

      Activation du redimensionnement d’instance (vers une taille plus grande) : le redimensionnement d’instance permet de redéfinir la version d’une instance sur une version de plus grande taille. L’opération de redimensionnement n’est pas effectuée lors du fonctionnement de l’instance : cette dernière est arrêtée, puis redémarrée. Pour plus d’informations, voir ce document.

      Notes sur l’édition à venir

      Les notes sur l’édition seront publiées régulièrement afin de fournir des détails sur les modifications apportées à l’offre et sur la manière dont ces modifications pourraient avoir un impact sur votre utilisation de l’offre IBM Bluemix Private Cloud. Par exemple, lorsqu’une nouvelle version d’OpenStack est installée et qu’elle est susceptible d’activer de nouvelles fonctionnalités, vous recevrez les notes sur l’édition contenant les détails.
