      Activation de vos licences Windows dans IBM Bluemix Private Cloud

      Le catalogue d’images fourni par IBM® contient des images Microsoft® Windows regroupées sous la forme d’un modèle BYOL (apportez votre propre licence). Conformément aux Droits d’utilisation du fournisseur de services Microsoft, IBM peut ne pas fournir les services d’activation KMS (services de gestion de clés) pour votre image BYOL Windows.

      Vous pouvez activer vos licences Windows de différentes manières :

      • Créer un serveur KMS dans votre environnement IBM Bluemix Private Cloud
      • Utiliser un serveur KMS existant sur votre site ou à un autre emplacement
      • Procéder à l’activation directe en ligne avec Microsoft
      • Procéder à l’activation hors ligne par téléphone avec Microsoft

      Vous pouvez utiliser une ou plusieurs de ces méthodes selon le cas. Tenez compte des points suivants au moment de choisir votre méthode :

      • Sur les serveurs KMS, le nombre de clients pouvant être activés est limité par un seuil minimal. Si votre service KMS est hébergé sous Windows Server 2008 ou Windows Server 2012, vous devez activer cinq clients au minimum. Si vous activez moins de cinq licences, vous ne pourrez pas utiliser KMS dans IBM Bluemix Private Cloud.
      • Pour utiliser un serveur KMS existant situé en dehors de votre environnement IBM Bluemix Private Cloud, vous devez établir une connectivité adaptée. Vous devez notamment activer les pare-feu Bluemix Private Cloud et les pare-feu permettant aux instances Windows de Bluemix Private Cloud d’initier le trafic vers le serveur KMS sur le port tcp 1688.
      • Pour utiliser l’activation en ligne directe, vos instances Windows doivent disposer d’une connectivité sortante à Internet. Les instances ne disposent pas d’une connectivité Internet par défaut, mais vous pouvez demander un accès dans votre commande initiale ou soumettre un ticket de demande de service afin de mettre à jour l’accès à tout moment.

      Création d’un serveur KMS dans IBM Blue Box Cloud

      Vous pouvez configurer n’importe quelle instance Windows Server 2008 ou ultérieur de sorte à activer le service KMS. Pour agir en tant que serveur KMS, l’instance Windows doit disposer d’une connectivité sortante à Internet pour pouvoir communiquer avec les serveurs de licences Microsoft. Vous devez également acheter une clé d’hôte KMS auprès de Microsoft pour recevoir une autorisation spécifique.

      Pour convertir une instance Windows 2008 R2 en KMS dans IBM Bluemix Private Cloud, procédez comme suit :

      1. Téléchargez ce fichier de mise à jour Microsoft sur votre instance.

      2. Installez la mise à jour.

      3. Enregistrez la clé d’hôte KMS à l’aide de cette commande :

         %SystemRoot%\System32\CScript.exe %windir%\system32\slmgr.vbs /ipk <votre_clé_hôte_KMS>

      4. Activez la nouvelle clé d’hôte KMS à l’aide de cette commande :

         %SystemRoot%\System32\CScript.exe %windir%\system32\slmgr.vbs /ato
      5. Redémarrez le service de gestion de licences des logiciels à l’aide de cette commande : 
         net stop sppsvc && net start sppsvc
      6. Ouvrez le port 1688 afin que les clients KMS puissent s’enregistrer auprès de l’hôte KMS à l’aide de cette commande : 
         netsh firewall set portopening tcp 1688 KMS enable

      Utilisation d’un serveur KMS existant

      Si vous disposez déjà d’un serveur KMS sur votre site ou à un autre emplacement, vous pouvez l’utiliser à condition que les connexions réseau appropriées aient été établies. Pour que le serveur Windows puisse être activé avec le serveur KMS, il doit pouvoir initier la communication avec le serveur KMS (généralement sur le port TCP 1688). Créez un ticket de demande de service pour appliquer les règles statiques appropriées dans le pare-feu IBM Bluemix Private Cloud.

      Activation de votre serveur Windows par le biais d’un serveur KMS

      Pour activer votre serveur Windows par le biais d’un serveur KMS, vous avez besoin d’activer l’adresse IP et la connectivité du réseau sur le serveur KMS, ainsi que les clés de configuration du client KMS appropriées à partir de Microsoft Technet.

      Après avoir créé une instance à partir de l’image Microsoft Windows Server, connectez-vous à cette instance et procédez comme suit :

      1. Installez une clé de client KMS sur votre instance Windows. Ouvrez une fenêtre d’invite de commande et exécutez la commande suivante : 
         slmgr /ipk <clé_client_KMS>

        Consultez la page Annexe A : Clés d’installation du client KMS. Par exemple, pour activer une instance Windows 2008 R2 Standard, utilisez YC6KT-GKW9T-YTKYR-T4X34-R7VHC.

      2. Définissez l’adresse IP et le port du serveur KMS sur votre instance Windows. Le port TCP par défaut utilisé par les clients pour se connecter à l’hôte KMS est 1688 : 
         slmgr /skms <adresse_IP_serveur_KMS>:<port_serveur_KMS>
      3. Synchronisez l’heure avec votre serveur KMS : 
         net start w32time
 w32tm /config /manualpeerlist:<adresse_IP_serveur_KMS> /update
 w32tm /resync
      4. Activez votre instance Windows : 
         slmgr.vbs /ato
      5. Restaurez l’heure du serveur par défaut : 
         w32tm /config /manualpeerlist:time.windows.com /update

      Pour plus de détails sur l’activation des licences Windows, utilisez les ressources suivantes :

      • Understanding KMS (en anglais)
      • Deploying KMS Activation (en anglais)
      • Determining Product Key needs (en anglais)
      • Annexe A : Clés d’installation du client KMS

      Activation directe

      Si vous ne devez activer qu’un très petit nombre de serveurs Windows, il se peut que vous ne souhaitiez pas utiliser un serveur KMS. Vous pouvez activer le serveur directement auprès de Microsoft, soit en contactant Microsoft par téléphone, soit en activant le serveur en ligne. Pour activer votre serveur en ligne, votre instance doit disposer d’une connectivité à Internet sortante afin de contacter Microsoft License Service.
