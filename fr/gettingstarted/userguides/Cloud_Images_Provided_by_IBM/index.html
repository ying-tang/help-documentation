      Guide d'utilisation des images de cloud (Community Edition)

      • by Yanqi Dou ● 9 mars 2016

      • Tags:

        • userguides

        • userguides(guides d'utilisation)

        • images(images)

        • glance(glance)

        • image-create(image-create)

        • instance(instance)

      IBM Bluemix Private Cloud 3.0 est pré-rempli avec des images x86_64 Cirros 0.3.3. IBM Bluemix Private Cloud fournit d’autres images de cloud téléchargeables pour que vous puissiez exécuter des instances avec le système d’exploitation de votre choix. Le présent document décrit la procédure de téléchargement de ces images ainsi que leur utilisation. Pour vérifier que vous utilisez les images les plus récentes, vous pouvez consulter les Notes sur l’édition des images de cloud à tout moment.

      Pour plus d’informations sur l’utilisation d’images avec OpenStack, consultez le site Web d’OpenStack : https://docs.openstack.org/image-guide/

      Prérequis

      1. Si vous prévoyez d’utiliser les outils de ligne de commande, assurez-vous d’avoir installé le client Glance d’OpenStack. Vous trouverez des instructions d’installation dans notre base de connaissances : Initiation à l’API OpenStack.

      2. Assurez-vous d’avoir accès à Box Panel pour pouvoir obtenir les URL de téléchargement temporaires des images de cloud fournies par IBM.

      3. Le transfert d’une image vers Glance en utilisant tempUrl comme emplacement d’image est uniquement pris en charge sur Bluemix Private Cloud 2.1.0 et ultérieur.

      Concepts de base

      Paramètre Description
      Image Une image de machine virtuelle, appelée simplement “image” dans le présent document, est un fichier unique qui contient un disque virtuel sur lequel un système d’exploitation amorçable est installé. Utilisez les images pour créer des instances de machine virtuelle au sein du cloud.
      Instance Les instances sont des machines virtuelles qui s’exécutent dans le cloud OpenStack.
      Version Le terme version décrit la taille d’une instance. Elle définit les spécifications de ressource pré-déterminées par rapport au volume d’UC virtuelle, de mémoire RAM et d’espace disque disponible sur une instance.

      Sélection d’une image de cloud alternative fournie par IBM Bluemix Private Cloud

      L’instance d’IBM Bluemix Private Cloud de cette édition est pré-remplie avec des images x86_64 Cirros 0.3.3. Des images de cloud supplémentaires sont disponibles, fournies à titre gracieux au client par IBM Bluemix Private Cloud.

      Vous pouvez transférer et installer un logiciel de système d’exploitation acquis séparément, ou mettre à disposition l’une des images de système d’exploitation suivantes, disponibles en tant qu’instance de machine virtuelle. Vous êtes chargé de respecter toutes les dispositions du contrat de licence du système d’exploitation applicable et d’acquérir les autorisations d’utilisation appropriées pour chaque instance de machine virtuelle.

      • CentOS 6.8 x86_64
      • CentOS 7.2 x86_64
      • Ubuntu Server 12.04 LTS x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      Les dispositions client de n’importe quelle instance virtuelle de calcul pour le système d’exploitation Windows sont les suivantes : le client doit posséder et gérer son propre serveur de licences Microsoft Windows pour pouvoir activer ces instances virtuelles de calcul pour le système d’exploitation Windows. Pour plus d’informations, voir Comment activer des licences Windows dans IBM Bluemix Private Cloud.

      Obtention des téléchargements d’image de cloud à partir de Box Panel

      1. Connectez-vous à Box Panel à l’aide des données d’identification Box Panel. La page Images de cloud de Box Panel est accessible si vous disposez au moins d’un cloud et si vous êtes un contact client Principal ou Technique.

      2. A partir du tableau de bord Box Panel, cliquez sur Services et sélectionnez Images de cloud.

      3. La page Images de cloud s’affiche. Chaque image apparaît sous forme de carte.

      4. Cliquez sur la carte correspondant à l’image qui vous intéresse. Une fenêtre modale, contenant votre adresse tempURL, s’affiche.

      5. L’adresse tempURL qui s’affiche reste valide pendant 24 heures pour que vous puissiez télécharger l’image de cloud.

      Transfert d’une image vers le référentiel Glance d’IBM Bluemix Private Cloud à l’aide de l’adresse tempURL de votre image de cloud

      Remarques :

      • Seuls les utilisateurs dotés du rôle cloud_admin pourront rendre une image publique aux autres utilisateurs et au projet.
      • Le transfert d’une image vers Glance en utilisant l’adresse tempUrl directement comme emplacement d’image n’est pas applicable à IBM Bluemix Private Cloud qui a bloqué l’accès Internet sortant à partir du plan de contrôle OpenStack.
      • Le transfert d’une image vers Glance en utilisant l’adresse tempUrl directement comme emplacement d’image est uniquement pris en charge sur IBM Bluemix Private Cloud 2.1.0 et ultérieur, pas sur les éditions ANTERIEURES à 2.1.0.
      • Pour les instances d’IBM Bluemix Private Cloud ANTERIEURES à 2.1 ou sans accès Internet sortant, suivez les instructions de la rubrique Transfert d’une image vers le référentiel Glance d’IBM Bluemix Private Cloud à l’aide des fichiers image téléchargés.

      Vous pouvez transférer une image à l’aide du tableau de bord OpenStack ou du client de ligne de commande.

      Transfert d’une image à l’aide du tableau de bord OpenStack

      1. Connectez-vous au tableau de bord OpenStack d’IBM Bluemix Private Cloud.
      2. Sous le panneau Projet, développez Calcul et cliquez sur Images.

      3. Cliquez sur Créer une image dans la page. Spécifiez les paramètres suivants dans la nouvelle page, puis cliquez sur Créer une image.

        Paramètre Description
        Nom Saisissez un nom pour l’image.
        Description Saisissez éventuellement une courte description de l’image.
        Source de l’image Sélectionnez Emplacement de l’image.
        Emplacement de l’image Spécifiez l’adresse URL temporaire (tempURL) de téléchargement de l’image de cloud.
        Format Sélectionnez le format approprié (par exemple, QCOW2) à l’image.
        Architecture Spécifiez l’architecture (par exemple, i386 pour une architecture 32 bits ou x86_64 pour une architecture 64 bits).
        Taille minimale du disque (Go) Pour une image Linux, utilisez 5. Pour une image Windows, utilisez 30.
        Mémoire RAM minimale (Mo) Pour une image Linux, utilisez 512. Pour une image Windows, utilisez 2048.
        Copier les données Cette option doit être sélectionnée pour copier les données image dans le service d’image.
        Public Cochez cette case pour rendre l’image publique à tous les utilisateurs sur tous les projets.
        Protégé Cochez cette case pour que seuls les utilisateurs dotés de droits puissent supprimer l’image.

        Par exemple :

         Nom : Ubuntu Server 14.04 LTS x86_64 - 20160301
 Description : Image de cloud Ubuntu Server 14.04 LTS x86_64 publiée le 01/03/2016
 Source de l'image : Emplacement de l'image
 Fichier image : <tempURL_pour_QCOW2>
 Format : QCOW2 - QEMU Emulator
 Architecture : x86_64
 Taille minimale du disque (Go) : 5
 Mémoire RAM minimale (Mo) : 512
 Copier les données : True
 Public : True
      4. Vérifiez que l’image créée figure dans la liste des images. L’image est mise en file d’attente pour être transférée. Le changement de statut de “Mis en file d’attente” à “Actif” peut prendre un certain temps.

      Remarque : seuls les utilisateurs dotés du rôle cloud_admin pourront rendre une image publique aux autres utilisateurs et au projet.

      Transfert d’une image à l’aide du client de ligne de commande OpenStack

      Vous pouvez transférer les images par le biais du client Glance. Pour plus d’informations, voir Installation des clients OpenStack.

      Créez une image.

      • Pour les images Linux, utilisez -- min-disk 5 --min-ram 512

      • Pour les images Windows, utilisez --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <nom_image> --min-disk <disque_min> --min-ram <ram_min> --disk-format qcow2 --container-format bare --is-public True --copy-from  <URLtemp_vers_fichier_image_qcow2>

      Vérifiez si l’image a été créée avec succès. L’image est mise en file d’attente pour être transférée. Le changement de statut de “Mis en file d’attente” à “Actif” peut prendre un certain temps.

      $ glance image-show <ID-image>

      Transfert d’une image vers le référentiel Glance d’IBM Bluemix Private Cloud à l’aide des fichiers image téléchargés

      Remarques :

      • Seuls les utilisateurs dotés du rôle cloud_admin pourront rendre une image publique aux autres utilisateurs et au projet.

      • Lorsque vous téléchargez l’image de cloud en utilisant l’adresse tempUrl, vérifiez que le jeton tempUrl est mis en échappement ou entre guillemets. Par exemple :

        curl -f -o <fichier_sortie> '<tempUrl_téléchargement_image_cloud>'
curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      Procédez comme suit :

      1. Téléchargez les fichiers .qcow2 et .md5sum de l’image.

      2. Téléchargez le fichier image .qcow2 et le fichier de total de contrôle .md5sum en utilisant l’adresse tempURL et placez ces 2 fichiers sous le même dossier. Par exemple :

         ubuntu-guest-image-14.04-20160301-x86_64.qcow2
 ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      3. Vérifiez le fichier image .qcow2 téléchargé à l’aide du fichier de total de contrôle .md5sum. Par exemple :

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

        Si le résultat renvoie “OK”, l’image qcow2 téléchargée est valide.

      4. Transférez l’image vers Glance. Vous pouvez transférer l’image à l’aide du tableau de bord OpenStack ou du client de ligne de commande.

      Connexion aux instances Linux

      Connectez-vous à votre instance via SSH ou VNC. L’ID utilisateur par défaut des images Linux fournies par IBM est ibmcloud. Une fois connecté à votre instance Linux, vous pouvez utiliser la commande sudo pour exécuter des commandes qui nécessitent en temps normal des droits d’accès d’utilisateur root. Sudo est configuré pour ne pas demander de mot de passe pour l’ID utilisateur par défaut.

      • A l’aide de votre client SSH favori, ouvrez une connexion à l’adresse IP appropriée de l’instance de machine virtuelle en utilisant la clé SSH privée de la paire de clés spécifiée au cours de la mise à disposition :

          ssh -i <chemin_de_votre_fichier_clé_privée_SSH> <ID_utilisateur>@<IP_instance>

      • Affichez la console VNC de l’instance en cliquant sur le nom de l’instance dans le tableau de bord OpenStack d’IBM Bluemix Private Cloud, puis cliquez sur l’onglet Console. La console VNC vous connecte via HTTPS.

      Remarque : pour pouvoir vous connecter via la console à une nouvelle instance Linux en tant qu’utilisateur ibmcloud, vous devez avoir spécifié un mot de passe dans le script de personnalisation lors de la mise à disposition.

      Connexion aux instances Windows

      Connectez-vous à votre instance via Remote Desktop ou VNC. Vous devrez utiliser l’ID utilisateur et le mot de passe spécifiés dans le script de personnalisation lors de la mise à disposition de la machine virtuelle. Lorsque vous vous connectez pour la première fois, une invite vous demandant de changer votre mot de passe apparaît. Faites-le.

      • Utilisez votre client Remote Desktop favori pour vous connecter.
      • Affichez la console VNC de l’instance en cliquant sur le nom de l’instance dans le tableau de bord OpenStack d’IBM Bluemix Private Cloud, puis cliquez sur l’onglet Console. La console VNC vous connecte via HTTPS.

      Mise à jour d’une image

      Lorsque vous recevez une notification de la part du personnel des opérations IBM Bluemix Private Cloud indiquant que les images de cloud sont mises à jour, vous pouvez désactiver l’image existante et recréer une image en utilisant l’URL de téléchargement obtenue à partir de Box Panel pour l’image mise à jour.

      Utilisation du client de ligne de commande OpenStack pour désactiver l’image existante

      1. Obtenez l’ID de l’image existante :

         $ glance image-list

      2. Désactivez l’image :

        $ glance image-deactivate <ID_image>

      3. Affichez le statut de l’image :

        $ glance image-show <ID_image>

      4. Après avoir désactivé l’image existante, re-créez la nouvelle image en procédant comme décrit à la section Transfert d’une image vers le référentiel Glance d’IBM Bluemix Private Cloud à l’aide de l’adresse tempURL de votre image de cloud du présent guide.

      Obtention d’aide relative aux images de cloud auprès du personnel des opérations IBM Bluemix Private Cloud

      Pour obtenir de l’aide auprès du personnel des opérations IBM Bluemix Private Cloud à propos d’une image de cloud spécifique, il est utile de fournir le buildID de l’image. Obtenez le buildID de l’image, puis créez un ticket de demande de service avec l’aide du personnel des opérations IBM Bluemix Private Cloud en mentionnant le buildID. Voici comment procéder :

      • A partir du nom du fichier image :
        Exemple : ubuntu-guest-image-14.04-20160301-x86_64.qcow2

      • A partir du fichier buildID dans la machine virtuelle mise à disposition à partir de l’image de cloud :

        • Pour les instances Linux : le fichier buildID se trouvera à l’emplacement : /etc/buildID

        • Pour les instances Windows : le fichier buildID se trouvera à l’emplacement : C:\buildID
