      Box Panel, contrôleur de site et IBM Bluemix Private Cloud Local

      • by Leslie Lundquist et Tom Spoonemore ● 20 septembre 2016

      • Tags:

      Le présent document présente les concepts de base et la terminologie relatifs à Box Panel et au contrôleur de site et décrit la manière dont ils fonctionnent ensemble pour contrôler et gérer votre environnement IBM Bluemix Private Cloud Local. Le document inclut des liens et des pointeurs vers des documents externes qui fournissent davantage d’informations. Certains termes définis dans la section “Foire aux questions : terminologie” sont plus techniques que ce que vous devriez savoir. Il peut toutefois s’avérer utile de les connaître si vous avez l’opportunité de communiquer avec notre équipe de support.

      Définition de Box Panel

      Box Panel est une suite d’applications utilisée pour distribuer IBM Bluemix Private Cloud. Box Panel est utilisé par le personnel d’assistance interne ainsi que nos clients IBM. Il s’agit de la méthode de gestion de l’infrastructure de cloud.

      A l’instar d’autres fonctionnalités, Box Panel est un système d’enregistrement destiné à tous les centres de données, actifs de machine, clusters, adresses IP et informations sur le compte client.

      La suite Box Panel est distribuée sous forme d’application SaaS accessible via un navigateur Web et des API personnalisées. Elle inclut des applications de support, de demande de service et de discussion, ainsi que des liens vers la documentation du produit.

      Pour plus d’informations sur Box Panel, consultez le Guide d’utilisation de Box Panel.

      Définition du contrôleur de site Box Panel

      Le contrôleur de site Box Panel est un outil logiciel qui fournit des services d’opérations de plan de contrôle distribué, de gestion des configurations et d’automatisation à la suite Box Panel ainsi qu’aux équipes de support et d’ingénieurs Bluemix Private Cloud.

      Le contrôleur de site Box Panel s’exécute dans chaque centre de données local et dédié, où il communique avec le logiciel sous forme de services (SaaS) Box Panel par l’intermédiaire d’une capsule de contrôle centrale.

      Les fonctionnalités du contrôleur de site Box Panel sont les suivantes :

      • Accès administratif à distance
      • Proxy IPMI
      • Mise à disposition Bare Metal
      • Reproduction de package central
      • Surveillance/Alerte
      • Sensu
      • Graphite
      • PagerDuty
      • Gestion des journaux (ELK)
      • Services d’automatisation
      • Services proxy HTTP

      Foire aux questions : terminologie

      Q : Qu’est-ce qu’une file d’attente Box Panel ?

      R : La file d’attente Box Panel correspond à la file d’attente de travaux destinée aux fonctions administratives issues de Box Panel. Ces fonctions incluent la correction logicielle automatisée, le déploiement d’un nouveau noeud et le réamorçage de serveurs.

      Q : Qu’est-ce qu’un agent Box Panel ?

      R : L’agent est un processus qui reçoit les demandes de travail de la file d’attente Box Panel, exécute ces demandes, puis renvoie les résultats à la file d’attente Box Panel.

      Q : A quoi correspond Sensu sur la capsule de contrôle centrale ?

      R : Sensu permet de surveiller les contrôleurs de site distant dans les centres de données appartenant au client et IBM Cloud.

      Q : A quoi correspond Sensu sur un contrôleur de site ?

      R : Sensu permet de surveiller les noeuds physiques Bluemix Private Cloud (pNodes) et renvoie des alertes à la capsule de contrôle centrale. Les noeuds physiques incluent les noeuds de contrôle, de calcul et de stockage.

      Q : Qu’est-ce que PagerDuty ?

      R : Il s’agit de l’outil de génération de rapports d’incident qui nous réveille la nuit lorsque votre cloud exige une attention particulière.

      Q : Qu’est-ce que Graphite ?

      R : Il s’agit d’un outil graphique surtout utilisé pour créer des rapports SLA qui indiquent si les API sont opérationnelles. Vous pouvez visualiser ces informations dans Horizon étant donné que Graphite alimente les tableaux de bord Grafana avec les données de temps d’activité.

      Q : Qu’est-ce que le proxy IPMI ?

      R : Il fournit un accès IPMI sécurisé aux noeuds du cloud.

      Q : Qu’est-ce que le proxy HTTP ?

      R : Il fournit une interface HTTP aux interfaces utilisateur ELK et Sensu sur le contrôleur de site.

      Q : Qu’est-ce que Mirror ?

      R : Il s’agit d’un référentiel logiciel central destiné au logiciel des opérations du cloud. Ce référentiel n’inclut pas les charges de travail du client.

      Q : Qu’est-ce que Bastion ?

      R : L’hôte Bastion fournit un accès distant sécurisé aux contrôleurs de site et aux noeuds du cloud client, en utilisant le contrôle d’accès basé sur les rôles (RBAC). Pour plus d’informations sur le contrôle d’accès basé sur les rôles, voir les documents suivants :

      • Gestion des utilisateurs et des projets
      • Définitions du contrôle d’accès basé sur les rôles
      • Services d’identité fédérée

      Q : Qu’est-ce que Yama ?

      R : Yama fournit un processus d’authentification multi-facteur permettant d’accéder aux clouds du client, ainsi qu’un processus de journalisation de l’accès par le service d’assistance du cloud.

      Q : Qu’est-ce que ELK (Elasticsearch, Logstash & Kibana) ?

      R : ELK est utilisé pour la gestion des journaux des ressources de cloud. Ces journaux restent sur le site client.

      Q: Qu’est-ce que Boot-Strapper ?

      R : Permet la mise à disposition bare metal des serveurs. Boot-Strapper est une réplique du référentiel Mirror.

      Foire aux questions : sécurité

      Q : Qui a accès à Bluemix Private Cloud Local à partir du support IBM ?

      R : Les membres des équipes d’ingénieurs et d’opérations d’OpenStack ont accès aux contrôleurs, au service de calcul, aux noeuds, à l’API OpenStack et à Horizon. Ils utilisent cet accès pour déployer et exploiter le service de cloud privé. Ces ingénieurs de support n’ont aucun accès aux données clients ou aux charges de travail au-dessus de l’hyperviseur.

      Q : Les représentants du support IBM peuvent-ils avoir accès aux ressources client ?

      R : Les équipes d’ingénieurs et d’opérations d’IBM Bluemix Private Cloud ne disposent d’aucune donnée d’identification leur permettant d’accéder aux machines virtuelles de cloud ou aux applications déployées dans le cloud.

      Q : Quelles sont les informations qui circulent sur le réseau privé virtuel ?

      R : Seuls les messages de commande et de contrôle requis pour déployer et exploiter le service de cloud privé circulent sur le réseau privé virtuel. Il n’y a pas de routage entre le réseau privé virtuel et le plan de données clients (machines virtuelles, charges de travail, stockage, réseaux Neutron). Les sessions d’accès administratif distant sécurisé initiées par le service d’assistance OpenStack. Les progiciels utilisés pour installer et corriger les serveurs de cloud privé. Les alertes de surveillance et les messages initiés par les noeuds du cloud. Toutes les données de journalisation sont conservées sur l’équipement des locaux du client.

      Q : Qui peut obtenir ces informations ?

      R : Les équipes d’ingénieurs et d’opérations OpenStack d’IBM Bluemix Private Cloud ont accès à ces informations à des fins de gestion et de traitement des incidents liés au cloud.

      Q : Quelle est la règle de contrôle d’accès de l’équipe de support ?

      R : L’équipe de support suit des règles de sécurité IBM Cloud strictes lui permettant d’accorder, de ré-évaluer et de révoquer l’accès aux clouds des clients.

      Q : Bluemix Private Cloud peut-il utiliser un processus d’enregistrement client ?

      R : Bluemix Private Cloud ne prend actuellement pas en charge ce processus ; nos processus sont conformes à la norme ISO27002.

      Q : Que gère Bluemix Private Cloud ?

      R : Bluemix Private Cloud gère le cloud OpenStack, de bare metal jusqu’à l’hyperviseur. Nos processus, règles et outils sont conformes à IBM Cloud Security Policy (CSP). CSP est conforme à la norme de l’industrie ISO27002, qui est la norme des technologies de l’information en pratique pour le management de la sécurité de l’information (c’est-à-dire, pour les techniques de sécurité).
