      Comment transférer une image

      1. Dans le tableau de bord, sélectionnez Projet > Calcul > Images.
      2. Cliquez sur Créer une image.
      3. Renseignez les valeurs, puis cliquez sur Créer une image une fois terminé.

      Pour obtenir une image Red Hat, cliquez sur le lien suivant pour vous connecter à votre compte Red Hat et obtenir une image :

      https://access.redhat.com/downloads/content/69/ver=/rhel—7/7.3/x86_64/product-software

      Tableau : Options d’image

      Zone Remarques
      Nom Nom de l’image. Le nom doit être unique au sein du projet.
      Description Brève description pour identifier l’image.
      Source de l’image Emplacement de l’image ou Fichier image. La zone suivante qui s’affiche dépend de votre sélection.
      Emplacement de l’image ou Fichier image Sélectionnez l’option Emplacement de l’image pour spécifier l’emplacement de l’image en utilisant cette URL. Sélectionnez l’option Fichier image pour transférer une image à partir de votre disque local.
      Format Format de l’image (par exemple, qcow2).
      Architecture Architecture de l’image. Pour une architecture 64 bits, x86_64 est pris en charge.
      Taille minimale du disque (Go) Taille minimale du disque nécessaire pour amorcer l’image. Si cette zone n’est pas spécifiée, la valeur par défaut est 0 (pas de minimum).
      Mémoire RAM minimale (Mo) Taille minimale de la mémoire nécessaire pour amorcer l’image. Si cette zone n’est pas spécifiée, la valeur par défaut est 0 (pas de minimum).
      Public Si cette option est sélectionnée, rend l’image publique à tous les utilisateurs dotés d’un accès au projet.
      Protégé Si cette option est sélectionnée, seuls les utilisateurs dotés de droits spécifiques peuvent supprimer cette image.

      Une fois que l’image a été transférée avec succès, son statut passe à actif, ce qui signifie que l’image peut être utilisée. Le service d’image peut traiter des images volumineuses dont le transfert prend du temps, probablement plus de temps que la durée de vie du jeton du service d’identité qui a été utilisé au début du transfert. Le service d’image commence par créer un lien de confiance avec le service d’identité pour qu’un nouveau jeton puisse être obtenu et utilisé une fois le transfert terminé, puis le statut de l’image doit être mis à jour.

      Remarque : vous pouvez également utiliser la commande glance image-create avec l’option property pour transférer une image. Vous pouvez utiliser davantage de valeurs sur la ligne de commande. Pour en obtenir la liste complète, voir Image Configuration Parameters (en anglais).

      Mise à jour d’une image

      1. Dans le tableau de bord, sélectionnez Projet > Calcul > Images.
      2. Cliquez sur Editer l’image dans la liste déroulante. Remarque : l’option Editer l’image n’est disponible que lorsque vous vous connectez en tant qu’utilisateur admin. Lorsque vous vous connectez en tant qu’utilisateur demo, vous pouvez Lancer une instance ou Créer un volume.
      3. Mettez à jour les zones et cliquez sur Mettre à jour l’image une fois terminé. Vous pouvez mettre à jour les valeurs suivantes - Nom, Description, ID noyau, ID disque, Architecture, Format, Taille minimale du disque, Mémoire RAM minimale, Public, Protégé. (Les options sont fournies dans le tableau ci-dessus.)
      4. Cliquez sur le menu déroulant et sélectionnez l’option Mettre à jour les métadonnées.
      5. Spécifiez les métadonnées en transférant des éléments de la colonne de gauche vers la colonne de droite. La colonne de gauche contient des définitions de métadonnées issues du catalogue de métadonnées du service d’image. Sélectionnez Autre pour ajouter des métadonnées associées à la clé de votre choix et cliquez sur Sauvegarder une fois terminé.

      Remarque : vous pouvez également utiliser la commande glance image-update avec l’option property pour mettre à jour une image. Vous pouvez utiliser davantage de valeurs sur la ligne de commande ; pour en obtenir la liste complète, voir Image Configuration Parameters (en anglais).

      Suppression d’une image

      1. Dans le tableau de bord, sélectionnez Projet > Calcul > Images.
      2. Sélectionnez l’image que vous voulez supprimer et cliquez sur Supprimer des images.

      Purge de vos données image supprimées

      Les informations relatives à une image sont conservées dans la base de données du service d’image OpenStack, même une fois que l’image a été supprimée. Au fil du temps, la base de données peut donc augmenter de taille, son utilisation devenir lente et sa mise à niveau difficile. Contactez l’équipe de support pour obtenir de l’aide sur le retrait des images supprimées.
