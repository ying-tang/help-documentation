      Comment créer une instance avec une IP statique

      • by Jason Kennedy ● 2 septembre 2016

      • Tags:

        • openstack

        • openstack(openstack)

        • static ip(IP statique)

        • instance(instance)

      Le présent article décrit comment créer un port dans OpenStack et comment le connecter manuellement aux instances lors de l’amorçage, dans le but d’affecter une IP “statique” à une instance.

      Une grande partie de la présente méthode sera exécutée sur la ligne de commande, car il s’agit d’un outil très polyvalent pour l’utilisation d’IBM Bluemix Private Cloud. Pour rendre les instructions plus compréhensibles, je suis prêt à revenir en arrière et à répéter la procédure en utilisant cette fois-ci l’interface CLI et l’API OpenStack. Quoiqu’il en soit, passons au vif du sujet. Cette procédure est effectuée sur un cluster de test, mais elle peut également être répétée dans une session en partage d’écran sur votre propre cluster si vous le jugez nécessaire.

      En premier lieu, les clusters Bluemix Private Cloud disposent de deux réseaux pré-générés : internal et external. Nous souhaitons créer un nouveau port sur le réseau internal, jetons donc un rapide coup d’oeil à la commande associée via la ligne de commande :

      bash-3.2$ openstack network list | grep internal
| 81e829d0-5e35-4808-9f7a-d22159faa9d1 | internal | c6560f00-ae29-40b5-93bb-6e7b1ffd3720

      Nous obtenons ainsi l’ID du réseau internal. Concernant l’interface CLI, j’utilise python-openstackclient ; il s’agit du client “le plus récent”. L’ID est moins utilisé avec ce client, mais la possibilité d’obtenir l’ID pour un réseau, un titulaire, ou autre, s’avère importante si vous avez l’occasion d’utiliser des outils plus anciens (python-novaclient, etc.). Mais n’entrons pas dans les détails à ce stade.

      Nous devons connaître le sous-réseau que nous allons utiliser dans ce scénario :

      bash-3.2$ openstack network show internal

      Il s’agit bien du sous-réseau dont nous avons besoin. Examinons-le de plus près :

      L’IP appropriée lui a été affectée, mais fonctionne-t-elle ? Attribuons-lui une IP flottante et testons-la.

      bash-3.2$ openstack ip floating add 169.45.125.46 test1
bash-3.2$ openstack server list | grep test1
| 1e78035f-e48b-48f7-a94d-8b402d451c44 | test1         | ACTIVE | internal=10.220.3.100, 169.45.125.46  |
