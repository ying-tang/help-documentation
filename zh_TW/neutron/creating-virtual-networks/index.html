      在 IBM Bluemix Private Cloud 上建立虛擬網路

      • by Jason Kennedy ● September 16th, 2015

      • Tags:

        • neutron

        • neutron

        • 建立虛擬網路

      在 OpenStack 叢集中管理及維護虛擬網路是一項挑戰。您多半不需要修改 IBM Bluemix Private Cloud 的預設配置。

      IBM Bluemix Private Cloud OpenStack 環境通常會有一個內部及外部網路：「內部」網路將客戶的內部 IP 範圍放在 IBM Bluemix 基礎架構上，而「外部」網路則擁有該客戶的浮動 IP 範圍儲存區。在大部分環境中，這樣的安排就夠了。傳輸隔離可透過使用安全群組來處理，它會在執行實例與應用程式之間進行資料傳送。建議使用這個方法來處理這類需求，因為它可以進行微調來符合您的需求和期望。

      建置您自己的網路

      那麼，您何時要建立多個網路？最簡單的回答如下：只有當您需要在執行實例或應用程式之間具備乾淨的網路時，例如針對一些開發/測試環境，其中您的正式作業網路的 VLAN 必須已清空。其他範例可能包括安全與合規實務練習，其中網路隔離是絕對必要項目。

      假設您現有的基礎架構是利用 VXLAN 設定（這是標準作法，舊客戶可能需要詢問其帳戶管理員），則設定新網路的程序如下所示：

      在專案的網路區段中，您會在 Horizon 的右邊看到建立網路。請按一下它。在第一個蹦現功能表上，您可以選取網路名稱和網路的管理狀態（將它保留為 UP，這是預設值）。按下一步會帶您前往子網路區段，您需要填寫它（如我在下面所填寫的）：

      確定您選擇的子網路尚未在您的網路上使用，因為這種雙重配置可能會造成問題和衝突，以及變成有效的子網路。

      完成這個區段並選擇下一步之後，您就會在子網路詳細資料區段中。您可以在這裡設定 IP 範圍的特定「配置」儲存區，以及這項新增作業特有的 DNS 和路由資訊。

      完成這些選擇之後，您應該會看到您的網路，同時列出預設的內部和外部網路。我們快要完成了！我們只需建立一個路由器，以便在外部網路與我們建立的網路之間遞送連線。

      附註：將您的新網路連接至外部網路時，會同時採取外部網路中的浮動 IP 位址及新的子網路中的 IP，因為路由器需要這兩個介面上的位址。

      在 Horizon 左邊的網路下方，您會看到路由器區段。請注意，如果您想要將私有網路橋接到其他網路，則路由器是必要項目。由於多個網路可以連接至執行實例，所以這裡不需要橋接。您的網路是私有的位址空間，應該以此方式來處理它。您可以使用這種方式來建立「氣隙 (air gapped)」網路。請從該處右邊選取建立路由器。完成之後（只有名稱是選項），您會在新建的路由器右邊看到一個下拉功能表，其中已強調顯示設定閘道。請選取該選項，然後從後續的蹦現功能表中選取外部（它將是唯一選項），如下所示：

      先前的一系列步驟將我們新建的路由器連接至外部網路，現在我們只需要將它連接到我們建立的網路。若要完成此動作，最簡單的地方就是透過 Horizon 中您的專案下的網路拓蹼區段。現在，您會看到路由器已連接至外部網路，如果您將滑鼠移至其上，就會出現蹦現功能表，其中顯示可讓您新增介面的選項，如下所示：

      在新增介面區段中，從子網路下拉清單中選取您的網路，然後從 IP 位址下的範圍中，為它指派一個特定的 IP 位址。（如果您沒有挑選，則會選擇第一個可用的位址。）按一下新增介面，就應該設定完畢了。網路拓蹼顯示畫面現在應該會顯示您的網路透過路由器連接到外部網路。

      現在，您應該能夠利用網路啟動執行實例，將浮動 IP 附加至執行實例，然後像平常一樣連接到該執行實例。
