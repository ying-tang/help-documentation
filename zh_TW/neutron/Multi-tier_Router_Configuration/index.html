      IBM Bluemix Private Cloud 內的多層路由器配置

      • by Ulysses Kanigel and Leslie Lundquist ● August 31, 2016

      • Tags:

        • neutron

        • neutron

        • 網路

        • 路由器

        • 配置

        • 多層

        • add_interface

      本文說明如何測試「多層路由器配置」，以及推薦您使用它的原因。

      在下列其中一個使用案例中，您可能會想要為雲端設定多層路由器配置：

      1. 您想要防止這些（子）子網路上的主機使用浮動 IP。
      2. 您想要透過在主要路由器之下提供這些網路自己的（子）路由器，來為這些（子）網路建立與您雲端內的其他專案或租戶網路之間的網路隔離。
      3. 您需要每一個隔離的專案網路可以傳入及傳出網際網路，同時又能與您雲端內的其他網路彼此保持隔離，而且也不使用浮動 IP。

      最初建立多層網路的方式是建立從主要路由器到每一個「子」路由器的靜態路由。此測試的主要構想是要確定這些靜態路由能持續保存。在這個範例中，我們會設定這種類型的兩個子網路，稱為 child-net1 和 child-net2。

      讓我們假定您想要測試多層路由器配置，並確認您所設定的路由遇到失效接手時會持續保存。

      以下是您要執行測試時所需採取的步驟：

      
# 建立網路和子網路

neutron net-create parent-net

neutron net-create child-net1

neutron net-create child-net2

neutron subnet-create parent-subnet 162.16.0.0/24

neutron subnet-create child-subnet1 162.16.1.0/24

neutron subnet-create child-subnet2 162.16.2.0/24

# 建立路由器

neutron router-create parent-router

neutron router-create child-router1

neutron router-create child-router2

# 設定外部網路的母路由器的閘道，並在 parent-router 子網路上新增介面

neutron router-gateway-set parent-router external

neutron router-interface-add parent-subnet b490f691-364f-4f7c-866c-1c45e7b8f8cf  # 子網路的唯一 ID

# 新增 parent-router、child-router1 及 child-router2 子網路的介面

neutron port-create parent-router

neutron port-create child-router1

neutron port-create child-router2

neutron router-interface-add child-subnet1 port=c1b305a1-f634-4828-a6ce-d5a6a058824e  # 埠的唯一 ID

neutron router-interface-add child-subnet1 2476ad51-e2cb-4d4e-bbd8-db18cc06ea88   # 子網路的唯一 ID

neutron router-interface-add child-subnet2 port=c1b467b1-f444-4325-a5dc-d4b3a04624d   # 埠的唯一 ID

neutron router-interface-add child-subnet2 2378ad51-e4cc-2d3e-bcc5-db34cd05fa48    # 子網路的唯一 ID

# 將子路由器上的預設路由設為 parent-router 子網路閘道位址：這兩者都與母路由器對話

neutron router-update child-router1 --routes type=dict list=true destination=0.0.0.0/0,nexthop=162.16.0.1

neutron router-update child-router2 --routes type=dict list=true destination=0.0.0.0/0,nexthop=162.16.0.1

      將此設計擴增到包括每一個子路由器所連接的多個子網路應該沒有問題，因為遞送會到達子路由器。

      附註：請記住，任何已連接至子路由器的執行實例都無法有相關聯的浮動 IP，因為子路由器未直接連接至外部網路。

      疑難排解：錯誤：無法進行 add_interface：

      如果您嘗試將介面連接至 tenant-B-network 的 tenant-A-router： 錯誤：無法進行 add_interface：要求失敗：處理您的要求時發生內部伺服器錯誤。

      我們的 Neutron 原則實際上允許將路由器介面新增至具有 admin、cloud_admin 或 project_admin 角色的使用者，但是此作業失敗，因為 Neutron API 已明確編寫禁止對不是向其鑑別您的租戶所擁有的資源進行作業，除非您有 admin 角色，如下列程式碼所反映：

      https://github.com/openstack/neutron/blob/stable/mitaka/neutron/api/v2/attributes.py#L971

      如果您需要的網路是來自連接到不同租戶之路由器的某個租戶，則支援團隊需要代表您執行該作業。請提供您嘗試執行但擲出錯誤的 CLI 指令，我們會為您執行它。
