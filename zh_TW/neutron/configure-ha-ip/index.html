      如何配置高可用性 IP

      • by Eric French ● September 21, 2016

      • Tags:

        • neutron

        • neutron

        • 配置 ip

        • ha

        • 高可用性

        • 容許的位址對組

        • l2 pop

      本文提供說明如何在 IBM Bluemix Private Cloud（本端）環境內設定 HA 浮動 IP 的資源。這些指示均假設您已安裝 OpenStack Python API 用戶端。

      附註：此功能只能與 IBM Bluemix Private Cloud（本端）搭配使用。不適用於 IBM Bluemix Private Cloud（專用雲端）供應項目。IBM Bluemix 資料中心網路目前要求使用 L2 POP，此驅動程式是用來支援調整成大型網路。下面引述我們其中一位工程師的話：

      「我們在 IBM Bluemix 資料中心網路上執行 VXLAN 所使用的 L2Population 機制驅動程式，與 HA IP 方法使用的 MAC 學習或 ARP 回應不相容。IP 會透過兩種轉換機制對映至主機：ARP 將 IP 對映至 MAC 位址，而交換器則瞭解與每一個埠相關聯的 MAC。在失效接手期間，任何 HA IP 都必須至少變更這兩種對映的其中之一。」

      可能的暫行解決方法為：在 OpenStack Mitaka 版本中，可以選擇停用一半的 L2 POP 回應端，這可讓您設定 HA 的無償 ARP 類型。（請參閱此文章：https://review.openstack.org/278597）如果您真的需要 HA，最好使用「負載平衡器」，例如 Neutron LBaaS 第 2 版。另一種可能的暫行解決方法是：如果雲端部署在單一 SoftLayer Pod 中，而且未來的大小絕不可能超過單一 Pod，則您或許可以透過自訂方式實作部分類型的 HA IP。
