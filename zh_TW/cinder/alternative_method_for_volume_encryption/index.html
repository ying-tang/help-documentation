      區塊儲存空間磁區加密的替代方法（使用 Ubuntu 訪客 VM）

      • by Ruben Orduz ● October 17, 2016

      • Tags:

        • cinder

        • cinder

        • 區塊儲存空間

        • 加密

        • ubuntu

      不論是在雲端或是內部部署，某些狀況和工作負載需要較高階的資料隱私和安全。截至撰寫本文之時，OpenStack 計算 (Nova) 不支援連接 LUKS 磁區 – 但 OpenStack 管理的磁區層次加密需要它們。本文聚焦在雲端中資料加密的使用案例，更具體地說，它提供在 Linux 進行自助式磁區加密的程序。

      開始之前，有一些非常重要的 警告事項，讀者應該留意：

      • 如果已加密的磁區已毀損或不穩定，資料很可能會流失。
      • 加密會有效能「負擔」，不過在許多情況下，它可能微不足道。
      • 加密之後，Snapshot 或備份之類可能沒有可預期的行為。
      • IBM Blue Box 不支援加密的磁區或檔案系統。
      • 這裡所說明的方法是針對新建/空白磁區。請不 要在包含現有資料的磁區上嘗試使用這些指令，那些資料將被消除，並永遠消失。

      本文也假設下列必要條件：

      • Ubuntu 14.04 或更新版本
      • 使用者可以在其雲端中建立及連接區塊儲存空間磁區。
      • 使用者在其即將用於此程序的虛擬機器中具有管理者專用權。

      程序

      步驟 1）第一步是使用 Ubuntu 14.04 或更新版本佈建一個執行實例（可以是小型的執行實例）（如果您尚未這麼做的話），方法是使用 API 或 Horizon 儀表板。

      步驟 2）第二步是建立區塊儲存空間磁區。同樣地，這項作業可以透過 CLI 或 Horizon 儀表板來達成。不需要任何特殊參數，只要選取一個敘述性名稱，選取所要的容量，並確定它可供存取，而且是位於您剛才在前一個步驟中啟動的 VM 的同一個可用性區域中。

      步驟 3）當區塊儲存空間磁區順利佈建之後，將它連接至您在步驟 1 建立的 VM。

      步驟 4）登入您在步驟 1 中建立的 VM。
