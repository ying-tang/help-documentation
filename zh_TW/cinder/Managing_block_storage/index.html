      管理區塊儲存空間

      • by Ying Tang ● November 21, 2016

      本文改編自下列 Bluemix 部落格文章：使用 IBM Bluemix Private Cloud 管理 Cinder 磁區。

      IBM Bluemix Private Cloud 提供多個儲存空間模型，以因應我們客戶的獨特性需求。如果您有諸如可擴充的檔案系統及資料管理系統之類在 Bluemix Private Cloud 上管理的工作負載，則區塊儲存空間可能適用於您環境特有的使用案例。本文說明如何開始使用我們的區塊儲存空間模型。

      OpenStack Cinder

      區塊儲存空間是獨立式持續性儲存空間，您可以在虛擬伺服器中將它連接和分離。有了區塊儲存空間，您的資料就可以持續保存超過虛擬伺服器的生命週期。例如，如果您的虛擬伺服器非預期地停止，您儲存在區塊儲存空間中的資料會受到保護，原封不動。

      因為 Bluemix Private Cloud 是採用 OpenStack 技術，所以我們以區塊儲存空間選項提供 OpenStack Cinder 給我們的用戶端。Cinder 讓開發人員有能力同時使用 OpenStack Horizon 儀表板使用者介面和標準化的 OpenStack API 來管理其區塊磁區。

      對於執行 IBM Bluemix Private Cloud 3.1.0 版以及更新版本的客戶，Cinder 支援多個後端：其排程器可找到正確的磁區類型，而且很有彈性。依預設，Cinder 區塊儲存空間是配置為使用 Ceph 叢集。

      Ceph 如何與 Bluemix Private Cloud 搭配使用

      Ceph 是一個以「可靠自主分散式物件儲存庫 (RADOS)」為基礎的分散式儲存空間解決方案，它可以在單獨統一的儲存空間叢集中為應用程式提供物件、區塊及檔案系統儲存空間。

      Ceph 叢集通常是由監視節點及儲存節點（OSD 節點）組成。監視節點保留一份叢集對映的副本以確保高可用性，並追蹤叢集性能。

      搭配 Ceph 時，Bluemix Private Cloud 提供的區塊儲存空間選項能增加容量、提供卓越效能及降低成本。根據您的特定工作負載需求，也可以為 Ceph 叢集自訂下列節點類型：

      • 具有僅限 SSD 的 OSD 磁碟和並置的 Ceph 日誌登載的僅限 SSD 選項，具有最佳 I/O 效能。它適合您預期為高 IOPS 的儲存空間工作負載。

      • 具有 SATA OSD 磁碟和 SSD 型 Ceph 日誌登載的混合式選項有良好的 I/O 效能及較低成本，適合中等 IOPS 工作負載。

      • 混合使用僅限 SSD 節點和混合式節點的多層選項很適合大型環境。

      以適當的多後端和多層選項配置區塊儲存空間之後，您就可以開始使用 OpenStack Horizon 儀表板或 OpenStack Cinder 用戶端來建立 Cinder 磁區。

      使用 OpenStack API 建立 Cinder 磁區

      若要使用 OpenStack 用戶端建立磁區，請執行下列 Cinder 指令：

      $  cinder create <size> --display-name <name> --volume-type <type> \
--image-id <imageid> --availablity-zone <az>

      下列指令顯示的範例說明如何為您的 Bluemix Private Cloud 自訂新建的磁區。

      • <size>：磁區大小，以十億位元組 (GiB) 為單位。
      • <name>：磁區名稱。
      • <type>：CEPH_HYBRID 或 CEPH_SSD。您可以使用 cinder type-list 指令來取得可用的磁區類型的清單。
      • <imageid>：（選用）指定要用於磁區的映像檔 ID。您可以使用 nova image-list 指令來列出現有的映像檔。
      • <az>：（選用）指定要在其中建立磁區的可用性區域。您可以使用 cinder availability-zone-list 指令來取得可用性區域的清單。

      如需此主題的相關資訊，請參閱下列文件：使用指令行建立磁區。

      使用 Horizon 建立 Cinder 磁區

      若要使用 OpenStack 儀表板 Horizon 建立區塊儲存空間磁區，您可以從下列步驟開始進行：

      1. 登入 Horizon 儀表板。

      2. 在專案區段下，選取計算，然後按一下磁區。如果您具有現有磁區，則可以從儀表板使用者介面檢視磁區的清單及其詳細資料（例如，名稱、說明、大小及狀態）。

      按一下 Horizon 儀表板右上方的建立磁區。即會顯示「建立磁區」對話框。

      提供磁區的下列詳細資料：

      • 磁區名稱：鍵入磁區的名稱。
      • 說明：鍵入磁區的說明。
      • 磁區來源：沒有來源、空白磁區 或映像檔。如果選擇空白磁區，空白磁區不會包含檔案系統或分割區表格。如果選擇映像檔選項，則會顯示使用映像檔作為來源的新欄位，因此您可以從清單中選取映像檔。
      • 類型：Ceph_SSD 或 Ceph_Hybrid。
      • 大小 (GiB)：以十億位元組 (GiB) 為單位選取磁區大小，例如 1。
      • 可用性區域：選擇可用性區域，例如 nova。

      完成時，按一下建立磁區。您現在可以將 Cinder 磁區連接至虛擬伺服器。

      如需此主題的相關資訊，請參閱下列文件：從 Horizon 儀表板建立磁區。

      在建立 Cinder 磁區並將其連接至虛擬伺服器之後，您可以隨著容量需求增加，輕鬆調整及擴充其磁區大小。您也可以將區塊磁區分離，再重新連接至另一個需要持續性儲存空間的虛擬伺服器，而您儲存的資料仍然受到保護。

      疑難排解

      openstack volume list –name 與 cinder list –name

      如果您的使用者角色不是 cloud_admin，則 openstack volume list --name 指令無法正確運作。若要克服這個問題，您可以讓具有 cloud_admin 角色的使用者依名稱過濾磁區，也可以使用對任何使用者都有效的 cinder list --name 指令。我們已針對此問題開啟了內部問題報告 374319，我們正在與工程團隊致力解決此問題。

      執行 cinder 或 openstack volume 指令時，發生找不到 volumev2 服務的 publicURL 端點的錯誤

      如果您發生這個錯誤，它通常是因為您的現行叢集中沒有任何區塊儲存空間（Cinder 磁區）節點。如果您想要訂購區塊儲存空間，請開啟問題單進行要求，我們的銷售團隊將協助您新增區塊儲存空間。

      找不到 volumev3 服務的 publicURL 端點

      當您的端點清單中有舊版的磁區服務時，便可能發生此錯誤。如果要解決此問題，請將 export OS_VOLUME_API_VERSION=2 新增至 RC 檔。

      為何在 Cinder 中配置的所有磁區的總磁區大小不符合 Ceph 的原始總計？

      如果 Ceph 中有被刪除但仍位於儲存區內的磁區，而這些磁區在被刪除之前，有其他磁區是從這些磁區建立的，便可能發生此問題。這是 Ceph 運作的方式，複製的磁區是其複製來源磁區的寫入時複製子磁區，這表示母磁區必須一直保留到其所有的子磁區（以及孫磁區、曾孫磁區等）都被刪除為止。話雖如此，但其使用量並不等於目前作用中磁區的大小總和。

      如需 Ceph 寫入時複製層的相關資訊，請參閱：http://docs.ceph.com/docs/hammer/dev/rbd-layering/
