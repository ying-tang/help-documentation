      OpenStack 網路概念

      • by Ying Tang ● May 25, 2016

      • Tags:

        • OpenStack

        • OpenStack

        • 概念

      本文件針對一些常見的網路概念，提供基本介紹。您在使用 IBM Bluemix Private Cloud 時就會用到這些概念。

      本文件使用的術語

      概念 說明
      L2 ISO 7 層網際網路架構的第 2 層，封包在此層中遞送。
      VLAN 虛擬區域網路 (Virtual Local Area Network)，基本上是一個子網路。
      IP 網際網路通訊協定 (Internet Protocol)，通常用作 IP 位址的縮寫，指定封包在網路上的目的地。
      IPv4 網際網路通訊協定第 4 版 (Internet Protocol, version 4)，格式為 xxx.xxx.xxx 的網際網路位址，現在通常會使用稱為 CIDR 的通訊協定，在尾端加入 /xx 來表示子網路大小。
      CIDR 無類別內部網域遞送 (Classless Inter-Domain Routing) 或超級網路連線，此方法可以延伸 IPv4 位址的使用年限，這些位址在廣域儲存區很稀少，因此通常必須購買才能取得。如需 CIDR 的相關資訊，請使用此鏈結：http://searchnetworking.techtarget.com/definition/CIDR。下列鏈結可提供 CIDR 子網路大小計算機，方便您瞭解要從特定範圍內取得多少個 IP：http://www.subnet-calculator.com/cidr.php（請記住，部分 IP 會預先配置，無法供您的雲端使用）。
      IPv6 網際網路通訊協定第 6 版 (Internet Protocol, version 6)，較新的網際網路定址通訊協定，可提供大量網址，但會讓網路架構變得複雜。
      NIC 網路介面卡 (Network Interface Card)，可將伺服器連接至網路的一種硬體組件。此外還有「虛擬 NIC」，基本上，這是雲端常用的一種軟體組件，作用與 NIC 類似，而「網路介面」一詞通常是指虛擬 NIC，除非環境定義指出它是專指硬體。
      MAC 類似硬體序號的個別 ID，可唯一識別網路上的每個主機。

      基本網路概念

      概念 說明
      網路 隔離的第 2 層 (L2) 區段，與一般網路中的 VLAN 類似。一個網路可能有一個或多個子網路。
      子網路 IPv4 位址的範圍。
      埠 將單一裝置（例如，虛擬伺服器的 NIC）連接至虛擬網路的連線點。「埠」一詞也說明相關聯的網路配置，例如，要用於該埠的 MAC 和 IP 位址。
      固定 IP 位址 開機時指派給執行實例的 IP 位址，例如，需要連接公用網際網路的執行實例所使用的網路 IP 位址。
      浮動 IP 位址 依需求從儲存區指派給執行實例的 IP 位址，通常是 IPv6 格式。
      配額 配額會限制雲端的可用資源數量，例如計算或磁碟配置。
      安全性群組 專案特有的一組網路過濾器規則，這些規則會指定哪些 IP 可以或不可以傳送封包給另一個 IP 或埠，藉以套用於執行實例的網路互動。
      安全性群組規則 通訊協定、埠及 IP 位址規格清單，這些規格會建立容許資料流量在網路上進出任何特定執行實例的方式；這些是安全群組的特有項目。

      雲端網路類型

      通常，雲端會有一個或多個內部及外部網路。您可以將此概念想成是雲端中的執行實例可以在內部網路中彼此「交談」，而且可以與私有雲之外的外部網路來源「交談」。

      可以使用各種術語來說明內部和外部網路。例如，內部網路有時稱為「租戶網路」。用來與公用網際網路通訊的外部網路，有時稱為「提供者網路」。

      內部網路

      內部網路在雲端內「交談」；它可以在所有連接的虛擬網路介面 (NIC) 之間提供連線功能。每個內部網路介面都與虛擬機器 (VM) 相關聯，且單一 VM 可以有多個相關聯的網路介面，這表示它可以在許多網路上「交談」。

      您可以將一組 VM 連接至相同的內部網路，就可以在這些 VM 之間啟用通訊。內部網路與單一專案（租戶）相關聯，只有該專案可以看見及存取這些內部網路。

      外部網路

      外部網路可以在您的雲端之中和之外「交談」。外部網路可包含公用網際網路、您雲端的外部網路，以及含有「IBM Cloud 資料中心」路由器的「IBM Cloud 資料中心」網路。

      • 雲端的「公有網路」或「外部網路」提供進出公用網際網路的存取。
      • SoftLayer 私有網路的目的在於提供 SoftLayer 中其他服務（包括您可能想要與之通訊的其他雲端）的存取。（如需存取 SoftLayer 私有網路的相關資訊，請參閱版本注意事項。）
      • SoftLayer 路由器網路連接至「SoftLayer 後端核心路由器 (BCR)」，讓您可以連接自己的私有雲環境外的資源、可能位於您網站 (Blue Box Local) 上資料中心內的資源、您所擁有的另一個私有雲環境中的資源，或位於 SoftLayer 資料中心或其他地方（Blue Box Dedicated 或您自己的舊式雲端）中的資源。
      • 公有和 SoftLayer 私有網路都是實體網路。

      不同私有子網路上虛擬機器之間的連線功能？

      我們假設您有兩個私有網路。您已在每一個子網路上建立虛擬機器，且已新增必要的安全群組，讓它們可以接受送入的資料流量。您要如何在不同私有子網路上的 VM 之間實現連線功能？假設您有一個名為 operations 的網路，以及一個稱為 internal 的網路。您可以使用數種方式在兩者間進行通訊：

      • 將 operations 網路連接至預設路由器。此連線可讓資料流量在預設的 internal 網路和 operations 網路之間流動。現在，您可以將執行實例部署至 internal 網路，指派公用浮動 IP 位址給它，然後使用此執行實例以連接至目標執行實例。
      • 或者，您可以使用類似的方法，在 internal 網路和 operations 網路之間建立新的內部路由器。在 internal 網路中使用具有公用 IP 的執行實例，即可存取 operations 網路上的 VM。透過建立內部路由器，您可以確保 operations 網路上的 VM 沒有任何連至網際網路的外部連線功能。
      • 另一個可能方法是建立多網卡執行實例。建立連接至 internal 和 operations 網路的執行實例。指派公用浮動 IP 給該執行實例。使用此 VM，透過第二個專用介面存取 operations 網路上的執行實例。

      應對具有公用浮動 IP 的機器採取哪些安全預防措施？

      您可以從下面一些不錯的作法開始：

      • 配置 sshd 僅允許金鑰鑑別，然後停用 root ssh 存取。
      • 請勿在 SSH 中使用標準或預設的系統使用者名稱（ubuntu、deploy 及類似名稱）。
      • 使用系統服務（如 fail2ban 或 denyhosts）來封鎖潛在的暴力攻擊。這兩種工具都已經包裝在最常用的 Linux 發行套件中。
      • 使用安全群組規則，將 SSH 存取限制為有限的 IP 位址數目。
      • 定期更新在公用執行實例上執行的系統。
      • 請勿在公用執行實例上執行任何不必要的服務（例如 http、ftp、smb …）
