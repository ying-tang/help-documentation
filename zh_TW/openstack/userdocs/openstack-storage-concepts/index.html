      基本概念

      概念 說明
      磁區 磁區是附加到執行實例以啟用永續性儲存的區塊儲存裝置。終止執行實例時不會刪除永續性儲存空間。您可以隨時將磁區附加到執行中的執行實例，或卸除磁區再附加到另一個執行實例。您也可以刪除磁區。只有管理使用者才能建立磁區類型。
      暫時儲存空間 與執行實例聯結的硬碟是暫時性的，這表示終止執行實例時會刪除資料。從執行中的執行實例所新增的 Snapshot 將會保留，但前次 Snapshot 之後加入至暫時儲存空間的其他任何資料都會遺失。

      下表比較三種 OpenStack 儲存空間概念：

        暫時儲存空間 區塊儲存空間 物件儲存空間
      用來： 執行作業系統及臨時空間 將其他永續性儲存空間加入至執行實例 儲存資料，包括執行實例映像檔
      存取方式： 檔案系統 可分割、格式化及掛載的區塊裝置（例如 /dev/vdc） REST API
      可存取來源： 執行實例內 執行實例內 任何地方
      管理者： OpenStack 計算 (nova) OpenStack 區塊儲存空間 (cinder) OpenStack 物件儲存空間 (swift)
      永續性截止時間： 執行實例終止 由使用者刪除 由使用者刪除
      規格決定者： 管理者配置大小設定，稱為虛擬硬體範本 由使用者在初始請求中指定 可用的實體儲存空間數量
      一般用法範例 第一個硬碟 10 GB，第二個硬碟 30 GB 1 TB 硬碟 資料集儲存空間有數十 TB

      交換空間

      • 佈建交換空間的計算主機與執行交換的虛擬機器所在的主機相同。
      • 交換空間在虛擬機器中會顯示為裝載的裝置 - 例如，透過 udev 裝載於 /dev/vdb 中。

      為什麼 openstack host show 不會顯示實體主機的硬碟使用情形？

      openstack host show 只會顯示在主機上執行的執行實例的虛擬硬體範本相關資訊。我們的其中一位客戶對於是否併入交換使用情形有疑問。目前，並未併入該資訊。如需 Mitaka 版本所併入項目的資訊，請參閱 http://docs.openstack.org/admin-guide/common/nova-show-usage-statistics-for-hosts-instances.html

      disk_gb 直欄 [在 openstack host show 輸出中] 顯示在主機上執行的執行實例的根磁碟和暫時磁碟大小總和（以 GB 為單位）。… [此值的] 計算方式是使用在主機上執行的執行實例的虛擬硬體範本相關資訊。此指令不會查詢 … 實體主機的硬碟使用情形。

      如需實際的虛擬硬體範本計算，請參閱 https://github.com/openstack/nova/blob/c521ba7fa679b6e9c790d84ea22f04fb999987db/nova/notifications/base.py#L448-L454

      disk_gb = root_gb + ephemeral_gb

      ephemeral_gb = 次要暫時資料磁碟大小。這是空的未格式化磁碟，它只存在於該執行實例的效期內。預設值為 0。root_gb = 用於根 (/) 分割區的磁碟空間數量（以 GB 為單位）。

      Nova 的 disk_gb 目前「不」包含交換（執行實例的選用交換空間配置。預設值為 0。）

      在 Newton 中提出的工作開始併入這個選用的交換空間，請參閱：https://specs.openstack.org/openstack/nova-specs/specs/newton/implemented/resource-providers-allocations.html：「當計算節點利用執行實例磁碟的本端儲存空間或從磁區啟動時，使用的值應為虛擬硬體範本的 root_gb、ephemeral_gb 及 swap 欄位值總和。resource_provider_uuid 應為計算節點的 UUID。請注意，若為從磁區啟動的執行實例，則 root_gb 值會是 0。」

      另請參閱 https://derpops.bike/2016/10/07/openstack-nova-and-hypervisor-disk-consumption/
