        • api

      今天我們要逐步執行必要的程序，將浮動 IP 位址新增到您的其中一個執行實例。我會在透過 Horizon 畫面在 OpenStack 中建立其他網路的結尾處簡述如何完成此作業，因此，我們要透過終端機來執行此範例。要考慮的第一件事，就是建立 浮動 IP 實際上並未建立位址，而只是從指派的儲存區中配置一個位址而已。我們來看看指派了什麼：

      現在已建立 IP，我們可以繼續並將它附加至執行實例。我們需要執行實例內部 IP 位址的埠 ID，以及上面所看到的浮動 IP 的 ID。

      以上就是我們的執行實例（格式方面請包涵）及其內部 IP。現在，我們要使用該資訊來尋找埠。

      好了！現在我們只需要浮動 IP 的 ID。它就在上面我們所建立的 floating_network_id 之下：953cda96-1a33-4645-a4e2-1b00d6e06736。最後一個步驟是建立關聯。

      nova list 指令現在應該顯示與您的執行實例相關聯的兩個 IP 位址：

      透過執行後面接著所談論位址 ID 的指令，您也可以隨時取消與浮動 IP 的關聯。我們會在下面把剛才指派給執行實例的位址取消關聯。

      您會看到該位址不再與您的執行實例相關聯。

      • 在 OpenStack 中，浮動 IP 是可公開遞送的 IP，您可以將其指派給您的 VM。新增了浮動 IP 之後，VM 內部會發生什麼情況？答案是… 什麼都不會發生。如果您透過 SSH 登入，並顯示網路配置，您會看到還是有一個網路介面已配置固定 IP。所有設定已在計算 Hypervisor 上自行完成。OpenStack Neutron 會在 VM 的固定 IP 與浮動 IP 之間設定 NAT。

      浮動 IP 的最大好處之一是：如果 VM 當掉，您可以將浮動 IP 附加至另一個 VM，即可重複使用它。
