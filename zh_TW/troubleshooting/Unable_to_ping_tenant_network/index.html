      為何無法在我的租戶網路之間執行 ping？

      網路疑難排解範例：無法在租戶網路之間執行 ping

      問：我已建立自己的網路 (private_network) 及子網路 (private_subnet)。也建立了路由器 (my_router)，將此網路連接至現有的內部網路。不過，連接至 private_network 的執行實例無法 ping 到內部網路上的執行實例。它們可以 ping 到閘道 IP 位址 (10.0.0.1)，也可以 ping 到路由器上內部網路的介面 IP 位址 (192.168.1.113)… 但是無法 ping 到內部網路上的任何執行實例。我可以從另一個方向執行 ping，亦即，從內部網路上的執行實例 ping 到 private_network 上的執行實例… 但相反方向就不行。

      以下是我為了在 OpenStack 中建立相關元件而執行的一些指令（我最多只能記住這些）：

      1. neutron net-create private_network
      2. neutron subnet-create private_network --name private_subnet --gateway 10.0.0.1 10.0.0.0/24
      3. neutron router-create my_router
      4. neutron router-interface-add my_router private_subnet
      5. neutron port-create internal
      6. neutron router-interface-add my_router port=

      我需要以手動方式在內部網路上建立額外的埠，以將路由器連接至內部網路，因為路由器已連接至它。

      答：只是在 my_router 上新增內部網路的介面就遇到問題，亦即，內部網路上的執行實例有預設閘道 192.168.0.1，這是在預設路由器上，而預設路由器並不知道如何遞送回到 private_network。

      除了從相反的子網路將埠新增至每一個路由器之外，您還需要新增路由，讓它們可以雙向遞送…

      neutron router-update default --routes type=dict list=true destination=10.0.0.0/24,nexthop=10.0.0.1
neutron router-update my_router --routes type=dict list=true destination=192.168.0.0/22,nexthop=192.168.0.1

      問：嘗試執行第一個指令失敗，並發生下列錯誤：

      路由的格式無效：[{u'destination': u'10.0.0.0/24', u'nexthop': u'10.0.0.1'}]，nexthop 未與路由器連接

      我需要修改指令才能順利執行它（如下所示），其中 192.168.1.119 是我在內部網路上建立來連接 my_router 的埠。

      neutron router-update default --routes type=dict list=true destination=10.0.0.0/24,nexthop=192.168.1.119

      結果這樣就足以讓資料流量從 private_network 所連接的執行實例傳送到內部網路所連接的執行實例。不過，看起來這種配置會使網路資料流量出現一個額外的中繼站，也就是說，資料流量先遞送至預設閘道 (192.168.0.1)，然後再重新導向至 192.168.1.119。如果我從內部網路上的執行實例 ping 到 private_network 上的執行實例，就會看到下列結果：

      ubuntu@test:~$ ping 10.0.0.13
PING 10.0.0.13 (10.0.0.13) 56(84) bytes of data.
64 bytes from 10.0.0.13: icmp_seq=1 ttl=63 time=0.783 ms
From 10.0.0.13: icmp_seq=2 Redirect Host(New nexthop: 10.0.0.13)
64 bytes from 10.0.0.13: icmp_seq=2 ttl=63 time=0.624 ms
From 10.0.0.13: icmp_seq=3 Redirect Host(New nexthop: 10.0.0.13)

      看起來更好的方式是使用我們的 Ansible Script，直接以手動方式將路由新增至內部網路上的每一個主機，如下所示：

      route add -net 10.0.0.0/24 gw 192.168.1.119

      關於這項配置我所遇到的下一個問題是，private_network 上的執行實例無法透過內部網路連接至公用網際網路，例如，我無法 ping 到 8.8.8.8。您是否恰巧知道它的原因以及我需要做什麼才能啟用它？

      對了，我忘記提到在嘗試 ping 到公用網際網路上的主機時出現的錯誤訊息為「無法連接到目的地網路」，這意味著該要求有到達閘道 (10.0.0.1) 這麼遠，但是它不知道要如何遞送要求。

      答：您說得對，透過 my_router 遞送到達內部網路會產生另一個網路中繼站，但此影響可能微不足道。在每一個執行實例上新增靜態路由可刪除該中繼站，但會增加大量管理額外負擔及很多可能的失敗點。以我的意願來說，我會建議在路由器層次維護路由。

      至於 private_network 上的執行實例為何無法連接到網際網路，這是因為路由器未設定外部閘道（請參閱 external_gateway_info）…

      如果您執行 neutron router-gateway-set the_router external，它會將浮動 IP 配置給路由器，並容許它連接到外界。

      問：my_router 上缺乏外部閘道是刻意為之。我們想要儘可能對外界「隱藏」private_network。我們希望能夠透過預設路由器來遞送針對公用網際網路指定的資料流量，而預設路由器已配置了外部閘道。您知道它是否可行？如果是的話，可以告訴我們要如何達成此目的嗎？

      答：將預設路由器新增至 my_router 是可行的，如下所示：

      neutron router-update my_router --routes type=dict list=true destination=0.0.0.0/0,nexthop=192.168.0.1

      這可有效提供與直接連接到外部網路的路由器相同的功能，只不過它沒有配置浮動 IP 的能力。直接將路由器連接到外部網路會更有彈性，控制存取時只要將浮動 IP 配置給需要從網際網路存取的執行實例即可，並且使用安全群組規則來彼此保護不同的伺服器「區域」。

      我已確認，將未連接到外部網路的路由器上的預設路由設為指向預設路由器的 IP 可讓它遞送到網際網路。以下是它在我們實驗室環境中的面貌：

      討論：參考用… 我們要連接至 private_network 的執行實例確實是內部的，也就是說，它們絕不會配置浮動 IP。實際上我們大部分的執行實例就是這樣 … 在我們的供應項目中，只有大約 4 個執行實例需要有浮動 IP。

      目的是要把內部網路視為 DMZ 對等項目，並將需要浮動 IP 的執行實例連接到它。我們所有其他的執行實例都會連接至 private_network，而且只能透過預設路由器遞送資料流量來存取公用網際網路… 使用您所識別的配置。
