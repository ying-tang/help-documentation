      管理使用者和專案

      基本概念

      使用者：使用者是具有 OpenStack 環境登入認證的個人或服務。一個使用者具有一或多個角色，且隸屬於一個或多個專案。

      專案：專案是雲端中的組織單位，也稱為租戶或帳戶。每一個使用者都是一個或多個專案的成員。在專案內，使用者可以新增和管理資源，例如執行實例和磁區。對於允許專案建立多少資源（執行實例、vCPU、磁區及其他），管理員可以設定限制。

      角色：角色是一組專用權，可讓使用者執行一組特定作業。

      群組：群組扮演使用者與角色之間的鏈結。一個使用者可以是一或多個群組的成員，而一個群組可以與一或多個角色聯結。

      您的 IBM Bluemix Private Cloud 安裝隨附四個預先定義的角色：cloud_admin、project_admin、_member_ 及 heat_stack_owner。系統會自動指派第五個角色 heat_stack_user。

      角色 說明
      cloud_admin 此角色容許雲端層次存取控制。它可讓您執行 API 執行作業，而不必考慮您的專案。此角色可以建立及管理配額、群組、使用者和專案，還可以執行管理磁區動作。
      project_admin 此角色容許專案層次存取控制。此使用者可以在指定的專案內執行使用者和專案管理。
      _member_ 此角色可讓使用者利用配置給專案的資源（例如執行實例和磁區）。
      heat_stack_owner 此角色可讓使用者在專案中建立 Heat 堆疊。此角色必須手動指派，若使用者已獲指派 heat_stack_user 角色，則絕不能同時將此角色指派給該使用者。
      heat_stack_user 此角色是由 Heat 在建立使用者時自動指派給使用者。此角色受到所有 API 存取限制，所以絕不應明確指派給任何使用者。

      cloud_admin 角色限於雲端和專案管理動作。較高的一組專用權專門授與 admin 角色，包括服務、網域和角色、網路及身分提供者的管理動作。admin 角色限於 IBM，不會授與客戶。

      安裝 IBM Bluemix Private Cloud 後，您會取得一個具有 cloud_admin 專用權的使用者，此使用者可以建立其他專案和使用者。cloud_admin 可以授與使用者 cloud_admin 及具有較低專用權的角色。

      需求

      此角色結構需要執行 Keystone API 第 3 版。使用 Keystone API 第 2 版除了會造成 _member_ 使用者的錯誤權限行為外，還會導致 cloud_admin 和 project_admin 角色的專用權流失。

      透過 Horizon 儀表板發出 API 呼叫並不需要採取任何特殊動作，但執行指令行呼叫必須視您執行的 IBM Bluemix Private Cloud 版本而變更執行指令 (RC) 檔。

      使用 Keystone 第 3 版群組功能

      搭配 IBM Bluemix Private Cloud 的第 3 版 Keystone API 引進了群組概念。您可以輕鬆使用群組同步執行多項指派。

      例如，您的支援團隊可能需要存取環境中的每個專案。不用再將每個成員個別新增至每個專案，您只要建立一個稱為支援的群組，在所有專案中給與此群組 _member_ 角色，然後將所有支援人員加入該群組即可。

      另一個常見的使用案例是建立 cloud_admin 群組，並將環境中每一個專案的 cloud_admin 角色指派給此群組。此安排只要暫時將人員加入 cloud_admin 群組，就可以讓 cloud_admin 輕鬆提升使用者專用權。可用於休假代理之類的狀況。

      停用專案的結果

      具有 cloud_admin 角色的使用者可以啟用和停用專案。停用專案會有下列結果：

      • 在儀表板中，使用者再也無法從「專案」標籤的「現行專案」清單中存取專案。
      • 如果使用者僅是已停用專案的成員，則無法再登入。
      • 您無法啟動已停用專案的執行實例。已在執行中的執行實例不會自動終止。必須手動停止。
      • 已停用專案的資料仍然存在，以便您可以隨時重新啟用此專案。

      使用儀表板

      下一節說明如何使用儀表板來新增、更新、修改及刪除專案或使用者帳戶。

      新增專案

      1. 在導覽列中，開啟「身分」畫面，然後按一下專案。
      2. 按一下新增專案。
      3. 在「建立專案」視窗的「專案資訊」標籤中，輸入該專案的名稱和說明。依預設會啟用專案。
      4. 在「專案成員」標籤上，將成員加入至專案並指派適當的角色。
      5. 在「專案群組」標籤上，將使用者群組加入至專案並指派適當的角色。
      6. 在「配額」標籤上，編輯配額值。
      7. 按一下新增專案。「專案」清單會顯示專案，包括獲指派的 ID。

      更新專案

      您可以更新專案來變更其名稱或說明，以及啟用或暫時停用它。

      1. 在導覽列中，開啟「身分」畫面，然後按一下專案。
      2. 選取您要更新的專案。
      3. 從「動作」直欄下的下拉清單選取編輯專案。
      4. 在「編輯專案」視窗中，您可以更新專案來變更其名稱或說明，以及啟用或暫時停用它。依預設會啟用專案。若要暫時停用，請清除已啟用勾選框。若要啟用已停用的專案，請選取已啟用勾選框。
      5. 按一下儲存。

      修改指派給專案的使用者

      如果您具有 cloud_admin 角色，則可以在任何專案中指派角色。如果您具有 project_admin 角色，則可以在此 project_admin 角色適用的專案中修改指派。

      1. 在導覽列中，開啟「身分」畫面，然後按一下專案。
      2. 針對您要修改的專案，按一下管理成員按鈕。

      「編輯專案」視窗會顯示下列使用者清單：

      • 全部的使用者：可指派給現行專案的使用者。
      • 專案成員：已指派給現行專案的使用者。您也可以在「專案群組」標籤中指派使用者群組。

      刪除專案

      1. 在導覽列中，開啟「身分」畫面，然後按一下專案。
      2. 選取您要刪除的專案。
      3. 按一下刪除專案以確認刪除。

      建立使用者帳戶

      如果您具有 cloud_admin 或 project_admin 角色，則可以建立新的使用者帳戶。新增使用者帳戶時，您必須指派帳戶給主要的專案。您也可以選擇指派帳戶給其他專案。

      1. 在導覽列中，開啟「身分」畫面，然後按一下使用者。
      2. 按一下新增使用者。
      3. 在新增使用者視窗中，輸入使用者的使用者名稱、電子郵件信箱及初步的密碼。確認密碼。從「主要的專案」清單中選擇一個專案。 從「角色」清單中選擇使用者的角色。預設值是 _member_。
      4. 按一下新增使用者以確認您所做的變更。儀表板會指派 ID 給使用者，且使用者會出現在「使用者」種類中。

      停用或啟用使用者

      如果您具有 cloud_admin 或 project_admin 角色，則可以啟用和停用使用者帳戶。停用使用者帳戶時，該使用者就無法再登入。但該使用者的資料仍然存在，以便您可以隨時重新啟用此使用者。

      1. 在導覽列中，開啟「身分」畫面，然後按一下使用者。
      2. 在「使用者」清單中，找出您要停用或啟用的使用者。
      3. 從「動作」直欄下的下拉清單選取停用使用者或啟用使用者。在「已啟用」直欄中，已啟用的值會更新為「是」或「否」。
