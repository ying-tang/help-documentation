      RHOSP 的映像檔配置參數

      • by Leslie Lundquist ● March 28, 2017

        • 內容索引鍵

      使用 IBM Bluemix Private Cloud with Red Hat 時，下列索引鍵可以與 glance image-update 和 glance image-create 指令的適當選項一起使用。

      $ glance image-update IMG-UUID --property architecture=x86_64

      附註：使用映像檔內容設定的行為會置換使用虛擬硬體範本所設定的行為。

      內容索引鍵

      適用於 索引鍵 說明 支援的值
      全部 architecture Hypervisor 必須支援的 CPU 架構。例如，x86_64 是目前支援的 CPU 類型。 x86_64 - IA-32 的 64 位元延伸
      全部 instance_uuid 若為 Snapshot 映像檔，則這是用來建立此映像檔的伺服器 UUID。 有效的伺服器 UUID
      全部 kernel_id 儲存在「映像檔服務」中的映像檔 ID，該「映核心像檔服務」會在啟動 AMI 樣式映像檔時用作核心。 有效的映像檔 ID
      全部 os_distro 小寫字體的作業系統發行套件通用名稱（使用與 libosinfo 專案相同的資料詞彙）。只在此欄位中指定已辨識的值。下面列出這些值，以協助您搜尋已辨識的值。 （請參閱下面的清單）
      全部 os_version 經銷商指定的作業系統版本。 版本號碼（例如，”11.10”）
      全部 ramdisk_id 儲存在「映像檔服務」中的映像檔 ID，該「映像檔服務」會在啟動 AMI 樣式映像檔時用作 ramdisk。 有效的映像檔 ID
      全部 vm_mode 虛擬機器模式。這代表用於虛擬機器的主機/訪客伺服器 ABI（應用程式二進位介面）。 HVM 完全虛擬化。這是 QEMU 及 KVM 所使用的模式。
      libvirt API 驅動程式 hw_disk_bus 指定磁碟裝置所要連接的磁碟控制器類型。 scsi、virtio、ide 或 usb。
      libvirt API 驅動程式 hw_qemu_guest_agent 訪客代理程式支援。如果設為 yes，且同時安裝了 qemu-ga，則可以靜止（凍結）檔案系統並自動建立 Snapshot yes / no
      libvirt API 驅動程式 hw_rng_model 將亂數產生器裝置新增至映像檔執行實例。雲端管理者可以配置執行實例的虛擬硬體範本，藉以啟用及控制裝置行為。依預設：產生器裝置為停用，並使用 /dev/random 作為預設熵來源。若要指定實體 HW RNG 裝置，請在 nova.conf 檔案中使用下列選項：rng_dev_path=/dev/hwrng virtio 或其他支援裝置。
      libvirt API 驅動程式 hw_scsi_model 啟用 VirtIO SCSI (virtio-scsi) 以提供計算執行實例的區塊裝置存取權；依預設，執行實例會使用「VirtIO 區塊」(virtio-blk)。VirtIO SCSI 是一種半虛擬化的 SCSI 控制器裝置，提供已改良的可調整性和效能，並支援高階 SCSI 硬體。 virtio-scsi
      libvirt API 驅動程式 hw_video_model 使用的影像映像檔驅動程式。 vga、cirrus、vmvga、xen 或 qxl
      libvirt API 驅動程式 hw_video_ram 影像映像檔的 RAM 上限。只有在虛擬硬體範本的 extra_specs 中設定了 hw_video:ram_max_mb 值，且該值高於 hw_video_ram 所設定的值時，才使用此索引鍵。 以 MB 為單位的整數（例如，’64’）
      libvirt API 驅動程式 hw_watchdog_action 啟用當伺服器當機時執行指定動作的虛擬硬體監視器裝置。此監視器會使用 i6300esb 裝置（模擬 PCI Intel 6300ESB）。如果沒有指定 hw_watchdog_action，則會停用 監視器。 disabled - 裝置未連接。可讓使用者停用映像檔的監視器，即使已使用映像檔的虛擬硬體範本啟用該監視器亦如此。此參數的預設值是 disabled。
      reset - 強制重設訪客伺服器。poweroff - 強制關閉訪客伺服器電源。pause - 暫停訪客伺服器。none- 僅啟用監視器；如果伺服器當機，則不執行任何動作。      
      libvirt API 驅動程式 os_command_line libvirt 驅動程式所使用的核心指令行，而非使用預設值。若為「Linux 容器 (LXC)」，則此值會用作起始設定的引數。此索引鍵只適用於 Amazon 核心、ramdisk 或機器映像檔（aki、ari 或 ami）。  
      libvirt API 驅動程式 hw_vif_model 指定要使用的虛擬網路介面裝置機型。有效選項需視配置的 Hypervisor 而定。 KVM 及 QEMU：e1000、ne2k_pci、pcnet、rtl8139 及 virtio。
