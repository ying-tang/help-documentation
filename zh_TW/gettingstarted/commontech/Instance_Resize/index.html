      如何調整執行實例大小？

      功能：執行實例調整大小（增加）

      說明：從 1.3 版的 Bluemix Private Cloud 開始，我們已支援執行實例調整大小（增加）。調整大小作業不是即時作業：它會 停止再重新啟動執行實例。調整大小可發生在同一部主機上，這對於小型環境很有用。

      執行實例調整大小可讓執行實例將其虛擬硬體範本變更為較大的虛擬硬體範本。執行實例只能調整為較大的虛擬硬體範本。當執行實例已停止、移轉至大小較大的虛擬硬體範本後再重新啟動時，此變更會引起短暫的中斷。這項調整大小作業也可能會將執行實例重新定位至新的 Hypervisor。

      當調整大小程序在進行中時，請勿更新目標虛擬硬體範本（執行實例將調整成的虛擬硬體範本），否則調整大小作業可能會失敗，且執行實例可能會進入「錯誤」狀態。

      調整大型執行實例的大小可能有點困難，因為如果沒有足夠的磁碟空間供新的磁碟映像檔使用，它可能會導致失敗。

      附註：一旦執行實例調整大小或移轉並處於 VERIFY_RESIZE 狀態，在網路回復之前可能會有短暫的延遲。

      如需相關資訊，請參閱 http://docs.openstack.org/user-guide/enduser/cli_change_the_size_of_your_server.html，以取得如何執行調整大小的說明文件。

      疑難排解

      如果大型執行實例的調整大小作業需要很長的時間（例如，有些日誌顯示作業開始於 09:30，而鑑別錯誤出現在 14:41），則 Nova 用來完成此作業的鑑別記號實際上會在這段期間內到期。

      如果您試圖起始新的調整大小作業，很可能會再次發生相同的錯誤。在這種情況下，建議使用不同的方法。如果您的叢集已啟用區塊儲存空間，我們建議執行下列步驟：

      1. 關閉目標 VM。
      2. 使用 nova image-create 產生目標 VM 的 Snapshot。
      3. 使用 cinder create --image-id <SNAPSHOT UUID> 從該 Snapshot 中建立磁區。
      4. 使用 OpenStack image delete 或 glance image-delete 刪除 Snapshot。
      5. 使用 nova boot --boot-volume，從該磁區啟動新的 VM。

      如果您要保留相同的 IP 位址，則可以執行下列動作：

      1. 使用 neutron port-list | grep <IP of new VM>（第一個直欄是埠 UUID）取得已啟動磁區之 VM 的埠 UUID。

      2. 使用 nova interface-detach，從舊的 VM 中移除該埠

      3. 使用 neutron port-update --fixed-ip ip_address=<old VM IP> <port UUID> 來更新新 VM 上的埠，以使用舊的 VM IP

      當您滿意這項切換作業之後，您可以使用 nova delete 清除舊的 VM。

      此程序更加錯綜複雜，您可能不會很喜歡。不過，切換至區塊儲存空間作為訪客後端有許多好處：

      • 虛擬硬體範本調整大小會更快發生，因為將跳過磁碟調整大小。
      • 使用區塊儲存空間移轉至新的 Hypervisor 會非常快（新的 Hypervisor 只連接至區塊儲存空間叢集）。
      • 您可以使用 cinder extend 指令快速延伸 VM 的磁碟。

      如果您沒有區塊儲存空間，但想要調整成更小或更大的虛擬硬體範本，您可以使用您想要的大小虛擬硬體範本來建立新的執行實例，並將您需要的資料從舊的執行實例 rsync（遠端同步處理）到新的執行實例。

      為什麼 OpenStack 不容許您將執行實例大小調低？

      您可以參閱 https://blueprints.launchpad.net/nova/+spec/no-downward-resize 來瞭解其原因 - 基本上，容許調低大小的作業可能會影響資料完整性，因此不被允許。作為安全的暫行解決方法，您仍然可以使用大小較小的虛擬硬體範本來建立新的 VM，並將您需要的資料從較大的執行實例遠端同步處理到較小的執行實例。

      當我調整從磁區啟動的執行實例的大小時，虛擬硬體範本變更，但大小維持不變。為什麼？

      這是預期的行為。從磁區啟動時，虛擬硬體範本的磁碟大小並不是根磁碟的磁碟大小。您可以檢查這個無效的 OpenStack 錯誤 (https://bugs.launchpad.net/nova/+bug/1412993)，它對此有進一步的說明。

      若要調整磁碟的大小，請參閱 https://docs.openstack.org/user-guide/dashboard-manage-volumes.html#edit-a-volume 來調整磁區大小。如果基於某個原因而無法運作，您也可以嘗試執行 https://mcwhirter.com.au/craige/blog/2014/Resizing_a_Root_Volume_for_an_Openstack_Instance/ 所列出的步驟
