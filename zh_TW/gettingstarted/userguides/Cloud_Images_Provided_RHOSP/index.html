      雲端映像檔的使用手冊 (IBM Bluemix Private Cloud with Red Hat)

      • by Ying Tang, Xiong Chen ● March 31, 2017

      • Tags:

        • userguides

        • 使用手冊

        • 映像檔

        • glance

        • image-create

        • 執行實例

        • rhosp

        • newton

      IBM Bluemix Private Cloud with Red Hat 已預先移入 Cirros 0.3.3 x86_64 映像檔。

      如需 IBM 所提供映像檔的詳細資料，您可以查閱雲端映像檔版本注意事項。

      若要使用 Red Hat 訪客映像檔，您可以從下列 URL 下載映像檔：

      • Red Hat Enterprise Linux 7 KVM 訪客映像檔
      • Red Hat Enterprise Linux 6 KVM 訪客映像檔

      如需在 OpenStack 中使用映像檔的相關資訊，請參閱 OpenStack 網站：https://docs.openstack.org/image-guide/

      必要條件

      如果您打算使用指令行 (CLI) 工具，請確定您已安裝「OpenStack Glance 用戶端」。如需安裝指示，請參閱我們的知識庫：開始使用 OpenStack API。

      在 IBM Bluemix Private Cloud with Red Hat 現行版本中，您可以透過指令行或從 Horizon 儀表板上傳映像檔。不支援透過映像檔位置或 URL 上傳。

      選取 IBM Bluemix Private Cloud 提供的替代雲端映像檔

      此版本的 IBM Bluemix Private Cloud with Red Hat 已預先移入 Cirros 0.3.3 x86_64 映像檔。還有其他雲端映像檔可供使用，這些映像檔是 IBM Bluemix Private Cloud 基於對客戶的善意而提供。

      • CentOS 6.8 x86_64
      • CentOS 7.3 x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Ubuntu Server 16.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      將映像檔上傳至 Glance 儲存庫

      您可以上傳和安裝個別取得的作業系統軟體，例如，Red Hat Enterprise Linux 7 KVM 用戶映像檔，或佈建我們以虛擬機器執行實例提供給您的任何作業系統映像檔。您有責任遵循所有適用的作業系統授權條款，以及取得每個虛擬機器執行實例的適當授權。

      從 Box Panel 取得 IBM 提供的雲端映像檔下載

      1. 使用 Box Panel 認證登入 Box Panel。如果您至少有一個雲端，且您是主要或技術客戶聯絡人，則可以使用 Box Panel 中的「雲端映像檔」頁面。

      2. 在「Box Panel 儀表板」中，按一下「服務」，然後選取「雲端映像檔」。

      3. 即會顯示「雲端映像檔」頁面。每個映像檔都會顯示為一張卡。

      4. 按一下您感興趣的映像檔所在的卡片。即會蹦現限制模式視窗，其中含有您的 tempURL。

      5. 顯示的 tempURL 有效期限為接下來的 24 小時，所以您可以下載雲端映像檔。

      6. 請確定 tempUrl 記號已跳出或以引號指定。例如：

          curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
  curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      7. 下載映像檔的 .qcow2 及 .md5sum 檔案。

      8. 使用 tempURL 下載 .qcow2 映像檔及 .md5sum 總和檢查檔案，並將這兩個檔案放在相同的資料夾。例如：

          ubuntu-guest-image-14.04-20160301-x86_64.qcow2
  ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      9. 使用 .md5sum 總和檢查檔案驗證下載的 .qcow2 映像檔。例如：

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

      10. 如果結果傳回 OK，則下載的 qcow2 映像檔有效。

      使用指令行上傳映像檔

      您可以透過 Glance 用戶端上傳映像檔。如需相關資訊，請參閱安裝 OpenStack 用戶端。

      建立映像檔。

      • 若為 Linux 映像檔，請使用 -- min-disk 5 --min-ram 512

      • 若為 Windows 映像檔，請使用 --min-disk 30 --min-ram 2048

        $ glance image-create --os-version <os_version> --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --file <image_file>

      檢查映像檔是否已順利建立。映像檔即排入佇列等候上傳。可能需要一些時間，狀態才會從「已排入佇列」變更為「作用中」。

      $ glance image-show <image-id>

      如需此指令的相關資訊，請參閱 Glance 指令行用戶端。

      使用 OpenStack 儀表板上傳映像檔

      1. 登入 IBM Bluemix Private Cloud OpenStack 儀表板。

      2. 在專案畫面下，展開計算，然後按一下映像檔。

      3. 按一下頁面中的建立映像檔。在新頁面中指定下列參數，然後勾選建立映像檔。

        項目 說明
        名稱 輸入映像檔的名稱。
        說明 選擇性地輸入映像檔的簡短說明。
        檔案 選擇映像檔。
        格式 選取映像檔的正確格式（例如，QCOW2）。
        核心 選擇性地指定儲存在 Glance 中的映像檔核心 ID，該映像檔應在啟動 AMI 樣式映像檔時用作核心。
        Ramdisk 選擇性地指定儲存在 Glance 中的映像檔 ID，該映像檔應在啟動 AMI樣式映像檔時用作核心。
        架構 您可以選擇性地指定架構（例如，i386 代表 32 位元架構，或 x86_64 代表 64 位元架構）。
        磁碟下限 (GB) 若為 Linux 映像檔，請使用 5；若為 Windows 映像檔，請使用 30
        RAM 下限 (MB) 若為 Linux 映像檔，請使用 512；若為 Windows 映像檔，請使用 2048
        公用 選擇此勾選框，表示映像檔設為公用，供所有專案的所有使用者使用。
        保護 選擇此勾選框，以確保只有具備權限的使用者才能刪除映像檔。

        例如：

         名稱：Ubuntu Server 14.04 LTS x86_64 - 20160301	   說明：Ubuntu Server 14.04 LTS x86_64 雲端映像檔，發行日期為 2016/03/01		
 檔案：C:\Documents\ubuntu-guest-image-14.04-20160301-x86_64.qcow2	
 格式：QCOW2 - QEMU 模擬器
 架構：x86_64	
 磁碟下限 (GB)：5	
 RAM 下限 (MB)：512	
 公用：True

      4. 檢查建立的映像檔是否顯示在映像檔清單中。映像檔即排入佇列等候上傳。可能需要一些時間，狀態才會從「已排入佇列」變更為「作用中」。
