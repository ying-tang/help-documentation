      雲端映像檔的使用手冊（社群版）

      • by Yanqi Dou ● March 9th, 2016

      • Tags:

        • userguides

        • 使用手冊

        • 映像檔

        • glance

        • image-create

        • 執行實例

      IBM Bluemix Private Cloud 3.0 已預先移入 Cirros 0.3.3 x86_64 映像檔。IBM Bluemix Private Cloud 提供其他可下載的雲端映像檔，讓您可以使用選擇的作業系統來執行這些執行實例。本文件說明您在下載及使用這些映像檔時必須知道的資訊。若要確保您使用的是最新的映像檔，您可以隨時查閱雲端映像檔版本注意事項。

      如需在 OpenStack 中使用映像檔的相關資訊，請參閱 OpenStack 網站：https://docs.openstack.org/image-guide/

      必要條件

      1. 如果您打算使用指令行 (CLI) 工具，請確定您已安裝「OpenStack Glance 用戶端」。如需安裝指示，請參閱我們的知識庫：開始使用 OpenStack API。

      2. 請確定您可以存取 Box Panel，這樣才能取得 IBM 所提供之雲端映像檔的暫時下載 URL。

      3. 只有 Bluemix Private Cloud 2.1.0 及之後的版本才支援使用 tempUrl 作為映像檔位置，將映像檔上傳至 Glance。

      基本概念

      項目 說明
      映像檔 虛擬機器映像檔（本文中簡稱為映像檔）是單一檔案，內含已安裝可開機作業系統的虛擬硬碟。請使用映像檔在雲端內新增虛擬機器執行實例。
      執行實例 執行實例是在 OpenStack 雲端內部執行的虛擬機器 (VM)。
      虛擬硬體範本 虛擬硬體範本 一詞說明執行實例的大小，並定義執行實例可以使用的預先決定資源規格，例如 vCPU、RAM 及磁碟空間的數量。

      選取 IBM Bluemix Private Cloud 提供的替代雲端映像檔

      此版本的 IBM Bluemix Private Cloud 已預先移入 Cirros 0.3.3 x86_64 映像檔。還有其他雲端映像檔可供使用，這些映像檔是 IBM Bluemix Private Cloud 基於對客戶的善意而提供。

      您可以上傳及安裝個別取得的作業系統軟體，也可以佈建下列任何作業系統映像檔，我們會將這些映像檔作為虛擬機器執行實例提供給您。您有責任遵循所有適用的作業系統授權條款，以及取得每個虛擬機器執行實例的適當授權。

      • CentOS 6.8 x86_64
      • CentOS 7.2 x86_64
      • Ubuntu Server 12.04 LTS x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      對於任何 Microsoft Windows 作業系統虛擬計算執行實例而言，這些是特定客戶規定：客戶必須取得及維護自己的 Microsoft Windows 授權伺服器，才能啟動這類 Windows 作業系統虛擬計算執行實例。如需相關資訊，請參閱如何在 IBM Bluemix Private Cloud 中啟動 Windows 授權。

      從 Box Panel 取得雲端映像檔下載

      1. 使用 Box Panel 認證登入 Box Panel。如果您至少有一個雲端，且您是主要或技術客戶聯絡人，則可以使用 Box Panel 中的「雲端映像檔」頁面。

      2. 在「Box Panel 儀表板」中，按一下服務，然後選取雲端映像檔。

      3. 即會顯示「雲端映像檔」頁面。每個映像檔都會顯示為一張卡。

      4. 按一下您感興趣的映像檔所在的卡片。即會蹦現限制模式視窗，其中含有您的 tempURL。

      5. 顯示的 tempURL 有效期限為接下來的 24 小時，所以您可以下載雲端映像檔。

      使用您的雲端映像檔 tempURL，將映像檔上傳至 IBM Bluemix Private Cloud Glance 儲存庫

      附註：

      • 只有具有 cloud_admin 角色的使用者可以將映像檔設為公用，供其他使用者和專案存取。
      • 如果 IBM Bluemix Private Cloud 已在 OpenStack 控制面板中封鎖出埠網際網路存取，則不能在 IBM Bluemix Private Cloud 中直接使用 tempUrl 作為映像檔位置，將映像檔上傳至 Glance。
      • 只 有 IBM Bluemix Private Cloud 2.1.0 以及更新版本，才支援直接使用 tempUrl 作為映像檔位置將映像檔上傳至 Glance，2.1.0 之前的版本則不支援。
      • 若 IBM Bluemix Private Cloud 的版本在 2.1 版之前或沒有出埠網際網路存取權，請遵循使用下載的映像檔，將映像檔上傳至 IBM Bluemix Private Cloud Glance 儲存庫中的指示。

      您可以使用 OpenStack 儀表板或指令行用戶端來上傳映像檔。

      使用 OpenStack 儀表板上傳映像檔

      1. 登入 IBM Bluemix Private Cloud OpenStack 儀表板。
      2. 在專案畫面下，展開計算，然後按一下映像檔。

      3. 按一下頁面中的建立映像檔。在新頁面中指定下列參數，然後按一下建立映像檔。

        項目 說明
        名稱 輸入映像檔的名稱。
        說明 選擇性地輸入映像檔的簡短說明。
        映像檔來源 選擇映像檔位置。
        映像檔位置 指定「雲端映像檔」下載 tempURL。
        格式 選取映像檔的正確格式（例如，QCOW2）。
        架構 指定架構（例如，i386 代表 32 位元架構，或 x86_64 代表 64 位元架構）。
        磁碟下限 (GB) 若為 Linux 映像檔，請使用 5；若為 Windows 映像檔，請使用 30。
        RAM 下限 (MB) 若為 Linux 映像檔，請使用 512；若為 Windows 映像檔，請使用 2048。
        複製資料 必須勾選此選項，以便將映像檔資料複製到映像檔服務。
        公用 選擇此勾選框，表示映像檔設為公用，供所有專案的所有使用者使用。
        保護 選擇此勾選框，以確保只有具備權限的使用者才能刪除映像檔。

        例如：

         名稱：Ubuntu Server 14.04 LTS x86_64 - 20160301
 說明：Ubuntu Server 14.04 LTS x86_64 雲端映像檔，發行日期為 2016/03/01
 映像檔來源：映像檔位置
 映像檔：<tempURL_for_QCOW2>
 格式：QCOW2 - QEMU 模擬器
 架構：x86_64
 磁碟下限 (GB)：5
 RAM 下限 (MB)：512
 複製資料：True
 公用：True
      4. 檢查建立的映像檔是否顯示在映像檔清單中。映像檔即排入佇列等候上傳。可能需要一些時間，狀態才會從「已排入佇列」變更為「作用中」。

      附註：只有具有 cloud_admin 角色的使用者才能將映像檔設為公用，供其他使用者和專案存取。

      使用 OpenStack 指令行用戶端上傳映像檔

      您可以透過 Glance 用戶端上傳映像檔。如需相關資訊，請參閱安裝 OpenStack 用戶端。

      建立映像檔。

      • 若為 Linux 映像檔，請使用 -- min-disk 5 --min-ram 512

      • 若為 Windows 映像檔，請使用 --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --copy-from  <tempURL_to_qcow2_image_file>

      檢查映像檔是否已順利建立。映像檔即排入佇列等候上傳。可能需要一些時間，狀態才會從「已排入佇列」變更為「作用中」。

      $ glance image-show <image-id>

      使用下載的映像檔，將映像檔上傳至 IBM Bluemix Private Cloud Glance 儲存庫

      附註：

      • 只有具有 cloud_admin 角色的使用者可以將映像檔設為公用，供其他使用者和專案存取。

      • 使用 tempUrl 下載雲端映像檔時，請確定 tempUrl 記號已跳出或以引號指定。例如：

        curl -f -o <output_file> '<cloud_image_download_tempUrl>'
curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      請遵循下列步驟：

      1. 下載映像檔的 .qcow2 及 .md5sum 檔案。

      2. 使用 tempURL 下載 .qcow2 映像檔和 .md5sum 總和檢查檔案，並將這 2 個檔案放在相同的資料夾。例如：

         ubuntu-guest-image-14.04-20160301-x86_64.qcow2
 ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      3. 使用 .md5sum 總和檢查檔案驗證下載的 .qcow2 映像檔。例如：

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

        如果結果傳回 OK，則表示下載的 qcow2 映像檔有效。

      4. 將映像檔上傳至 Glance。您可以使用 OpenStack 儀表板或使用指令行用戶端上傳映像檔。

      使用 OpenStack 儀表板上傳映像檔

      1. 登入 IBM Bluemix Private Cloud OpenStack 儀表板。

      2. 在專案畫面下，展開計算，然後按一下映像檔。

      3. 按一下頁面中的建立映像檔。在新頁面中指定下列參數，然後勾選建立映像檔。

        項目 說明
        名稱 輸入映像檔的名稱。
        說明 選擇性地輸入映像檔的簡短說明。
        映像檔來源 選擇映像檔。
        映像檔 瀏覽至二進位映像檔的目錄。
        格式 選取映像檔的正確格式（例如，QCOW2）。
        架構 指定架構（例如，i386 代表 32 位元架構，或 x86_64 代表 64 位元架構）。
        磁碟下限 (GB) 若為 Linux 映像檔，請使用 5；若為 Windows 映像檔，請使用 30
        RAM 下限 (MB) 若為 Linux 映像檔，請使用 512；若為 Windows 映像檔，請使用 2048
        公用 選擇此勾選框，表示映像檔設為公用，供所有專案的所有使用者使用。
        保護 選擇此勾選框，以確保只有具備權限的使用者才能刪除映像檔。

        例如：

         名稱：Ubuntu Server 14.04 LTS x86_64 - 20160301	
 說明：Ubuntu Server 14.04 LTS x86_64 雲端映像檔，發行日期為 2016/03/01	
 映像檔來源：映像檔
 映像檔：C:\Documents\ubuntu-guest-image-14.04-20160301-x86_64.qcow2	
 格式：QCOW2 - QEMU 模擬器
 架構：x86_64	
 磁碟下限 (GB)：5	
 RAM 下限 (MB)：512	
 公用：True

      4. 檢查建立的映像檔是否顯示在映像檔清單中。映像檔即排入佇列等候上傳。可能需要一些時間，狀態才會從「已排入佇列」變更為「作用中」。

      如需相關資訊，請參閱安裝 OpenStack 用戶端。

      建立映像檔。

      • 若為 Linux 映像檔，請使用 -- min-disk 5 --min-ram 512

      • 若為 Windows 映像檔，請使用 --min-disk 30 --min-ram 2048

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --file <path_to_qcow2_image_file>

      檢查映像檔是否已順利建立。映像檔即排入佇列等候上傳。可能需要一些時間，狀態才會從「已排入佇列」變更為「作用中」。

      $ glance image-show <image-id>

      附註：若為大型映像檔，則 Web 上傳可能會逾時。對於這些檔案，我們建議使用者在其雲端啟動執行實例，將映像檔下載至該執行實例，然後從該執行實例上傳至 Glance。所有資料流量都會停留在 SoftLayer 內，而上傳會在他們自己的雲端中。

      附註：請確定已定義適當的安全群組規則，以透過 ssh 或 rdp 連接至執行實例。

      1. 登入 IBM Bluemix Private Cloud OpenStack 儀表板。

      連接至 Linux 執行實例

      透過 SSH 或 VNC 連接至您的執行實例。IBM 所提供的 Linux 映像檔的預設 userid 是 ibmcloud。連接至您的 Linux 執行實例之後，您可以使用 sudo 指令來執行通常需要 root 存取權的指令。Sudo 已配置為預設 userid 不需要密碼。

      • 使用您慣用的 SSH 用戶端，並利用您在佈建期間指定的金鑰組之專用 SSH 金鑰，開啟連至 VM 執行實例正確 IP 位址的連線：

          ssh -i <path_of_Your_SSH_private_key_file> <userid>@<ip_of_instance>

      • 在 IBM Bluemix Private Cloud OpenStack 儀表板中按一下執行實例名稱，然後按一下主控台標籤，檢視執行實例的 VNC 主控台。 VNC 主控台會透過 HTTPS 連接。

      附註：若要以 ibmcloud 使用者身分透過主控台登入新的 Linux 執行實例，您必須在佈建時於「自訂作業 Script」中指定密碼。

      連接至 Windows 執行實例

      透過「遠端桌面」或 VNC 連接至執行實例。您必須使用虛擬機器佈建時在「自訂作業 Script」中指定的 userid 和 password。 第一次登入時，會出現提示畫面，要求您變更密碼。請變更密碼。

      • 使用您慣用的「遠端桌面」用戶端來連接。
      • 在 IBM Bluemix Private Cloud OpenStack 儀表板中按一下執行實例名稱，然後按一下主控台標籤，檢視執行實例的 VNC 主控台。VNC 主控台會透過 HTTPS 連接。

      映像檔更新

      收到來自 IBM Bluemix Private Cloud 作業的通知，指出雲端映像檔已更新時，您可以停用現有的映像檔，然後使用取自 Box Panel 的更新映像檔下載 URL 來重建新的映像檔。

      使用 OpenStack 指令行用戶端來停用現有的映像檔

      1. 取得現有的映像檔 ID：

         $ glance image-list

      2. 停用映像檔：

        $ glance image-deactivate <image_id>

      3. 顯示映像檔狀態：

        $ glance image-show <image_id>

      4. 停用現有的映像檔之後，請遵循本手冊中使用您的雲端映像檔 tempURL，將映像檔上傳至 IBM Bluemix Private Cloud Glance 儲存庫中的步驟，重建新映像檔。

      若要在 IBM Bluemix Private Cloud 作業中取得有關特定雲端映像檔的支援，最好能提供映像檔的 buildID。請取得映像檔的 buildID，然後使用 IBM Bluemix Private Cloud 作業建立支援問題單並提供 buildID。方法如下：

      • 在映像檔名稱中：
        範例：ubuntu-guest-image-14.04-20160301-x86_64.qcow2

      • 在使用雲端映像檔佈建的虛擬機器中，取得 buildID 檔案：
