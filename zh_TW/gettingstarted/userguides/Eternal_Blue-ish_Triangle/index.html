      Box Panel、網站控制器及 IBM Bluemix Private Cloud（本端）

      • by Leslie Lundquist and Tom Spoonemore ● September 20, 2016

      • Tags:

        • userguides

        • 使用手冊

        • box panel

        • 網站控制器

        • 常見問題

        • 雲端安全

      本文件介紹有關 Box Panel、「網站控制器」的基本概念及術語，以及它們如何分工合作來控制及管理 IBM Bluemix Private Cloud（本端）環境。本文件含有連至外部文件的鏈結和指標，讓您可以尋找更多資訊。「術語常見問題」一節中定義的部分術語，可能會比您日常須知的詞彙更具技術性，但知道這些，可能會在您有機會與支援團隊談話時派上用場。

      何謂 Box Panel？

      Box Panel 是一種應用程式套組，用來交付 IBM Bluemix Private Cloud。Box Panel 由內部支援人員及 IBM 客戶使用。這是我們用來管理雲端基礎架構的方式。

      搭配其他功能使用，Box Panel 就成了所有資料中心、機器資產、叢集、IP 位址及客戶帳戶資訊的記錄系統。

      Box Panel 套組可作為 SaaS 應用程式交付，而此應用程式可透過 Web 瀏覽器和自訂 API 存取。「Box Panel 套組」包含「支援」、「問題單」及「會談」的應用程式，同時還含有產品說明文件的鏈結。

      如需 Box Panel 的相關資訊，您可以檢閱 Box Panel 使用手冊。

      何謂 Box Panel 網站控制器？

      「Box Panel 網站控制器」是一種軟體工具，提供分散式控制面板作業、配置管理及自動化服務給「Box Panel 套組」與 Bluemix Private Cloud 支援和工程團隊。

      「Box Panel 網站控制器」會在每個「本端」及「專用」資料中心內執行，在該資料中心內，它會透過「中央控制 Pod」與「Box Panel 軟體即服務 (SaaS)」通訊。

      「Box Panel 網站控制器」的功能包括：

      • 管理遠端存取
      • IPMI Proxy
      • 裸機佈建
      • 中央套件鏡映
      • 監視/警示
      • Sensu
      • Graphite
      • PagerDuty
      • 日誌管理 (ELK)
      • 自動化服務
      • HTTP Proxy 服務

      下圖顯示部署時 Box Panel 與其相關聯「網站控制器」之間的關係概觀。

      術語常見問題：何謂…..

      問：Box Panel 佇列？

      答：「Box Panel 佇列」是用於來自 Box Panel 之管理功能的工作佇列。這些功能包括自動化軟體修補、新節點部署及重新啟動伺服器。

      問：Box Panel 工作者？

      答：「工作者」是一種程序，可接收來自「Box Panel 佇列」的工作要求並執行這些要求，然後將結果傳回給「Box Panel 佇列」。

      問：「中央控制 Pod」上的 Sensu？

      答：用來監視 IBM Cloud 及客戶擁有的資料中心內的遠端網站控制器。

      問：「網站控制器」上的 Sensu？

      答：監視 Bluemix Private Cloud 實體節點 (pNode)，並將警示傳回給「中央控制 Pod」。pNode 包括控制、計算及儲存節點。

      問：PagerDuty？

      答：突發事件報告工具，會在晚上需要注意雲端時喚醒我們。

      問：Graphite？

      答：我們所使用的一種圖形工具，特別是用於製作 SLA 報告以顯示 API 是否已啟動。您可以在 Horizon 中查看此資訊，因為 Graphite 會將執行時間資料回饋到 Grafana 儀表板。

      問：IPMI Proxy？

      答：提供雲端節點的安全 IPMI 存取。

      問：HTTP Proxy？

      答：為「網站控制器」上的 ELK 和 Sensu 使用者介面提供 HTTP 介面。

      問：Mirror？

      答：雲端作業軟體的集中式軟體儲存庫。此儲存庫不包含用戶端工作負載。

      問：Bastion？

      答：Bastion 主機使用「角色型存取控制 (RBAC)」，為「網站控制器」和客戶雲端節點提供安全遠端存取。如需 RBAC 的相關資訊，請參閱下列文件：

      • 管理使用者和專案
      • RBAC 定義
      • 聯合識別服務

      問：Yama？

      答：對客戶雲端的存取提供「多因子鑑別」，並記載雲端支援人員的存取。

      問：ELK（Elasticsearch、Logstash 及 Kibana）？

      答：用於雲端資源的日誌管理。這些日誌不會離開客戶網站。

      問：啟動載入器？

      答：提供伺服器的裸機佈建。「啟動載入器」是 Mirror 儲存庫的抄本。

      安全常見問題：

      問：誰可以從「IBM 支援中心」存取 Bluemix Private Cloud（本端）？

      答：OpenStack 工程和作業團隊的成員有權存取「控制器」、「計算和節點」、OpenStack API 以及 Horizon。他們使用此存取權來部署及操作私有雲服務。這些支援工程師對於 Hypervisor 上的客戶資料或工作負載沒有任何存取權。

      問：IBM 支援中心代表可以取得客戶資源的存取權嗎？

      答：IBM Bluemix Private Cloud 工程和作業團隊沒有認證，無法讓他們存取雲端上部署的任何雲端 VM 或應用程式。

      問：哪些資訊會流經 VPN？

      答：只有部署及操作私有雲服務所需的那些「指令」和「控制項」訊息會流經 VPN。在 VPN 與客戶資料面（VM、工作負載、儲存空間、Neutron 網路）之間沒有任何路由。「安全管理遠端存取」階段作業是由「OpenStack 支援中心」人員起始。套裝軟體用來安裝及修補私有雲伺服器。警示和訊息的監視則由雲端節點起始。所有記載資料均會保留在客戶駐地設備中。

      問：誰可以取得此資訊？

      答：IBM Bluemix Private Cloud OpenStack 工程和作業人員可以基於雲端管理和疑難排解的目的來存取此資訊。

      問：何謂支援團隊的存取控制原則？

      答：支援團隊遵循嚴格的「IBM Cloud 安全原則」來授與、重新評估及撤銷對客戶雲端的存取權。

      問：Bluemix Private Cloud 可以使用客戶登錄程序嗎？

      答：Bluemix Private Cloud 目前不支援，因為我們的程序需符合 ISO27002。

      問：Bluemix Private Cloud 管理的項目？

      答：Bluemix Private Cloud 可管理 OpenStack 雲端，範圍從裸機直到 Hypervisor。我們的程序、原則及工具均符合「IBM Cloud 安全原則 (CSP)」。CSP 符合業界標準 ISO27002，這是資訊安全控制實務的資訊技術標準，也就是說，它是用於安全技術。
