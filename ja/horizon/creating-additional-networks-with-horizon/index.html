      Horizon パネルを使用した OpenStack での追加ネットワークの作成

      • by Jason Kennedy ● August 5th, 2015

      • Tags:

        • 入門

        • 追加ネットワーク

      新しく導入した OpenStack クラウドの環境において、開発/テスト環境と実動ネットワークを分離する必要があったため、VXLAN を使用してクラウドをセットアップするよう具体的に要求されました。最初に、これは、セキュリティー・グループを使用して実行できることでしょうか? 答えはいいえですね? それでは、以下をお読みください。

      通常の IBM Bluemix Private Cloud には、デフォルトでセットアップされる内部ネットワークと外部ネットワーク、およびそれらをブリッジするデフォルト・ルーター (default という名前) があります。独自のネットワークを作成するには、このセットアップをミラーする必要があります。

      (特記事項: この手順には注意が必要です。これを行う前に実際に VXLAN を使用しているか確認します。そうでない場合は、適切に処理されません。また、非常に大きな問題が発生するリスクがあります。)

      最初にネットワークを作成します。整合性を図るために、コマンド・ライン・インターフェース (CLI) と Horizon の両方の方法について役立つ手順を示します。最初に Horizon の方法を示します。CLI の方法は別の投稿で簡潔に示します。

      プロジェクトで、「ネットワーク」タブに移動し、「ネットワーク」を選択します。前述した内部ネットワークおよび外部ネットワークが表示されます。

      ここから、右上にある「ネットワークの作成 (Create Network)」をクリックして、「ネットワークの作成 (Create Network)」ポップアップ画面を表示します。

      「ネットワーク名 (Network Name)」を入力し (「管理状態」は「アップ (UP)」のままにします)、「サブネット」タブをクリックします (または「次へ」を選択します)。

      「サブネット名 (Subnet Name)」(例えば test_subnet など。「ネットワーク名 (Network Name)」には test などを使用します)、「ネットワーク・アドレス (Network Addresses)」(例えば 192.168.10.0/28 など)、および「ゲートウェイ IP (Gateway IP)」(例えば 192.168.10.1 など) を入力します。

      次に「サブネット詳細」または「次へ」をクリックします。この画面で、プールの一部のみ割り振る場合、あるいは DNS サーバーまたはホスト・ルートを指定する場合は、それらを行います。

      後は、青色の「作成」ボタンをクリックするだけです。「ネットワーク」セクションは次のようになります。

      次にルーターを追加します。

      ナビゲーションの「ネットワーク」で「ルーター」をクリックします。まだルーターはないため、「ルーターの作成 (Create Router)」をクリックしてルーターを作成します。「ルーターの作成 (Create Router)」ダイアログ・ウィンドウが表示されます。実行する必要があるのは、「ルーター名 (Router Name)」を入力することのみです (例えば test_router などを使用します)。「ルーターの作成 (Create Router)」をクリックします。以下のような画面が表示されます。

      ルーターを見ると、外部ネットワークに接続されていないことに気付きます。ルーターで処理を行うためには、これを行う必要があります。右側の「アクション」で「ゲートウェイの設定 (Set Gateway)」をクリックします。「ゲートウェイの設定 (Set Gateway)」ウィンドウが表示されます。「外部ネットワーク (External Network)」ドロップダウンで、以下を行います。「外部 (External)」を選択し、その後、青色の「ゲートウェイの設定 (Set Gateway)」ボタンをクリックします。これで、ゲートウェイが外部ネットワークに接続されます。

      ここから、手順を戻って、これまで行ったことを検査できます。左側の「ネットワーク・トポロジー (Network Topology)」をクリックすると、ネットワークが存在していること、および作成したルーターが存在していることを確認できます。このルーターは、予定通りに外部ネットワークに接続されています。この時点で足りないのは、このルーターをご使用のネットワークに接続することのみです。これにより、インスタンス・トラフィックがルーティングされます。ルーターにマウス・カーソルを合わせると、「ネットワーク・トポロジー (Network Topology)」セクションでそのルーターに関する詳細を表示できます (「インターフェースの追加 (Add Interface)」および「ルーターの削除 (Delete Router)」機能も含まれます)。インターフェースを追加する必要がある場合は、以下を行います。

      左側で「ルーター」をクリックし、ご使用のルーターをクリックします。以下のような「ルーターの詳細 (Router Details)」画面が表示されます。

      この画面では、ルーターに関する多くの情報が提供され、外部ゲートウェイ・インターフェースが追加されたことも確認できます。「インターフェースの追加 (Add Interface)」をクリックして「インターフェースの追加 (Add Interface)」ダイアログ・ボックスを表示します。「サブネット」ドロップダウンで、作成したネットワークを選択します。オプションで、このインターフェースに IP を割り当てることができます。また、クラウドで自動的にこれを行うこともできます。IP アドレスは空白のままにして、自動的に割り当てられるようにします。青色の「インターフェースの追加 (Add Interface)」ボタンをクリックして終了します。

      続行する前に、インターフェースおよび IP アドレスについてここで説明するのが適切なので説明します。ネットワークに追加する各インターフェースには IP が必要です。前述したように、ルーターを外部ネットワークに接続するには IP アドレスが必要でした。外部ゲートウェイ・インターフェースでは、浮動 IP 範囲の固定 IP が使用されます。作成した内部インターフェースには、作成した内部ネットワーク・プールの専用固定 IP が必要です。「状況」は最初は「ダウン」ですが、完了すると「アクティブ」になります。

      ここで、左側の「ネットワーク・トポロジー (Network Topology)」に戻ると、外部ネットワークと作成したネットワークの間の接続が表示されます。各ネットワークには作成したルーターが接続され、そのルーターを介して接続されています。これで、このネットワークでインスタンスを開始し、そのインスタンスに浮動 IP を割り当てると、そのインスタンスに外部からアクセスできます。

      「プロジェクト」の「コンピュート」から、「インスタンス」をクリックします。右側で「インスタンスの起動 (Launch Instance)」をクリックします。これにより、「インスタンスの起動 (Launch Instance)」ダイアログ・ウィンドウが表示されます。「インスタンス名 (Instance Name)」 (この例では test_instance) を入力し、「インスタンスのブート・ソース (Instance Boot Source)」ドロップダウンから「イメージからブート」を選択します。単純な Cirros インスタンスを作成するため、「イメージ名」ドロップダウンから「Cirros (12.6MB)」を選択します。

      次に、上部で青色の「アクセスおよびセキュリティー (Access & Security)」をクリックします。鍵ペアは既にあるため、それが選択されていることを確認します。また、適切なセキュリティー・グループを使用していることを確認し、上部にある次の青色の「ネットワーキング」セクションをクリックします。

      ここで、テスト・ネットワークを選択します。「起動 (Launch)」をクリックします。次のような画面が表示されます。

      IP アドレス 192.168.10.1 は、ルーターの内部インターフェース・ゲートウェイに対して使用されているため、インスタンスに対しては使用できません。インスタンス名をクリックして「概要」に移動します。ここから、「コンソール」タブをクリックします。このタブが表示されたら、「接続済み」という暗い青色のセクションをクリックし、Enter キーを押して、ログイン・プロンプトを表示します。デフォルトの Cirros ログイン (ユーザー名 cirros、パスワード cubswin) を使用してインスタンスにログインします。

      外部アドレスに対して ping を実行すること、および IP を確認することができます。これで、実行する必要があるのは、浮動 IP を割り当てることのみです。

      右側の「アクション」で「浮動 IP の関連付け (Associate Floating IP)」を選択します。これにより、「浮動 IP の関連付けの管理 (Manage Floating IP Associations)」ダイアログ・ボックスが開きます。初めて IP を関連付ける場合、リストが空であるため、ドロップダウンの横の「+」アイコンを選択する必要があります。プールから外部 IP を選択し、青色の「IP の割り振り (Allocate IP)」をクリックし、「関連付ける」をクリックします。これで、インスタンスに関連付けられた 2 つの IP アドレスが表示されます。

      これで、ssh を介し、浮動 IP と関連する鍵を使用して、独自ネットワーク上のインスタンスに独自ルーターを介して接続することができます。
