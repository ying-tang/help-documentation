      ユーザーおよびプロジェクトの管理

      基本概念

      ユーザー: OpenStack 環境へのログイン認証情報を持つ、個別のユーザーまたはサービス。ユーザーは 1 つ以上のロールを持ち、1 つ以上のプロジェクトに属します。

      プロジェクト: プロジェクトはクラウドにおける組織単位であり、「テナント」や「アカウント」と呼ばれることもあります。各ユーザーは 1 つまたは複数のプロジェクトのメンバーです。プロジェクト内で、ユーザーはインスタンスやボリュームのようなリソースを作成して管理します。管理者はプロジェクトが作成できるリソース (インスタンス、vCPU、ボリュームなど) の数に上限を設定できます。

      ロール: ロールは、一連の特定の操作の実行をユーザーに許可する、管理権限のグループです。

      グループ: グループはユーザーとロールの間のリンクの働きをします。ユーザーは 1 つ以上のグループのメンバーである場合があり、グループには 1 つ以上のロールが割り当てられている場合があります。

      IBM Bluemix Private Cloud インストールでは、cloud_admin、project_admin、¥member¥、および heat_stack_owner の 4 つの役割があらかじめ定義されています。5 番目の役割 heat_stack_user は自動的に割り当てられます。

      |ロール| 説明 |:———|:———– | cloud_admin | この役割では、クラウド・レベルのアクセス制御が可能です。プロジェクトに関係なく、API 実行タスクを実行できます。この役割は、割り当て量、グループ、ユーザー、およびプロジェクトを作成および管理でき、管理ボリューム処置を実行できます。 | project_admin | この役割では、プロジェクト・レベルのアクセス制御が可能です。このユーザーは、指定のプロジェクト内でユーザーおよびプロジェクトの管理を実行できます。 | _member_ ** | この役割では、ユーザーはそのプロジェクトに割り振られたリソース (インスタンスやボリュームなど) を使用できます。 | **heat_stack_owner | この役割では、ユーザーはそのプロジェクトで Heat スタックを作成できます。この役割は、手動で割り当てる必要があります。heat_stack_user 役割が割り当てられているユーザーには、この役割を割り当てないでください。 | heat_stack_user | これは、Heat が作成するユーザーに対して、Heat によって自動的に割り当てられる役割です。この役割では、すべての API アクセスが制限されます。この役割をユーザーに明示的に割り当てないでください。 cloud_admin 役割は、クラウドおよびプロジェクトの管理アクションのみ実行できます。より高位の一連の特権は、admin 役割にのみ付与されます。これには、サービス、ドメインと役割、ネットワーク、および ID プロバイダーの管理アクションが含まれます。admin 役割は IBM のみ使用でき、お客様には付与されません。

      IBM Bluemix Private Cloud のインストールでは、他のプロジェクトおよびユーザーを作成できる cloud_admin 特権を持つユーザーは 1 人です。cloud_admin は、cloud_admin およびそれより下位の特権役割をユーザーに付与できます。

      Keystone v3 グループ関数の使用

      IBM Bluemix Private Cloud の Keystone API のバージョン 3 では、グループの概念が導入されています。簡単にグループを使用して、複数の割り当てを同時に行うことができます。

      例えば、環境内のあらゆるプロジェクトに対するアクセス権限が必要なサポート・チームがあるとします。各メンバーをすべてのプロジェクトに個別に追加する代わりに、サポートという名前のグループを作成し、そのグループにすべてのプロジェクトに対する _member_ 役割を割り当て、その後そのグループにサポート・スタッフを全員追加します。

      もう 1 つの一般的なユース・ケースは、環境内のあらゆるプロジェクトに対する cloud_admin ロールを付与された cloud_admin グループを作成する方法です。この編成により、cloud_admin は、cloud_admin グループにユーザーを一時的に追加することで、そのユーザーの特権を簡単に拡大できます。これは、休暇対応などの状況で使用できます。

      次の 2 つの図でアプローチの違いを説明します。

      この図は、Keystone v2 API でサポートされている従来の役割の割り当ての例を示しています。cloud_admin ユーザーには、プロジェクト 1 (Project 1)、プロジェクト 2 (Project 2)、およびプロジェクト 3 (Project 3) に対する cloud_admin 役割があります。ユーザー 1 (User 1) には、プロジェクト 2 (Project 2) に対する project_admin 役割があります。ユーザー 2 (User 2) とサポート・ユーザー 1 (Support User 1) にはプロジェクト 2 (Project 2) の _member_ 役割があります。サポート・ユーザー 1 (Support User 1) には、プロジェクト 3 (Project 3) の _member_ 役割もあります。

      プロジェクトを無効化した場合の結果

      cloud_admin 役割を持つユーザーは、プロジェクトを有効化および無効化することができます。プロジェクトを無効化した場合、以下のような結果になります。

      • ダッシュボードで、ユーザーは「プロジェクト」タブの「現在のプロジェクト」リストからそのプロジェクトにアクセスできなくなります。
      • 無効化されたプロジェクトのみのメンバーであるユーザーは、ログインできなくなります。
      • 無効化されたプロジェクトのインスタンスを起動できなくなります。既に実行中のインスタンスは自動的に終了しません。これらは手動で停止する必要があります。
      • 無効化されたプロジェクトのデータは、いつでもそのプロジェクトを再有効化できるように維持されます。

      ダッシュボードの使用

      次のセクションでは、ダッシュボードを使用して、プロジェクトまたはユーザー・アカウントを追加、更新、変更、および削除する方法について説明します。

      プロジェクトの追加

      1. ナビゲーション・バーで、「ID (Identity)」パネルを開き、「プロジェクト」をクリックします。
      2. 「プロジェクトの作成 (Create Project)」をクリックします。
      3. 「プロジェクトの作成」ウィンドウの「プロジェクトの情報」タブで、プロジェクトの名前と説明を入力します。デフォルトで、プロジェクトは有効になっています。
      4. 「プロジェクトのメンバー」タブで、プロジェクトのメンバーを追加し、適切なロールを割り当てます。
      5. 「プロジェクト・グループ」タブで、ユーザーのグループをプロジェクトに追加し、適切なロールを割り当てます。
      6. 「クォータ」タブで、クォータの値を編集します。
      7. 「プロジェクトの作成 (Create Project)」をクリックします。「プロジェクト」リストに、割り当てられた ID を含めてプロジェクトが表示されます。

      プロジェクトの更新

      プロジェクトを更新して、プロジェクトの名前または説明を変更したり、プロジェクトを有効化または一時的に無効化したりすることができます。

      1. ナビゲーション・バーで、「ID (Identity)」パネルを開き、「プロジェクト」をクリックします。
      2. 更新するプロジェクトを選択します。
      3. 「アクション」列のドロップダウン・リストから「プロジェクトの編集」を選択します。
      4. 「プロジェクトの編集」ウィンドウで、プロジェクトを更新してプロジェクトの名前または説明を変更したり、プロジェクトを有効または一時的に無効にすることができます。デフォルトで、プロジェクトは有効になっています。一時的に無効にするには、「有効」チェック・ボックスのチェック・マークを外します。無効になっているプロジェクトを有効にするには、「有効」チェック・ボックスを選択します。
      5. 「保存」をクリックします。

      プロジェクトのユーザー割り当ての変更

      cloud_admin 役割を持っている場合は、どのプロジェクトにも役割を割り当てることができます。project_admin ロールを持っている場合は、project_admin ロールが適用されているプロジェクトの割り当てを変更することができます。

      1. ナビゲーション・バーで、「ID (Identity)」パネルを開き、「プロジェクト」をクリックします。
      2. 変更するプロジェクトの「メンバーの管理 (Manage Members)」ボタンをクリックします。

      「プロジェクトの編集」ウィンドウに、次のユーザー・リストが表示されます。

      • すべてのユーザー: 現在のプロジェクトへの割り当てが可能なユーザー。
      • プロジェクトのメンバー: 現在のプロジェクトに割り当てられているユーザー。さらに、「プロジェクト・グループ」タブで、ユーザーのグループを割り当てることもできます。

      プロジェクトの削除

      1. ナビゲーション・バーで、「ID (Identity)」パネルを開き、「プロジェクト」をクリックします。
      2. 削除するプロジェクトを選択します。
      3. 「プロジェクトの削除」をクリックして削除を確認します。

      ユーザー・アカウントの作成

      cloud_admin 役割または project_admin 役割を持っている場合、新しいユーザー・アカウントを作成できます。ユーザー・アカウントを作成したら、そのアカウントを主プロジェクトに割り当てる必要があります。さらに、アカウントを追加のプロジェクトに割り当てるオプションもあります。

      1. ナビゲーション・バーで、「ID (Identity)」パネルを開き、「ユーザー (Users)」をクリックします。
      2. 「ユーザーの作成」をクリックします。
      3. 「ユーザーの作成」ウィンドウで、ユーザー名、E メール、およびユーザーの初期パスワードを入力します。パスワードを確認してください。「主プロジェクト」リストからプロジェクトを選択します。 「ロール」リストからユーザーのロールを選択します。デフォルトは _member_ です。
      4. 「ユーザーの作成」をクリックして変更を確認します。ダッシュボードがユーザーに ID を割り当て、ユーザーが「ユーザー」カテゴリーに表示されます。

      ユーザーの無効化または有効化

      cloud_admin または project_admin ロールを持っている場合、ユーザー・アカウントを有効化または無効化することができます。ユーザー・アカウントを無効化すると、ユーザーはログインできなくなります。ただし、ユーザーのデータは、いつでもそのユーザーを再有効化できるように維持されます。

      1. ナビゲーション・バーで、「ID (Identity)」パネルを開き、「ユーザー (Users)」をクリックします。
      2. 「ユーザー」リストで、無効化または有効化するユーザーを見つけます。
      3. 「アクション」列のドロップダウン・リストから「ユーザーの無効化」または「ユーザーの有効化」を選択します。 「有効」列で、有効値が Yes または No に更新されます。
