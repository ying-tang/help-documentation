      Box Panel、サイト・コントローラー、および IBM Bluemix Private Cloud Local

      • by Leslie Lundquist and Tom Spoonemore ● September 20, 2016

      • Tags:

        • ユーザー・ガイド

      本資料では、Box Panel とサイト・コントローラー、およびそれらが IBM Bluemix Private Cloud Local 環境を制御および管理するために連動するしくみに関する基本概念と用語の概要を示します。本資料には、外部資料へのリンクやポインターが記載されており、さらに詳しい情報を調べることもできます。『用語の FAQ』セクションで定義されている用語の中には、日常的な操作に際して知っておく必要のある用語よりも専門性の高いものが含まれていますが、これらの用語について知っておくことで、サポート・チームと対話する機会が生じたときに役立つ場合があります。

      Box Panel とは

      Box Panel は、IBM Bluemix Private Cloud を提供するために IBM が使用するアプリケーション・スイートです。Box Panel は、IBM のお客様だけでなく、内部サポート・スタッフも使用します。IBM は、Box Panel を使用してクラウド・インフラストラクチャーを管理します。

      Box Panel は、すべてのデータ・センター、マシン資産、クラスター、IP アドレス、およびお客様アカウント情報の記録システムであり、他にもいくつかの機能を備えています。

      Box Panel スイートは、Web ブラウザーやカスタム API を通じてアクセス可能な SaaS アプリケーションとして提供されます。Box Panel スイートは、サポート、チケット発行、およびチャットのためのアプリケーションで構成されており、製品資料へのリンクも含まれています。

      Box Panel の詳細については、『Box Panel ユーザーズ・ガイド』を参照してください。

      Box Panel は運用スタック全体を管理する

      Box Panel サイト・コントローラーとは

      Box Panel サイト・コントローラーは、コントロール・プレーンの分散運用、構成の管理、および自動化の各サービスを、Box Panel スイートと Bluemix Private Cloud のサポート・チームおよびエンジニアリング・チームに提供するソフトウェア・ツールです。

      Box Panel サイト・コントローラーは、各ローカル・データ・センターと専用データ・センターで実行され、Box Panel Software-as-a-Service (SaaS) と中央制御ポッドを通じて通信します。

      Box Panel サイト・コントローラーは以下のような機能を備えています。

      • 管理リモート・アクセス
      • IPMI プロキシー
      • ベア・メタル・プロビジョニング
      • 中央パッケージ・ミラー
      • モニター/アラートの発行
      • Sensu
      • Graphite
      • PagerDuty
      • ログ管理 (ELK)
      • 自動化サービス
      • HTTP プロキシー・サービス

      用語の FAQ: 各用語の意味

      Q: Box Panel キューとは

      A: Box Panel キューとは、Box Panel から送信される管理機能のジョブ・キューです。これらの機能には、ソフトウェア・パッチの自動適用、新規ノードのデプロイ、サーバーのリブートなどがあります。

      Q: Box Panel ワーカーとは

      A: ワーカーは、Box Panel キューからジョブ要求を受け取り、これらの要求を実行して、その結果を Box Panel キューに返すプロセスです。

      Q: 中央制御ポッド上の Sensu とは

      A: IBM クラウドおよびお客様所有のデータ・センター内のリモート・サイト・コントローラーをモニターします。

      Q: サイト・コントローラー上の Sensu とは

      A: Bluemix Private Cloud の物理ノード (pNode) をモニターし、中央制御ポッドにアラートを返します。pNode には、制御ノード、コンピュート・ノード、ストレージ・ノードなどがあります。

      Q: PagerDuty とは

      A: 夜間にクラウドが注意を要する状態になったときに警告を発するインシデント報告ツール。

      Q: Graphite とは

      A. API が機能しているかどうかを示す SLA レポートを作成することを主な目的として使用されるグラフィック・ツール。Graphite は、実行可能時間データを Grafana ダッシュボードにフィードするため、ユーザーはこの情報を Horizon で確認できます。

      Q: IPMI プロキシーとは

      A: クラウド・ノードへのセキュアな IPMI アクセスを提供します。

      Q: HTTP プロキシーとは

      A: サイト・コントローラー上の ELK および Sensu の各ユーザー・インターフェースへの HTTP インターフェースを提供します。

      Q: ミラーとは

      A: クラウド運用ソフトウェアの中央ソフトウェア・リポジトリー。このリポジトリーにはお客様のワークロードは含まれません。

      Q: 要塞とは

      A: 要塞ホストは、ロール・ベースのアクセス制御 (RBAC) を使用することで、サイト・コントローラーおよびお客様のクラウド・ノードへのセキュアなリモート・アクセスを実現します。RBAC の詳細については、以下の資料を参照してください。

      • ユーザーとプロジェクトの管理
      • RBAC の定義
      • フェデレーテッド ID サービス

      Q: Yama とは

      A: ユーザーのクラウドにアクセスするための多要素認証を提供し、クラウドのサポート・スタッフによるアクセスのロギングを保持します。

      Q: ELK (Elasticsearch、Logstash、および Kibana) とは

      A: クラウド・リソースのログ管理に使用されます。これらのログがお客様の敷地外へ移動されることはありません。

      Q: ブートストラッパーとは

      A: サーバーのベア・メタル・プロビジョニングを行います。ブートストラッパーはミラー・リポジトリーのレプリカです。

      セキュリティーの FAQ:

      Q: Bluemix Private Cloud Local へのアクセス権限を保持している IBM サポートのメンバーは誰ですか?

      A: OpenStack のエンジニアリング・チームと運用チームのメンバーが、コントローラー、コンピュート・ノード、OpenStack API、および Horizon へのアクセス権限を保持しています。これらのメンバーは、プライベート・クラウド・サービスのデプロイと運用を行うために、このアクセス権限を使用します。これらのサポート・エンジニアは、お客様のデータや、ハイパーバイザーよりも上位のワークロードに対するアクセス権限は保持していません。

      Q: IBM のサポート担当員が顧客のリソースへのアクセス権限を与えられることはありますか?

      A: IBM Bluemix Private Cloud のエンジニアリング・チームと運用チームは、クラウド VM やクラウドにデプロイされたアプリケーションに対するアクセス権限をこれらの担当者に付与するための資格情報を保持していません。

      Q: VPN 上ではどのような情報がやり取りされますか?

      A: VPN 上でやり取りされるのは、プライベート・クラウド・サービスのデプロイと運用に必要なコマンド・メッセージと制御メッセージだけです。VPN とお客様のデータ・プレーン (VM、ワークロード、ストレージ、Neutron ネットワーク) との間でルーティングが行われることはありません。OpenStack サポート・スタッフによって開始されたセキュアな管理リモート・アクセス・セッション。プライベート・クラウド・サーバーのインストールやパッチ適用に使用されるソフトウェア・パッケージ。クラウド・ノードによって開始されたモニター・アラートやメッセージ。すべてのロギング・データは、お客様の敷地内の設備で維持されます。

      Q: この情報を入手できるのは誰ですか?

      A: IBM Bluemix Private Cloud OpenStack エンジニアリング・チームと運用チームのスタッフが、クラウドの管理とトラブルシューティングのために、この情報へのアクセス権限を保持しています。

      Q: サポート・チームのアクセス制御ポリシーを教えてください。

      A: サポート・チームは、IBM の厳格なクラウド・セキュリティー・ポリシーに従って、お客様のクラウドに対するアクセス権限の付与、再評価、および取り消しを行います。

      Q: Bluemix Private Cloud は顧客登録プロセスを使用できますか?

      A: Bluemix Private Cloud は、現時点では顧客登録プロセスをサポートしていません。IBM のプロセスは ISO27002 に準拠しています。

      Q: Bluemix Private Cloud の管理対象を教えてください。

      A: Bluemix Private Cloud は、ベア・メタルからハイパーバイザーまでの OpenStack クラウドを管理します。IBM のプロセス、ポリシー、およびツールは、IBM クラウド・セキュリティー・ポリシー (CSP) に準拠しています。CSP は、業界標準である ISO27002 に準拠しています。この標準は、情報セキュリティー制御の手法、つまりセキュリティー技術を定めた情報技術標準です。
