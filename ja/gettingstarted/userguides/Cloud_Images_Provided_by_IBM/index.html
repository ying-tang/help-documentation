      クラウド・イメージのユーザーズ・ガイド (コミュニティー版)

      • by Yanqi Dou ● March 9th, 2016

      • Tags:

        • ユーザー・ガイド

        • イメージ

        • glance

        • image-create

        • インスタンス

      IBM Bluemix Private Cloud 3.0 には、Cirros 0.3.3 x86_64 イメージが事前設定されています。ただし、IBM Bluemix Private Cloud には他のダウンロード可能なクラウド・イメージも用意されているため、ユーザーは任意のオペレーティング・システムを使用してインスタンスを実行できます。本資料では、これらのイメージをダウンロードして操作するために必要な情報を示します。最新のイメージを使用するために、クラウド・イメージのリリース・ノートを必ず確認してください。

      OpenStack でのイメージの使用に関する詳細情報は、OpenStack の Web サイト (https://docs.openstack.org/image-guide/) から入手できます。

      前提条件

      1. コマンド・ライン (CLI) ツールを使用することを予定している場合は、OpenStack Glance クライアントがインストールされていることを確認してください。インストール手順については、知識ベースの記事『OpenStack API の概要』を参照してください。

      2. IBM が提供するクラウド・イメージの一時ダウンロード URL を取得できるようにするために、Box Panel へのアクセス権限を保持していることを確認します。

      3. イメージの場所として tempUrl を使用して Glance にイメージをアップロードできるのは、Bluemix Private Cloud 2.1.0 以降だけです。

      基本概念

      項目 説明
      イメージ 仮想マシン・イメージをこの文書では単に「イメージ」と呼びますが、これは、ブート可能なオペレーティング・システムがインストールされている仮想ディスクが含まれた、1 つのファイルです。イメージを使用して、仮想マシンのインスタンスをクラウド内に作成します。
      インスタンス インスタンスは、OpenStack クラウド内で稼働する仮想マシン (VM) です。
      フレーバー フレーバー という用語はインスタンスのサイズを表し、インスタンスが使用可能な vCPU、RAM、およびディスク・スペースの量を定めた設定済みのリソース仕様を定義します。

      IBM Bluemix Private Cloud が提供する代替クラウド・イメージからの選択

      このリリースの IBM Bluemix Private Cloud には、Cirros 0.3.3 x86_64 イメージが事前設定されています。ただし、IBM Bluemix Private Cloud では、ユーザーの便宜のために、追加のクラウド・イメージが用意されています。

      個別に取得したオペレーティング・システム・ソフトウェアをアップロードしてインストールすることも、IBM が提供する以下のオペレーティング・システム・イメージのうちいずれかを、仮想マシン・インスタンスとしてプロビジョンすることもできます。ただし、オペレーティング・システムの該当するすべてのライセンス条項の遵守、および各仮想マシン・インスタンスに対する適切なライセンスの取得は、お客様の責任において行ってください。

      • CentOS 6.8 x86_64
      • CentOS 7.2 x86_64
      • Ubuntu Server 12.04 LTS x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      Microsoft Windows オペレーティング・システムの仮想コンピューティング・インスタンスについては具体的な顧客条項があり、上記の Windows オペレーティング・システムの仮想コンピューティング・インスタンスをアクティブ化するには、顧客が独自の Microsoft Windows ライセンス・サーバーを所有および維持している必要があります。詳しくは、『IBM Bluemix Private Cloud での Windows ライセンスのアクティブ化』を参照してください。

      Box Panel からのクラウド・イメージ・ダウンロードの取得

      1. Box Panel の資格情報を使用して Box Panel にログインします。Box Panel の「クラウド・イメージ」ページが使用可能になるのは、ユーザーが 1 つ以上のクラウドを保持しており、「主要」または「技術」のお客様連絡先に該当する場合です。

      2. Box Panel ダッシュボードで、「サービス」をクリックして「クラウド・イメージ」を選択します。

      3. 「Cloud Images (クラウド・イメージ)」のページが表示されます。各イメージは、カードとして表示されます。

      4. 関心があるイメージが表示されているカードをクリックします。tempURL を示すモーダル・ウィンドウがポップアップします。

      5. 表示される tempURL の有効期限は 24 時間で、ユーザーはこれを使用してクラウド・イメージをダウンロードできます。

      クラウド・イメージの tempURL を使用した IBM Bluemix Private Cloud Glance リポジトリーへのイメージのアップロード

      注:

      • cloud_admin 役割を持つユーザーのみが、イメージを他のユーザーやプロジェクトに対して公開できます。
      • OpenStack コントロール・プレーンからアウトバウンド・インターネット・アクセスをブロックしている IBM Bluemix Private Cloud では、tempUrl をイメージの場所として直接使用して Glance にイメージをアップロードすることができません。
      • tempUrl をイメージの場所として直接使用して Glance にイメージをアップロードできるのは、IBM Bluemix Private Cloud 2.1.0 以降のみです。2.1.0 よりも前のリリースでは、この機能がサポートされていません。
      • リリース 2.1 よりも前の IBM Bluemix Private Cloud や、アウトバウンド・インターネット・アクセスが許可されていない IBM Bluemix Private Cloud を使用する場合は、『ダウンロード済みのイメージ・ファイルを使用した IBM Bluemix Private Cloud Glance リポジトリーへのイメージのアップロード』に記載された手順に従ってください。

      イメージのアップロードは、OpenStack ダッシュボードまたはコマンド・ライン・クライアントから実行できます。

      OpenStack ダッシュボードを使用したイメージのアップロード

      1. IBM Bluemix Private Cloud OpenStack ダッシュボードにログインします。
      2. 「プロジェクト」トピック下で、「コンピュート」を展開して「イメージ」をクリックします。

      3. ページで「イメージの作成 (Create Image)」をクリックします。新規ページに以下のパラメーターを指定し、「イメージの作成 (Create Image)」をクリックします。

        項目 説明
        名前 イメージ名を入力します。
        説明 イメージの簡単な説明を入力します。(オプション)
        イメージ・ソース イメージの場所を選択します。
        イメージの場所 クラウド・イメージのダウンロードの tempURL を指定します。
        フォーマット イメージの適切なフォーマットを選択します。(例: QCOW2)
        アーキテクチャー アーキテクチャーを指定します。(例: 32 ビット・アーキテクチャーの場合 i386。64 ビット・アーキテクチャーの場合 x86_64。)
        最小ディスク (GB) Linux イメージの場合 5 を使用。Windows イメージの場合 30 を使用。
        最小 RAM (MB) Linux イメージの場合 512 を使用。Windows イメージの場合 2048 を使用。
        データのコピー (Copy Data) イメージ・データをイメージ・サービスにコピーするには、このオプションにチェック・マークを付ける必要があります。
        パブリック イメージをすべてのプロジェクトのすべてのユーザーに公開する場合、このチェック・ボックスを選択します。
        保護 (Protected) 許可されたユーザーのみがイメージを削除できるようにする場合、このチェック・ボックスを選択します。

        例:

         Name: Ubuntu Server 14.04 LTS x86_64 - 20160301
 Description: Ubuntu Server 14.04 LTS x86_64 cloud image released on 2016/03/01
 Image Source: Image Location
 Image File: <tempURL_for_QCOW2>
 Format: QCOW2 - QEMU Emulator
 Architecture: x86_64
 Minimum Disk(GB): 5
 Minimum RAM(MB): 512
 Copy Data: True
 Public: True
      4. 作成したイメージが、イメージ・リストに表示されることを確認します。イメージはアップロードのキューに入っています。状況が “待機中” から “アクティブ” に変わるまでに時間がかかる場合があります。

      注: イメージを他のユーザーやプロジェクトに対して公開できるのは cloud_admin 役割を持つユーザーだけです。

      OpenStack コマンド・ライン・クライアントを使用したイメージのアップロード

      Glance クライアントを介してイメージをアップロードできます。詳しくは、Installing OpenStack Clientsを参照してください。

      イメージを作成します。

      • Linux イメージの場合、-- min-disk 5 --min-ram 512 を使用します。

      • Windows イメージの場合、--min-disk 30 --min-ram 2048を使用します。

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --copy-from  <tempURL_to_qcow2_image_file>

      イメージが正常に作成されたことを確認します。イメージはアップロードのキューに入っています。状況が “待機中” から “アクティブ” に変わるまでに時間がかかる場合があります。

      $ glance image-show <image-id>

      ダウンロード済みのイメージ・ファイルを使用した IBM Bluemix Private Cloud Glance リポジトリーへのイメージのアップロード

      注:

      • cloud_admin 役割を持つユーザーのみが、イメージを他のユーザーやプロジェクトに対して公開できます。

      • tempUrl を使用してクラウド・イメージをダウンロードするときは、必ず tempUrl トークンをエスケープするか、引用符で囲んでください。例:

        curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      次の手順を実行します。

      1. イメージの .qcow2 and .md5sum ファイルをダウンロードします。

      2. tempURL を使用して .qcow2 イメージ・ファイルと .md5sum チェックサム・ファイルをダウンロードし、これら 2 つのファイルを同じフォルダーに保存します。例:

         ubuntu-guest-image-14.04-20160301-x86_64.qcow2
 ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      3. .md5sum チェックサム・ファイルを使用して、ダウンロードした .qcow2 イメージ・ファイルを検証します。例:

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

        「OK」の結果が返された場合は、ダウンロードした qcow2 イメージは有効です。

      4. イメージを Glance にアップロードします。イメージのアップロードは、OpenStack ダッシュボードまたはコマンド・ライン・クライアントから実行できます。

      OpenStack ダッシュボードを使用したイメージのアップロード

      1. IBM Bluemix Private Cloud OpenStack ダッシュボードにログインします。

      2. 「プロジェクト」トピック下で、「コンピュート」を展開して「イメージ」をクリックします。

      3. ページで「イメージの作成 (Create Image)」をクリックします。新規ページに以下のパラメーターを指定し、「イメージの作成 Create Image.

        項目 説明
        名前 イメージ名を入力します。
        説明 イメージの簡単な説明を入力します。(オプション)
        イメージ・ソース イメージ・ファイルを選択します。
        イメージ・ファイル イメージ・バイナリー・ファイルのディレクトリーを参照します。
        フォーマット イメージの適切なフォーマットを選択します。(例: QCOW2)
        アーキテクチャー アーキテクチャーを指定します。(例: 32 ビット・アーキテクチャーの場合 i386。64 ビット・アーキテクチャーの場合 x86_64。)
        最小ディスク (GB) Linux イメージの場合 5 を使用。Windows イメージの場合 30 を使用。
        最小 RAM (MB) Linux イメージの場合 512 を使用。Windows イメージの場合 2048 を使用。
        パブリック イメージをすべてのプロジェクトのすべてのユーザーに公開する場合、このチェック・ボックスを選択します。
        保護 (Protected) 許可されたユーザーのみがイメージを削除できるようにする場合、このチェック・ボックスを選択します。

        例:

         Name: Ubuntu Server 14.04 LTS x86_64 - 20160301	
 Description: Ubuntu Server 14.04 LTS x86_64 cloud image released on 2016/03/01	
 Image Source: Image File	
 Image File: C:\Documents\ubuntu-guest-image-14.04-20160301-x86_64.qcow2	
 Format: QCOW2 - QEMU Emulator	
 Architecture: x86_64	
 Minimum Disk(GB): 5	
 Minimum RAM(MB): 512	
 Public: True

      4. 作成したイメージが、イメージ・リストに表示されることを確認します。イメージはアップロードのキューに入っています。状況が “待機中” から “アクティブ” に変わるまでに時間がかかる場合があります。

      OpenStack コマンド・ライン・クライアントを使用したイメージのアップロード

      詳しくは、Installing OpenStack Clientsを参照してください。

      イメージを作成します。

      • Linux イメージの場合、-- min-disk 5 --min-ram 512 を使用します。

      • Windows イメージの場合、--min-disk 30 --min-ram 2048を使用します。

        $ glance --os-image-api-version 1 image-create --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --is-public True --file <path_to_qcow2_image_file>

      イメージが正常に作成されたことを確認します。イメージはアップロードのキューに入っています。状況が “待機中” から “アクティブ” に変わるまでに時間がかかる場合があります。

      $ glance image-show <image-id>

      注: イメージのサイズが大きい場合は、Web アップロードがタイムアウトになることがあります。このような場合は、ユーザーが自分のクラウド内でインスタンスを起動し、そのインスタンスにイメージをダウンロードしたうえで、そのインスタンスから Glance にアップロードすることをお勧めします。これにより、すべてのトラフィックが SoftLayer 内で行われ、アップロードがユーザーのクラウド内で行われます。

      アップロード済みのイメージを使用した VM インスタンスの作成

      OpenStack ダッシュボードを使用した VM インスタンスの作成

      手順の詳細については、『Horizon ダッシュボードからのインスタンスの起動』を参照してください。

      作成後のタブでの初期ユーザー名とパスワードの設定

      「カスタマイズ・スクリプト (Customization Script)」フィールドは、インスタンスの起動後に実行されるカスタマイズ・スクリプトを指定します。

      **Linux インスタンスの場合: **

      コンソールおよび SSH を通じたパスワード認証を有効にするには、「カスタマイズ・スクリプト (Customization Script)」に以下のスクリプト・データを使用します (<YOUR_PASSWORD> には該当するパスワードを指定)。

      #cloud-config 
password: <YOUR_PASSWORD> 
chpasswd: { expire: False } 
ssh_pwauth: True

      **Windows インスタンスの場合: **
      初期ユーザー名とパスワードを設定するには、「カスタマイズ・スクリプト (Customization Script)」に以下のスクリプト・データを使用します (_<YOUR_USERNAME>_ と _<YOUR_PASSWORD>_ には該当するユーザー名とパスワードを指定)。

      OpenStack コマンド・ライン・クライアントを使用した VM インスタンスの作成

      手順の詳細については、『Nova コマンド・ライン・クライアントを使用したインスタンスの作成方法』を参照してください。

      初期ユーザー名とパスワードの設定

      セッション『OpenStack ダッシュボードを使用した VM インスタンスの作成』で説明したカスタマイズ・スクリプトを 1 つのファイル (userdata.txt など) に保存し、--user-data userdata.txt オプションを nova boot コマンドに渡します。以下のように指定します。

      インスタンスへの接続

      注: ssh または rdp を使用してインスタンスに接続するためには、適切なセキュリティー・グループ・ルールが定義されていることを確認してください。

      1. IBM Bluemix Private Cloud OpenStack ダッシュボードにログインします。

      2. 「プロジェクト」パネルを展開して左側のナビゲーション・エリアで「コンピュート」タブを選択し、「インスタンス」をクリックします。

      3. 実行中のインスタンスのリストでインスタンスを見つけ、IP アドレス (例: 192.168.100.237) をメモします。

      4. IP アドレスを指定して、インスタンスへの接続を作成します。

      Linux インスタンスへの接続

      SSH または VNC を介してインスタンスに接続します。IBM から提供される Linux イメージのデフォルトの userid は、ibmcloud です。Linux インスタンスに接続されると、通常であれば root アクセス権限がなければ実行できないコマンドを sudo コマンドを使用して実行できるようになります。Sudo は、デフォルトの userid 用のパスワードを要求しないように構成されています。

      • 任意の SSH クライアントを使用して、VM インスタンスの正しい IP アドレスへの接続を開きます。このとき、プロビジョン時に指定した鍵ペアの SSH 秘密鍵を使用します。以下のように指定します。

          ssh -i <path_of_Your_SSH_private_key_file> <userid>@<ip_of_instance>

      • IBM Bluemix Private Cloud OpenStack ダッシュボードでインスタンス名をクリックしてインスタンスの VNC コンソールを表示し、「コンソール (Console)」タブをクリックします。VNC コンソールを使用して HTTPS 接続が可能になります。

      注: コンソールから新規 Linux インスタンスに ibmcloud ユーザーとしてログインできるようにするには、プロビジョン時にカスタマイズ・スクリプト内でパスワードを指定しておく必要があります。

      Windows インスタンスへの接続

      リモートデスクトップ または VNC を介してインスタンスに接続します。VM のプロビジョン時にカスタマイズ・スクリプトで指定した userid と password を使用する必要があります。初めてのログイン時はプロンプトが表示され、パスワードを変更するように要求されます。指示に従ってください。

      • お気に入りのリモートデスクトップ・クライアントを使用して接続します。
      • IBM Bluemix Private Cloud OpenStack ダッシュボードでインスタンス名をクリックしてインスタンスの VNC コンソールを表示し、「コンソール (console)」タブをクリックします。VNC コンソールを使用して HTTPS 接続が可能になります。

      イメージの更新

      IBM Bluemix Private Cloud 運用チームからクラウド・イメージが更新されたという通知が届いた場合は、既存のイメージを非アクティブ化し、Box Panel から入手した更新イメージ用のダウンロード URL を使用して、新規イメージを再作成することができます。

      OpenStack コマンド・ライン・クライアントを使用した既存イメージの非アクティブ化

      1. 以下のコマンドを使用して既存イメージの ID を取得します。

         $ glance image-list

      2. 以下のコマンドを使用してイメージを非アクティブ化します。

        $ glance image-deactivate <image_id>

      3. 以下のコマンドを使用してイメージの状況を表示します。

        $ glance image-show <image_id>

      4. 既存のイメージを非アクティブ化したら、本資料の『クラウド・イメージの tempURL を使用した IBM Bluemix Private Cloud Glance リポジトリーへのイメージのアップロード』に記載された手順に従って、新規イメージを再作成します。

      クラウド・イメージに関する IBM Bluemix Private Cloud 運用チームからのサポートの利用

      特定のクラウド・イメージについて、IBM Bluemix Private Cloud 運用チームからのサポートを利用する場合は、イメージの buildID をご提供いただけると助かります。イメージの buildID を取得し、IBM Bluemix Private Cloud 運用チームに対するサポート・チケットを作成して、その buildID を指定してください。buildID を判断する方法は以下のとおりです。

      • イメージ・ファイル名から判断する。例:
        例: ubuntu-guest-image-14.04-20160301-x86_64.qcow2

      • クラウド・イメージからプロビジョンされた VM の buildID ファイルから判断する。

        • Linux インスタンスの buildID ファイルの場所: /etc/buildID

        • Windows インスタンスの buildID ファイルの場所: C:\buildID
