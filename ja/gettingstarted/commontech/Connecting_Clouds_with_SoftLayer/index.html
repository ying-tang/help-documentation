      プライベート・ネットワークの使用による複数のクラウド環境間でのネットワーク接続のセットアップ

      • by Shaival Choski ● March 22nd, 2016

      • Tags:

        • はじめに

        • ネットワーク

        • プライベート・ネットワーク

      概要:

      この文書では、IBM クラウド・データ・センターのプライベート・ネットワークを使用して、IBM Bluemix Private Cloud の複数のインストール済み環境間のネットワーク接続をセットアップする方法について説明します。

      IBM クラウド・データ・センターのプライベート・ネットワーク内で転送されるトラフィックは計量されないため、これは複数の IBM Bluemix Private Cloud 間で優先される通信方法です。IBM クラウド・データ・センターのプライベート・ネットワークの詳細については、ここを参照してください。

      IBM クラウド・データ・センターのプライベート・ネットワーク・トラフィックは公共のインターネットおよび他の SoftLayer の顧客から分離されますが、トラフィックは 暗号化されない という点に留意してください。プライベート・ネットワークのコンテキストであっても暗号化が必要な場合は、クライアント・サイドの暗号化を構成に追加してください。

      前提条件:

      この文書では、接続しようとしている各 IBM Bluemix Private Cloud が同じ SoftLayer アカウントでデプロイされていることを前提としています。単一の IBM Bluemix Private Cloud 顧客として複数のクラウドを注文している場合、この前提条件は自動的に満たされます。

      既知の制限事項:

      浮動 IP が関連付けられているインスタンスからのトラフィックは、IBM クラウド・データ・センターのプライベート・ネットワークを通過する前に SNAT 変換が実行されます。顧客のアプリケーションがソース・アドレスの変換による影響を受ける場合、浮動 IP を使用するインスタンスは使用できません。

      ネットワーク接続を開始するには

      1. IBM Bluemix Private Cloud が既にデプロイされている場合、サポート・チケットをオープンして IBM クラウド・データ・センターのプライベート・ネットワークへのアクセスを要求します。

      2. IBM Bluemix Private Cloud が新規デプロイメントの場合 (新規ユーザー向け) は、Bluemix Private Cloud を注文する際に IBM クラウド・データ・センターのプライベート・ネットワーク接続を要求します。

      プライベート・ネットワークを介した仮想マシン間の通信

      IBM クラウド・データ・センターのプライベート・ネットワークを要求すると、sl-private-network という名前の OpenStack 内部共有ネットワークが作成されます。このネットワークには次のようないくつかの条件があります。

      • このネットワークは、IBM Bluemix Private Cloud 内のすべての OpenStack プロジェクトにわたって全ユーザーに表示されます。
      • すべてのユーザーはこのネットワークに仮想マシンを直接接続でき、このネットワークに接続されている他のすべての仮想マシンにネットワーク接続されます。
      • このネットワーク上の仮想マシンは標準の OpenStack セキュリティー・グループを使用して保護されます。
      • 仮想マシンが sl-private-network にデプロイされると、10.x.x.x の IP アドレスが割り当てられます。この IP アドレスは、IBM クラウド・データ・センター間でルーティング可能です。
      • IBM Bluemix Private Cloud はそれぞれ独自の 10.x.x.x サブネットを持ち、IBM Cloud Data Center はそのプライベート・ネットワークを介してサブネット間のトラフィックをルーティングします。

      通信に使用する各インスタンスで、以下のステップを実行します。

      ステップ 1. 通常どおりに仮想マシンを作成します。右側のプルダウン・メニューを見つけます。ここでは通常、浮動 IP アドレスを VM に接続します。同じプルダウン・メニューで、「インターフェースの接続 (Attach Interface)」を選択し、sl-private-network-01 を選択します。

      ステップ 2. 上記で作成した仮想マシンにログインします。eth1 インターフェースに IP を手動で構成します。以下に例を示します。

      vi /etc/network/interfaces

# Configure Loopback
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet dhcp

auto eth1
iface eth1 inet static
address 10.168.6.87
netmask 255.255.255.192
gateway 10.168.6.65

      ステップ 3. ip route add 10.0.0.0/8 via 10.168.6.65 のように指定して、10 ネットへの静的ルートを追加し、ifdown eth1 && ifup eth1 を指定します

      各インスタンスに上記の 3 つのステップを行っていない場合、他のロケーションへの接続を取得できません。Neutron は NIC カード上で割り当てられると、IP アドレスを浮動 IP に自動的に変更します。
