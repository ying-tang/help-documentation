      静的 IP を使用したインスタンスの作成方法

      • by Jason Kennedy ● September 2, 2016

      • Tags:

        • openstack

        • 静的 IP

        • インスタンス

      この記事では、OpenStack で新規ポートを作成し、「静的」IP をインスタンスに割り当てるためにブート時にそのポートをインスタンスに手動で関連付ける方法について説明します。

      まず、この方法の大部分は、コマンド・ラインで行います。これは、コマンド・ラインが IBM Bluemix Private Cloud で作業するためのはるかに多用途のツールであるためです。必要な場合には、この一連の説明が分かりやすくなるように、OpenStack CLI & API の使用についても多少補って説明します。それでは、この実行方法について見ていきましょう。この作業はテスト・クラスター上で行いますが、ご使用の独自のクラスターにおいて screenshare でステップを繰り返すこともできます (それが正当な場合)。

      まず、Bluemix Private Cloud クラスターには、2 つの事前に生成されたネットワーク internal および external があります。ここでの目的のために、内部ネットワーク上に新規ポートを作成します。以下のように、コマンド・ラインで内部ネットワークを簡単に見てみましょう。

      bash-3.2$ openstack network list | grep internal
| 81e829d0-5e35-4808-9f7a-d22159faa9d1 | internal | c6560f00-ae29-40b5-93bb-6e7b1ffd3720

      これにより、内部ネットワークの ID が得られます。少しの間、CLI について話すと、この作業では、python-openstackclient を使用します。これは、新しいクライアントです。このクライアントでは ID はあまり使用しませんが、特定のネットワーク、テナントなどの ID を見つけることができることは、必要に応じて古いツール (python-novaclient など) を使用する機会が生じたときに重要になります。ここでは、あまり詳細にこだわることは止めておきましょう。

      この例で作業するサブネットを確認する必要があるので、以下のように確認してみましょう。
