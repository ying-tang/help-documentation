      高可用性 IP の構成方法

      • by Eric French ● September 21, 2016

      • Tags:

        • neutron

        • ip の構成

        • ha

        • 高可用性

        • 許可済みアドレス・ペア

        • l2 pop

      この記事では、IBM Bluemix Private Cloud Local 環境内で HA 浮動 IP をセットアップする方法について説明します。これらの説明は、OpenStack Python API クライアントが既にインストール済みであることを前提としています。

      注: この機能は、IBM Bluemix Private Local Cloud でのみ動作します。IBM Bluemix Private Cloud (専用クラウド) オファリングには適用されません。IBM Bluemix データ・センター・ネットワークでは、現在、L2 POP (大規模ネットワークへのスケーリングをサポートするために設計されたドライバー) を使用する必要があります。以下は、ある IBM のエンジニアの発言です。

      「IBM Bluemix データ・センター・ネットワークで VXLAN を動作させるために使用する必要がある L2Population メカニズム・ドライバーは、HA IP 方式で使われる MAC ラーニングおよび ARP 応答と互換性がありません。IP は、2 つの変換メカニズムを介してホストにマップされます。つまり、ARP によって IP が MAC アドレスにマップされ、スイッチによってどの MAC が各ポートに関連付けられているか学習されます。フェイルオーバー時には、HA IP によってこれらの 2 つのマッピングのうち少なくとも 1 つが変更される必要があります。」

      可能な対処法として、OpenStack Mitaka リリースでは、L2 POP レスポンダーの半分を無効にするというオプションがあります。これにより、Gratuitous ARP タイプの HA をセットアップできます (https://review.openstack.org/278597 の記事を参照してください)。HA が実際に必要な場合、Neutron LBaaS v2 などの、HA 用のロード・バランサーを使用するのが最も適切です。もう 1 つの可能な対処法として、単一の SoftLayer Pod 内にクラウドがデプロイされており、その単一の Pod を超えてクラウドを拡大させることがない場合、一部のタイプの HA IP をカスタマイズして実装できます。
