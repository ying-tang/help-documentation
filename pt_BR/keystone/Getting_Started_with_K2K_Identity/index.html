      Getting Started with Keystone Federated Identity

      • by Elvin Tubillara ● March 6, 2017

      • Tags:

        • keystone

        • federation

        • k2k

        • groups

      A Identidade Federada Keystone-to-Keystone (K2K) usa o componente do OpenStack Keystone v3, que é comumente usado para autenticação. Se você estiver acabando de iniciar o OpenStack, poderá aprender mais sobre o Keystone e outros componentes principais do OpenStack em outro artigo, A Força do OpenStack.

      Nossa equipe de especialistas configura automaticamente a Identidade Federada K2K mediante solicitação. Depois da configuração, um administrador, ou usuário cloud_admin, pode gerenciar identidades federadas gerenciando permissões por meio de grupos e gerenciando mapeamentos de usuários remotos, do Provedor de Identidade para grupos locais nos Service Providers. Os Service Providers usam mapeamentos para selecionar os atributos do usuário do OpenStack do Provedor de Identidade e, em seguida, usam esses atributos para mapear o usuário para um grupo local. Depois que o usuário efetua login usando a Identidade Federada K2K, o usuário recebe as funções designadas ao grupo, automaticamente.

      Os exemplos a seguir mostram como usuários federados que pertencem a um projeto demo no Provedor de Identidade têm acesso ao projeto demo no provedor de serviços.

      Criando grupos para usuários federados

      Vamos criar um grupo chamado demo_member_group e designar a ele a função _member_ no projeto demo.

      Execute os seguintes comandos como o usuário cloud_admin do Provedor de Serviços:

      $ openstack group list
$ openstack group create demo_member_group --or-show
$ openstack project list
$ openstack role list
$ openstack role add _member_ --project demo --group demo_member_group

      O demo_member_group agora terá acesso ao projeto demo.

      Criando mapeamentos para usuários federados

      Depois que um grupo é criado, você pode criar mapeamentos para mapear usuários remotos do Provedor de Identidade para um grupo local no Provedor de Serviços.

      O exemplo a seguir mostra como criar um arquivo de mapeamento que mapeia usuários do Provedor de Identidade, com a função _member_ designada no projeto demo, para demo_member_group no Provedor de Serviços.

      Primeiro, será necessário criar um arquivo chamado federated_demo_mapping.json:

      [
  {
    "local": [
      {
        "user": {
          "name": "{0}"
        }
      },
      {
        "group": {
          "domain": {
            "name": "Default"
          },
          "name": "demo_member_group"
        }
      }
    ],
    "remote": [
      {
        "type": "openstack_user"
      },
      {
        "type": "openstack_project",
        "any_one_of": [
          "demo"
        ]
      },
      {
        "type": "openstack_roles",
        "any_one_of": [
          "_member_"
        ]
      }
    ]
  }
]

      Em seguida, configure o mapeamento executando os seguintes comandos como o usuário cloud_admin do Provedor de Serviços:

      $ openstack mapping list
$ openstack mapping set mapping-for-k2k-federation --rules federated_demo_mapping.json
$ openstack mapping show mapping-for-k2k-federation --format json

      Depois de executar essas etapas, você tem regras mapeadas para usuários federados.

      Tentando no Horizon

      Os clientes do IBM Bluemix Private Cloud podem usar a Identidade Federada Keystone-to-Keystone para conectar ao Horizon, o painel do OpenStack. Ao conectar ao Provedor de Identidade, você é capaz de usar os serviços no Provedor de Serviços.

      É importante assegurar que haja um usuário no Provedor de Identidade para login federado. No contexto do nosso projeto demo, esse é o usuário com a função _member_.

      O usuário cloud_admin do Provedor de Identidade deve executar os seguintes comandos:

      $ openstack user create demo_member_user --project demo --password &lt;secure password&gt; --or-show
$ openstack role add _member_ --user demo_member_user --project demo
$ openstack role assignment list --name --user demo_member_user

      Os usuários agora são capazes de efetuar login no painel do Horizon.

      No Horizon, você tem acesso fácil aos serviços do OpenStack, gerenciados pelo Keystone do Provedor de Serviços.
