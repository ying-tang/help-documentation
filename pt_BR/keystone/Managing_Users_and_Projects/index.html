      Gerenciando usuários e projetos

      Conceitos básicos

      Usuários: um usuário é um indivíduo ou serviço com credenciais de login para o ambiente do OpenStack. Um usuário tem um ou mais papeis e pertence a um ou mais projetos.

      Projetos: projetos são unidades organizacionais na nuvem e também são conhecidos como locatários ou contas. Cada usuário é membro de um ou mais projetos. Dentro de um projeto, um usuário cria e gerencia recursos, como instâncias e volumes. Administradores podem configurar limites quanto ao número de recursos (instâncias, vCPU, volumes e mais) que um projeto tem permissão de criar.

      Funções: uma função é um grupo de privilégios que permitem que um usuário execute um conjunto específico de operações.

      Grupos: grupos funcionam como o link entre usuários e funções. Um usuário pode ser um membro de um ou mais grupos e um grupo pode ter uma ou mais funções associadas a ele.

      A instalação do IBM Bluemix Private Cloud vem com quatro funções predefinidas: cloud_admin, project_admin, _member_ e heat_stack_owner. Uma quinta função, heat_stack_user, é designada automaticamente.

      Função Descrição
      cloud_admin Esta função permite o controle de acesso de nível de nuvem. Ela permite realizar tarefas de execução de API, independentemente do projeto. Esta função pode criar e gerenciar cotas, grupos, usuários e projetos e pode executar ações de volume administrativas.
      project_admin Esta função permite o controle de acesso de nível de projeto. Esse usuário pode executar o gerenciamento do usuário e do projeto dentro do projeto especificado.
      _member_ Esta função permite que o usuário utilize os recursos (como instâncias e volumes) alocados para o projeto.
      heat_stack_owner Esta função permite que o usuário crie pilhas do Heat no projeto. Esta função deve ser designada manualmente e nunca deve ser designada a um usuário que também tem a função heat_stack_user designada.
      heat_stack_user Esta é uma função automaticamente designada pelo Heat aos usuários que ela cria. Esta função é restrita de todo o acesso de API e nunca deve ser designada a nenhum usuário explicitamente.

      A função cloud_admin é restrita a ações de gerenciamento de nuvem e projeto. Um conjunto mais alto de privilégios é concedido exclusivamente à função admin, que inclui ações administrativas para serviços, domínios e funções, redes, e provedores de identidade. A função admin é restrita à IBM e não é concedida ao cliente.

      Com a instalação do IBM Bluemix Private Cloud, você receberá um usuário com privilégios cloud_admin que pode criar outros projetos e usuários. O cloud_admin pode conceder aos usuários cloud_admin e funções privilegiadas inferiores.

      Requisitos

      Essa estrutura de função requer que o Keystone API v3 seja executado. Usar o Keystone API v2 resultará em uma perda de privilégios para as funções cloud_admin e project_admin, além de causar comportamento de permissões irregulares para usuários _member_.

      Fazer chamadas API por meio do painel do Horizon não requer nenhuma ação especial, mas executar chamadas da linha de comandos necessita de mudanças no arquivo run command (RC) dependendo de qual versão do IBM Bluemix Private Cloud você está executando.

      • No IBM Bluemix Private Cloud versão 2.1.1, os arquivos RC gerados pelo painel ativam o Keystone v2.
      • Assim, em 2.1.1, alternar para v3 requer mudanças no arquivo RC padrão. O OS_AUTH_URL deve ser mudado de v2.0 para v3, e OS_IDENTITY_API_VERSION deve ser criado e configurado para 3. Um arquivo RC de amostra é mostrado abaixo.

(Há um problema conhecido que os comandos da CLI de keystone atualmente não funcionam previsivelmente como parte da V3.)

      Usando a função de grupo do Keystone v3

      A Versão 3 do Keystone API com IBM Bluemix Private Cloud introduz o conceito de grupos. Os grupos podem ser facilmente usados para fazer várias designações simultaneamente.

      Por exemplo, você pode ter uma equipe de suporte que precisa de acesso a cada projeto em seu ambiente. Em vez de incluir individualmente cada membro em cada projeto, você pode criar um grupo chamado Suporte, dar ao grupo a função _member_ em todos os seus projetos e, em seguida, incluir toda a sua equipe de suporte no grupo.

      Outro caso de uso comum seria criar um grupo cloud_admin que recebe a função cloud_admin em cada projeto no ambiente. Esse acordo permite que cloud_admin eleve facilmente os privilégios de um usuário, incluindo-os temporariamente no grupo cloud_admin. Ele pode ser usado em situações como cobertura de férias.

      Esta figura mostra um exemplo de designações de funções tradicionais conforme suportado pela API do Keystone v2.       Consequências de se desabilitar projetos

      Os usuários com a função cloud_admin podem ativar e desativar projetos. Quando você desabilita um projeto, isso tem as seguintes consequências:

      • No painel, os usuários não têm mais acesso ao projeto na lista CURRENT PROJECT na guia Projeto.
      • Usuários que são membros apenas do projeto desativado não podem mais efetuar login.
      • Não é possível ativar instâncias para um projeto desativado. Instâncias que já estão em execução não são finalizadas automaticamente. Deve-se pará-las manualmente.
      • Os dados para um projeto desativado são mantidos para que seja possível reativar o projeto a qualquer momento.

      Usando o painel

      A próxima seção informa como usar o painel para incluir, atualizar, modificar e excluir um projeto ou uma conta de usuário.

      Incluindo um projeto

      1. Na barra de navegação, abra o painel Identidade e clique em Projetos.
      2. Clique em Criar Projeto.
      3. Na guia Informações sobre o Projeto na janela Criar Projeto, insira um nome e descrição para o projeto. Por padrão, o projeto fica ativado.
      4. Na guia Membros do seu Projeto, adicione membros no projeto e designe os papeis apropriados.
      5. Na guia Grupos de Projeto, adicione grupos de usuários no projeto e designe os papeis apropriados.
      6. Na guia Cota, edite os valores de cota.
      7. Clique em Criar Projeto. A lista Projetos mostra o projeto, incluindo seu ID designado.

      Atualizando um projeto

      É possível atualizar um projeto para alterar seu nome ou descrição e para habilitá-lo ou desabilitá-lo temporariamente.

      1. Na barra de navegação, abra o painel Identidade e clique em Projetos.
      2. Selecione o projeto que deseja atualizar.
      3. Selecione Editar Projeto na lista suspensa na coluna Ações.
      4. Na janela Editar Projeto, é possível atualizar um projeto para alterar seu nome ou descrição e para habilitá-lo ou desabilitá-lo temporariamente. Por padrão, o projeto fica habilitado. Para desabilitá-lo temporariamente, limpe a caixa de seleção Ativado. Para ativar um projeto desabilitado, selecione a caixa de seleção Ativado.
      5. Clique em Salvar.

      Modificando designações de usuário para um projeto

      Se tiver a função cloud_admin, você poderá designar funções em qualquer projeto. Se tiver a função project_admin, você poderá modificar designações em projetos nos quais tem a função project_admin aplicada.

      1. Na barra de navegação, abra o painel Identidade e clique em Projetos.
      2. Clique no botão Gerenciar Membros para o projeto que deseja modificar.

      A janela Editar Projeto mostra estas listas de usuários:

      • Todos Usuários: usuários que estão disponíveis para serem designados ao projeto atual.
      • Membros do Projeto: usuários que são designados ao projeto atual. Também é possível designar grupos de usuários na guia Grupos de Projeto.

      Exclusão de projetos

      1. Na barra de navegação, abra o painel Identidade e clique em Projetos.
      2. Selecione os projetos que deseja excluir.
      3. Clique em Excluir Projetos para confirmar a exclusão.

      Criando uma Conta de Usuário

      Se tiver a função cloud_admin ou project_admin, você poderá criar novas contas de usuários. Ao criar uma conta do usuário, você deve designar a conta a um projeto primário. Você também terá a opção de designar a conta a projetos adicionais.

      1. Na barra de navegação, abra o painel Identidade e clique em Usuários.
      2. Clique em Criar Usuário.
      3. Na janela Criar Usuário, insira um nome de usuário, e-mail e senha preliminar para o usuário. Confirme a senha. Selecione um projeto na lista Projeto Primário. Escolha um papel para o usuário na lista Papel. O padrão é _member_.
      4. Clique em Criar Usuário para confirmar suas mudanças. O painel designa um ID ao usuário, que aparece na categoria Usuários.

      Desativando ou ativando um usuário

      Se tiver a função cloud_admin ou project_admin, você poderá ativar e desativar as contas de usuários. Quando você desabilita uma conta do usuário, o usuário não pode mais logar. No entanto, os dados para o usuário são mantidos para que seja possível reativar o usuário a qualquer momento.

      1. Na barra de navegação, abra o painel Identidade e clique em Usuários.
      2. Localize o usuário que deseja desabilitar ou habilitar na lista Usuários.
      3. Selecione Desativar Usuário ou Ativar Usuário na lista suspensa na coluna Ações. Na coluna Ativado, o valor ativado é atualizado para Yes ou No.
