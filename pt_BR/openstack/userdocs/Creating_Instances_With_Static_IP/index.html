      Como criar uma instância com IP estático

      • by Jason Kennedy ● September 2, 2016

      • Tags:

        • openstack

        • static ip

        • instance

      Este artigo mostra como é possível criar uma nova porta no OpenStack, e anexá-la manualmente a instâncias no momento da inicialização, para designar um IP “estático” a uma instância.

      Primeiro, muito desse método acontecerá na linha de comandos, uma vez que é muito mais versátil uma ferramenta para trabalhar com o IBM Bluemix Private Cloud. Se necessário, estou disposto a fazer backup e avançar usando a CLI & API do OpenStack para fazer este conjunto de instruções parecer mais razoável. Dito isso, vamos passar diretamente para como fazer isso. Este trabalho está sendo feito em um cluster de teste, mas essas etapas podem também ser repetidas em um compartilhamento de tela em seu próprio cluster se isso parecer seguro.

      Primeiro, os clusters do Bluemix Private Cloud têm duas redes pré-geradas: interna e externa. Para nossos propósitos, nós queremos criar essa nova porta na rede interna, então, vamos rapidamente para isso por meio da linha de comandos:

      bash-3.2$ openstack network list | grep internal
| 81e829d0-5e35-4808-9f7a-d22159faa9d1 | internal | c6560f00-ae29-40b5-93bb-6e7b1ffd3720

      Isso nos dá o ID da rede interna. Para discutir a CLI por um momento, estou usando python-openstackclient para este trabalho; é o “mais novo” dos clientes. O ID é usado menos com esse cliente, mas conseguir encontrar o ID para uma determinada rede, locatário ou o que quer que seja é importante uma vez que há ocasião para usar as ferramentas mais antigas (python-novaclient, etc.) quando necessário. Não vamos entrar em muitos detalhes nesse ponto.

      Precisamos ver a sub-rede com a qual vamos trabalhar neste caso, então, vamos dar uma olhada nisso:

      Há a nossa sub-rede; CIDR é 10.220.0.0/22. Vamos usar 10.220.3.100 para o nosso teste. Vamos criar manualmente a porta com essa reserva IP.
