      Hoje veremos o processo necessário para a inclusão de um endereço IP flutuante em uma de suas instâncias. A conclusão deste processo é mencionada no final de Criando redes adicionais no OpenStack através do painel do Horizon, vamos executar este exemplo através do terminal. Observe que a criação de um IP flutuante não cria realmente o endereço, apenas realiza a alocação de um endereço do conjunto designado. Vejamos o que foi designado:

      Agora que o IP foi criado, é possível anexá-lo à instância. Vamos precisar do ID da porta do endereço IP interno da instância e do ID do IP flutuante, indicado acima.

      Aqui estão a instância (ignore a formatação) e seu IP interno. Vamos usar essas informações para localizar a porta.

      Pronto! Agora, precisamos apenas do ID do IP flutuante. Já o possuímos no floating_network_id quando ele foi criado: 953cda96-1a33-4645-a4e2-1b00d6e06736. A etapa final é associá-los.

      O comando nova list deve mostrar agora os dois endereços IP associados à sua instância:

      Quando desejar, também é possível desassociar um IP flutuante executando o comando seguido do ID do endereço em questão. O endereço que acabamos de associar à instância abaixo será desassociado.

      Você notará que o endereço não está mais associado à sua instância.

      • No OpenStack, os IPs flutuantes são IPs que podem ser roteados publicamente e que são designados às MVs. O que acontece dentro da MV após a inclusão do IP flutuante? A resposta é… nada. Ao efetuar login por meio de SSH e exibir a configuração de rede, você verá que ainda há uma interface de rede única com um IP fixo configurado. Toda configuração é feita pelo próprio hypervisor de cálculo. O OpenStack Neutron configura o NAT entre os IPs fixos e flutuantes da MV.

      Uma das grandes vantagens do uso de IPs flutuantes é: se uma MV ficar inativa, é possível reutilizar o endereço IP flutuante, anexando-o a outra MV.

      Para obter mais informações, consulte: https://www.mirantis.com/blog/configuring-floating-ip-addresses-networking-openstack-public-private-clouds/
