      Guia do usuário para Imagens na Nuvem (IBM Bluemix Private Cloud com Red Hat)

      • by Ying Tang, Xiong Chen ● March 31, 2017

      • Tags:

        • userguides

        • images

        • glance

        • image-create

        • instance

        • rhosp

        • newton

      O IBM Bluemix Private Cloud com Red Hat vem pré-preenchido com imagens do Cirros 0.3.3 x86_64.

      É possível verificar as Notas sobre a liberação da imagem da nuvem para obter mais detalhes sobre as imagens fornecidas pela IBM.

      Para trabalhar com imagens guest do Red Hat, faça download dos arquivos de imagem das seguintes URLs:

      • Red Hat Enterprise Linux 7 KVM Guest Image
      • Red Hat Enterprise Linux 6 KVM Guest Image

      Informações adicionais sobre como usar imagens com o OpenStack estão disponíveis no website do OpenStack: https://docs.openstack.org/image-guide/

      Pré-requisitos

      Caso planeje usar ferramentas da linha de comandos (CLI), certifique-se de que o OpenStack Glance Client esteja instalado. É possível encontrar instruções de instalação na base de conhecimento: Introdução à API do OpenStack.

      Na liberação atual do IBM Bluemix Private Cloud com Red Hat, é possível fazer upload de um arquivo de imagem por meio da linha de comandos ou do painel do Horizon. O upload por meio de um local de imagem ou URL não é suportado.

      Selecione a partir das imagens de nuvem alternativas fornecidas pelo IBM Bluemix Private Cloud

      O IBM Bluemix Private Cloud com Red Hat nesta liberação vem pré-preenchido com imagens do Cirros 0.3.3 x86_64. Há imagens de nuvem adicionais disponíveis, fornecidas pelo IBM Bluemix Private Cloud como cortesia para o cliente.

      • CentOS 6.8 x86_64
      • CentOS 7.3 x86_64
      • Ubuntu Server 14.04 LTS x86_64
      • Ubuntu Server 16.04 LTS x86_64
      • Windows Server 2008 R2 SP1 Datacenter
      • Windows Server 2008 R2 SP1 Enterprise
      • Windows Server 2008 R2 SP1 Standard
      • Windows Server 2012 R2 Datacenter
      • Windows Server 2012 R2 Standard

      Faça upload de uma imagem para o repositório Glance

      É possível fazer upload e instalar um software do sistema operacional adquirido separadamente, por exemplo, o Red Hat Enterprise Linux 7 KVM Guest Image, ou fornecer qualquer uma das imagens do sistema operacional que são disponibilizadas a você como uma instância de máquina virtual. Você é responsável por obedecer a todos os termos da licença do sistema operacional e por adquirir autorizações adequadas para cada instância da máquina virtual.

      Obter downloads de imagens da nuvem fornecidas pela IBM do Box Panel

      1. Efetue login no Box Panel com as credenciais do Box Panel. A página Imagens da nuvem no Box Panel fica disponível se houver pelo menos uma nuvem e se você for um contato de cliente Primário ou Técnico.

      2. No painel do Box Panel, clique em Serviços e selecione Imagens da Nuvem.

      3. A página Imagens da nuvem será exibida. Cada imagem é exibida como uma placa.

      4. Clique na placa que exibe a imagem de seu interesse. Uma janela modal será exibida contendo a tempURL.

      5. A tempURL exibida será válida pelas próximas 24 horas para que seja possível fazer download da imagem da nuvem.

      6. Certifique-se de que o token da tempUrl tenha caracteres de escape ou seja especificado entre aspas. Por exemplo:

          curl -f -o <output_file> '<cloud_image_download_tempUrl>'  
  curl -f -o ubuntu-guest-image-14.04-20160301-x86_64.qcow2 'https://dal05.objectstorage.softlayer.net/v1/AUTH_2201d7be-5d96-431c-9bd0-ec3ed5b62b19/cloud_images/test/ubuntu-guest-image-14.04-20160301-x86_64.qcow2?temp_url_sig=9e9678ad2d81489cebd032dff2332ec8ee50ebba&temp_url_expires=1458956867'

      7. Faça download dos arquivos .qcow2 e .md5sum da imagem.

      8. Faça download do arquivo de imagem .qcow2 e do arquivo de soma de verificação .md5sum usando a tempURL e coloque esses 2 arquivos na mesma pasta. Por exemplo:

          ubuntu-guest-image-14.04-20160301-x86_64.qcow2
  ubuntu-guest-image-14.04-20160301-x86_64.md5sum

      9. Verifique o arquivo de imagem .qcow2 transferido por download usando o arquivo de soma de verificação .md5sum. Por exemplo:

         md5sum -c ubuntu-guest-image-14.04-20160301-x86_64.md5sum
 ubuntu-guest-image-14.04-20160301-x86_64.qcow2: OK

      10. Se o resultado retornar “OK”, a imagem qcow2 transferida por download será válida.

      Faça upload de uma imagem usando a linha de comandos

      É possível fazer upload de imagens por meio do cliente Glance. Consulte Instalando OpenStack Clients para obter mais informações.

      Crie uma imagem.

      • Para imagens do Linux, use -- min-disk 5 --min-ram 512

      • Para imagens do Windows, use --min-disk 30 --min-ram 2048

        $ glance image-create --os-version <os_version> --name <image_name> --min-disk <min-disk> --min-ram <min-ram> --disk-format qcow2 --container-format bare --file <image_file>

      Verifique se a imagem foi criada com êxito. A imagem é enfileirada para upload. Pode demorar um pouco para que o status mude de “Enfileirado” para “Ativo”.

      $ glance image-show <image-id>

      Consulte cliente da linha de comandos Glance para obter mais informações sobre este comando.

      Faça upload de uma imagem usando o painel do OpenStack

      1. Efetue login no painel do IBM Bluemix Private Cloud OpenStack.

      2. No painel Projeto, expanda Calcular e clique em Imagens.

      3. Clique em Criar imagem na página. Especifique os parâmetros a seguir na nova página e marque Criar imagem.

        Item Descrição
        Nome Insira um nome para a imagem.
        Descrição Opcionalmente, insira uma breve descrição da imagem.
        **Arquivo  
        ** Escolha Arquivo de imagem.
        Tipo de Selecione o formato correto (por exemplo, QCOW2) para a imagem.
        Kernel Como opção, especifique o ID do kernel da imagem armazenada no Glance que deve ser usado como o kernel ao inicializar uma imagem de estilo AMI.
        Ramdisk Como opção, especifique o ID da imagem armazenada no Glance que deve ser usado como o kernel ao inicializar uma imagem de estilo AMI.
        Arquitetura Como opção, especifique a arquitetura (por exemplo, i386 para uma arquitetura de 32 bits ou x86_64 para uma arquitetura de 64 bits).
        Disco Mínimo (GB) Para uma imagem do Linux, use 5; para uma imagem do Windows, use 30
        RAM Mínima (MB) Para uma imagem do Linux, use 512; para uma imagem do Windows, use 2048
        Público Selecione esta caixa de seleção para tornar a imagem pública para todos os usuários em todos os projetos.
        Protegido Selecione esta caixa de seleção para assegurar que apenas os usuários com permissões possam excluir a imagem.

        Por exemplo:

         Name: Ubuntu Server 14.04 LTS x86_64 - 20160301	   Description: Ubuntu Server 14.04 LTS x86_64 cloud image released on 2016/03/01		
 File: C:\Documents\ubuntu-guest-image-14.04-20160301-x86_64.qcow2	
 Format: QCOW2 - QEMU Emulator	
 Architecture: x86_64	
 Minimum Disk(GB): 5	
 Minimum RAM(MB): 512	
 Public: True

      4. Verifique se a imagem criada aparece na lista de imagens. A imagem é enfileirada para upload. Pode demorar um pouco para que o status mude de “Enfileirado” para “Ativo”.
