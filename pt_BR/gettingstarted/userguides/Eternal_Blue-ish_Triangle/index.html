      Box Panel, Site Controller e IBM Bluemix Private Cloud Local

      • by Leslie Lundquist and Tom Spoonemore ● September 20, 2016

      • Tags:

        • user guides

        • box panel

        • site controller

        • faq

        • cloud security

      Este documento oferece uma introdução aos conceitos fundamentais e à terminologia do Box Panel, Site Controller e como eles funcionam juntos para controlar e gerenciar o ambiente do IBM Bluemix Private Cloud Local. O documento inclui links e indicadores para documentos externos, para que você possa encontrar ainda mais informações. Alguns dos termos definidos na seção Perguntas frequentes de terminologia serão mais técnicos do que você pode precisar saber diariamente, mas saber que eles existem pode ser útil se você tiver uma oportunidade de falar com nossa equipe de suporte.

      O que é o Box Panel?

      Box Panel é um conjunto de aplicativos que utilizamos para entregar o IBM Bluemix Private Cloud. O Box Panel é usado pela equipe de suporte interno e também por nossos clientes IBM. É a maneira como nós gerenciamos a infraestrutura de nuvem.

      Juntamente com outras funcionalidades, o Box Panel é um sistema de registro de todos os datacenters, ativos de máquina, clusters, endereços IP e informações da conta do cliente.

      O conjunto do Box Panel é entregue como um aplicativo SaaS acessível por meio de um navegador da web e de APIs customizadas. O conjunto do Box Panel inclui aplicativos para suporte, chamado e bate-papo, e inclui links para a documentação do produto.

      Para obter mais informações sobre o Box Panel, veja novamente nosso Guia do Usuário do Box Panel.

      O que é o Site Controller do Box Panel?

      O Site Controller do Box Panel é uma ferramenta de software que fornece operações de plano de controle distribuído, gerenciamento de configuração e serviços de automação para o conjunto do Box Panel e para as equipes de engenharia e suporte do Bluemix Private Cloud.

      O Site Controller do Box Panel é executado em cada datacenter Local e Dedicado, no qual se comunica com o Software como Serviço (SaaS) do Box Panel através de um pod de controle central.

      Os recursos do Site Controller do Box Panel incluem:

      • Acesso remoto administrativo
      • Proxy IPMI
      • Fornecimento Bare Metal
      • Espelho de pacote central
      • Monitoramento / Alerta
      • Sensu
      • Graphite
      • PagerDuty
      • Gerenciamento de log (ELK)
      • Serviços de automação
      • Serviços de proxy HTTP

      Perguntas frequentes de terminologia: o que é…..

      P: fila do Box Panel?

      R: A Fila do Box Panel é a fila de tarefas para funções administrativas vindas do Box Panel. Essas funções incluem correção de software automatizada, nova implementação de nó e reinicialização de servidores.

      P: trabalhador do Box Panel?

      R: O Trabalhador é um processo que recebe solicitações de tarefas da Fila do Box Panel e executa essas solicitações, em seguida, retorna os resultados para a Fila do Box Panel.

      P: Sensu no pod de controle central?

      R: Fornece monitoramento dos controladores de sites remotos no IBM Cloud e nos datacenters de propriedade dos clientes.

      P: Sensu em um Site Controller?

      R: Monitora os nós físicos (pNodes) do Bluemix Private Cloud e fornece alertas de volta para o pod de controle central. Os pNodes incluem os nós de controle, cálculo e armazenamento.

      P: PagerDuty?

      R: A ferramenta de relatório de incidentes que nos acorda à noite quando a nuvem precisa de atenção.

      P: Graphite?

      R. Uma ferramenta gráfica que usamos, especialmente para fazer relatórios SLA que mostram se as APIs estão ativas. É possível ver essas informações no Horizon porque o Graphite alimenta dados de tempo de atividade nos painéis de Grafana.

      P: Proxy IPMI?

      R: Fornece acesso IPMI seguro aos nós da nuvem.

      P: Proxy HTTP?

      R: Fornece interface HTTP para as Interfaces com o Usuário Sensu e ELK no Site Controller.

      P: espelho?

      R: Um repositório de software central para o software de operações do Cloud. Esse repositório não inclui cargas de trabalho do cliente.

      P: Bastion?

      R: O host Bastion fornece acesso remoto seguro aos Site Controllers e aos nós de nuvem do cliente, usando Role-Based Access Control (RBAC). Para obter mais informações sobre o RBAC, consulte esses documentos:

      • Gerenciando usuários e projetos
      • Definições do RBAC
      • Serviços de identidade federada

      P: Yama?

      R: Fornece autenticação de diversos fatores para acesso às nuvens do cliente e tem registro de acesso pelo pessoal de suporte da nuvem.

      P: ELK (Elasticsearch, Logstash & Kibana)?

      R: Usado para gerenciamento de log de recursos em nuvem. Esses logs não deixam o site do cliente.

      P: Boot-strapper?

      R: Oferece fornecimento bare metal de servidores. O Boot-Strapper é uma réplica do repositório do Mirror.

      Perguntas frequentes de segurança:

      P: quem tem acesso ao Bluemix Private Cloud Local do suporte IBM?

      R: Membros das equipes de engenharia e operações do OpenStack têm acesso aos Controladores, Cálculo e Nós, API do OpenStack e Horizon. Eles usam esse acesso para implementar e operar o serviço de nuvem privada. Esses engenheiros de suporte não têm nenhum acesso aos dados ou cargas de trabalho do cliente acima do hypervisor.

      P: os representantes de suporte IBM podem obter acesso aos recursos do cliente?

      R: As equipes de engenharia e operações do IBM Bluemix Private Cloud não têm credenciais para lhes dar acesso a nenhuma MV ou aplicativo do Cloud implementado na nuvem.

      P: quais informações fluem pela VPN?

      R: Apenas essas mensagens de comando e controle requeridas para implementar e operar o serviço de nuvem privada fluem pela VPN. Não há nenhum roteamento entre a VPN e o plano de dados do cliente (MVs, cargas de trabalho, armazenamento, redes Neutron). Sessões de Acesso Remoto Administrativo Seguro iniciadas pela equipe de suporte do OpenStack. Pacotes de software usados para instalar e corrigir os servidores de nuvem privada. Mensagens e alertas de monitoramento iniciados pelos nós da nuvem. Todos os dados de criação de log são mantidos no equipamento das instalações do cliente.

      P: quem pode obter essas informações?

      R: A equipe de engenharia e operações do IBM Bluemix Private Cloud do OpenStack tem acesso a essas informações para fins de gerenciamento de nuvem e resolução de problemas.

      P: qual é a política de controle de acesso para a equipe de suporte?

      R: A equipe de suporte segue uma estrita Política de Segurança do IBM Cloud para conceder, reavaliar e revogar o acesso às nuvens dos clientes.

      P: o Bluemix Private Cloud pode usar um processo de registro do cliente?

      R: O Bluemix Private Cloud não suporta atualmente e nossos processos são alinhados com a ISO27002.

      P: o que o Bluemix Private Cloud gerencia?

      R: O Bluemix Private Cloud gerencia a nuvem do OpenStack do bare metal até, e incluindo, o hypervisor. Nossos processos, políticas e ferramentas estão em conformidade com a Política de Segurança do IBM Cloud (CSP). A CSP se alinha com o padrão de mercado ISO27002, que é o padrão de tecnologia de informações de prática para controles de segurança de informações—ou seja, para técnicas de segurança.
