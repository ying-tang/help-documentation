      Ativando as licenças do Windows no IBM Bluemix Private Cloud

      O catálogo de imagens fornecido pela IBM® contém as imagens do Microsoft® Windows oferecidas em um modelo BYOL (bring your own license). De acordo com os Direitos de uso do provedor de serviços da Microsoft, a IBM não pode fornecer serviços de ativação de Serviços de gerenciamento de chaves (KMS) para a imagem BYOL do Windows.

      Há várias maneiras de ativar as licenças do Windows:

      • Crie um servidor KMS no ambiente do IBM Bluemix Private Cloud
      • Use um servidor KMS existente no site ou em outro local
      • Ative online diretamente com a Microsoft
      • Ative offline por telefone com a Microsoft

      É possível usar um ou mais desses métodos, dependendo das circunstâncias. Itens a serem considerados ao selecionar o método:

      • Os servidores KMS possuem limites mínimos quanto ao número de clientes que eles ativarão. Se o serviço KMS estiver hospedado no Windows Server 2008 ou no Windows Server 2012, você deverá ativar no mínimo cinco clientes. Se estiver ativando menos de cinco licenças, não é possível usar o KMS dentro do IBM Bluemix PRivate Cloud.
      • Para usar um servidor KMS existente localizado fora do ambiente do IBM Bluemix Private Cloud, deve-se estabelecer a conectividade adequada. Isso inclui firewalls do Bluemix Private Cloud e os firewalls que permitem que as instâncias do Windows do Bluemix Private Cloud iniciem o tráfego com o servidor KMS na porta 1688 do tcp.
      • Para usar a ativação online direta, as instâncias do Windows deverão possuir conectividade de saída com a Internet. Por padrão, as instâncias não possuem conectividade com a Internet, mas é possível solicitar acesso no pedido inicial ou enviar um chamado de suporte para atualizar o acesso a qualquer momento.

      Criando um servidor KMS no IBM Blue Box Cloud

      É possível configurar qualquer Windows Server 2008 ou uma instância mais recente para fornecer o serviço KMS. Para agir como um servidor KMS, a instância do Windows precisa de conectividade de saída com a Internet para comunicar-se com os Servidores de licença da Microsoft. Você também deve comprar uma Chave do host KMS da Microsoft para receber uma autorização específica.

      Para converter uma instância do Windows 2008 R2 em um KMS no IBM Bluemix Private Cloud, use estas etapas:

      1. Faça download deste arquivo de atualização da Microsoft na instância.

      2. Instale a atualização.

      3. Registre a chave do host KMS usando este comando:

         %SystemRoot%\System32\CScript.exe %windir%\system32\slmgr.vbs /ipk <your_KMS_host_key>

      4. Ative a nova chave do host KMS usando este comando:

         %SystemRoot%\System32\CScript.exe %windir%\system32\slmgr.vbs /ato
      5. Reinicie o Serviço de Licenciamento de Software usando este comando: 
         net stop sppsvc && net start sppsvc
      6. Use este comando para abrir a porta 1688 para que os clientes KMS possam se registrar no host KMS: 
         netsh firewall set portopening tcp 1688 KMS enable

      Usando um servidor KMS existente

      Se você já tiver um servidor KMS existente no site ou em outro local, ele poderá ser usado, desde que possua as conexões de rede apropriadas estabelecidas. Para que o servidor Windows possa ser ativado com o servidor KMS, ele deverá ter a capacidade de iniciar a comunicação com o servidor KMS (geralmente, na porta TCP 1688). Crie um chamado de suporte para que as regras estáticas adequadas sejam aplicadas no firewall do IBM Bluemix Private Cloud.

      Ativando o servidor Windows por meio de um servidor KMS

      Para ativar o servidor Windows por meio de um servidor KMS, são necessários o endereço IP e a conectividade de rede com o servidor KMS, além das chaves de configuração do cliente KMS do Microsoft Technet adequadas.

      Após criar uma nova instância a partir da imagem do Servidor Microsoft Windows, conecte-se a ela e siga estas etapas:

      1. Instale uma chave do cliente KMS em sua instância do Windows. Abra uma janela de prompt de comandos e execute este comando: 
         slmgr /ipk <KMS_client_key>

        Visualize as chaves de configuração do cliente KMS on-line. Por exemplo, para ativar a instância do Windows 2008 R2 Standard, você usaria YC6KT-GKW9T-YTKYR-T4X34-R7VHC.

      2. Configure o endereço IP e a porta do servidor KMS na instância do Windows. A porta TCP padrão usada pelos clientes para se conectar ao host KMS é 1688: 
         slmgr /skms <KMS_server_ip_address>:<KMS_server_port>
      3. Sincronize o tempo com o servidor KMS: 
         net start w32time
 w32tm /config /manualpeerlist:<KMS_server_ip_address> /update
 w32tm /resync
      4. Ative sua instância do Windows: 
         slmgr.vbs /ato
      5. Altere o servidor de horário de volta para o padrão: 
         w32tm /config /manualpeerlist:time.windows.com /update

      Para obter mais detalhes sobre a ativação do Windows, use estes recursos:

      • Entendendo o KMS
      • Implementando a ativação do KMS
      • Determinando as necessidades de chave do produto
      • Chaves de configuração do cliente KMS

      Ativação direta

      Se houver somente um pequeno número de Servidores Windows a serem ativados, não será necessário usar um servidor KMS. Será possível ativar o servidor diretamente com a Microsoft, entrando em contato com a Microsoft por telefone ou ativando o servidor online. Para ativar o servidor online, a instância deverá possuir conectividade de saída com a Internet para entrar em contato com o Microsoft License Service.
